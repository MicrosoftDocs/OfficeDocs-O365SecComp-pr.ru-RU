# <a name="use-the-pst-collection-tool-to-find-copy-and-delete-pst-files-in-your-organization"></a>Используйте средство коллекции PST-файлов для поиска, копирование и удаление PST-файлов в вашей организации

Можно использовать средство PST семейства Microsoft для поиска PST-файлы сети вашей организации. Средство помогает получить перечень PST-файлов, которые находятся в вашей организации. После найти PST-файлов, можно использовать средство PST коллекции скопировать их в одном месте. Использование PST-файлов в одном месте, затем позволяет импортировать их в Exchange Online почтовые ящики (или к одному почтовому ящику Exchange Online), где можно применить широкий набор возможностей соответствием требованиям в Office 365. Это включает в себя Импорт PST-файлов в архив, который содержит почтовые ящики, поиск определенного сообщения в PST-файлов, которые импортированы с помощью средств поиска обнаружения электронных данных, хранением сообщений с помощью обнаружения электронных данных и политики хранения Office 365 пользователей и управление ими жизни цикл этих сообщений с помощью системы обмена сообщениями записывает функции управления и в Exchange Online. После вы уверены в том, что PST-файлов, которые вы собраны были успешно импортирована в Office 365, можно использовать средство для удаления их из исходного расположения в сети. 
  
Другой, что можно делать с средство PST коллекции — запретить пользователям создание новых файлов PST и изменение существующего PST-файлов, которые можно найти в сети. Эти возможности «block» позволяют найти, собрать и импорт известные набора PST-файлов в Office 365 и предотвратить будущих распространение PST-файлов в вашей организации. 
  
## <a name="how-the-pst-collection-tool-works"></a>Принципы работы средства коллекции PST-файлов

Ниже приведен краткий обзор процесса с использованием средства коллекции PST-файлов для поиска, управления, сбор и удаление PST-файлов в вашей организации.
  
![Общие сведения о процессе средство коллекции PST-файлов](media/67a29f27-f83c-4f0a-9df4-7ed92d3086fe.png)
  
1. **[Шаг 1: Поиск PST-файлов в сети](#step-1-find-pst-files-on-your-network)** - при выполнении tool, чтобы найти PST-файлов, укажите расположение, например подразделения, содержат объекты Active Directory для клиентских и серверных компьютеров. Можно также выполнить поиск определенных компьютеров или общих сетевых папок. При запуске средства «lightweight» агент семейства установлен на конечных компьютерах. Данный агент выполняет на целевом компьютере для PST-файлов и отправляет данные средство PST коллекции о любой PST-файл, найденный. Средство создает файлы журнала, которые содержит сведения о файлах PST-файлов, найденных в указанном расположении. Эти файлы используются при запуске средства на последующих этапах. 
    
2. **[(Необязательно) шаг 2: управления доступом к PST-файлов](#optional-step-2-control-access-to-pst-files)** -средство создает объект групповой политики (GPO) с параметрами, которые не позволяют пользователям Создание или изменение PST-файлов. Этот объект групповой Политики применяется к каждому пользователю в домене. Это необязательное действие помогает «заблокировать» PST-файлов, обнаруженных в шаге 1, чтобы собирать, Импорт и удалять их без необходимости PST-файлы созданы или изменены существующие файлы PST-файлов. 
    
3. **[Шаг 3: скопируйте PST-файлы в папку коллекции](#step-3-copy-the-pst-files-to-a-collection-location)** — это позволяет собирать PST-файлов в одном месте, чтобы импортировать их в почтовые ящики Exchange Online с помощью службы Office 365 импортировать на шаге 4. При запуске средство в режиме «сбор» каждому агенту коллекции копирует PST-файлов на целевом компьютере, агент установлен на расположение семейства сайтов. 
    
4. **[Шаг 4: Импорт PST-файлов в Office 365](#step-4-import-the-pst-files-to-office-365)** -после копирования PST-файлов в одном месте, вы готовы к их импорт в почтовые ящики Exchange Online. 
    
5. **[Шаг 5: удаление PST-файлов, найденные в сети](#step-5-delete-the-pst-files-found-on-your-network)** - после файлов PST-файлов, что найти и сбора были импортированы почтовых ящиков Exchange Online в Office 365, можно использовать средство PST коллекции для удаления PST-файлы из исходного расположения которых они найдено в шаге 1. 

## <a name="before-you-begin"></a>Перед началом работы

- Выполните следующие действия, чтобы загрузить средство PST коллекции на локальный компьютер. 
    
    1. [Загрузите средство PST семейства сайтов](https://aka.ms/pstcollectiontool).
    
    2. Во всплывающем окне, нажмите кнопку **Сохранить** \> **Сохранить как** для сохранения файла PSTCollectionTool.zip в папку на локальном компьютере. 
    
    3. Извлеките файл PSTCollectionTool.zip в папку на локальном компьютере; Имя папки по умолчанию — PSTCollectionTool.
    
- Чтобы запустить средство коллекции PST-файлов в любой режим (поиск, блока, копия или удаление), необходимо быть членом группы администраторов домена в вашем домене Active Directory. 

## <a name="step-1-find-pst-files-on-your-network"></a>Шаг 1: Поиск PST-файлов в сети

Первым шагом является запуск средства коллекции PST-файлов для поиска файлов PST-файлов в вашей организации. Можно использовать средство поиска следующие типы расположений. 
  
- Подразделения (OU) в домене Active Directory локальной. Средство выполняется поиск всех компьютерах, содержащиеся в указанном Подразделении. 
    
- Клиентские и серверные компьютеры. Производит указанных компьютерах. 
    
- Общие папки. Производит общие указанной папки. 
    
В разделе Описание для `Locations` параметр в таблице в следующей процедуре примеры синтаксиса для каждого из этих типов размещения. 
  
> [!IMPORTANT]
> Необходимо запустить средство коллекции PST-файлов в режиме поиска перед выполнением других действий, например блокировки, сбор или удаление PST-файлов. 
  
1. Откройте командную строку (Запуск от имени администратора) на локальном компьютере.
    
2. Перейдите к папке PSTCollectionTool (или к папке, где были извлечены файл PSTCollectionTool.zip, чтобы).
    
3. Перейдите в каталог DataCollectorMaster.
    
4. Выполните следующую команду, чтобы найти PST-файлов в заданном расположении.
    
    ```
    DataCollectorMaster.exe -DataSource Pst -Mode Find -JobName <Name> -Locations <Locations to search for PSTs> -LogLocation <Location to store log files> -ConfigurationLocation <Location to store configuration files>
    ```

    | `Mode` <br/> |Указывает тип операции, которая будет выполнять средство. Используйте значение `Find` для поиска PST-файлов в указанном расположении. Через некоторое время последнее сообщение о состоянии показывает общее число PST-файлов, которые были найдены ли завершения задания и были возникновения ошибок. Же сообщения о состоянии, копируются в файл журнала.
    
### <a name="results-of-running-datacollectormasterexe-in-the-find-mode"></a>Результаты выполнения DataCollectorMaster.exe в режиме поиска

После успешного выполнения средство коллекции PST-файлов в режиме поиска, следующие файлы создаются и хранятся в папке, указанной с `LogLocation` и `ConfigurationLocation` параметры. 
  
- **\<Имя_задания\>_Найти_\<DateTimeStamp\>.log** -файл журнала содержит сообщения о состоянии, которые были отображаются. Этот файл создается в папке, указанной в `LogLocation` параметр. 
    
- **\<Имя_задания\>_Найти_\<DateTimeStamp\>.csv** -CSV-файл содержит строку для каждого PST-файл, который был найден. Сведения для каждого PST включает в себя компьютер, где найден PST-файл, полный путь из PST-файл, владелец PST-файл и размер (в килобайтах, статьи базы знаний) PST-файл. Этот файл создается в папке, указанной в `LogLocation` параметр. 
    
    > [!TIP]
    > Используйте средство Автосумма в Excel для вычисления общий размер всех файлов PST-файлов, перечисленных в CSV-файл (в КБ). Затем можно использовать калькулятор преобразования для преобразования общий размер в мегабайтах (МБ) или гигабайт (ГБ). 
  
- **\<Имя_задания\>_Найти_\<DateTimeStamp\>.xml** — XML-файл, содержащий сведения о параметр со значениями, где используется при запуске команды средство в режиме поиска. Этот файл также содержит сведения о каждом PST-файл, который был найден. Данные в этот файл будет использоваться при запуске повторно запустить средство для того же задание для блокировки, собирать или удалите PST файлы, которые были найдены. Этот файл создается в папке, указанной в `ConfigurationLocation` параметр. 
    
    > [!IMPORTANT]
    > Не переименовать, изменить или переместить этот файл. Он используется с помощью данного средства коллекции PST-файлов при повторно запустить средство в блоке копию, или удалить режим того же задания. 

## <a name="optional-step-2-control-access-to-pst-files"></a>(Необязательно) Шаг 2: Управление доступом к PST-файлов

Это необязательное действие позволяет «заблокировать» PST-файлов, обнаруженных в шаге 1, чтобы собрать и импорт известные набора PST-файлов в Office 365. При запуске средства коллекции PST-файлов в режиме блокировки организуется следующим образом: 
  
- Средство создает объект (Групповой) с именем *Элементы управления использование PST -файлов* . Этот объект групповой Политики связан с вашего домена и применяется ко всем прошедшим проверку подлинности пользователям в вашей организации. 
    
- Объект групповой Политики PST использования элементов управления создает параметры реестра на компьютерах в организации. В зависимости от параметра, который можно использовать можно создать параметр реестра, чтобы запретить пользователям создавать новые PST-файлов и параметров реестра, который запрещает пользователям изменять существующие файлы PST-файлов.
    
> [!NOTE]
> Если управление доступом к PST-файлов слишком аварийные для вашей организации, может потребоваться пропуск это действие и выполняется шаг 3 для копирования PST-файлов в центральном расположении. А затем повторите шаг 1 для того же задания (с помощью `ForceRestart` параметра) для поиска дополнительных файлов PST-файлов, которые были созданы после копирования файлов PST-файлов в расположение семейства сайтов. При обнаружении новых файлов PST-файлов, их можно скопировать в папку коллекции. При использовании `ForceRestart` параметр при повторно запустить средство в режиме поиска, результаты из предыдущей операции поиска для задания будут отменены и средство будет повторно проверить указанного расположения. 

Чтобы заблокировать доступ к PST-файлов:

1. Откройте командную строку (Запуск от имени администратора) на локальном компьютере.
    
2. Перейдите в каталог, в которую было загружено средство коллекции PST-файлов для.
    
3. Выполните следующую команду для блокировки доступа к PST-файлов, обнаруженных в шаге 1.

    ```
    DataCollectorMaster.exe -DataSource Pst -Mode Block -JobName <Name of job from Step 1> -ConfigurationLocation <Location of configuration files from Step 1> -BlockChangesToFiles -BlockNewFiles
    ```

    Ниже перечислены параметры и их требуемые значения при выполнении команды DataCollectorMaster.exe для блокирования Создание и изменение PST-файлов. 
    
    |Параметр ***|****Описание****|Примеры ***|
    |:-----|:-----|:-----|
    | `DataSource` <br/> |Указывает тип данных для поиска. В настоящее время можно использовать средство PST семейства сайтов для поиска файлов PST-файлов.  <br/> | `-DataSource Pst` <br/> |
    | `Mode` <br/> |Указывает тип операции, которая будет выполнять средство. Используйте значение `Block` для запрета создания новых файлов PST-файлов и внесение изменений в существующий PST-файлов.<br/> | `-Mode Block` <br/> |
    | `JobName` <br/> |Указывает имя существующего задания коллекции PST-файлов. Необходимо использовать этот же имя задания, которая использовалась при запуске команды средство в режиме поиска на шаге 1. Это имя задания также добавляется к имени файла журнала, созданный при запустить средство в режиме блокировки.  <br/> | `-JobName PstSearch1` <br/> |
    | `ConfigurationLocation` <br/> |Указывает, что папка содержит XML-файл конфигурации, который был создан при запуске команды средство в режиме поиска.
