# <a name="use-network-upload-to-import-rms-encrypted-pst-files-to-office-365"></a>Импорт зашифрованных с помощью RMS PST-файлов в Office 365 с помощью отправки по сети

**Эта статья предназначена для администраторов. Вы пытаетесь Импорт PST-файлов для почтового ящика? Просмотреть [импорта электронной почты, контактов и календаря из Outlook PST-файл](https://go.microsoft.com/fwlink/p/?LinkID=785075)**
   
Использование сети отправка параметр и службы Office 365 импорта для импорта PST-файлов для почтовых ящиков пользователей. Отправка сети означает, что отправка PST-файлов временная область в облаке Майкрософт. Затем службы Office 365 импорта копирует PST-файлов из области хранилища почтовых ящиков пользователей целевой. Новая функция импорта службы позволяет зашифровать PST-файлов, прежде чем они загружаются и хранятся в облаке Майкрософт. Эти файлы будут без зашифрованных при импорте к почтовым ящикам пользователей.

[Шаг 4: Загрузите средства импорта PST-файлов и скопируйте URL-адрес SAS](#step-4-download-the-pst-import-tools-and-copy-the-sas-url)

[Шаг 5: Шифрования и отправить файлы PST-файлов в Office 365](#step-5-encrypt-and-upload-your-pst-files-to-office-365)

[(Необязательно) Шаг 6: Представление списка файлов PST-файлов было выгрузить в Office 365](#optional-step-6-view-a-list-of-the-pst-files-uploaded-to-office-365)

Роль Импорт и экспорт почтового ящика можно добавить в группу ролей управления организацией. Или можно создать новую группу ролей, назначение роли почтового ящика Импорт и экспорт и добавьте себя в качестве члена. Для получения дополнительных сведений см. «Добавить роли, группы ролей» или «Создать группу ролей» разделов в [Управление группами ролей](https://go.microsoft.com/fwlink/p/?LinkId=730688).
    
    Кроме того, чтобы создать импорта заданий в безопасности Office 365 &amp; должны иметь положительный ответ центре соответствия требованиям, одно из следующих действий:
    
  - Необходимо назначить роль получателей почты в Exchange Online. По умолчанию эта роль назначается группы ролей управления организацией и управления получателя.
    
    Второй вариант:
    
  - Необходимо быть глобальным администратором в организации Office 365.
    
  > [!TIP]
  > Рассмотрите возможность создания новой группы ролей в Exchange Online, специально предназначенный для импорта PST-файлов в Office 365. Для минимальный уровень привилегий, необходимым для импорта PST-файлов, назначение ролей получателей почты и импорт экспорт почтового ящика в новую группу ролей, а затем добавьте членов.

На шаге 5 будет выполняться ImportTool 365 Office, в которой будет шифрования и загрузки файлов PST-файлов, которые хранятся на файловом сервере или в общей папке в Office 365.

Эти сведения будут использоваться на шаге 5 для шифрования и отправить файлы PST-файлов. Убедитесь, что примите предварительные меры для защиты этих так же, как бы защиты паролей и другие сведения, связанные с безопасностью. Например можно сохранять их на документ Microsoft Word, защищенных паролем или сохранять их на зашифрованные USB-диска. Пример этих разделов, идентификаторы и URL-адреса в разделе [Дополнительные сведения](#more-information) .

Это делается путем указания идентификатора GUID неактивного почтового ящика в `Mailbox` параметр в файле сопоставления Импорт PST-файлов. Дополнительные сведения см в [шаге 7](#step-7-create-the-pst-import-mapping-file) .

Помещение почтового ящика на удержание хранения позволит получить времени владелец почтового ящика для управления эти сообщения только что импортированный или присвойте времени для изменения параметров хранения для почтового ящика. ## <a name="step-1-set-up-azure-rights-management-for-pst-import"></a>Действие 1. Настройте службу управления правами Azure для импорта PST-файлов. 

Импорт PST-файлов с помощью шифрования функциональные возможности, предоставляемые службой управления правами Azure (Azure RMS) в Office 365. Это позволяет для шифрования файлов PST-файлов перед передачей их в Office 365. 
  
Настройка службы управления правами Azure для импорта PST-файлов состоит из трех этапов:
  
- [Активация службы управления правами Azure](#activate-azure-rms)
    
- [Настройка службы управления правами в Exchange Online](#configure-rms-in-exchange-online)
    
- [Установка клиента службы управления правами Active Directory](#install-the-active-directory-rms-client)
    
### <a name="activating-azure-rms"></a>Активация службы управления правами Azure

По умолчанию отключен Azure службы управления правами, но вы или другой администратора в вашей организации могут активировать его. Следуйте инструкциям на [Активация службы управления правами Azure](https://docs.microsoft.com/azure/information-protection/deploy-use/activate-service) для установки и активации управления цифровыми правами Azure.
  
### <a name="configuring-rms-in-exchange-online"></a>Настройка службы управления правами в Exchange Online

После активирован службы управления правами, следующим шагом является установите копирование управления правами на доступ к данным (IRM) в Exchange Online для использования службы управления правами Azure. Дополнительные сведения можно [Настроить IRM так, чтобы использовать службу управления правами Azure](https://go.microsoft.com/fwlink/p/?LinkId=394816).
  
1. [Подключение к Exchange Online с помощью удаленной оболочки PowerShell](https://go.microsoft.com/fwlink/p/?LinkId=396554 ).
    
2. Чтобы задать URL-адрес для общего доступа к ключам RMS, выполните указанную ниже команду.
    
    ```
    Set-IRMConfiguration -RMSOnlineKeySharingLocation <RMS key sharing location>
    ```

    Чтобы определить правильное расположение для общего доступа к ключам RMS для расположения вашей организации, воспользуйтесь таблицей ниже.
    
    |**Расположение**|**Расположение для общего доступа к ключам RMS**|
    |:-----|:-----|
    |Северная Америка  <br/> | `https://sp-rms.na.aadrm.com/TenantManagement/ServicePartner.svc` <br/> |
    |Европейский Союз  <br/> | `https://sp-rms.eu.aadrm.com/TenantManagement/ServicePartner.svc` <br/> |
    |Азия  <br/> | `https://sp-rms.ap.aadrm.com/TenantManagement/ServicePartner.svc` <br/> |
    |Южная Америка  <br/> | `https://sp-rms.sa.aadrm.com/TenantManagement/ServicePartner.svc` <br/> |
    |Office 365 для государственных учреждений (облако сообщества госучреждений)  <br/> | `https://sp-rms.govus.aadrm.com/TenantManagement/ServicePartner.svc`<sup>1</sup> <br/> |
   
    > [!NOTE]
    > <sup>1</sup> только те клиенты, которые приобрели Office 365 для государственных учреждений (облако сообщества госучреждений) следует использовать это расположение для обмена ключами RMS. 
  
    Например эта команда настраивает ключам RMS Online, поиск расположения в Exchange Online для клиента, расположенной в Северной Америке.
    
    ```
    Set-IRMConfiguration -RMSOnlineKeySharingLocation "https://sp-rms.na.aadrm.com/TenantManagement/ServicePartner.svc"
    ```

3. Выполните следующую команду, чтобы импортировать доверенный домен публикаций (TPD) из службы RMS Online в организации Office 365. 
    
    ```
    Import-RMSTrustedPublishingDomain -RMSOnline -Name "RMS Online"
    ```

    Доверенный домен публикации содержит параметры, необходимые для использования функций RMS в организации.  
    
4. Выполните следующую команду, чтобы включить IRM для организации Office 365.
    
    ```
    Set-IRMConfiguration -InternalLicensingEnabled $true
    ```

### <a name="installing-the-active-directory-rms-client"></a>Установка клиента службы управления правами Active Directory

В этом разделе остается Загрузите клиент службы управления правами (RMS) 2.1. Это программное обеспечение помогает защитить доступ к службы управления правами Azure и обеспечивает защиту информации, проходящие через приложения, использующие Azure RMS. Установка клиента службы управления правами на том же компьютере, который используется для шифрования и загрузки файлов PST-файлов на шаге 5.

## <a name="step-2-generate-an-encryption-key-for-pst-import"></a>Действие 2. Создайте ключ шифрования для импорта PST-файлов.

После настройки службы управления правами Azure следующим шагом является создание ключа шифрования (называемые симметричный ключ), который будет использоваться для шифрования файлов PST-файлов, которые передаются в Office 365. Будем делать это путем добавления службы Импорт PST-файлов в качестве службы участника в Azure Active Directory. Добавление этого приложения как участников служб позволяет службе Импорт PST-файлов для проверки подлинности непосредственно в Azure Active Directory при отправке зашифрованных файлов PST-файлов в расположение хранилища Azure на шаге 5.
  
1. Запустите модуля Azure Active Directory для Windows PowerShell.
    
2. Выполните указанную ниже команду, чтобы подключиться к службе Microsoft Online.
    
    ```
    Connect-MsolService
    ```

3. Введите учетные данные для учетной записи администратора в организации Office 365 и нажмите кнопку **ОК**.
    
4. Чтобы создать ключ шифрования (симметричный ключ), выполните указанную ниже команду. Вы сделаете это, создав субъект шифрования PST-файлов.
    
    ```
    New-MsolServicePrincipal -DisplayName PstEncryptionPrincipal
    ```

    Система отобразит симметричный ключ и свойства нового субъекта шифрования PST-файлов.
    
    ![Копирование и сохранение отображаемого симметричного ключа](media/0003fdea-dace-41d2-b49d-f5c98c6230ca.png)
  
5. Скопируйте симметричный ключ в текстовый файл или в файл Word. Как было сказано выше, необходимо предпринять меры для защиты этого файла. Так как это единственный раз, когда отображается симметричный ключ, можно на всякий случай сделать снимок этого окна и сохранить его в вышеупомянутый файл.  
    
    > [!IMPORTANT]
    > После создания субъекта шифрования PST-файлов вам не удастся получить симметричный ключ с помощью командлета **Get-MsolServicePrincipal**. Поэтому важно сохранить ключ. 
  
Будьте Azure модуль Active Directory для Windows PowerShell открыто и подключено к службе Microsoft Online. В это окно на следующем этапе будет запуск команды.

## <a name="step-3-obtain-rms-tenant-id-and-licensing-url"></a>Шаг 3: Получение идентификатора клиента службы управления правами и лицензирования URL-адрес

Следующим шагом является получение клиента идентификатор и лицензирования URL-адрес расположения для службы управления правами Azure для вашей организации. Скопируйте и сохраните эти сведения в тот же файл, содержащий симметричного ключа в шаге 2. Идентификатора и URL-адрес будет использоваться на шаге 5 для шифрования файлов PST-файлов.
  
1. В Azure Active Directory модуля для Windows PowerShell (подключенный к службе Microsoft Online) выполните следующую команду, чтобы подключиться к службе управления правами Azure в организации Office 365.
    
    ```
    Connect-AadrmService 
    ```

2. Введите учетные данные для учетной записи администратора в организации Office 365 и нажмите кнопку **ОК**.
    
3. Выполните следующую команду, чтобы отображать идентификатор клиента службы управления правами Azure в организации Office 365.
    
    ```
    Get-AadrmConfiguration | FL BPOSId
    ```

    Скопируйте и сохраните значение `BPOSId` свойства. 
    
4. Выполните следующую команду, чтобы отобразить лицензирования расположение службы управления правами Azure.
    
    ```
    Get-AadrmConfiguration | FL LicensingIntranetDistributionPointUrl
    ```

    Скопируйте и сохраните значение `LicensingIntranetDistributionPointUrl` свойства. 

## <a name="step-4-download-the-pst-import-tools-and-copy-the-sas-url"></a>Шаг 4: Загрузите средства импорта PST-файлов и скопируйте URL-адрес SAS

Теперь, настройки службы управления правами Azure и получить идентификаторы, необходимые для шифрования файлов PST-файлов, следующим шагом является загрузка и установка средств, которые будут выполняться на шаге 5 для шифрования и PST отправляемых файлов в Office 365. Эти средства, Azure AzCopy средства и средства шифрования данных Office 365. Также будет скопируйте URL-адрес SAS для вашей организации. Этот URL-адрес состоит из URL-адрес сетевого расположения хранилища Azure в облаке Майкрософт для вашей организации и общего доступа к подписи (SAS) ключ. Этот раздел содержит необходимые разрешения для загрузки файлов PST места хранения Azure. Сохраните его в тот же файл, что сведения для копирования в шаге 2 и шаг 3. Как было сказано ранее примите предварительные меры для защиты URL-адрес SAS. 
  
> [!IMPORTANT]
> Необходимо использовать AzCopy Azure версии 5.0 успешно Отправка PST-файлов в расположение хранилища Azure. Новая версия средства AzCopy не поддерживаются для импорта PST-файлов в Office 365. Убедитесь, что загрузить средство AzCopy со страницы **загрузки файлов по сети** с помощью процедур на этом этапе. 
  
1. Последовательно выберите пункты [https://protection.office.com](https://protection.office.com).
    
2. Войдите в Office 365 с помощью учетных данных для учетной записи администратора в организации Office 365.
    
3. В левой области нажмите кнопку **управления данными** и нажмите кнопку **Импорт**.
    
4. На странице **Импорт** щелкните **Перейти к службе импорта**.
    
5. На странице **Импорт данных в Office 365** нажмите кнопку **Создать задание** ![добавить значок](media/ITPro-EAC-AddIcon.gif)и нажмите кнопку **Отправить сообщения электронной почты (PST-файлы)**.

## <a name="step-5-encrypt-and-upload-your-pst-files-to-office-365"></a>Шаг 5: Шифрования и отправить файлы PST-файлов в Office 365

После завершения шаг 1 – 4 шаг можно использовать средство O365ImportTool.exe для шифрования и загрузки файлов PST-файлов в Office 365. Это средство шифрует PST-файлов и затем загружает и сохраняет их в Azure хранилище в облаке Майкрософт. Чтобы выполнить это действие, PST-файлов обязательно должна находиться в общей папке или на файловом сервере в вашей организации. Это называется исходного каталога в следующей процедуре. Каждый раз при запуске средства O365ImportTool.exe будет можно указать разные исходного каталога. 
  
1. Откройте окно командной строки на локальном компьютере.
    
2. Перейдите в каталог, в котором вы установили средство O365ImportTool.exe в действии 4.
    
3. Выполните следующую команду для шифрования и загрузки файлов PST-файлов в Office 365.
    
    ```
    O365ImportTool.exe /srcdir:<Location of PST files> /protect-rmsserver:<RMS licensing location> /protect-tenantid:<BPOSId> /protect-key:<Symmetric key> /transfer:upload /upload-dest:<Network upload URL> /upload-destSAS:<SAS key>
    ```

## <a name="step-8-create-a-pst-import-job-in-office-365"></a>Действие 8. В Office 365 создайте задание на импорт PST-файлов.

Последним шагом является создание задание импорта PST-файлов в службе импорта в Office 365. Как объяснялось ранее будут отправлять файл сопоставления Импорт PST-файлов, созданный на шаге 7. После создания нового задания импорта службы будет использовать информацию в файл сопоставления для отмены-шифрования и импорт PST-файлов (, загруженные в Office 365 на шаге 5) для указанного почтового ящика пользователя. 
  
1. Последовательно выберите пункты [https://protection.office.com](https://protection.office.com).
    
2. Войдите в Office 365 с помощью учетных данных для учетной записи администратора в организации Office 365.
    
3. В левой области нажмите кнопку **управления данными** и нажмите кнопку **Импорт**.
    
4. На странице **Импорт** щелкните **Перейти к службе импорта**.
    
5. На странице **Импорт данных в Office 365** нажмите кнопку **Создать задание**![добавить значок](media/ITPro-EAC-AddIcon.gif)и нажмите кнопку **Отправить сообщения электронной почты (PST-файлы)**.
    
6. На странице **Отправка файлов по сети** установите флажки **Отправка файлов завершена** и **У меня есть доступ к файлу сопоставления**, а затем щелкните **Далее**.  
    
7. Введите имя для задания на импорт PST-файлов и щелкните **Далее**.
    
8. Нажмите кнопку **Добавить** ![добавить значок](media/ITPro-EAC-AddIcon.gif) для выберите сопоставление PST-файл, созданный на шаге 7. 
    
9. Когда имя CSV-файла появится в списке, выберите его и щелкните **Проверить**, чтобы проверить CSV-файл на наличие ошибок.  
    
    > [!NOTE]
    > В предыдущем описано, когда зашифрованные PST-файлов, `.pfile` расширение добавляется к имени файла PST-файлов. Необходимо добавить `.pfile` расширение имени файла PST-файлов в CSV-файл. Если вы не завершится с ошибкой проверки в CSV-файле. 
  
    Чтобы можно было создать задание на импорт PST-файлов, необходимо успешно выполнить проверку CSV-файла. Если не удалось выполнить проверку, в столбце **Состояние** щелкните ссылку **Недопустимый**. Откроется копия файла сопоставления для импорта PST-файлов. В этом файле для каждой строки со сбоем будет соответствующее сообщение об ошибке. 
    
10. После успешной проверки файла сопоставления для импорта PST-файлов прочитайте документ с условиями, а затем установите флажок.
    
11. Чтобы отправить задание, щелкните **Готово**. 
    
    Задание отображается в списке заданий Импорт PST-файлов на странице **Импорт данных в Office 365** . 
    
12. Выберите задание и нажмите кнопку **Обновить**![значок обновления](media/O365-MDM-Policy-RefreshIcon.gif) требуется обновить сведения о состоянии, который отображается в области сведений. 
    
13. В области сведений щелкните **Просмотреть сведения**, чтобы получить последние сведения о состоянии выбранного задания. 
 
## <a name="more-information"></a>Дополнительные сведения

- Почему Импорт PST-файлов в Office 365?
    
  - Рекомендуется миграция электронной почты вашей организации в Office 365.
    
  - Это поможет обеспечить соответствие требованиям в организации, позволяя:
    
  - включить архивные почтовые ящики, чтобы обеспечить пользователям дополнительное место хранения в почтовом ящике;
    
  - ставить почтовые ящики на удержание, чтобы сохранить их содержимое;
    
  - использовать средства обнаружения электронных данных для поиска содержимого в почтовых ящиках;
    
  - применять политики хранения для управления сроками хранения содержимого почтовых ящиков;
    
  - Поиск в журнале аудита Office 365 для событий, связанных с почтовых ящиков.
    
  - Он обеспечивает защиту от потери данных. PST-файлов, которые требуется импортировать в почтовые ящики Office 365 наследовать функции высокой доступности Exchange Online, в отличие от хранения данных на компьютере пользователя.
    
  - Данные доступны пользователю на любых устройствах, так как они хранятся в облаке.

- Как объяснялось ранее службы Office 365 Import включает удержания хранения, Настройка (неограниченное время) после импорта PST-файлов в почтовый ящик. Это означает, что свойство *RentionHoldEnabled* задано значение `True` , чтобы не будут обрабатываться политику хранения, назначенные в почтовый ящик. Это дает время владельца почтового ящика для управления сообщениями только что импортированный, предотвращая выполнение удаления или политики архивации, удаление или архивацию старых сообщений. Ниже приведены некоторые действия, которые нужно выполнить для управления этой удержание. 
    
  - После определенного периода времени, вы можете отключить удержания хранения, выполнив `Set-Mailbox -RetentionHoldEnabled $false` команды. Сведения содержатся в разделе [почтовый ящик на хранение хранение](https://go.microsoft.com/fwlink/p/?LinkId=544749).
    
  - Удержание хранения можно настроить таким образом, чтобы он выключен на некоторые даты в будущем. Это можно сделать, выполнив `Set-Mailbox -EndDateForRetentionHold <date>` команды. Например, если предполагается, что текущая дата — 1 июля 2016 и требуется удержания хранения, отключена в течение 30 дней, вы выполните следующую команду: `Set-Mailbox -EndDateForRetentionHold 8/1/2016`. В этом сценарии будет оставить это свойство *RentionHoldEnabled* , задайте значение `True`. Для получения дополнительных сведений см [Set-Mailbox](https://go.microsoft.com/fwlink/p/?LinkId=150317).
    
  - Можно изменить параметры политики хранения, назначенная в почтовый ящик, чтобы старые элементы, которые были импортированы не будут сразу же удаления или перемещения в архивный почтовый ящик пользователя. Например может увеличить срок хранения для удаления или архива политики, назначенной для почтового ящика. В этом сценарии может отключить удержания хранения для почтового ящика после изменения параметров политики хранения. Для получения дополнительных сведений см [набор архивации и удаления политики почтовых ящиков в организации Office 365](set-up-an-archive-and-deletion-policy-for-mailboxes.md).
