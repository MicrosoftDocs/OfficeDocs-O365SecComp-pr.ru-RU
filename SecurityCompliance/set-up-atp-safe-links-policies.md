# <a name="set-up-office-365-atp-safe-links-policies"></a>Настройка политик Office 365 ATP безопасных ссылки

> [!IMPORTANT]
> Эта статья предназначена для предприятий. Если вы являетесь домашних пользователей, Дополнительные сведения о безопасном ссылки в Outlook, см [Advanced Outlook.com](https://support.office.com/article/advanced-outlook-com-security-for-office-365-subscribers-882d2243-eab9-4545-a58a-b36fee4a46e2).

[Безопасные ссылок анализа](atp-safe-links.md), компонента [Защиты расширенного Threat Office 365](office-365-atp.md) (ATP), могут помочь защитить организацию от вредоносного ссылок, используемых в фишинга и другие атаки. Если у вас есть необходимые [разрешения для системы безопасности Office 365 &amp; центре соответствия требованиям](permissions-in-the-security-and-compliance-center.md), можно настроить политики ATP безопасных ссылки помогут вам, когда пользователи щелкают веб-адреса (URL-адреса), защищенных вашей организации. Можно настроить политиках безопасных ссылок анализа для проверки URL-адреса электронной почты и URL-адреса в документах Office.
  
[Новые возможности постоянно добавляются для анализа](office-365-atp.md#new-features-are-continually-being-added-to-atp). При добавлении новых функций, может потребоваться внести изменения в существующих политик безопасных ссылок анализа.

## <a name="what-to-do"></a>Действия 
  
1. [Просмотрите необходимого программного обеспечения](#review-the-prerequisites).
    
2. [Просмотреть и изменить ссылки безопасных ATP политики по умолчанию, которая применяется для всех](#define-an-atp-safe-links-policy-that-applies-to-everyone). Например можно [настроить настраиваемый список заблокированных URL-адреса для безопасных ссылок анализа](set-up-a-custom-blocked-urls-list-wtih-atp.md).
    
3. [Добавление или редактирование политик для конкретных электронной почты получателей](#add-a-policy-for-specific-email-recipients), включая [настройку настраиваемый «Не rewrite» URL-адреса, список для безопасных ссылок анализа](set-up-a-custom-do-not-rewrite-urls-list-with-atp.md).
    
4. [Дополнительные сведения о параметрах политики безопасных ссылок анализа](#learn-about-atp-safe-links-policy-options) (в этой статье) включая параметры для изменения.
    
## <a name="step-1-review-the-prerequisites"></a>Шаг 1: Просмотрите необходимого программного обеспечения

- Убедитесь в том, что в вашей организации есть [Защиту от угроз для Office 365 расширенный](office-365-atp.md).
    
- Убедитесь в том, что имеются необходимые разрешения. (Или изменение) ATP политик, вам должна быть назначена одна из ролей описаны в следующей таблице: <br>

    |Роль  |Где и как назначен  |
    |---------|---------|
    |Глобальный администратор Office 365 |Человека, который регистрируется приобрести Office 365 — это глобального администратора по умолчанию. ( [Роли администраторов об Office 365](https://docs.microsoft.com/office365/admin/add-users/about-admin-roles) для получения дополнительных сведений см.)         |
    |администратор безопасности (Security Administrator). |Центр администрирования Azure Active Directory ([https://aad.portal.azure.com](https://aad.portal.azure.com))|
    |Управление Exchange Online организацией |Центр администрирования Exchange ([https://outlook.office365.com/ecp](https://outlook.office365.com/ecp)) <br>или <br>  Командлеты PowerShell (см. [Exchange Online PowerShell](https://docs.microsoft.com/powershell/exchange/exchange-online/exchange-online-powershell?view=exchange-ps)) |

    Чтобы узнать больше о роли и разрешения, обратитесь к разделу [разрешения безопасности Office 365 &amp; центре соответствия требованиям](permissions-in-the-security-and-compliance-center.md).

- Убедитесь в том, что клиенты Office настроены на использование [Проверки подлинности на современном](https://docs.microsoft.com/office365/enterprise/modern-auth-for-office-2013-and-2016) (используется для защиты ATP безопасных ссылок в документах Office).
    
- [Дополнительные сведения о параметрах политики безопасных ссылок анализа](#learn-about-atp-safe-links-policy-options) (в этой статье). 

- Разрешить до 30 минут для новых или обновленных политики для распространения для всех центров обработки данных Office 365.
    
## <a name="step-2-define-or-review-the-atp-safe-links-policy-that-applies-to-everyone"></a>Шаг 2: Определение (или просмотрите) политики безопасных ссылок анализа, которое применяется ко всем пользователям

При наличии [Защиты расширенного Threat Office 365](office-365-atp.md)будут иметь политику безопасных ссылок анализа по умолчанию, которое применяется ко всем пользователям в организации. Убедитесь в том просмотреть и при необходимости изменить политики по умолчанию.
  
1. Последовательно выберите пункты [https://protection.office.com](https://protection.office.com) и войдите с учетной записи рабочего или школы. 
    
2. Выберите в левой панели навигации в разделе **Threat management** **политики \> ** **Безопасных ссылки**.
    
3. Раздел **политики, которые применяются во всей организации** выберите вариант **по умолчанию**и выберите команду **Изменить** ("Изменить" представляет Карандаш).<br/>![Нажмите кнопку Изменить для изменения политики по умолчанию для защиты безопасных ссылки](media/d08f9615-d947-4033-813a-d310ec2c8cca.png)
  
4. В разделе **Блокировка следующие URL-адреса** укажите один или несколько URL-адресов, которые вы хотите запретить сотрудникам вашей организации из странице веб-узла. (См. [Настройка настраиваемого заблокированных список URL-адресов с помощью безопасных ссылок анализа](set-up-a-custom-blocked-urls-list-wtih-atp.md)).
    
5. В разделе **Параметры, которые применяются к контенту, за исключением электронной почты** установите (или снимите) параметры, которые вы хотите использовать. (Рекомендуется выбрать все параметры.) 
    
6. Нажмите кнопку **Сохранить**.
    
## <a name="step-3-add-or-edit-atp-safe-links-policies-that-apply-to-specific-email-recipients"></a>Шаг 3: Добавление (или изменение) безопасных ссылок анализа политик, которые применяются к получателям определенных сообщений электронной почты

После просмотра (или изменять) безопасных ссылок анализа политики по умолчанию, которое применяется ко всем пользователям, следующим шагом является определение дополнительных политик, которые будут применяться к определенным получателям. К примеру можно указать исключения в политику по умолчанию, определив Дополнительные политики. 
  
1. Последовательно выберите пункты [https://protection.office.com](https://protection.office.com) и войдите с учетной записи рабочего или школы. 
    
2. В левой панели навигации в разделе **Управление угроз**выберите **политику**.
    
3. Выбор **надежных ссылки**.
    
4. В разделе **политики, которые применяются к определенным получателям** , нажмите кнопку **Создать** (новой кнопки имеет следующий вид: знак плюс ( **+**)).<br/>![Нажмите кнопку Создать, чтобы добавить политику безопасных ссылки для конкретных электронной почты получателей](media/01073f42-3cec-4ddb-8c10-4d33ec434676.png)
  
5. Укажите имя, описание и параметры политики.<br/>**Пример:** Чтобы настроить политику, называется «без прямого щелкните через», которая не позволяет пользователям в определенной группой в организации, щелкните через для определенного веб-сайта без защиты ATP безопасных ссылок, можно указать следующие рекомендуемые параметры: 
    
  - В поле **имя** введите нет прямого Пройдите по.
    
  - В поле **Описание** введите описание как запрещает людей в определенных группах возникающий через веб-сайта, без проверки безопасных ссылок анализа.
    
  - **В разделе **Выберите действие** выберите команду.**
    
  - Выберите **Использовать безопасные вложения для сканирования загружаемые материалы**.
    
  - Если этот параметр доступен, выберите **Применить безопасных ссылки для сообщений, отправленных в рамках организации**.
    
  - Выберите **не разрешает пользователю перейдите по исходный URL-адрес**.
    
  - (Необязательно) В разделе **не переопределить следующие URL-адреса** укажите один или несколько URL-адресов, которые должны рассматриваться как безопасные для вашей организации. (См. [Настройка настраиваемого «Не rewrite» URL-адреса списка с помощью надежных ссылок анализа](set-up-a-custom-do-not-rewrite-urls-list-with-atp.md))
    
  - В разделе **Применить к** выберите **получатель входит в состав**и выберите группы, которые необходимо включить в политике. Последовательно выберите пункты **Добавить**и затем нажмите **кнопку ОК**.
    
6. Нажмите кнопку **Сохранить**.
    
## <a name="step-4-learn-about-atp-safe-links-policy-options"></a>Шаг 4: Сведения о безопасном ссылки ATP политики

Настройка или изменение политик ATP безопасных ссылок, будут видеть доступно несколько вариантов. В случае, если вы хотите узнать о, эти параметры доступны, следующей таблице описывается каждый из них и влияние. Помните, что существует два основных типа политик безопасных ссылок анализа для задания или изменения:
- [Политика по умолчанию](#default-policy-options) , которое применяется ко всем пользователям 
- Дополнительные [политики, которые определены для конкретных получателей](#policies-that-apply-to-specific-email-recipients) 

### <a name="default-policy-options"></a>Параметры политики по умолчанию

Параметры политики по умолчанию применяются к всем пользователям в организации.

|Этот параметр  |Обеспечивает следующее  |
|---------|---------|
| **Блокировать следующие URL-адреса** <br/>    | Позволяет организации имеют настраиваемый список URL-адресов, которые блокируются автоматически. Когда пользователи щелкают URL-адрес в этом списке, они будете перенаправлены на [страницу предупреждение](atp-safe-links-warning-pages.md) , описывается, почему заблокировано URL-адрес.<br/> Для получения дополнительных сведений, см. [Настройка настраиваемого заблокированных список URL-адресов с помощью безопасных ссылок анализа      |
| **Office 365 профессиональный плюс, Office для iOS и Android (en)** <br/>    | Если выбран этот параметр, безопасные ссылки ATP защиты применяется к URL-адреса в документах, откройте в Office 365 профессиональный плюс (Word, Excel и PowerPoint в Windows или Mac OS), документов Office на операций ввода-вывода или устройства Android, 2016 Visio в ОС Windows и Office Online (для Word Сети, сети PowerPoint, Excel Online и OneNote Online), при условии, что пользователь вошел в Office 365. <br/><br/>Если отображаются только **2016 Office на Windows**, затем обновления компонента не достигнуто среде Office 365 еще (и они готовится к выпуску). До тех пор безопасных ссылок анализа защиты применяется к Word 2016, Excel 2016, PowerPoint 2016 или Visio 2016, запущенные на Windows.            |
| **Не отслеживать, когда пользователи щелкают ATP безопасных ссылки** <br/>  | Если выбран этот параметр, выберите данные для URL-адреса в документы Word, Excel, PowerPoint и Visio не сохраняется.  <br/> |
|**Не позволяйте щелкните через ATP безопасных ссылки на исходный URL-адрес** <br/> |Если выбран этот параметр, пользователи не может продолжать процесс прошлые [Предупреждение страницы](atp-safe-links-warning-pages.md) на URL-адрес, который определяется как вредоносных.  <br/> |

### <a name="policies-that-apply-to-specific-email-recipients"></a>Политики, которые применяются к получателям определенных сообщений электронной почты

|Этот параметр  |Обеспечивает следующее  |
|---------|---------|
|**Выкл.** <br/> |Не сканирует URL-адресов в сообщениях электронной почты.  <br/> Позволяет определить правило исключений, таких как правило, которое не сканирует URL-адресов в сообщениях электронной почты для определенной группе получателей.  <br/> |
|**На** <br/> |Перезапись URL-адреса для маршрута пользователей с помощью надежных ссылки ATP защиты при нажатии URL-адресов в сообщениях электронной почты.  <br/> Проверяет URL-адрес при нажатии этой кнопки в списке заблокированных или вредоносного URL-адресов.  <br/> |
|**Используйте безопасные вложения для проверки загружаемые материалы** <br/> |Если выбран этот параметр, URL-адресов, которые указывают на загружаемые материалы проверяются.  <br/> |
|**Применение безопасных ссылки для сообщений, отправленных в рамках организации** <br/> | Когда этот параметр, доступные и выбранные, защиты безопасных ссылок анализа применяется к электронной почты, сообщения, отправленные между людьми в организации, предоставленных учетных записей электронной почты, размещенные в Office 365.  <br/> |
|**Не отслеживать пользователь нажимает кнопку** <br/> |Если выбран этот параметр, выберите данные для URL-адреса электронной почты от внешних отправителей не сохраняется. Щелкните URL-адрес, отслеживания для ссылок в пределах сообщений электронной почты, отправленных в рамках организации в настоящее время не поддерживается.  <br/> |
|**Не разрешать пользователям перейдите по исходный URL-адрес** <br/> |Если выбран этот параметр, пользователи не может продолжать процесс прошлые [Предупреждение страницы](atp-safe-links-warning-pages.md) на URL-адрес, который определяется как вредоносных.  <br/> |
|**Не переопределить следующие URL-адреса** <br/> |Покидает URL-адреса как есть. Сохраняет настраиваемый список безопасных URL-адресов, которым не требуется сканирования для определенной группе получателей электронной почты в вашей организации.  См [в настраиваемых «Не rewrite» URL-адреса, список с помощью ATP безопасных ссылки](set-up-a-custom-do-not-rewrite-urls-list-with-atp.md) для получения дополнительных сведений, включая последние изменения для поддержки для подстановочные знаки звездочки (\*).<br/> |
