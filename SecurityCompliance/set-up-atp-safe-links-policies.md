# <a name="set-up-office-365-atp-safe-links-policies"></a>Настройка политик Office 365 ATP безопасных ссылки

[Безопасные ссылок анализа](atp-safe-links.md) , компонента [Защиты расширенного Threat Office 365](office-365-atp.md) (ATP), могут помочь защитить организацию от вредоносного ссылок, используемых в фишинга и другие атаки. Если у вас есть необходимые [разрешений, назначенных безопасности Office 365 &amp; центре соответствия требованиям](permissions-in-the-security-and-compliance-center.md), можно настроить политики ATP безопасных ссылки помогут вам, когда пользователи щелкают веб-адреса (URL-адреса), защищенных вашей организации. Можно настроить политиках безопасных ссылок анализа для проверки URL-адреса электронной почты и URL-адреса в документах Office.
  
[Новые возможности постоянно добавляются в безопасном ссылки анализа](atp-safe-links.md#new-features-are-continually-being-added-to-atp-safe-links). При добавлении новых функций, может потребоваться внести изменения в существующих политик безопасных ссылок анализа.

## <a name="what-to-do"></a>Действия 
  
1. [Просмотрите необходимого программного обеспечения](#review-the-prerequisites).
    
2. [Просмотреть и изменить ссылки безопасных ATP политики по умолчанию, которая применяется для всех](#define-an-atp-safe-links-policy-that-applies-to-everyone). Например можно [настроить настраиваемый список заблокированных URL-адреса для безопасных ссылок анализа](set-up-a-custom-blocked-urls-list-wtih-atp.md).
    
3. [Добавление или редактирование политик для конкретных электронной почты получателей](#add-a-policy-for-specific-email-recipients), включая [настройку настраиваемый «Не rewrite» URL-адреса, список для безопасных ссылок анализа](set-up-a-custom-do-not-rewrite-urls-list-with-atp.md).
    
4. [Дополнительные сведения о параметрах политики безопасных ссылок анализа](#learn-about-atp-safe-links-policy-options) (в этой статье) включая параметры для изменения
    
## <a name="step-1-review-the-prerequisites"></a>Шаг 1: Просмотрите необходимого программного обеспечения

- Убедитесь в том, что в вашей организации есть [Защиту от угроз для Office 365 расширенный](office-365-atp.md).
    
- Убедитесь, что необходимые разрешения для задания или изменения политики анализа. Просмотреть [разрешения безопасности Office 365 &amp; центре соответствия требованиям](permissions-in-the-security-and-compliance-center.md).
    
- [Дополнительные сведения о параметрах политики безопасных ссылок анализа](#learn-about-atp-safe-links-policy-options) (в этой статье). 

- Убедитесь в том, что клиенты Office настроены на использование [Проверки подлинности на современном](https://docs.microsoft.com/office365/enterprise/modern-auth-for-office-2013-and-2016).
    
- Разрешить до 30 минут для новых или обновленных политики для распространения для всех центров обработки данных Office 365.
    
## <a name="step-2-define-or-review-the-atp-safe-links-policy-that-applies-to-everyone"></a>Шаг 2: Определение (или просмотрите) политики безопасных ссылок анализа, которое применяется ко всем пользователям

При наличии дополнительные защиту от угроз в Office 365 для предприятия будут иметь политику безопасных ссылок анализа по умолчанию, которое применяется ко всем пользователям в организации. 
  
1. Последовательно выберите пункты [https://protection.office.com](https://protection.office.com) и войдите с учетной записи рабочего или школы. 
    
2. Выберите в левой панели навигации в разделе **Threat management** **политики \> ** **Безопасных ссылки**.
    
3. Раздел **политики, которые применяются во всей организации** выберите вариант **по умолчанию**и выберите команду **Изменить** ("Изменить" представляет Карандаш). 
    
    ![Нажмите кнопку Изменить для изменения политики по умолчанию для защиты безопасных ссылки](media/d08f9615-d947-4033-813a-d310ec2c8cca.png)
  
4. В разделе **Блокировка следующие URL-адреса** укажите один или несколько URL-адресов, которые вы хотите запретить сотрудникам вашей организации из странице веб-узла. (См. [Настройка настраиваемого заблокированных список URL-адресов с помощью безопасных ссылок анализа](set-up-a-custom-blocked-urls-list-wtih-atp.md)).
    
5. В разделе **Параметры, которые применяются к контенту, за исключением электронной почты** установите (или снимите) параметры, которые вы хотите использовать. (Рекомендуется выбрать все параметры.) 
    
6. Нажмите **Сохранить**.
    
## <a name="step-3-add-or-edit-atp-safe-links-policies-that-apply-to-specific-email-recipients"></a>Шаг 3: Добавление (или изменение) безопасных ссылок анализа политик, которые применяются к получателям определенных сообщений электронной почты

После просмотра политики безопасных ссылок анализа для всех пользователей, рекомендуется определить дополнительные политики для определенных групп получателям сообщений электронной почты. Это позволяет указать исключения в политику по умолчанию. 
  
1. Последовательно выберите пункты [https://protection.office.com](https://protection.office.com) и войдите с учетной записи рабочего или школы. 
    
2. В левой панели навигации в разделе **Управление угроз**выберите **политику**.
    
3. Выбор **надежных ссылки**.
    
4. В разделе **политики, которые применяются к определенным получателям** , нажмите кнопку **Создать** (новой кнопки имеет следующий вид: знак плюс ( **+**)).<br/>![Нажмите кнопку Создать, чтобы добавить политику безопасных ссылки для конкретных электронной почты получателей](media/01073f42-3cec-4ddb-8c10-4d33ec434676.png)
  
5. Укажите имя, описание и параметры политики.<br/>**Пример:** Чтобы настроить политику, называется «без прямого щелкните через», которая не позволяет пользователям в определенной группой в организации, щелкните через для определенного веб-сайта без защиты ATP безопасных ссылок, можно указать следующие рекомендуемые параметры: 
    
  - В поле **имя** введите нет прямого Пройдите по.
    
  - В поле **Описание** введите описание как запрещает людей в определенных группах возникающий через веб-сайта, без проверки безопасных ссылок анализа.
    
  - **В разделе **Выберите действие** выберите команду.**
    
  - Выберите **Использовать безопасные вложения для сканирования загружаемые материалы**.
    
  - Если этот параметр доступен, выберите **Применить безопасных ссылки для сообщений, отправленных в рамках организации**.
    
  - Выберите **не разрешает пользователю перейдите по исходный URL-адрес**.
    
  - (Необязательно) В разделе **не переопределить следующие URL-адреса** укажите один или несколько URL-адресов, которые должны рассматриваться как безопасные для вашей организации. (См. [Настройка настраиваемого «Не rewrite» URL-адреса списка с помощью надежных ссылок анализа](set-up-a-custom-do-not-rewrite-urls-list-with-atp.md))
    
  - В разделе **Применить к** выберите **получатель входит в состав**и выберите группы, которые необходимо включить в политике. Последовательно выберите пункты **Добавить**и затем нажмите **кнопку ОК**.
    
6. Нажмите **Сохранить**.
    
## <a name="step-5-learn-about-atp-safe-links-policy-options"></a>Шаг 5: Сведения о безопасном ссылки ATP политики

Настройка или изменение политик ATP безопасных ссылок, будут видеть доступно несколько вариантов. В случае, если вы хотите узнать о, эти параметры доступны, следующей таблице описывается каждый из них и влияние. 

Существует два основных типа политик для задания или изменения: политику по умолчанию, которое применяется ко всем пользователям и дополнительных политик, которые определены для конкретных получателей. Те, отличаются в столбце **для этой политики** в следующей таблице.
  
|**Для этой политики**|**Этот параметр**|**Обеспечивает следующее**|
|:-----|:-----|:-----|
|По умолчанию (после определения политики по умолчанию применяется ко всем пользователям в организации)  <br/> |**Блокировать следующие URL-адреса** <br/> |Позволяет организации имеют настраиваемый список URL-адресов, которые блокируются автоматически. Когда пользователи щелкают URL-адрес в этом списке, они будете перенаправлены на [страницу предупреждение](atp-safe-links-warning-pages.md) , описывается, почему заблокировано URL-адрес.<br/> См [настраиваемых заблокированных список URL-адресов с помощью ATP безопасных ссылки](set-up-a-custom-blocked-urls-list-wtih-atp.md) для получения дополнительных сведений, таких как только что добавлена поддержка до трех подстановочные знаки звездочки (\*).  <br/> |
|По умолчанию  <br/> |**Office 365 профессиональный плюс, Office для iOS и Android (en)** <br/> |Если выбран этот параметр, безопасные ссылки ATP защиты применяется к URL-адреса в документах, откройте в Office 365 профессиональный плюс (Word, Excel и PowerPoint в Windows или Mac OS), документов Office на операций ввода-вывода или устройства Android, 2016 Visio в ОС Windows и Office Online (для Word Сети, сети PowerPoint, Excel Online и OneNote Online), при условии, что пользователь вошел в Office 365. <br/><br/>Если отображаются только **2016 Office на Windows**, затем обновления компонента не достигнуто среде Office 365 еще (и они готовится к выпуску). До тех пор безопасных ссылок анализа защиты применяется к Word 2016, Excel 2016, PowerPoint 2016 или Visio 2016, запущенные на Windows.           |
|По умолчанию  <br/> |**Не отслеживать, когда пользователи щелкают ATP безопасных ссылки** <br/> |Если выбран этот параметр, выберите данные для URL-адреса в документы Word, Excel, PowerPoint и Visio не сохраняется.  <br/> |
|По умолчанию  <br/> |**Не позволяйте щелкните через ATP безопасных ссылки на исходный URL-адрес** <br/> |Если выбран этот параметр, пользователи не может продолжать процесс прошлые [Предупреждение страницы](atp-safe-links-warning-pages.md) на URL-адрес, который определяется как вредоносных.  <br/> |
|Политики, созданные для конкретного электронной почты получателей  <br/> |**Выкл.** <br/> |Не сканирует URL-адресов в сообщениях электронной почты.  <br/> Позволяет определить правило исключений, таких как правило, которое не сканирует URL-адресов в сообщениях электронной почты для определенной группе получателей.  <br/> |
|Политики, созданные для конкретного электронной почты получателей  <br/> |**На** <br/> |Перезапись URL-адреса для маршрута пользователей с помощью надежных ссылки ATP защиты при нажатии URL-адресов в сообщениях электронной почты.  <br/> Проверяет URL-адрес при нажатии этой кнопки в списке заблокированных или вредоносного URL-адресов.  <br/> |
|Политики, созданные для конкретного электронной почты получателей  <br/> |**Используйте безопасные вложения для проверки загружаемые материалы** <br/> |Если выбран этот параметр, URL-адресов, которые указывают на загружаемые материалы проверяются.  <br/> |

Сохраняет настраиваемый список безопасных URL-адресов, которым не требуется сканирования для определенной группе получателей электронной почты в вашей организации.
