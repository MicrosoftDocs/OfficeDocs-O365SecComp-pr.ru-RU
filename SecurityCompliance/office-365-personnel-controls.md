# <a name="office-365-personnel-controls"></a>Элементы управления персоналом в Office 365 

Персонал блокировки, которые — это процесс просмотра и проверка последние поведение и состояние пользователя, является элементом управления важные смягчение ему помешать службы Office 365. Последние поведение не является идеальным прогнозом будущих поведение другого пользователя, она помогает определить потенциальные субъекты очень плохо. Блокировки стандартных корпорации Майкрософт персонал применяется ко всем Microsoft сотрудников, помощника и временного персонала, участвующих в разработки, операции или доставки online services для государственных организаций или коммерческие облачных клиентов. Блокировки стандартов для Национальный облака среды, в которых не обслуживается Microsoft определяются эксплуатационных персонал партнера для каждой конкретной среды.

## <a name="microsofts-personnel-screening-standard"></a>Блокировки стандартный численности специалистов корпорации Майкрософт

Персонал корпорации Майкрософт, блокировки и рекомендации для Office 365 выровненные с корпоративными стандартами корпорации Майкрософт и национального института стандартов и технологий (NIST) элементов управления для блокировки персонала. Microsoft персонала, которым требуется доступ к таким компонентам могут быть блокировки стандартных корпорации Майкрософт персонал:
- Физический доступ к центров обработки данных, совместного расположений, с защитой комнат, вместо, стойки сервера или пограничного сервера сайты, которые обеспечивают инфраструктуры поддержки Интернет-служб для государственных организаций или коммерческие облачных клиентов.
- Логическая доступа государственных организаций или коммерческие облачных данные клиента с помощью определенного управляемые среды.
    - Сетевой доступ управления для устройств и службы, которые транспорта или хранить государственных организаций или коммерческие облачных данные клиента.

Определенные события, связанные с персонала, которые активируют отбора требования относятся:
- Новых сотрудников Майкрософт, расположенные в роли, где отбора — определенные требования.
- Внутренний сотрудников Майкрософт передачи или перемещение к существующей роли, то в настоящее время включает в себя отбора как определенное требование.
- Существующие роли, которые изменяют область действия для включения блокировки как определенное требование.

## <a name="screening-enforcement-criteria"></a>Принудительное применение критериев фильтрации

Чтобы обеспечить доступ к данным клиента или сред, требующих отбора только утвержденные персонала, применяется следующим критериям принудительного применения:

В **Соединенных Штатах Америки облачных средах только**:
- Доступ к данным клиента или сред, которые требуют соблюдения блокировки должен быть разрешен только после завершения adjudication и передаются отбора требования.
- Сотрудников Майкрософт, пользователей, которые больше не требуется доступ к данным клиента или связанных с ними сред должна иметь доступ удалены при покидать Microsoft или перемещения в новую роль.
- Microsoft персонала необходимо оставить промежуточной среды смарт-карты с помощью управления перед выходом из США.

**Национальный облачных средах**:
- Персонал доверенного объекта оператор или данных сторонних производителей несете ответственность за управление и обеспечение доступа для страны облачных средах.

В корпорации Майкрософт облачных служб сред доступ разрешен только на основе роли пользователя и тип данных, как описано в следующей таблице. Полные или неполные персонал расположены за пределами США могут не имеют доступ к данным клиента в облаке США. Доступ к национальный облачных средах ограничен, чтобы сотрудники Майкрософт нет технические доступ к данным клиента или системы, содержащие данные клиента без утверждения с доверенного объекта оператор или данных сторонних производителей.

| Роль | Доступ к данным клиента | Доступ к данным системы |
|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------|---------------------------------|
| Квалифицированные специалисты расположены в США | Это разрешено | Это разрешено |
| Квалифицированные специалисты расположены за пределами США | Не допускается | Это разрешено |
| Международный исключение доступ для сотрудников корпорации Майкрософт — включает логической доступ для сотрудников корпорации Майкрософт, который находится в стране, где государственных организаций или коммерческие данные клиента — статических | Не допускается | Это разрешено |
| Значение без указания единицы персонал (нерастрированных персонала, которые требуют сопровождение квалифицированного персоналом) | Это разрешено авторизации | Это разрешено с надзор за сопровождение |


## <a name="microsoft-pre-employment-screening"></a>Блокировка предварительной занятости Microsoft

Где местным законом позволяет это, отдела безопасности корпорации Майкрософт Global проводит отбора предварительной сотрудников. Это расследования официальную фона, который включает в себя следующим критериям:
- Проверку resume соискателя (например, для полноты и точности)
- Подтверждение квалификации академический и профессиональный
- Исследование потери профессиональный учетные данные
- Проверка за последние три руководству
- Выполняется проверка записи полиции для убедительность felony
- Подтверждение идентификации из идентификации выдается государственных организаций
- При необходимости кредит проверки

Периодически rescreening и/или дополнительные проверки может быть требуется для определенных управления, безопасности или другие роли, включая, но не ограничиваясь на основе США сотрудников в роли, которые требуют доступа к данным клиента. Для временного персонала контракт со сторонних производителей приведены требования к корпорации Майкрософт для блокировки, который должен вестись по сторонних производителей. Для проверки фона компании сторонних производителей ответственность за предоставление Microsoft проверки, что была выполнена проверка фона. Результаты проверки фона обычно происходит получение по электронной почте от сторонних производителей отделу кадров. Международные сотрудников контракт персонала могут быть исключены из фона, блокировки процесс из-за законам стран, запретить фоновую проверку.

## <a name="microsoft-employment-screening"></a>Блокировка занятости Microsoft
Начиная с 2004 г. Корпорация Майкрософт необходимые отдельных пользователей для передачи экран международными записи семь года для felonies и misdemeanors и проверка их образования и опыте, как часть предварительной занятости для сотрудников и помощника по блокировке в Соединенных Штатах Америки.

В Соединенных Штатах Америки, перед назначением любого сотрудника Майкрософт или любой субподрядчика назначенные Майкрософт для предоставления служб, связанных с Office 365 Майкрософт будет проводить и вызвать его субподрядчика провести проверку фона, состоящий из социального страхования номера трассировки и международными записей проверки. Данные из этого проверку используется как коэффициент в принятии решения о найме. Проверка международными записи включает проверку семь летний felony и misdemeanor международными записей Федеральное, состояния и страны записей (при необходимости).

Как условие найма во время приема на работу все сотрудники Майкрософт требуются для входа, связанные с конфиденциальностью и соглашения о неразглашении и убедитесь, что они ознакомились журнал сотрудника Майкрософт.

## <a name="microsoft-cloud-background-check"></a>Microsoft Cloud проверку
Проверьте фона Microsoft Cloud является обязательным для кандидатов на работу как сотрудники, обеспечивающих работу служб, связанных с Office 365, в США. Сотрудники корпорации Майкрософт в Соединенных Штатах Америки с доступом к данным клиента необходимо выполнить процесс Microsoft Cloud фона проверьте, который необходим для всех служб Office 365. За пределами США процесс может отличаться. Например Microsoft Cloud для Германии использует модель утверждения доверенного объекта данных, которая разрабатывалась убедитесь, что доверенного объекта данных (немецкий компании), а не корпорацией Майкрософт, в элементе управления доступа к данным клиента. В облаке Майкрософт в Германии доставляется из центров обработки данных в Германии и центры операции (OC), содержащий технических специалистов доверенного объекта данных также находятся в Германии. Центр обработки данных и производственные объекты OC, используемых, ведутся и управляет доверенного объекта данных.

В следующей таблице приведены проверок, выполняемых в составе Microsoft Cloud фона проверки.

| Блокировки | Описание |
|--------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|
| Страховой номер для поиска | Проверяет допустимость предоставленного номера социального страхования. |
| Проверка данных о совершенных правонарушениях | 7 летний международными записей проверка felony и misdemeanor offenses на уровнях состояний, страны и локальной и при необходимости, Федеральное на уровне. |
| Office список внешних средств управления | Отдел финансов список отдельных пользователей и организаций, с которыми жителей Соединенных Штатах Америки и постоянное жителей не разрешено работать. |
| Бюро отрасли и безопасности списка | Список финансов отдельных пользователей и сущностей лишены участие в операции экспорта. |
| Office зашиты торговые список элементов управления Debarred лиц (*добавлен на 1 июля 2010 г.*) | Список отдела состояние отдельных пользователей и сущностей лишены участие в экспорт действия, связанные с отрасли защиту. |


Результаты из Microsoft Cloud фона проверьте хранятся в базе данных сотрудников, подключенный к системам управления доступа к нашей центра обработки данных. Если Microsoft Cloud фона проверки истечения срока действия и сотрудника не обновлять его, затем доступ к службам Office 365 отозванного и больше не доступен до Microsoft Cloud фона Проверка завершения еще раз. По окончании отношения приема на работу с Microsoft любого существующего центра обработки данных access сразу же был отозван.
