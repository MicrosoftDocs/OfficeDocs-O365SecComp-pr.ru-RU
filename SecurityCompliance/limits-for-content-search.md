# <a name="limits-for-content-search-in-the-office-365-security-amp-compliance-center"></a>Ограничения для поиска контента безопасности Office 365 &amp; центре соответствия требованиям

> [!NOTE]
> Ограничения в данном разделе, отличаются от текущего ограничения для электронного обнаружения на месте в Exchange Online, а также для центра обнаружения электронных данных в SharePoint Online. 
  
Различные ограничения применяются к компонент поиска контента в Office 365 безопасность &amp; центре соответствия требованиям. В этом include поисковые запросы, выполненные на странице **поиска контента** и поиска, связанных с вариантом eDiscovery. Эти ограничения позволяющих обеспечивать состояния и качества служб, предоставляемых организациями Office 365. Также существуют ограничения, связанные с индексирования сообщений электронной почты в Exchange Online для поиска. Нельзя изменять поиска контента или электронной почты, индексирование ограничения, но следует иметь в виду из них, чтобы можно было эти ограничения во внимание при планировании, выполнение и устранение неполадок в поиск.

## <a name="content-search-limits"></a>Ограничения для поиска контента
<a name="searchlimits"> </a>

В следующей таблице перечислены ограничения поиска в системы &amp; центре соответствия требованиям.
  
|**Описание ограничения**|**Ограничение**|
|:-----|:-----|
|Максимальное количество почтовых ящиков или сайты, которые можно искать в одном поиска контента  <br/> |Нет ограничений  <br/> |
|Максимальное количество поисков содержимого, можно запустить в то же время в организации.  <br/> |Нет ограничений  <br/> |
|Максимальное количество поисков содержимого, в то же время можно начать одного пользователя. Обратите внимание на то, что это скорее предела при попытке запуска нескольких поиска с помощью **Get-ComplianceSearch \| ComplianceSearch Пуск** в безопасности и соответствия требованиям центр PowerShell.<br/> |10   <br/> |
|Максимальное число элементов на один почтовый ящик пользователя, которые отображаются на странице предварительного просмотра при предварительном просмотре результатов поиска контента.  <br/> |100  <br/> |
|Максимальное число элементов, обнаруженных в всех почтовых ящиков пользователей, которые отображаются на странице предварительного просмотра при предварительном просмотре результатов поиска контента. Новые элементы будут отображаться.  <br/> |1,000  <br/> |
|Максимальное количество почтовых ящиков пользователей, которые можно будет просматривать для результатов поиска. При наличии более 1000 почтовых ящиков, которые содержат контента, отвечающей поисковому запросу, только верхнюю 1000 почтовых ящиков с большинства результаты поиска будут доступны для предварительного просмотра.  <br/> |1,000  <br/> |
|Максимальное число элементов, обнаруженных в SharePoint и OneDrive для бизнеса сайтов, которые отображаются на странице предварительного просмотра при предварительном просмотре результатов поиска контента. Новые элементы будут отображаться.  <br/> |200  <br/> |
|Максимальное число сайтов (в SharePoint и OneDrive для бизнеса), которые можно будет просматривать для результатов поиска. При наличии более чем 200 общее сайты, которые содержат контента, отвечающей поисковому запросу, только верхнюю 200 сайтов с большинства результаты поиска будут доступны для предварительного просмотра.  <br/> |200  <br/> |
|Максимальное количество элементов на почтовый ящик общедоступных папок, которые отображаются на странице предварительного просмотра при предварительном просмотре результатов поиска контента.  <br/> |100  <br/> |
|Максимальное число элементов, обнаруженных в всех почтовых ящиков общедоступных папок, которые отображаются на странице предварительного просмотра при предварительном просмотре результатов поиска контента.  <br/> |200  <br/> |
|Максимальное количество общедоступных почтовых ящиков, которые можно будет просматривать для результатов поиска. При наличии более 500 почтовых ящиков общедоступных папок, содержащих контента, отвечающей поисковому запросу, только верхнюю 500 почтовых ящиков общедоступных папок с большинства результаты поиска будут доступны для предварительного просмотра.  <br/> |500  <br/> |
|Максимальное количество символов для запроса поиска (в том числе операторы и условия) для поиска контента.  <br/> **Примечание:** Это ограничение вступает в силу после развертывания запроса, это значит, что запрос будет получить были развернуты для каждой из ключевых слов. Например если запрос поиска 15 ключевые слова и дополнительные параметры и условия, запрос возвращает были развернуты 15 раз, каждый из которых указанные параметры и условия в запросе. Несмотря на то, что количество символов в поисковый запрос может быть меньше значения, это расширенный запрос, который может привести к Превышение этого ограничения.<br/> |**Почтовых ящиков:** 10 000  <br/> **Сайты:** 4000 при поиске всех сайтов или 2 000 при поиске до 20 сайтов <sup>1</sup> <br/> |
|Максимальное число вариантов, возвращаемых при использовании префикса с подстановочными знаками для поиска точной фразы в запросе поиска или при работе с подстановочными знаками префикса и логический оператор **NEAR** и **ONEAR** .  <br/> |10 000<sup>2</sup> <br/> |
|Минимальное число альфа символов для подстановочные знаки префикса; например `time*`, `one*`, или `set*`.  <br/> |3   <br/> |
|Максимальное количество почтовых ящиков в поиск контента, который можно удалить элементы, расположенные по, выполните действия с «поиск и удаление» (с помощью **New ComplianceSearchAction-очистить** команды). Если поиск контента, который вы выполняете действие очистки для исходных почтовых ящиков превышает это ограничение, очистку завершится ошибкой. Дополнительные сведения о поиска и очищать можно [Поиск и удаление сообщений электронной почты в организации Office 365](search-for-and-delete-messages-in-your-organization.md).<br/> |50 000  <br/> |
   
> [!NOTE]
> <sup>1</sup> при поиске SharePoint и OneDrive для бизнеса расположений, символов в URL-адреса сайтов, которые считаются от этого ограничения. > <sup>2</sup> для запросов, не являющиеся фразу (ключевое значение, которое не использует двойные кавычки) мы используем специальный префикс индекса. Это сообщает нам о том, что слово происходит в документе, но не которых он встречается в документ. Для выполнения запроса фразу (значение ключевого слова в двойные кавычки), мы должны сравнить позицию в документе для ключевых слов в фразу. Это означает, что нельзя использовать префикс индекса для запросов фразы. В этом случае мы во внутренней сети разверните узел запросов со всех возможных слова, префикса при развертывании; например `"time*"` могут расширить до `"time OR timer OR times OR timex OR timeboxed OR …"`. 10 000 — максимальное число вариантов, которые слово можно развернуть на не число документов, соответствующих запросу. Нет верхний предел не фразу терминов.

## <a name="indexing-limits-for-email-messages"></a>Индексирование ограничения для сообщений электронной почты
<a name="indexinglimits"> </a>

В следующей таблице описываются индексирования ограничения, которые могут привести к в сообщении электронной почты, возвращаемых как неиндексированные элемента или частично индексированных элементов в списке результатов поиска контента.
  
|**Ограничение индексации**|**Примечания**|**Описание**|
|:-----|:-----|:-----|
|Максимальный размер вложения (за исключением файлов Excel)  <br/> |150 МБ  <br/> |Максимальный размер вложения электронной почты, который будет проанализировать для индексации. Вложения, размер которого превышает это ограничение не будут анализироваться для индексирования и сообщение с вложением будут помечены как частично индексированы.<br/> > [!NOTE]> Синтаксический анализ — это процесс, где служба индексирования извлекает текст из вложения, удаляет ненужные символы, такие как пробелов и знаков препинания и затем делит текста на слова (в процессе, называемом разметку), который затем сохраняются в индексе.           |
|Максимальный размер файлов Excel  <br/> |4 МБ  <br/> |Максимальный размер файла Excel на сайте или вложенного в сообщение электронной почты, которое будет анализироваться для индексации. Любой другой файл Excel, превышающие это ограничение не будут анализироваться и файл или сообщение электронной почты, сообщение с вложением файлов будут помечены как неиндексированные.  <br/> |
|Максимальное число вложений  <br/> |250  <br/> |Максимальное число файлов, вложенных в сообщение электронной почты, которое будет анализироваться для индексации. Если сообщение содержит более 250 вложений, анализа и индексируются первые 250 вложений, и сообщение помечено как частично индексируются по причине наличия дополнительных вложений, которые не были синтаксический анализ.  <br/> |
|Число уровней вложения максимальный  <br/> |30  <br/> |Максимальное количество вложенных вложений, которые выполняется синтаксический анализ. К примеру Если сообщение электронной почты имеет другое сообщение, подключенного к нему и вложенные сообщение имеет вложенный документ Word, будет индексируются документа Word и вложенное сообщение. Это поведение будет продолжаться до 30 вложенных вложений.  <br/> |
|Максимальное количество вложенных изображения  <br/> |0  <br/> |Изображение, вложенного в сообщение электронной почты пропускаются анализатором и не индексируется.  <br/> |
|Максимальное время, затраченное на синтаксический анализ элемента  <br/> |30 с  <br/> |Не более 30 секунд, затраченное синтаксический анализ элемента для индексации. Если синтаксического анализа времени превышает 30 секунд, элемент будет помечен как частично индексированы.  <br/> |
|Выходные данные максимальное средство синтаксического анализа  <br/> |2 миллиона символов  <br/> |Максимальный объем текста выходные данные средство синтаксического анализа, индексируются. Например если средство синтаксического анализа извлеченные 8 миллионов долларов США символов из документа, индексированы только первые 2 миллиона символов.  <br/> |
|Маркеры максимальное заметки  <br/> |2 миллиона  <br/> |При индексации сообщения электронной почты каждого слова помечается инструкции обработки, которые определяют, как должны индексироваться слова. Каждый набор инструкций обработки называется маркер примечания. Для поддержки качества обслуживания в Office 365, существует ограничение в 2 миллиона маркеры комментариев для сообщения электронной почты.  <br/> |
|Размер максимального текста в индекс  <br/> |67 миллиона символов  <br/> |Общее количество знаков в тексте сообщения электронной почты и его вложения. При индексации сообщения электронной почты, весь текст в тексте сообщения и все вложения объединены в одну строку. Максимальный размер этой строки, которая индексируется — 67 миллиона символов.  <br/> |
|Максимальная уникальных маркеров в тексте  <br/> |1 миллион  <br/> |Как уже упоминалось, маркеры основаны на результатах извлечение текста из содержимого, удаление пробелов и знаков препинания и затем разделив на слова (называемые маркеры), которые хранятся в индексе. Например, фразу `"cat, mouse, bird, dog, dog"` содержит 5 маркеры. Однако только 4 эти уникальные маркеры. Не более 1 миллиона уникальных маркеров на сообщение электронной почты, что помогает предотвратить индекс слишком большим с помощью маркеров в случайном порядке.<br/> |
