# <a name="searching-cloud-based-mailboxes-for-on-premises-users-in-office-365"></a>Поиск в облачных почтовых ящиках для локальных пользователей в Office 365

Если в организации используется гибридное развертывание Exchange с включенной поддержкой Microsoft Teams, пользователи могут использовать приложение Teams Chat для обмена мгновенными сообщениями. Для облачного пользователя данные чата в Teams (также называемые обсуждениями 1xN) сохраняются в основном облачном почтовом ящике. Если локальный пользователь использует приложение Team Chat, его основной почтовый ящик расположен локально. Чтобы обойти это ограничение, корпорация Майкрософт выпускала новую функцию, в которой создается облачная область хранения (называемая облачным почтовым ящиком для локальных пользователей) для хранения данных чата в Teams для локальных пользователей. Это позволяет использовать средство "поиск контента" в центре безопасности _Амп_ соответствия требованиям для поиска и экспорта данных чата в Teams для локальных пользователей. 
  
Ниже приведены требования и ограничения для настройки и настройки облачных почтовых ящиков для локальных пользователей, а также для настройки и поиска.
  
- Учетные записи пользователей в локальной службе каталогов (например, Active Directory) должны синхронизироваться с Azure Active Directory, службой каталогов в Office 365. Это означает, что учетная запись почтового пользователя создается в Office 365 и связана с пользователем, основной почтовый ящик которого находится в локальной организации.
    
- Облачный почтовый ящик для локальных пользователей используется только для хранения данных чата в Teams. Локальный пользователь не может войти в облачный почтовый ящик или в любой способ доступа. Она не может использоваться для отправки и получения сообщений электронной почты. 
    
- Необходимо отправить запрос в службу поддержки Майкрософт, чтобы разрешить вашей организации выполнять поиск данных чата в облачных почтовых ящиках для локальных пользователей. [Чтобы включить эту функцию](#filing-a-request-with-microsoft-support-to-enable-this-feature) в этой статье, ознакомьтесь со статьей Сохранение запроса в службе поддержки Майкрософт. 
    
 **Примечание:** Беседы в каналах Teams всегда хранятся в облачном почтовом ящике, связанном с командой. Это означает, что вы можете использовать поиск контента для поиска в беседах каналов, не обращаясь к запросу поддержки. Для получения дополнительных сведений о поиске по каналам маркетинговых программ, ознакомьтесь с разДелом [Поиск в группах Microsoft Teams и Office 365](content-search.md#searching-microsoft-teams-and-office-365-groups).
  
## <a name="how-it-works"></a>Как это работает

Если у пользователя с включенной поддержкой Microsoft Teams есть локальный почтовый ящик, а учетная запись пользователя или удостоверение синхронизированы с облаком, корпорация Майкрософт создает облачный почтовый ящик для хранения данных 1xN Teams. После того как данные чата в Teams хранятся в облачном почтовом ящике, он индексируется для поиска. Это позволяет использовать поиск контента (и результаты поиска, связанные с вариантами обнаружения электронных данных) для поиска, просмотра и экспорта данных чата в Teams для локальных пользователей. Вы также можете использовать ** \*** командлеты ComplianceSearch в PowerShell Security _амп_ Security Center для поиска данных чата в Teams для локальных пользователей. 
  
На приведенном ниже рисунке показан рабочий процесс, в котором данные чата в Teams для локальных пользователей доступны для поиска, предварительного просмотра и экспорта.
  
![Хранилище на основе облака для локальных пользователей в Microsoft Teams](media/895845f8-2ceb-47ed-96c9-5ab7f1aea916.png)
  
Помимо этой новой возможности, вы по-прежнему можете использовать поиск контента для поиска, просмотра и экспорта контента Teams на облачном сайте SharePoint и почтовом ящике Exchange, связанном с каждой командой Майкрософт и 1xN Teams Chat в почтовом ящике Exchange Online для облачные пользователи.

## <a name="filing-a-request-with-microsoft-support-to-enable-this-feature"></a>Включение этой функции в службу поддержки Майкрософт

Необходимо создать запрос в службе поддержки Майкрософт, чтобы разрешить в Организации использование графического пользовательского интерфейса в центре безопасности _Амп_ соответствия требованиям для поиска данных в Microsoft Teams в облачных почтовых ящиках для локальных пользователей. Обратите внимание, что эта функция доступна в PowerShell _Амп_ Security Center. Вам не нужно будет отправить запрос в службу поддержки, чтобы использовать PowerShell для поиска данных чата в Teams для локальных пользователей. 
  
Если вы отправите запрос в службу поддержки Майкрософт, включите следующие сведения:
  
- Доменное имя по умолчанию для организации Office 365.
    
- Имя клиента и идентификатор клиента организации Office 365. Их можно найти на портале Azure Active Directory (в разделе **Управление** \> **свойствами**). Сведения о том, как [найти свой идентификатор клиента Office 365](https://support.office.com/article/6891b561-a52d-4ade-9f39-b492285e2c9b).
    
- Следующий заголовок или описание назначения запроса поддержки: "Включение поиска содержимого приложения для локальных пользователей". Это поможет отправить запрос в службу проектирования обнаружения электронных данных Office 365, которая будет реализовывать запрос. 
    
После внесения изменений служба поддержки Майкрософт отправит вам оценочную дату развертывания. Обратите внимание, что после того как вы отправите запрос на поддержку, процесс развертывания обычно занимает 2-3 недель. 
  
### <a name="what-happens-after-this-feature-is-enabled"></a>Что происходит после включения этой функции?

После развертывания этой функции в организации Office 365 в поиске контента и поиске, связанном с вариантом обнаружения электронных данных в центре безопасности _Амп_ соответствием, вносятся следующие изменения:
  
- Флажок **Добавление содержимого приложений Office для локальных пользователей** добавляется в **местах** поиска контента. 
    
    ![В пользовательский интерфейс поиска контента добавляется флажок "добавить содержимое приложения Office для локальных пользователей"](media/599e751e-17bd-408d-a18c-127538de6e85.png)
  
- Локальные пользователи отображаются в средстве выбора расположений содержимого, которое используется для выбора почтовых ящиков пользователей для поиска. 
    

  
## <a name="searching-for-teams-chat-content-in-cloud-based-mailboxes-for-on-premises-users"></a>Поиск содержимого чата Teams в облачных почтовых ящиках для локальных пользователей

После включения компонента можно использовать поиск контента в центре безопасности _Амп_ соответствия требованиям для поиска данных в Microsoft Teams в облачных почтовых ящиках для локальных пользователей. 
  
1. В центре безопасности _амп_ соответствия требованиям выберите **** \> поиск **контента** поиска
    
2. На странице **Поиск** щелкните Добавить значок ![](media/8ee52980-254b-440b-99a2-18d068de62d3.gif) " **Новый поиск**".
    
    Как было сказано выше, флажок **Добавить содержимое приложения Office для локальных пользователей** отображается в разделе **расположения**. Обратите внимание, что он выбран по умолчанию.
    
3. Создайте запрос по ключевым словам и добавьте условия в поисковый запрос, если это необходимо. Для поиска только данных о беседах группы можно добавить следующий запрос в поле **Ключевые слова** : 
    
    ```
    kind:im
    ``` 

4. На этом шаге можно выбрать один из следующих параметров в разделе **расположения**:
    
    - **Все расположения** — выберите этот вариант, чтобы искать почтовые ящики всех пользователей в Организации. При выборе этого флажка также будет выполняться поиск всех облачных почтовых ящиков для локальных пользователей. 
    
    - **Определенные расположения** — выберите этот параметр, а затем нажмите кнопку **изменить** \> , выберите пользователя, группы или Teams, чтобы выполнить поиск в определенных почтовых ящиках. Как описывалось ранее, средство выбора расположений позволит искать локальных пользователей. 
    
5. Сохраните и выполните поиск. Все результаты поиска из облачных почтовых ящиков для локальных пользователей можно просматривать так же, как и любые другие результаты поиска. Кроме того, вы можете экспортировать результаты поиска (включая данные чата в Teams) в PST-файл. Дополнительные сведения см. в следующих разделах: 
    
    - [Создание нового поиска](content-search.md#create-a-new-search)
    
    - [Просмотр результатов поиска](content-search.md#preview-search-results)
    
    - [Экспорт результатов поиска контента](export-search-results.md)
    
## <a name="using-powershell-to-search-for-teams-chat-data-in-cloud-based-mailboxes-for-on-premises-users"></a>Поиск данных чата в облачных почтовых ящиках для локальных пользователей с помощью PowerShell

Командлеты **New – ComplianceSearch** и **Set – ComplianceSearch** можно использовать в PowerShell центра соответствия требованиям безопасности _амп_ для поиска облачного почтового ящика для локальных пользователей. Как было сказано ранее, вам не нужно будет отправить запрос в службу поддержки, чтобы использовать PowerShell для поиска данных чата в Teams для локальных пользователей. 
  
1. [Подключитесь к PowerShell центра безопасности _Амп_ соответствия требованиям](https://docs.microsoft.com/powershell/exchange/office-365-scc/connect-to-scc-powershell/connect-to-scc-powershell).
    
2. Выполните следующую команду PowerShell, чтобы создать новый контент, который выполняет поиск в облачных почтовых ящиках локальных пользователей.
    
    ```
    New-ComplianceSearch <name of new search> -ContentMatchQuery <search query> -ExchangeLocation <on-premises user> -IncludeUserAppContent $true -AllowNotFoundExchangeLocationsEnabled $true  
    ```
   
    Параметр *инклудеусераппконтент* используется для указания облачного почтового ящика для пользователей или пользователей, указанных с помощью параметра *ExchangeLocation* . *Алловнотфаундексчанжелокатионсенаблед* позволяет использовать облачные почтовые ящики для локальных пользователей. При использовании `$true` значения для этого параметра служба поиска не пытается проверить наличие почтового ящика перед его выполнением. Это необходимо для поиска в облачных почтовых ящиках локальных пользователей, так как эти типы почтовых ящиков не разрешаются как обычные почтовые ящики. 
    
    В следующем примере выполняется поиск сеансов Teams (которые являются мгновенными сообщениями), содержащих ключевое слово "редстоне" в облачном почтовом ящике Sara Davis, который является локальным пользователем в организации Contoso.
  
    ```
    New-ComplianceSearch "Redstone_Search" -ContentMatchQuery "redstone AND kind:im" -ExchangeLocation sarad@contoso.com -IncludeUserAppContent $true -AllowNotFoundExchangeLocationsEnabled $true  
    ```

   После создания нового поиска убедитесь, что для запуска поиска используется командлет **Start – ComplianceSearch** . 
  
Дополнительные сведения об использовании этих командлетов приведены в следующих статьях:
  
- [New-ComplianceSearch](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-content-search/new-compliancesearch)
    
- [Set-ComplianceSearch](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-content-search/set-compliancesearch)
    
- [Start-ComplianceSearch](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-content-search/start-compliancesearch)
    

## <a name="known-issues"></a>Известные проблемы

- В настоящее время вы можете выполнять поиск, предварительный просмотр и экспорт контента в облачных почтовых ящиках для локальных пользователей. Помещение облачного почтового ящика для локального пользователя в удержание, связанное с вариантом обнаружения электронных данных или назначение его политике хранения Office 365, не поддерживается. 
    
- В средстве выбора расположения контента для обнаружения электронных данных отображаются локальные пользователи, которые позволяют выбрать их. Тем не менее, как было объяснено ранее, оно не будет применено к локальному пользователю.
    
## <a name="frequently-asked-questions"></a>Вопросы и ответы

 **Где находятся облачные почтовые ящики для локальных пользователей?**
  
Облачные почтовые ящики создаются и хранятся в том же центре обработки данных, что и ваша организация Office 365. 
  
 **Существуют ли другие требования, отличные от отправки запроса поддержки?**
  
 Как было сказано ранее, удостоверения пользователей с локальными почтовыми ящиками должны быть синхронизированы с облачной организацией, чтобы для каждой локальной учетной записи пользователя в Office 365 была создана соответствующая учетная запись пользователя почты. кроме того, ваша организация должна иметь корпоративную подписку на office 365, например, подписку на office 365 корпоративный E1, E3 или подписку. 
  
 **Существует ли риск потери данных чата в Teams, если локальный почтовый ящик пользователя переносится в облако?**
  
Нет. При переносе основного почтового ящика локального пользователя в облако данные чата в Teams для этого пользователя будут перенесены в новый основной почтовый ящик на основе облака.
  
 **Можно ли применить политики хранения электронных данных или политики хранения Office 365 к локальным пользователям?**
  
Нет.
  
 **Может ли поиск контента находить старые сеансы команд для локальных пользователей до того момента, когда моя организация отправила запрос на включение этой функции?**
  
Корпорация Майкрософт приступила к хранению данных чата в Teams для локальных пользователей на 31 января 2018 г. Таким образом, если удостоверение локальных пользователей Teams было синхронизировано между Active Directory и Azure Active Directory с этой датой, то данные чата в Teams будут храниться в облачном почтовом ящике и будут использоваться для поиска контента. Кроме того, корпорация Майкрософт работает над хранением данных чата в Teams до 31 января 2018 в облачных почтовых ящиках для локальных пользователей. Дополнительные сведения об этом будут доступны в ближайшее время.
