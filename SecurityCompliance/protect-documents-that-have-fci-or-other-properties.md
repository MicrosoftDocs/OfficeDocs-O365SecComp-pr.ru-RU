<span data-ttu-id="71188-p106">Прежде чем можно будет использовать свойство Windows Server FCI или другое свойство в политике защиты от потери данных, необходимо создать управляемое свойство в центре администрирования SharePoint. Вот почему.</span><span class="sxs-lookup"><span data-stu-id="71188-p106">Before you can use a Windows Server FCI property or other property in a DLP policy, you need to create a managed property in the SharePoint admin center. <span data-ttu-id="71188-p107">Примеры</span><span class="sxs-lookup"><span data-stu-id="71188-p107">In SharePoint Online and OneDrive for Business, the search index is built up by crawling the content on your sites. <span data-ttu-id="71188-p108">Это важно, поскольку в политике защиты от потери данных в Office 365 поисковая программа-обходчик используется для обнаружения и классификации конфиденциальной информации на ваших сайтах и ее дальнейшего сохранения в защищенной части индекса поиска. Но для использования FCI или другого свойства в политике защиты от потери данных это свойство для обхода нужно сопоставить с управляемым свойством, чтобы контент с этим свойством хранился в индексе.</span><span class="sxs-lookup"><span data-stu-id="71188-p108">This is important because DLP in Office 365 uses the search crawler to identify and classify sensitive information on your sites, and then store that sensitive information in a secure portion of the search index. But to use an FCI or other property in a DLP policy, that crawled property needs to be mapped to a managed property so that content with that property is kept in the index.</span></span>
