# <a name="office-365-reporting-features"></a>Функции создания отчетов в Office 365 

## <a name="introduction"></a>Введение
Функция отчетов в Office 365 предоставляет разнообразные отчеты аудита для Azure Active Directory (AD), Exchange Online, управление устройствами, контролирующий просмотра и защита от потери данных (DLP). Это разные и отдельно от отчетов об активности Office 365.

## <a name="office-365-reports-dashboard"></a>Панель мониторинга отчетов Office 365
Панель мониторинга отчетов в режиме предварительного просмотра Центр администрирования Office 365 отображает данные об использовании системы в Office 365. Office 365 для глобальных администраторов, или Exchange Online, SharePoint Online или Скайп для администратора предприятия, можно получить фрагментарное понять использования этой службы. Отчеты можно предоставить полезные сведения о таких как количество пользователей, использование конкретной службы Office 365, количество пользователей, которые активации Office профессиональный плюс и сколько почта передается через организацию. Отчеты доступны за последние 7, 30, 90 и 180 дней.

Доступны следующие отчеты:

## <a name="azure-active-directory-reports"></a>Отчеты о Azure Active Directory
Office 365 с помощью Azure AD для проверки подлинности и управления удостоверениями. Office 365 для администраторов можно использовать в отчетах, создаваемых с Azure следует искать необычных мероприятий и несанкционированного доступа к своим данным. Отчеты об использовании и доступа к Azure AD позволяет обеспечить прозрачность целостности и безопасности directory вашей организации. С помощью этих сведений администратор можно лучше определить возможные угрозы безопасности может таким образом, чтобы они адекватно можете запланировать для устранения рисков.

Azure AD отчеты можно экспортировать в Microsoft Excel и связанной с другими данными из Office 365, например результаты поиска журнала аудита, чтобы предоставить информацию об access, проверки подлинности и действия на уровне приложения. Расширенные отчеты об использовании неполадок и ресурсов доступны при включении Azure AD Premium. Эти дополнительные отчеты, справки для повышения уровня и справки для организации безопасности организации реагировать на потенциальных угроз, используя анализа об использовании устройства доступа и приложения. Для получения дополнительных сведений см [отчетов Azure Active Directory](https://docs.microsoft.com/azure/active-directory/reports-monitoring/overview-reports/).

## <a name="exchange-online-audit-reports"></a>Отчеты по Exchange Online аудита
Exchange Online отчеты аудита включают сведения о доступ к почтовому ящику и изменения, внесенные администраторами организации Exchange Online клиента. После включения аудита почтовых ящиков можно использовать задачи в следующей таблице запускать отчеты и экспортировать Exchange Online журналов аудита.

>**Примечание**: необходимо включить почтовый ящик ведение журнала аудита для каждого почтового ящика, чтобы о событиях аудита сохраняются в журнале аудита для этого почтового ящика. Если аудита почтовых ящиков ведения журнала не включен для почтового ящика, события для этого почтового ящика не будут сохраняться в журнале аудита и не будет отображаться в отчетах аудита почтовых ящиков. Дополнительные сведения можно [Включить аудит почтового ящика](https://support.office.com/article/Enable-mailbox-auditing-in-Office-365-aaca8987-5b62-458b-9882-c28476a66918).

## <a name="device-compliance-reports"></a>Отчеты о соответствия устройств
Можно управлять и обеспечить безопасность мобильных устройств, при подключении к своей организации Office 365 с помощью Office 365 Mobile Device Management (MDM). Мобильные устройства, такие как смартфоны и планшетные ПК, которые используются для доступа к работы электронной почты, календарь, контакты и документы воспроизведение большая часть в проверке того, что сотрудников будет возможность работать в любое время и в любом месте. В результате важно защитить данные вашей организации. Office 365 MDM можно использовать настройки безопасности устройств правила политики и доступа и стирание мобильного устройства, если они в случае потери или кражи.

Отчеты соответствия MDM дается обзор политик, которые были настроены с организации для обеспечения безопасности мобильных устройств, которые получают доступ к данным Office 365. Отчет позволяет фильтрация устройств, состояние соответствия требованиям, переданные нарушения, заблокированные устройства и количество устройств, были очистить из-за политик безопасности. Для получения дополнительных сведений см [Обзор мобильных устройств управления для Office 365](https://support.office.com/article/Overview-of-Mobile-Device-Management-for-Office-365-faa7d8e5-645d-4d59-839c-c8d4c1869e4a).

## <a name="data-loss-prevention"></a>Защита от потери данных
Политики защиты от потери данных, используемые для управления безопасностью и потока информации в организации. Можно настроить политики для блокировки доступа к контенту, шифровать данные или уведомления пользователей о политике и нарушений политики, с помощью подсказки политики защиты от потери данных в приложение. Отчеты DLP дают понимание числа соответствия политике и правилу, переопределения и ложные срабатывания.

Чтобы просмотреть сведения о количестве сообщений, которые распознаются политик защиты от потери данных в формате графического диаграммы или таблицы можно использовать Центр администрирования Office 365. В частности соответствия политике защиты от потери данных для отправки и получения почты и соответствия правилам защиты от потери данных для отправки и получения почты. Можно также просмотреть количество совпадений, переопределения и ложные срабатывания для каждой политики за последние 24 часа с помощью центра администрирования Exchange. Тем не менее эти данные не доступен в качестве диаграммы. Если загрузить отчет в формате Excel, можно просмотреть даже более подробным, например, отправитель сообщений, в какой день и сопоставляет какие политики применено. Дополнительные сведения можно [просматривать отчеты об обнаруженных политики защиты от потери данных](https://technet.microsoft.com/en-us/library/jj889415(v=exchg.150).aspx).

## <a name="auditing-in-yammer-enterprise"></a>Аудит в корпоративной сети Yammer
Yammer, что Enterprise предоставляет администраторам возможность экспорта данных о действиях пользователей из своей сети Yammer с помощью [Yammer API экспорта данных](https://support.office.com/article/export-data-from-yammer-enterprise-b303d8f3-007d-4ad4-81f8-54fb1ecfb3f2)или вручную с помощью страницы администрирования сети Yammer. Возможность экспорта журналов ограничен сетевых администраторов в Yammer. (Все Office 365 для глобальных администраторов являются администраторами сети Yammer.)

Можно экспортировать следующие данные:

| Filename | Описание |
|----------------------------|-------------------------------------------------------------------------|
| Users.csv | Все новые отложенные и приостановленного пользователи в сети |
| Messages.csv | Все сообщения в сети |
| Files.csv (метаданные) | Метаданные, такие как имя файла, файла API URL-адрес, идентификатор Отгрузчик, загруженный в, и т.д. |
| Files.csv (исходными файлами) | ZIP-файл, исходных файлов, которые были отправлены пользователями в Yammer |
| Topics.csv | Разделы, созданные в сети |
| Pages.csv | Страницы (примечания), созданные пользователями в сети |
| Admins.csv | Все проверки администраторов в сети |
| Networks.csv | Все внешние сети Yammer |

*В таблице 3 - Yammer сети данных файлов, доступных для экспорта по клиентам*

Yammer корпоративных данных также доступен в отчеты об активности Office 365. Кроме того Yammer активно работает на предоставление дополнительной журнала с помощью API действия для управления Office 365 и на возможность причину отказа данных с помощью Power BI. Просмотрите [Схема Office](https://fasttrack.microsoft.com/roadmap?filters=yammer) для получения дополнительных сведений об этих возможностях.
