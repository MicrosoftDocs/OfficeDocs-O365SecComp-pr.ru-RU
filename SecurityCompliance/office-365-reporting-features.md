# <a name="office-365-reporting-features"></a>Функции создания отчетов в Office 365 

## <a name="introduction"></a>Введение
Функция отчетов в Office 365 предоставляет разнообразные отчеты аудита для Azure Active Directory, Exchange Online, управления устройствами, супервизорной проверки и защиты от потери данных (DLP). Они отличаются от отчетов об активности Office 365 и отличаются друг от друга.

## <a name="office-365-reports-dashboard"></a>Панель мониторинга отчетов Office 365
Панель мониторинга отчетов в предварительной версии центра администрирования Microsoft 365 отображает активность использования в Office 365. Глобальные администраторы Office 365, а также администратор Exchange Online, SharePoint Online или Skype для бизнеса могут получить подробные сведения об использовании этой службы. Отчеты могут предоставлять подробные сведения, такие как количество пользователей, использующих определенную службу Office 365, количество пользователей, которые активировали Office профессиональный плюс, и количество сообщений, передаваемых через организацию. Отчеты доступны за последние 7, 30, 90 и 180 дней.

Доступны следующие отчеты:

## <a name="azure-active-directory-reports"></a>Отчеты Azure Active Directory
Office 365 использует Azure AD для проверки подлинности и управления удостоверениями. Администраторы Office 365 могут использовать отчеты, созданные Azure, для поиска необычных действий и несанкционированного доступа к их данным. Вы можете использовать отчеты о доступе и использовании в Azure AD, чтобы получить представление о целостности и безопасности каталога организации. С помощью этих сведений администратор может определить, где могут возникнуть потенциальные риски безопасности, чтобы они могли надлежащим образом спланировать риски.

Отчеты Azure AD можно экспортировать в Microsoft Excel и сопоставить с другими данными из Office 365, например результатами поиска в журнале аудита, для предоставления сведений о действиях в доступе, проверке подлинности и уровне приложения. Расширенные отчеты об аномалиях и использовании ресурсов доступны, если включена служба Azure AD Premium. Эти расширенные отчеты помогут повысить уровень безопасности Организации и помочь организациям реагировать на потенциальные угрозы, используя аналитику, посвященную доступу к устройству и использованию приложений. Дополнительные сведения см. в статье [Azure Active Directory Reporting](https://docs.microsoft.com/azure/active-directory/reports-monitoring/overview-reports/).

## <a name="device-compliance-reports"></a>Отчеты о соответствии устройства
Вы можете управлять мобильными устройствами и защищать их при подключении к организации Office 365 с помощью управления мобильными устройствами Office 365 (MDM). Мобильные устройства, такие как смартфоны и Планшетные ПК, которые используются для доступа к рабочим электронным сообщениям, календарю, контактам и документам, играют значительную часть, чтобы сотрудники работали в любое время и в любом месте. В результате важно защитить сведения Организации. Вы можете использовать Office 365 MDM для настройки политик безопасности устройств и правил доступа, а также для очистки мобильных устройств в случае потери или кражи.

Отчеты о соответствии требованиям MDM предоставляют обзор политик, настроенных Организацией для защиты мобильных устройств, которые обращаются к данным Office 365. Этот отчет позволяет фильтровать устройства по состоянию соответствия требованиям, обнаруженным нарушениям, блокированным устройствам и количеству устройств, которые были стерты в результате применения политик безопасности. Дополнительные сведения см. в статье [Overview of Mobile Device Management for Office 365](https://support.office.com/article/Overview-of-Mobile-Device-Management-for-Office-365-faa7d8e5-645d-4d59-839c-c8d4c1869e4a).

## <a name="data-loss-prevention"></a>Защита от потери данных
Политики защиты от потери данных помогают управлять безопасностью и передачей информации в Организации. Можно настроить политики для блокирования доступа к содержимому, шифрования данных или уведомления пользователей о нарушениях политики и политики с помощью подСказок политики DLP в приложениях. Отчеты DLP предоставляют подробные сведения о количестве совпадений политик и правил, переопределений и ложных срабатываний.

Вы можете использовать центр администрирования Microsoft 365 для просмотра сведений о количестве сообщений, обнаруженных политиками защиты от потери данных в графическом формате диаграммы или таблицы. В частности, соответствие политике защиты от потери данных для отправленной и полученной почты, а правила защиты от потери данных для отправленной и полученной почты. Кроме того, можно просмотреть количество совпадений, переопределений и ложных срабатываний для каждой политики в течение более 24 часов с помощью центра администрирования Exchange. Но эти данные недоступны в виде диаграммы. Если вы загрузили отчет для использования в Excel, вы можете просмотреть еще больше сведений, например о том, кто отправил сообщение, на какой день и какие совпадения политики были активированы. Дополнительные сведения см. в статье [Просмотр отчетов об обнаружении политик защиты от потери](https://technet.microsoft.com/en-us/library/jj889415(v=exchg.150).aspx)данных.

## <a name="auditing-in-yammer-enterprise"></a>Аудит в Yammer Enterprise
Yammer Enterprise предоставляет администраторам возможность экспорта данных о действиях пользователей из сети Yammer с помощью [API экспорта данных Yammer](https://support.office.com/article/export-data-from-yammer-enterprise-b303d8f3-007d-4ad4-81f8-54fb1ecfb3f2)или вручную через страницу администратора сети Yammer. Возможность экспорта журналов ограничена сетевыми администраторами в Yammer. (Все глобальные администраторы Office 365 являются администраторами сети Yammer.)

Можно экспортировать следующие данные:

| Задан | Описание |
|----------------------------|-------------------------------------------------------------------------|
| Users.csv | Все новые, ожидающие и приостановленные пользователи в сети |
| Messages.csv | Все сообщения в сети |
| Files. CSV (метаданные) | Метаданные, такие как filename, URL-адрес файла API, идентификатор отгрузки, отправленные в и т. д. |
| Files. CSV (исходные файлы) | ZIP-файл исходных файлов, которые были отправлены пользователями в Yammer |
| Topics.csv | Темы, созданные в сети |
| Pages.csv | Страницы (заметки), созданные пользователями в сети |
| Admins.csv | Все проверенные администраторы сети |
| Networks.csv | Все внешние сети Yammer |

*Таблица 3. файлы данных сети Yammer, доступные для экспорта клиентами*

Данные Yammer Enterprise также доступны в отчетах об активности Office 365. Кроме того, Yammer активно работает над предоставлением дополнительного ведения журнала с помощью API действий управления Office 365, а также с возможностью обоснований данных с помощью Power BI. Дополнительные сведения об этих возможностях вы найдете в статье [Путеводитель по Office](https://fasttrack.microsoft.com/roadmap?filters=yammer) .
