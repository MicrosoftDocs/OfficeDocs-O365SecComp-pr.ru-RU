# <a name="set-up-attorney-client-privilege-detection-in-advanced-ediscovery"></a>Настройка обнаружения доверенных прав клиентов в Advanced eDiscovery

Основной и дорогостоящий аспект фазы проверки любого обнаружения электронных данных — рецензирование документов для привилегированного контента. Advanced eDiscovery обеспечивает обнаружение привилегированного содержимого с учетом машинного обучения, чтобы повысить эффективность этого процесса. Эта функция называется *обнаружением доверенности клиентов*.

> [!NOTE]
> Прежде чем можно будет использовать модель обнаружения доверенных прав клиентов, необходимо согласиться с ней. Для получения инструкций посетите [Шаг 1](#step-1-opt-in-to-attorney-client-privilege-detection) .

## <a name="how-does-it-work"></a>Как это работает?

Когда обнаружение доверенности клиентов включено, все документы в наборе проверки будут обрабатываться моделью обнаружения доверенности клиента, при [анализе данных](analyzing-data-in-review-set.md) в наборе проверки. Модель выполняет две вещи:

- Привилегированный контент — модель использует машинное обучение, чтобы определить вероятность того, что документ содержит контент, который является юридическим по природе.

- Участники: в рамках настройки обнаружения доверенности клиентов необходимо предоставить список юристов для вашей организации. Затем модель сравнивает участников документа со списком юриста, чтобы определить, имеет ли документ по крайней мере один участник юриста.

Модель создает следующие три свойства для каждого документа:

- **Атторнэйклиентпривилежескоре** — вероятность того, что документ является юридическим по природе; значения для оценки находятся в диапазоне от **0** до **1**.

- **Хасатторнэй** — для этого свойства задано **значение true** , если один из участников документа указан в списке юрист; в противном случае — значение **false**. Значение также задается равным **false** , если ваша организация не отправит список доверенности.

- **** GetProperty — для этого свойства задано значение **true** , если значение для **атторнэйклиентпривилежескоре** превышает пороговое значение, *или* если документ имеет доверенного участника; в противном случае задано значение **false**.

Эти свойства (и соответствующие им значения) добавляются в метаданные файлов документов в наборе рецензирования, как показано на следующем снимке экрана:

![Юрист — свойства полномочий клиентов, показанные в метаданных файла](../media/AeDAttorneyClientPrivilegeMetadata.png)

Эти три свойства также можно искать в наборе проверки. Дополнительные сведения см в разделе [Query Data в наборе рецензирования](review-set-search.md).

## <a name="set-up-the-attorney-client-privilege-detection-model"></a>Настройка модели обнаружения доверенных прав клиентов

Чтобы включить модель обнаружения доверенности для клиентов, ваша организация должна согласиться и отправить список доверенности.

### <a name="step-1-opt-in-to-attorney-client-privilege-detection"></a>Шаг 1: согласие на обнаружение доверенности для клиентского доступа

Как было сказано ранее, модель обнаружения доверенных прав клиента в предварительной версии находится в предварительной версии. Таким образом, пользователь в Организации обнаружения электронных данных в Организации (член группы администраторов обнаружения электронных данных в группе ролей "Диспетчер eDiscovery") должен принять участие в модели, доступной в расширенных случаях обнаружения электронных данных.

1. В центре безопасности & соответствия требованиям перейдите на **Обнаружение электронных данных > Advanced eDiscovery**.

2. На странице "расширенное расположение **электронных** данных" в разделе **Параметры** выберите пункт **Настройка экспериментальных функций**.

   ![Щелкните "Настройка экспериментальных функций"](../media/AeDExperimentalFeatures.png)

3. На вкладке **экспериментальные функции** выберите пункт **Управление уровнем доверенности — Клиент**.

4. В раскрывающемся списке **права доверенного клиента** щелкните переключатель, чтобы включить компонент, а затем нажмите кнопку **сохранить**.

### <a name="step-2-upload-a-list-of-attorneys-optional"></a>Шаг 2: Отправка списка юристов (необязательно)

Чтобы воспользоваться всеми преимуществами модели обнаружения доверенных прав клиентов и использовать результаты, описанные ранее **** в этой статье **** , рекомендуется отправить список адресов электронной почты для юристам и юридические сотрудники, которые работают в вашей организации. 

Чтобы отправить список доверенности для использования моделью обнаружения доверенных прав клиента, выполните следующие действия:

1. Создайте CSV-файл (без строки заголовка) и добавьте адрес электронной почты для каждого пользователя в отдельной строке. Сохраните этот файл на локальном компьютере.

2. На вкладке " **Расширенное обнаружение электронных** данных" в разделе **Параметры** выберите пункт **Настройка экспериментальных функций**, а затем выберите пункт **Управление юристом — Настройка прав клиента**.

   Отображается страница " **юрист-клиент** ", а также включен переключатель **обнаружения доверенных клиентских прав** .

   ![Юрист — всплывающая страница прав клиента](../media/AeDUploadAttorneyList.png)

3. Нажмите кнопку **Обзор** , а затем найдите и выберите CSV-файл, созданный в шаге 1.

4. Нажмите кнопку **сохранить** , чтобы отправить список доверенности.

## <a name="use-the-attorney-client-privilege-detection-model"></a>Использование модели обнаружения доверенных прав клиентов

Выполните действия, описанные в этом разделе, чтобы использовать обнаружение доверенности клиентов для документов в наборе рецензирования.

### <a name="step-1-create-a-smart-tag-group-with-attorney-client-privilege-detection-model"></a>Шаг 1: создание группы смарт-тегов с доверенной моделью обнаружения прав клиентов

Один из основных способов просмотра результатов обнаружения доверенности клиентов в процессе проверки заключается в использовании группы смарт-тегов. Группа смарт-тегов указывает на результаты обнаружения доверенности клиентов и отображает результаты в виде строки рядом с тегами в группе смарт-тегов. Это позволяет быстро определять потенциально привилегированные документы во время рецензирования документов. Кроме того, вы можете использовать теги в группе смарт-тегов, чтобы отмечать документы как привилегированные или непривилегированные. Дополнительные сведения о смарт-тегах приведены [в разделе Настройка смарт-тегов в Advanced eDiscovery](smart-tags.md).

1. В наборе проверки, содержащем документы, проанализированные на этапе 1, щелкните **Управление набором проверки** , а затем щелкните **Управление тегами**.
 
2. В разделе **теги**щелкните раскрывающийся список **Добавить группу** , а затем щелкните **Добавить группу смарт-тегов**.

   ![Нажмите кнопку "добавить группу смарт-тегов"](../media/AeDCreateSmartTag.png)

3. На странице **Выбор модели для смарт-тега** нажмите кнопку **выбрать** рядом с полем **привилегия доверенности клиента**.

   Отображается группа тегов с именем **юрист — клиент** . Он содержит два дочерних тега с именами " **положительный** " и " **отрицательный**", которые соответствуют возможным результатам, получаемым моделью.

   ![Юрист — группа смарт-тегов "права клиентов"](../media/AeDAttorneyClientSmartTagGroup.png)

3. Переименуйте группу тегов и теги в соответствии с проверкой. Например, вы можете переименование **положительных** и **отрицательных** в непривилегированные. **** ****

### <a name="step-2-analyze-a-review-set"></a>Шаг 2: анализ набора проверки

При анализе документов в наборе рецензирования также будет выполняться модель обнаружения доверенных прав клиентов и соответствующие свойства (описанные в разделе[как работает?](#how-does-it-work) будет добавлен в каждый документ в наборе проверки. Более подробную информацию об анализе данных в наборе рецензирования можно узнать в статье [анализ данных в наборе проверки в Advanced eDiscovery](analyzing-data-in-review-set.md).

### <a name="step-3-use-the-smart-tag-group-for-review-of-privileged-content"></a>Шаг 3: использование группы смарт-тегов для проверки привилегированного контента

После анализа набора проверки и настройки смарт-тегов необходимо просмотреть документы. Если модель определила, что документ является потенциально привилегированным, соответствующий смарт-тег в **панели маркировки** будет указывать на следующие результаты, полученные с помощью обнаружения прав клиента.

- Если в документе есть контент, который может быть юридическим, подпись будет **** отображаться рядом с соответствующим смарт-тегом (в данном случае это **положительный** тег по умолчанию).

- Если в документе есть участник, найденный в списке юриста Организации, то доверенность метки **** отображается рядом с соответствующим смарт-тегом (в данном случае это также **положительный** тег по умолчанию).

- Если документ содержит контент, который может быть юридическим в природе *и* содержит участника, найденного в списке юрист, отображаются как **юридические сведения** , так и метки **доверенности** . 

Если модель определяет, что документ не содержит контент, который является юридическим или не содержит участника из списка доверенности, то ни одна из этих меток не отображается в панели маркировки.

Например, на следующих снимках экрана показаны два документа; Первый из них содержит контент, который является юридическим и содержит участника, найденного в списке юристов; Вторая не содержит ни одной метки.

![Документ с метками юриста и юридических материалов](../media/AeDTaggingPanelLegalContentAttorney.png)

![Документ без меток](../media/AeDTaggingPanelNegative.png)

После просмотра документа, чтобы узнать, содержит ли он привилегированный контент, можно пометить документ с помощью соответствующего тега.
