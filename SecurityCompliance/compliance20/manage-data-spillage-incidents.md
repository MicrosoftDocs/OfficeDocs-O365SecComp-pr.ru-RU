# <a name="manage-a-data-spillage-incident-in-microsoft-365"></a>Управление инцидентом переноса данных в Microsoft 365 

При выпуске конфиденциального документа в ненадежной среде передается информация. При обнаружении инцидента с продолжением прохождения данных важно быстро оценить его размер и расположение, проанализировать действия пользователя, а затем окончательно удалить перенесенные данные из системы.

## <a name="scope-of-this-article"></a>Область действия этой статьи

В этой статье представлен список инструкций по удалению элементов из почтовых ящиков Office 365, которые больше не доступны пользователям и администраторам и не могут быть восстановлены. 

> [!NOTE]
> При удалении элементов, расположенных на сайте SharePoint или OneDrive для бизнеса, они сохраняются в течение 93 дней с момента их удаления из исходной папки.

## <a name="scenario"></a>Сценарий

Вы сообщаете об инциденте прокрутки данных, когда сотрудник неизвестен с очень конфиденциальным документом с несколькими людьми через электронную почту. Вы хотите быстро оценить, кто получил этот документ как внутри организации, так и за ее пределами. После изучения инцидента вы планируете поделиться своими сведениями с другими исследованиями, чтобы проверить, а затем окончательно удалить перенесенные данные из Office 365. После завершения расследования необходимо удалить все доказательства. 

## <a name="workflow"></a>Рабочий процесс

Ниже показан рабочий процесс для использования расследования данных (Предварительная версия) для управления инцидентом продолженности данных:

1.  Создание расследования данных.

2.  Поиск перенесенных данных.

3.  Просмотрите и изучите инцидент.

4.  Удаление перенесенных данных без возможности восстановления.

5.  Закройте или удалите расследования.


## <a name="before-you-begin"></a>Перед началом работы

- Используйте средство расследования данных (Предварительная версия) в центре безопасности Office 365 Security _Амп_ для создания расследования, поиска перенесенных данных, а также для просмотра и анализа. Затем вы будете использовать _Амп_ безопасности центра соответствия требованиям PowerShell для окончательного удаления перенесенных данных из Office 365. 

- Для создания расследования необходимо быть членом группы ролей "Администраторы соответствия требованиям" в центре безопасности _Амп_.

- Для удаления сообщений необходимо быть членом группы ролей Управление организацией в центре безопасности _Амп_ соответствия требованиям или назначить роль поиска и очистки. Сведения о добавлении пользователей в группу ролей приведены в статье [предоставление пользователям доступа к центру безопасности _Амп_ соответствия требованиям](../grant-access-to-the-security-and-compliance-center.md). 

- Чтобы определить, какие почтовые ящики пользователей и учетные записи OneDrive могут выполнять поиск в инвестигатор, ваша организация может настроить границы соответствия требованиям. Для получения дополнительных сведений [Настройте границы соответствия требованиям для исследований обнаружения электронных](../set-up-compliance-boundaries.md)данных. 

## <a name="step-1-create-a-data-investigation"></a>Шаг 1: создание расследования данных

Чтобы создать анализ данных, выполните указанные ниже действия.

1. Перейдите по ссылке [https://protection.office.com](https://protection.office.com).
    
2. Войдите в Office 365 с помощью своей рабочей или учебной учетной записи.
    
3. В центре безопасности _Амп_ соответствие выберите **расследования данных**.
 
4. На странице исследование **данных (предварительНая версия)** нажмите **создать новое исследование**.
    
5. В раскрывающейся окне **Новое исследование данных** укажите имя (обязательно), а затем введите дополнительный номер исследования и описание. Обратите внимание, что имя должно быть уникальным в Организации.

6. В разделе **вы хотите настроить дополнительные параметры после создания этого расследования?** выполните одно из следующих действий.

    - Нажмите кнопку **Да** , чтобы создать расследования, и отобразите страницу **Параметры** в новом случае. Это позволяет добавлять участников в исследование.
    
    - Нажмите кнопку **нет** , чтобы просто создать исследование и отобразить его в списке обращений на странице **расследования данных (Предварительная версия)** . Если выбран этот параметр, вы будете добавлены только в качестве единственного участника расследования и будут использоваться параметры поиска и аналитики по умолчанию. Вы можете добавить участников или изменить параметры в любое время после создания расследования.

7. Нажмите кнопку **сохранить** , чтобы создать расследование.

    Новое исследование отображается в списке на странице **расследования данных (предварительНая версия)** . 

8. Чтобы открыть исследование, щелкните имя расследования. 

    Отображается вкладка **Главная** для расследования. 

> [!TIP]
> Рекомендуется установить соглашение об именовании для расследования и предоставить как можно больше сведений, как в имени и описании, чтобы при необходимости можно было найти и ссылаться в будущем.
 
## <a name="step-2-search-for-the-spilled-data"></a>Шаг 2: Поиск перенесенных данных 
 
Если вы знаете, какие пользователи будут искать перенесенные данные, вы можете добавить их в качестве интересующих пользователей, чтобы сопоставить их источники данных с расследованиями и быстро найти их почтовые ящики и учетную запись OneDrive. Чтобы добавить людей, интересующих расследования, выберите **интересующих**людей, а затем нажмите кнопку **добавить людей**. 

На вкладке **поиски** можно создать поиск, чтобы найти перенесенные данные. Вы будете использовать тот же поисковый запрос, который использовался для поиска перенесенных данных для удаления тех же сообщений на [шаге 4](##step-4:-permanently-delete-the-spilled-data). Дополнительные сведения о создании поисковых запросов приведены [в разделе Создание поиска для сбора данных](create-search-to-collect-data.md).

После выполнения поиска вы можете просмотреть образцы результатов поиска и просмотреть статистику поиска, чтобы оценить Эффективность поискового запроса. Определив элементы, которые вы хотите удалить из Office 365, перейдите на вкладку инциденты, а **** затем создайте инцидент и добавьте результаты поиска, содержащие эти элементы. 

Для этого щелкните Поиск, который нужно исследовать. На всплывающей странице щелкните **Добавить результаты в инцидент** и следуйте инструкциям. Затем в инциденте можно просмотреть отдельные документы, выяснить, кто имел доступ к документам, и экспортировать документы. Чтобы просто удалить документы, а не просмотреть их, перейдите к [шагу 4](##step-4:-permanently-delete-the-spilled-data). 

> [!IMPORTANT]
> Ключевые слова, которые вы используете в поисковом запросе, могут содержать данные, которые вы ищете. Например, если вы ищете документы, содержащие номер социального страхования, и хотите использовать его в качестве ключевого слова в поисковых запросах, необходимо удалить запрос, чтобы избежать появления дополнительной системы. Вы можете удалить Поиск или удалить весь процесс расследования на [шаге 5](##step-5:-close-or-delete-investigation). 

## <a name="step-3-review-and-investigate"></a>Шаг 3: Проверка и исследование 

На вкладке исследование откройте вкладку **инциденты** и щелкните инцидент, созданный на предыдущем шаге. После завершения задания обработки и добавления результатов поиска в инцидент можно просмотреть отдельные документы в собственном формате, в текстовом формате или в близком формате. Вы можете создавать дополнительные запросы, чтобы сузить список документов, и отмечать документы тегами, чтобы показать результаты расследования.

Для группировки документов и получения дополнительной помощи по проверке нажмите кнопку **Управление инцидентом**. В плитке **аналитика** щелкните **анализ**. При этом будет выполняться Расширенная аналитика, например поиск повторяющихся данных, почтовые потоки и анализ темы. Более подробную информацию можно узнать в следующих статьях:

- [Обнаружение схожих документов (почти дубликатов)](near-duplicates.md)
- [Потоки почты](email-threading.md)
- [Темы](themes.md)

Чтобы определить, какие пользователи участвуют в проводе данных, вы можете создать новый запрос в инциденте, а затем использовать условия отПравителя/автора и получателей. При этом будет создан список всех отправителей, получателей и авторов, которые были найдены в собранных данных, добавленных в инцидент. Обязательно изучите список, чтобы определить, есть ли в списке внешние пользователи. Более подробную информацию можно узнать в статье [условия поиска](../keyword-queries-and-search-conditions.md#search-conditions).

## <a name="step-4-permanently-delete-the-spilled-data"></a>Шаг 4: окончательное удаление перенесенных данных

### <a name="deleting-mailbox-items"></a>Удаление элементов почтового ящика

После просмотра и проверки того, что результаты поиска содержат только сообщения электронной почты, которые необходимо удалить, можно удалить их, выполнив команду **New – ComplianceSearchAction – unnew – PurgeType определяет HardDelete** в соответствии с требованиями безопасности _амп_ Center PowerShell. Инструкции можно найти в статье [Поиск и удаление сообщений электронной почты](../search-for-and-delete-messages-in-your-organization.md). 

Обратите внимание, что если включено восстановление отдельных элементов для почтовых ящиков в Организации, удаленные элементы будут храниться в папке "элементы для восстановления" пользователя (и доступны администраторам) до завершения срока хранения удаленных элементов (по умолчанию — 14 дней). Кроме того, если какие-либо из почтовых ящиков, которые содержат перенесенные данные, находятся в судебном удержании или назначены политике хранения, очистка сообщения будет храниться в папке "элементы с возможностью восстановления" до тех пор, пока удержание не будет удалено или не будет исключено из политик хранения. Чтобы немедленно удалить сообщения, необходимо выполнить дополнительные задачи. Инструкции приведены в разделе [Удаление элементов в папке "элементы с возможностью восстановления" ](../delete-items-in-the-recoverable-items-folder-of-mailboxes-on-hold.md)в облачных почтовых ящиках на удержании.  

> [!IMPORTANT]
> Прежде чем удалять политику хранения или хранения, обратитесь к управлению записями или юридическим подразделениям. У вашей организации может быть политика, определяющая, имеет ли почтовые ящики на удержании или при проведении на них приоритет. 

### <a name="deleting-site-items"></a>Удаление элементов сайта

Чтобы окончательно удалить документ из сайта SharePoint или учетной записи OneDrive для бизнеса, необходимо удалить его, а затем удалить с сайта, а затем удалить из корзины семейства веб-сайтов. Инструкции см в разделе [Удаление документов в SharePoint и OneDrive](https://docs.microsoft.com/microsoft-365/compliance/gdpr-dsr-office365#deleting-documents-in-sharepoint-online-and-onedrive-for-business).

Кроме того, вы можете удалить все семейство веб-сайтов, в котором могут содержаться перенесенные данные. Инструкции см в разделе [Удаление семейства веб-сайтов](https://docs.microsoft.com/sharepoint/delete-site-collection).

## <a name="step-5-close-or-delete-the-investigation"></a>Шаг 5: закрытие или удаление расследования

После удаления документов в исходных расположениях содержимого (почтовые ящики или сайты) по-прежнему будут по-прежнему копироваться эти документы, добавленные в инциденты в ходе расследования. Чтобы избежать появления дополнительной информации, рекомендуется удалить расследования.

Удаление расследования:

1. На вкладке **Параметры** выберите **сведения об исследовании**.

2. Нажмите кнопку **Удалить обращение**. 

Если вы не хотите удалять расследования или вы хотите сохранить данные, собранные в ходе расследования, нажмите кнопку **Закрыть сценарий**. Позже вы можете повторно открыть закрытое расследование.
