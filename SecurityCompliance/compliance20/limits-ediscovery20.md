# <a name="limits-in-advanced-ediscovery"></a>Ограничения в Advanced eDiscovery

В этой статье описываются пределы в расширенном решении обнаружения электронных данных в Microsoft 365.

## <a name="case-limits"></a>Пределы обращений

В следующей таблице перечислены пределы вариантов в Advanced eDiscovery.

|**Описание ограничения**|**Ограничение**|
|:-----|:-----|
|Общее количество документов, которые можно добавить в обращение (для всех наборов проверок в случае случая).  <br/> |1 миллион  <br/> |
|Общий размер файлов для каждого набора нагрузок.  <br/> |100 ГБ  <br/> |
|Общий объем данных, загруженных в течение дня.<br/> |2 ТБ <br/> |
|Максимальное количество загрузок в наборе для каждого случая.  <br/> |означает <br/> |
|Максимальное количество наборов проверок для каждого случая.  <br/> |двадцать <br/> |

## <a name="search-limits"></a>Ограничения поиска

Приведенные в этом разделе пределы относятся к использованию средства поиска на вкладке **поиски** для сбора данных для обращения. Дополнительные сведения: [сбор данных для обращения в разделе Advanced eDiscovery](collecting-data-for-ediscovery.md).

|**Описание ограничения**|**Ограничение**|
|:-----|:-----|
|Максимальное количество почтовых ящиков или сайтов, в которых можно выполнять поиск в одной поиске.  <br/> |Нет ограничений  <br/> |
|Максимальное количество операций поиска, которые могут выполняться одновременно.  <br/> |Нет ограничений  <br/> | 
|Максимальное количество поисков, которое может начать один пользователь одновременно.  <br/> |10   <br/> | 
|Максимальное количество символов для поискового запроса (включая операторы и условия).  <br/> |**** Почтовые ящики: 10 000<br/>**Sites**: 4 000 при поиске на всех сайтах или 2 000 при поиске до 20 сайтов <sup>2</sup> <br/> |
|Минимальное количество буквенно-ых символов для подстановочных знаков префикса; Например: **1\* ** или **Set\***. <br/> |4  <br/> |  
|Максимальное число вариантов, возвращаемое при использовании подстановочного знака префикса для поиска точной фразы или при использовании подстановочного знака " **NEAR** " или " **ONEAR** ".  <br/> |10 000 <sup>3</sup> <br/> |
|Максимальное количество элементов в почтовом ящике пользователя, отображаемых на странице предварительного просмотра для поиска. Отображаются самые новые элементы.   <br/> |100  <br/> |
|Максимальное количество элементов из всех почтовых ящиков, отображаемых на странице предварительного просмотра для поиска.  <br/> |1,000  <br/> |
|Максимальное количество почтовых ящиков, которые можно просмотреть для результатов поиска.  Если имеется более 1000 почтовых ящиков, содержащих элементы, которые совпадают с поисковым запросом, для предварительного просмотра доступны только первые 1 000 почтовых ящиков с большинством результатов.<br/> |1,000  <br/> |
|Максимальное количество элементов из сайтов SharePoint и OneDrive для бизнеса, отображаемое на странице предварительный просмотр для поиска. Отображаются самые новые элементы.  <br/> |200  <br/> |
|Максимальное количество сайтов SharePoint и OneDrive для бизнеса, которые можно просмотреть для результатов поиска. При наличии более чем 200 сайтов, содержащих элементы, которые совпадают с поисковым запросом, для предварительного просмотра доступны только самые популярные сайты с наибольшим числом 200.  <br/> |200  <br/> |
|Максимальное количество элементов для почтового ящика общедоступных папок, отображаемое на странице предварительного просмотра при поиске.  <br/> |100  <br/> |
|Максимальное количество элементов, найденных во всех элементах почтового ящика общедоступных папок, отображаемых на странице предварительного просмотра для поиска.  <br/> |200  <br/> |
|Максимальное количество почтовых ящиков общедоступных папок, которые можно просмотреть для результатов поиска. Если более 500 почтовых ящиков общедоступных папок, содержащих элементы, которые совпадают с поисковым запросом, для предварительного просмотра доступны только первые 500 почтовых ящиков с большинством результатов.  <br/> |500  <br/> |

## <a name="review-set-download-limits"></a>Проверка пределов скачивания набора

|**Описание ограничения**|**Ограничение**|
|:-----|:-----|
|Общий размер файлов или максимальное количество документов, загруженных из набора рецензирования.  <br/> |3 МБ или 50 документов <sup>4</sup>|
|||

<br/>
<br/>

> [!NOTE]
> <sup>1</sup> . любой элемент, размер которого превышает ограничение на число файлов, будет отображаться как ошибка обработки.<br/>
> <sup>2</sup> при поиске расположений SharePoint и OneDrive для бизнеса символы в URL-адресах сайтов, для которых выполняется поиск, в соответствии с этим предельным числом.<br/>
> <sup>3</sup> для запросов без фразы (значение ключевого слова, не использующее двойные кавычки), используется особый Индекс префикса. Это говорит о том, что слово встречается в документе, но не в том месте, где он встречается в документе. Чтобы выполнить запрос фразы (значение ключевого слова с двойными кавычками), необходимо сравнить положение в документе для слов в фразе. Это означает, что мы не можем использовать индекс префикса для запросов фраз. В этом случае мы внутренним образом Раскройте запрос со всеми возможными словами, к которым префикс расширяется; Например, **время\* ** можно увеличить на **"время или таймер или время или Тимекс или тимебоксед или..."**. Ограничение 10 000 — максимальное число вариантов, до которых может быть расширено Word, а не число документов, которые соответствуют запросу. Отсутствует верхний предел для нефразных терминов.<br/>
> <sup>4</sup> это значение применяется для загрузки выбранных документов из набора рецензирования. Она не применяется к экспорту документов из набора рецензирования. Более подробную информацию о загрузке и экспорте документов можно узнать [в статье Export Data Case in Advanced eDiscovery](exporting-data-ediscover20.md). <br/>
