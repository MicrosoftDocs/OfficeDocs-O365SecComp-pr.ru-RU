# <a name="create-a-legal-hold-notice"></a>Создание удержания по юридическим причинам уведомление

С помощью расширенных обнаружения электронных данных (Просмотр) custodian связи, организации могут управлять их рабочий процесс вокруг связи с custodians. Через программу Communications юридических команды можно систематически отправить, сбор и отслеживать уведомления удержания по юридическим причинам. Процесс создания гибкие также позволяет группам для настройки рабочего процесса уведомления удержания и контента в уведомлений, отправленных custodians. 

В статье описываются действия, в рабочем процессе уведомления удержания.

## <a name="step-1-specify-communication-details"></a>Шаг 1: Укажите сведения о связи

Первый шаг — это указать соответствующие сведения для уведомления удержания по юридическим причинам или другие средства связи custodian. 

1. В центре соответствия требованиям & безопасности перейдите к **eDiscovery > расширенной обнаружения электронных данных (Просмотр)** для отображения списка случаев в вашей организации.
   
2. Перейдите на вкладку **коммуникаций** и нажмите кнопку **новый обмена данными**.
   
3. На странице " **Имя связи** " укажите следующие сведения о связи (обязательно).

    - **Имя**: имя для связи.
    
    - **Учет ответственным за**: В раскрывающемся списке отображается список вариантов членов. Каждого уведомления отправляются custodians будут отправляться сообщения от имени указанного выдающего ответственным за.

4. Нажмите кнопку **Далее**.

## <a name="step-2-define-the-portal-content"></a>Шаг 2: Определение контент портала

Затем можно создавать и добавлять содержимое уведомление о удержания. На странице **Определение контент портала** в мастере **создания связи** укажите содержимое уведомление о удержания. Этот контент будет автоматически добавлено на выдачу, повторно подготовить, напоминание и эскалации уведомления. Кроме того этот контент будет отображаться в портал custodian соответствия требованиям. 

Создание содержимого портала:

1. Тип (или вырезания и paster из другого документа) к удержанию Обратите внимание, в текстовом поле для содержимого портала. 

2. Вставка merge переменные в вашей уведомление о настраивать уведомления и общий доступ к порталу соответствия Custodian.

3. Нажмите кнопку **Далее**.

  >[!Tip]
  >Чтобы узнать дополнительные сведения об можно настроить содержимое и формат содержимого портала, обратитесь к разделу [с помощью редактора Communications](using-communications-editor.md).

## <a name="step-3-set-the-required-notifications"></a>Шаг 3: Задайте необходимые уведомления

После определения содержимое уведомление о удержания, можно настроить рабочие процессы, применимую к отправке и управление процессом уведомления. Уведомления — это сообщения электронной почты, отправленные для уведомления и наблюдение за ними с помощью custodians. Каждый custodian добавлена для передачи данных будет получать же уведомлений. 

Для установки и отправлять уведомление удержание, необходимо включить выдачу повторного присвоения и выпуска уведомлений.

### <a name="issuance-notification"></a>Выдачу уведомления 

После создания связи **Выдачу уведомления** инициируется ответственным за указанный выдачи. Выдачу уведомления — это первый передача данных отправляется custodian, информирующее о обязательства их хранения. 

Для создания выдачу уведомления о:

1. В иконку **выдачу** нажмите кнопку **Изменить**.
   
2. При необходимости добавьте дополнительные обращения члены или персонала для поля **«копия»** и **«СК»** . Чтобы добавить несколько пользователей в эти поля, разделяйте адреса электронной почты точкой с запятой.
   
3. Укажите **тему** уведомления (обязательно).
   
4. Укажите содержимое или Дополнительные инструкции, которые вы хотите предоставить custodian (обязательно). Обратите внимание на то, что контент портала, которую вы задали на шаге 2 добавляется в конец выдачу уведомления. 
   
5. Нажмите кнопку **Сохранить**. 

### <a name="re-issuance-notification"></a>RE выдачу уведомления 

В ходе case custodians может потребоваться сохранить данные дополнительные или меньше, чем было указано ранее. После обновления содержимого уведомление о удержания re выдачу уведомления оповещений custodians об изменении обязательства их хранения.

Чтобы создать re выдачу уведомления: 

1. В плитку **повторите** нажмите кнопку **Изменить**.
   
2. При необходимости добавьте дополнительные обращения члены или персонала для поля **«копия»** и **«СК»** . Чтобы добавить несколько пользователей в эти поля, разделяйте адреса электронной почты точкой с запятой.
   
3. Укажите **тему** уведомления (обязательно).
   
4. Укажите содержимое или Дополнительные инструкции, которые вы хотите предоставить custodian (обязательно). Обратите внимание на то, что контент портала, которую вы задали на шаге 2 добавляется в конец re выдачу уведомления.
   
5. Нажмите кнопку **Сохранить**.

>[!Note]
>Удержание уведомлений при изменении re выдачу уведомления будет автоматически отправляться все custodians назначается уведомление. После отправки уведомления custodians будет предложено повторно подтвердить их уведомление о удержания. Если выбрать любой напоминание или распространение рабочих процессов эти будет также повторно запустить. 

### <a name="release-notification"></a>Уведомление о выпуске

После разрешения вопрос или custodian больше не будет подвергаться на сохранение содержимого, могут освобождать custodian из дела. Если custodian ранее был отправлен уведомление удержания, уведомления о выпуске можно использовать для оповещения custodians, что они были выпущены из их обязанности.

Чтобы создать уведомление о выпуске: 

1. В **выпуске** плитку нажмите кнопку **Изменить**.
   
2. При необходимости добавьте дополнительные обращения члены или персонала для поля **«копия»** и **«СК»** . Чтобы добавить несколько пользователей в эти поля, разделяйте адреса электронной почты точкой с запятой.
   
3. Укажите **тему** уведомления (обязательно).
   
4. Укажите содержимое или Дополнительные инструкции, которые вы хотите предоставить custodian (обязательно).
   
5. Нажмите кнопку **Сохранить** и перейдите к следующему шагу. 

## <a name="optional-step-4-set-the-optional-notifications"></a>(Необязательно) Шаг 4: Установка дополнительных уведомлений

Кроме того можно упростить рабочий процесс для следующего с отвечает custodians путем создания и планирования автоматических напоминание и эскалации уведомлений.

### <a name="reminders"></a>Напоминания

После отправки уведомлений удержания можно исполнения с отвечает custodians, определение рабочего процесса напоминание. 

Планирование напоминания:

1. В плитку **напоминание** нажмите кнопку **Изменить**.
   
2. Для включения рабочего процесса **напоминание** для включения переключателя **состояние** (обязательно).
   
3. Укажите **напоминание интервал (в днях)** (обязательно). Это число дней до начала отправки уведомлений первого и последующего напоминание. Например если задать интервал напоминания о до 7 дней, затем первого напоминания о будет отправляться 7 дней после уведомления удержания первоначально был отправлен. Все последующие напоминания будут отправлены каждые 7 дней.
   
4. Укажите **число напоминаний** (обязательно). Это поле указывает, сколько напоминаний для отправки без реакции custodians. К примеру Если число напоминаний значение 3, custodian могут получать не более 3 напоминания. После custodian подтверждает уведомлений удержания, напоминания больше не будут отправляться данному пользователю.
   
5. Укажите **тему** уведомления (обязательно). 
   
6. Укажите содержимое или Дополнительные инструкции, которые вы хотите предоставить custodian (обязательно). Обратите внимание на то, что контент портала, которую вы задали на шаге 2 добавляется в конец уведомление о напоминание.
   
7. Нажмите кнопку **Сохранить** и перейдите следующему шагу.

### <a name="escalations"></a>Эскалации 

В некоторых случаях может потребоваться дополнительные способы обработки результатов с помощью custodians не отвечает. При custodian не подтвердить уведомление удержания после получения указанного числа напоминаний, юридических группы можно указать рабочего процесса для автоматической отправки на уведомление эскалации custodian и их manager.

Планирование эскалации:

1. В плитку **эскалации** нажмите кнопку **Изменить**.
   
2. Включение рабочего процесса **эскалации** , включив **состояние** переключателя.
   
3. Укажите **эскалации интервал (в днях)** (обязательно). 
   
4. Укажите **количество эскалации** (обязательно). Это поле указывает, сколько эскалации для отправки без реакции custodians. Например номер эскалации значение 3, затем на уведомление эскалации будет отправляться custodian и их руководитель не более 3 раза. После custodian подтверждает уведомлений удержания, эскалации больше не отправляются. 
   
5. Укажите **тему** уведомления (обязательно). 
   
6. Укажите содержимое или Дополнительные инструкции, которые вы хотите предоставить custodian (обязательно). Обратите внимание на то, что контент портала, которую вы задали на шаге 2 добавляется в конец уведомление о эскалации.
   
7. Нажмите кнопку **Сохранить** и перейдите следующему шагу.
   
## <a name="step-5-assign-custodians"></a>Шаг 5: Назначение custodians 

После завершения контент для уведомлений выберите custodians для отправки уведомлений. 

Чтобы добавить custodians:

1. Назначьте custodians передача данных, установив флажок рядом с их именами.

    После создания связи рабочего процесса уведомления автоматически применяется для выбранного custodians.
   
2. Нажмите кнопку **Далее** для просмотра параметров связи и сведений.
 
>[!NOTE]
>Можно добавить только custodians, которые были добавлены в рамках данного экземпляра и еще не был отправлен другое уведомление в случае.

## <a name="step-6-review-settings"></a>Шаг 6: Просмотр параметров

После того как просмотрите параметры и нажмите кнопку **Отправить** обмена данными система автоматически запускать связи рабочего процесса, отправив выдачу уведомления.
