# <a name="add-custodians-to-an-advanced-ediscovery-case"></a>Добавление custodians в расширенное дело обнаружения электронных данных

Используйте встроенное средство управления хранитель в Advanced eDiscovery для согласования рабочих процессов с управлением custodians и определения релевантных источников данных кустодиал, связанных с обращением. При добавлении хранитель система может автоматически определять и размещать удержание в своих почтовых ящиках Exchange и в учетной записи OneDrive для бизнеса. Во время процесса обнаружения вы также можете определить дополнительные источники данных (например, почтовые ящики, сайты или Teams), к которым хранитель обращался или который участвовал. В этом случае можно использовать средство управления хранитель, чтобы связать эти источники данных с определенным хранитель. После добавления custodians к случаю и связывания с ними другого источника данных можно быстро сохранить данные и выполнить поиск по данным кустодиал.

Используйте следующий рабочий процесс для добавления custodians в дополнительных случаях обнаружения электронных данных и управления ими. 

![Вкладка "Управление хранитель"](../media/CustodianMgtPage.png)

## <a name="before-you-begin"></a>Перед началом работы

Чтобы добавить custodians к случаю, необходимо быть участником группы ролей "Диспетчер обнаружения электронных данных". При этом будут предоставлены необходимые разрешения на добавление custodians к случаю и помещение удержания в источники данных кустодиал.


## <a name="step-1-add-potential-custodians"></a>Шаг 1: Добавление потенциальных custodians

Первый шаг — определение и добавление custodians в случае.

1. На домашней странице **Advanced eDiscovery (дополнительные функции обнаружения электронных** данных) выберите вариант, на который вы хотите добавить custodians. 
 
2. Перейдите на вкладку **custodians** и нажмите кнопку **+ Добавить custodians**.

3. Найдите custodians, который требуется добавить в обращение. Введите первую часть имени пользователя для отображения пользователей из Azure Active Directory вашей организации. Найдя нужного человека, щелкните его имя, чтобы добавить его в список.

   ![Определение потенциальных custodians](../media/AddCustodianStep1.png)
 
4. После добавления всех релевантных custodians нажмите кнопку **Далее** , чтобы выбрать основные источники данных custodians.
  
## <a name="step-2-select-custodian-data-sources"></a>Шаг 2: выбор источников данных хранитель

После добавления custodians средство хранитель поможет определить основные источники данных, принадлежащие каждому хранитель; в частности, эти расположения данных являются почтовым ящиком Exchange и учетной записью OneDrive хранитель. 

Идентификация источников данных хранитель: 

1. Чтобы выбрать почтовый ящик Exchange для всех custodians, установите флажок **Exchange** в верхней части столбца. Обратите внимание, что вы можете снять флажок для всех определенных хранитель, чтобы удалить почтовый ящик в качестве расположения кустодиал. Кроме того, можно оставить флажок **Exchange** в верхней части столбца невыбранным, а затем установить флажок для отдельных custodians. 
 
   ![Выбор источников данных Кустодиал](../media/AddCustodianStep2.png)
 
2. Повторите то же самое для учетных записей custodians "OneDrive. 

    После выбора источников данных хранитель система автоматически пытается идентифицировать и проверить эти источники данных, а затем добавляет их в качестве источников данных, связанных с custodians.
 
4. Нажмите кнопку **Далее** , чтобы начать связывание дополнительных источников данных с custodians в случае.

## <a name="step-3-associate-additional-data-sources-to-a-custodian"></a>Шаг 3: связывание дополнительных источников данных с хранитель

В зависимости от того, что вы изучаете, может также потребоваться поиск (и сохранение контента) почтовых ящиков, к которым можно получить доступ к определенным хранитель, группам Office 365, участником хранитель, или сайтам, к которым у хранитель также есть доступ. Поэтому в дополнение к основным источникам данных хранитель, указанным на предыдущем шаге, можно связать дополнительные источники данных Office 365 с помощью хранитель в случае. 

Сопоставление почтовых ящиков, сайтов или рабочих групп с определенным хранитель:

1. На странице " **Выбор дополнительных источников данных** " нажмите кнопку **Добавить** в строке для конкретного хранитель. 
  
   ![Сопоставление дополнительных источников данных](../media/AddCustodianStep3.PNG)

2. На всплывающей странице можно указать источник данных из любой из следующих служб Office 365:
  
   -  **Электронная почта Exchange** : выберите пункт **Выбор пользователей, групп или команд** , а затем снова нажмите кнопку **выбрать пользователей, группы или Teams** . Используйте поле поиска для поиска почтовых ящиков, которые необходимо связать с хранитель. Чтобы указать почтовые ящики, которые необходимо назначить выбранному хранитель, используйте поле поиска для поиска почтовых ящиков пользователей и групп рассылки. Можно также назначить связанный почтовый ящик для группы Office 365 или группы Майкрософт. Установите флажок Пользователь, группа, группа, нажмите кнопку **выбрать**, а затем нажмите кнопку **Готово**.

        > [!NOTE]
        > При нажатии кнопки выбрать пользователей, группы или Teams для указания почтовых ящиков, отображается пустое средство выбора почтовых ящиков. Это вызвано мерами по повышению скорости работы. Чтобы добавить почтовый ящик в этот список, введите в поле поиска имя или псевдоним (не менее 3 символов).
     
     - **Сайты SharePoint** — нажмите кнопку **выбрать сайты** , а затем нажмите кнопку **выбрать сайты** еще раз, чтобы отобразить список сайтов SharePoint в Организации. Чтобы связать сайт с хранитель, можно выбрать сайт в списке или ввести URL-адрес другого сайта или сайта, связанного с группой Office 365, Microsoft Team или учетной записью OneDrive.
     
     - **Teams** — щелкните **выбрать Teams (выбрать Teams** ), а затем нажмите кнопку **выбрать Teams** еще раз, чтобы отобразить список Microsoft Teams, участником которых является хранитель. Выберите команды, которые вы хотите добавить в свой хранитель. После выбора система автоматически определит _Амп_ выберите связанный сайт SharePoint и почтовые ящики групп, связанные с этой командой Майкрософт. Нажмите кнопку **выбрать**, а затем кнопку **Готово**.

       ![Сопоставление источников данных](../media/AddCustodianStep4.PNG)
        
      > [!NOTE]
      > Чтобы связать дополнительную команду с хранитель, необходимо отдельно добавить почтовый ящик и сайт, связанный с командой, с помощью адресов **почты Exchange** и **сайтов SharePoint** .

После завершения связывания дополнительных источников данных с custodians можно просмотреть общее количество почтовых ящиков, сайтов и команд, связанных с каждым хранитель, на **странице Выбор дополнительных источников данных**. После завершения работы с соответствующими источниками данных для определенного хранитель эта ассоциация будет поддерживаться и использоваться на этапах сбора, обработки и проверки в рабочем процессе обнаружения электронных данных.

## <a name="step-4-place-custodians-on-hold"></a>Шаг 4: помещение custodians на удержание

После завершения custodians и источников данных, которые необходимо добавить в это обращение, можно при необходимости поместить некоторые или все custodians на удержание. Когда вы помещаете хранитель на удержание, все содержимое всех расположений контента, связанных с хранитель, сохраняется до тех пор, пока вы не удалите удержание или освободите хранитель от. В некоторых случаях может потребоваться добавить custodians к случаю, не помещая их на удержание.

Чтобы поместить custodians и источники данных на удержание, выполните следующие действия:

1. В разделе **поместить блокировку на выбранную страницу custodians** установите флажок **удержание** в верхней части столбца, чтобы поместить все custodians на удержании. Обратите внимание, что вы можете снять флажок для всех определенных хранитель, которые необходимо удалить из удержания. Кроме того, вы можете оставить флажок **удержание** в верхней части столбца невыбранным, а затем установить флажок для отдельных custodians. 
 
   ![Размещение удержаний](../media/AddCustodianStep5.PNG)

2. Проверьте параметры хранения хранитель и нажмите кнопку **завершить**.

Если не поместить в Хранитель удержание, хранитель и связанные с ними источники данных будут добавлены к этому случаю, но контент в этих источниках данных не будет помещен в удержание.

После включения хранитель в удержание политика удержания хранитель, содержащая все источники кустодиал, будет автоматически создана. Чтобы просмотреть эту политику, выполните указанные ниже действия.

1. На **домашней** странице примера перейдите на вкладку **удержания** и выберите **кустодианхолд-GUID**,  

2. В раскрывающейся странице нажмите кнопку **изменить удержание** , чтобы просмотреть все источники данных хранитель, размещаемые на удержании.
