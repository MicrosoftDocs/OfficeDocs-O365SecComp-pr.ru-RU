# <a name="set-up-new-office-365-message-encryption-capabilities"></a>Настройка новых возможностей шифрования сообщений Office 365

Благодаря новым функциям шифрования сообщений Office 365 (OME), которые использовать функции защиты в Azure защита информации, организации могут совместно использовать защищенный электронной почты с кем-либо на любом устройстве. Пользователи могут отправлять и принимать защищенные сообщения с другими организациями Office 365, а также клиенты Office 365 с помощью Outlook.com, Gmail и другие службы электронной почты.
  
## <a name="get-started-with-ome-by-activating-azure-rights-management-part-of-azure-information-protection"></a>Начало работы с OME необходимо активировать Azure Rights Management частью защита информации Azure

Теперь можно легко приступить к работе с новыми возможностями OME. По состоянию на 2018 февраля Office 365 автоматически включает новые возможности OME для подходящими организаций в пределах нашего центров обработки данных. Вашей организации имеет право, если нового клиента Office 365 и вашей организации есть соответствующие подписки. **Если вы включили Azure Rights Management (службы управления правами Azure), частью защита информации Azure, а затем мы автоматически включить шифрования сообщений Office 365 для вас.** У вас нет необходимости выполнять дополнительные действия, чтобы включить OME. Служба управления правами Azure, см [Активация службы управления правами Azure](https://docs.microsoft.com/azure/information-protection/deploy-use/activate-service). Сведения о подписках содержатся в разделе «какие подписок необходимо использовать новые capabilities? OME» в [Вопросы и ответы шифрования сообщений Office 365](ome-faq.md). Для получения сведений о приобретении подписка на Azure Information Protection видеть [Защита информации Azure](https://azure.microsoft.com/services/information-protection/).
  
При использовании Exchange Online со службой управления правами Active Directory (AD RMS), вы не можете позволить этими новыми возможностями немедленно. Вместо этого необходимо сначала перенести из службы управления правами Azure защита информации. После завершения миграции можно успешно выполнить следующие действия.
  
Если вы решили использовать локальные службы управления правами с Exchange Online вместо миграции для защиты информации Azure, нельзя будет использовать эти новые возможности.
  
## <a name="how-the-new-capabilities-for-ome-work"></a>Как работают новые возможности для OME

Новые возможности шифрования сообщений Office 365 используйте возможности защиты, также называемая Azure Rights Management (службы управления правами Azure), из защита информации Azure. Этот компонент включает шифрование, идентификации и авторизации политик для защиты электронной почты. Можно шифровать сообщения с помощью шаблонов управления правами, [не пересылать параметр](https://docs.microsoft.com/information-protection/deploy-use/configure-usage-rights#do-not-forward-option-for-emails)и [параметр только для шифрования](https://docs.microsoft.com/information-protection/deploy-use/configure-usage-rights#encrypt-only-option-for-emails). Затем пользователи могут шифровать сообщения электронной почты и различных вложения Office 365 с помощью этих параметров. Полный список вложений в поддерживаемых типов в разделе [«типы файлов, обслуживаемые политики IRM присоединенные к сообщениям» в общие сведения о IRM для сообщений электронной почты](https://support.office.com/article/bb643d33-4a3f-4ac7-9770-fd50d95f58dc#FileTypesforIRM). Как администратор также можно определить правила потока обработки почты для применения защиты. Например можно определить правила, где все незащищенного сообщения, адресованные определенного получателя или, которые содержат определенные слова в строке темы защищены от несанкционированного доступа, а получатели не может копировать или печатать содержимое сообщения.
  
В отличие от предыдущей версии OME этими новыми возможностями предоставляют возможности объединенных отправителя, отправляете ли вы почты внутри организации или получателям за пределами Office 365. Кроме того получатели, которые получают защищенные по электронной почте сообщение, отправленное учетной записи Office 365 в Outlook 2016 или Outlook в Интернете, не требуется никаких дополнительных действий для просмотра сообщения. Он работает без. Также с помощью других клиентов электронной почты и поставщиков услуг электронной почты получателя быть улучшают. Сведения можно [узнать о защищенных сообщениях в Office 365](https://support.office.com/article/Learn-about-protected-messages-in-Office-365-2baf3ac7-12db-40a4-8af7-1852204b4b67) и [способ открытия защищенного сообщения](https://support.office.com/article/How-do-I-open-a-protected-message-1157a286-8ecc-4b1e-ac43-2a608fbf3098).
  
## <a name="steps-to-manually-set-up-the-new-capabilities-for-ome"></a>Действия, чтобы вручную установить новые возможности для OME

Если ваша организация не имеет OME включен автоматически или включить OME отключено, выполните следующие действия, чтобы вручную настроить новые возможности для OME.
  
### <a name="to-manually-set-up-the-new-capabilities-for-ome"></a>Чтобы вручную настроить новые возможности для OME

1. Убедитесь, что у правого подписки для вашей организации. Сведения о подписках содержатся «какой подписок необходимо использовать новые capabilities? OME» в разделе [вопросы и ответы шифрования сообщений Office 365.](ome-faq.md) Для получения сведений о приобретении подписка на Azure Information Protection видеть [Защита информации Azure](https://azure.microsoft.com/services/information-protection/).
    
2. Решите, следует ли Майкрософт для управления в корневом разделе для защиты информации Azure (по умолчанию), или создание и управление этот ключ самостоятельно (известную как собственный ключ или BYOK). Если вы хотите создать и самостоятельно управлять этот ключ, необходимо выполнить некоторые действия до настройки новые возможности для OME. Для получения дополнительных сведений см [Планирование и внедрение ключ клиента защита информации Azure](https://docs.microsoft.com/information-protection/plan-design/plan-implement-tenant-key). Корпорация Майкрософт рекомендует, выполните следующие действия, прежде чем настраивать OME.
    
3. Активация службы управления правами Azure для включения новых возможностей для OME. [Активация службы управления правами Azure](https://docs.microsoft.com/azure/information-protection/deploy-use/activate-service)инструкции, см. При этом Office 365 автоматически включает новые возможности OME.
    
    > [!TIP]
    > Outlook в Интернете кэширует его пользовательский Интерфейс, поэтому рекомендуется ожидания в день, прежде чем применения новых возможностей для OME сообщений электронной почты с помощью клиента. Перед пользовательского интерфейса обновлений в соответствии с новой конфигурацией и новые возможности для OME не будут доступны. После обновления пользовательского интерфейса пользователей можно защищать сообщения электронной почты, используя новые возможности для OME. 
  
4. (Необязательно) Настройка нового правила потока почты или обновите существующие правила потока обработки почты, которые определяют, как и когда требуется Office 365 для шифрования сообщений, отправляемых из вашей организации.
    
## <a name="verify-that-the-new-capabilities-for-ome-are-configured-properly-by-using-windows-powershell"></a>Убедитесь, что новые возможности для OME правильно настроены с помощью Windows PowerShell

Выполните следующие действия, чтобы убедиться, что клиент правильно настроено для использования новых возможностей для OME через Exchange Online PowerShell.
  
1. Использование рабочего или школы учетной записи, имеющей права глобального администратора в организации Office 365, начало сеанса Windows PowerShell и подключитесь к Exchange Online. Сведения содержатся в разделе [подключение к Exchange Online PowerShell](https://docs.microsoft.com/powershell/exchange/exchange-online/connect-to-exchange-online-powershell/connect-to-exchange-online-powershell?view=exchange-ps).
    
2. Командлет Test-IRMConfiguration, используя следующий синтаксис:
    
    ```Test-IRMConfiguration [-Sender <email address >]```  

   Пример:
    
    ```Test-IRMConfiguration -Sender securityadmin@contoso.com```

    Где адрес электронной почты — это адрес электронной почты пользователя в организации Office 365. Необязательный, предоставляя адрес электронной почты отправителя принудительно системы для выполнения дополнительных проверок.
    
    Результаты должен выглядеть эти:
    
    ```
    Results : Acquiring RMS Templates ...
                - PASS: RMS Templates acquired.  Templates available: Contoso  - Confidential View Only, Contoso  - Confidential, Do Not 
            Forward.
            Verifying encryption ...
                - PASS: Encryption verified successfully.
            Verifying decryption ...
                - PASS: Decryption verified successfully.
            Verifying IRM is enabled ...
                - PASS: IRM verified successfully.
            
            OVERALL RESULT: PASS
    ```

    Где *Contoso* заменяется именем организации Office 365. 
    
    Имена шаблонов по умолчанию, возвращаемых в результатах может отличаться от описания, отображаемых в результатах выше.
    
    Введение в шаблоны и сведения о шаблонах по умолчанию в разделе [Настройка и управление шаблонами для защиты информации Azure](https://docs.microsoft.com/information-protection/deploy-use/configure-policy-templates). Сведения о действие не пересылать параметр, параметр только для шифрования и каким образом, чтобы создать дополнительные шаблоны и узнайте, какие права включены в существующий шаблон, содержатся в разделе [Настройка правах на использование для управления правами Azure](https://docs.microsoft.com/information-protection/deploy-use/configure-usage-rights).
    
3. Выполните командлет Remove-PSSession, чтобы отключиться от службы управления правами.
    
    ```Remove-PSSession $session```

## <a name="next-steps-define-new-mail-flow-rules-that-use-the-new-ome-capabilities"></a>Дальнейшие действия: определить новые правила потока обработки почты, используйте новые возможности OME
<a name="Rules_1"> </a>

Этот шаг является необязательным для нового развертывания OME, тем не менее, этот шаг является обязательным для существующих развертываний OME, на которых уже поток правила для почты настроена на шифрование исходящей почты. Если вы хотите использовать новые возможности OME, необходимо обновить существующие правила поток почты. В противном случае пользователи будут предоставляться зашифрованных сообщений в формате предыдущей HTML вложения вместо нового эффективной работы OME.
  
Правила для потока обработки почты определить в разделе email какие условия должно быть зашифровано сообщения, а также условия для удаления шифрования. При задании действия для правила все сообщения, которые соответствуют условиям правила шифруются при отдельными.
  
Дополнительные сведения о правилах поток почты просмотрите [Определение правил поток обработки почты для шифрования сообщений электронной почты в Office 365](define-mail-flow-rules-to-encrypt-email.md).
