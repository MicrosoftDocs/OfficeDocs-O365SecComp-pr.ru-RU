# <a name="set-up-new-office-365-message-encryption-capabilities"></a>Настройка новых возможностей шифрования сообщений Office 365

Благодаря новым возможностям Microsoft 365 Message encryption (OME), которые используют функции защиты в Azure Information Protection, ваша организация может легко обмениваться защищенной электронной почтой с другими пользователями на любом устройстве. Пользователи могут отправлять и получать защищенные сообщения с другими организациями Office 365, а также клиентами, не являющимися клиентами Office 365, с помощью Outlook.com, Gmail и других почтовых служб.

Если вы просто ищете сведения о том, как отправлять или получать зашифрованные сообщения, ознакомьтесь со статьей, посвященной [шифрованИю сообщений Office 365 (OME)](ome.md) , и выберите статью, которая наилучшим образом соответствует вашим потребностям. |

## <a name="get-started-with-ome-by-activating-azure-rights-management-part-of-azure-information-protection"></a>Приступая к работе с OME, активируйте Azure Rights Management, часть Azure Information Protection

Теперь вы можете легко приступить к работе с новыми возможностями OME. По 2018 февраля Office 365 автоматически включает новые возможности OME для соответствующих организаций в наших центрах обработки данных. Ваша организация подходит только в том случае, если это новый клиент Office 365 и у вашей организации есть соответствующие подписки. **Если вы включили управление правами Azure (Azure RMS), часть Azure Information Protection, то мы автоматически включили шифрование сообщений Office 365.** Нет необходимости предпринимать какие-либо действия для включения OME. Чтобы активировать службу управления правами Azure, ознакомьтесь со статьей [Активация службы управления правами Azure](https://docs.microsoft.com/azure/information-protection/deploy-use/activate-service). Для получения сведений о подписках обратитесь к разделу "какие подписки необходимо использовать для создания нового OME Капабилитиес_км_" в статье [вопросы и ответы по шифрованИю сообщений Office 365](ome-faq.md). Сведения о приобретении подписки на Azure Information Protection можно узнать в статье [Azure Information Protection](https://azure.microsoft.com/services/information-protection/).
  
Если вы используете Exchange Online со службой управления правами Active Directory (AD RMS), вы не сможете включить эти новые возможности немедленно. Вместо этого необходимо сначала выполнить миграцию из службы управления правами Active Directory в Azure Information Protection. После завершения миграции вы можете успешно выполнить эти действия.
  
Если вы решили продолжить использование локальной службы AD RMS в Exchange Online вместо миграции на Azure Information Protection, вы не сможете использовать эти новые возможности.
  
## <a name="how-the-new-capabilities-for-ome-work"></a>Как работают новые возможности OME

Новые возможности шифрования сообщений Office 365 используют возможности защиты, также называемые Azure Rights Management (Azure RMS), из Azure Information Protection. К ним относятся политики шифрования, удостоверения и авторизации для защиты электронной почты. Сообщения можно шифровать с помощью шаблонов управления правами, [параметра](https://docs.microsoft.com/information-protection/deploy-use/configure-usage-rights#do-not-forward-option-for-emails)"не пересылать" и " [только шифровать](https://docs.microsoft.com/information-protection/deploy-use/configure-usage-rights#encrypt-only-option-for-emails)". С помощью этих параметров пользователи могут шифровать сообщения электронной почты и различные вложения Office 365. Полный список поддерживаемых типов вложений приведен [в разделе "типы файлов, включаемые политикаМИ IRM при их присоединении к сообщениям" в статье Введение в IRM для сообщений электронной почты](https://support.office.com/article/bb643d33-4a3f-4ac7-9770-fd50d95f58dc#FileTypesforIRM). Как администратор вы также можете определить правила для обработки почтового процесса, чтобы применить эту защиту. Например, можно определить правило, в котором все незащищенные сообщения, адресованные определенному получателю или содержащие определенные слова в строке темы, защищены от неавторизованного доступа, а получатели не могут копировать или печатать содержимое сообщения.
  
В отличие от предыдущей версии OME, эти новые возможности предоставляют единый интерфейс отправителя, если вы отправляете почту в пределах организации или получателям, не входящим в Office 365. Кроме того, получатели, которые получают защищенное сообщение электронной почты, отправленные в учетную запись Office 365 в Outlook 2016 или Outlook в Интернете, не должны предпринимать никаких дополнительных действий для просмотра сообщения. Он работает без проблем. Получатели, использующие другие почтовые клиенты и поставщики услуг электронной почты, также имеют улучшенные возможности. Сведения о защищенных сообщениях в Office 365 и о [том, как открыть защищенное сообщение, можно](https://support.office.com/article/How-do-I-open-a-protected-message-1157a286-8ecc-4b1e-ac43-2a608fbf3098)найти [в статье сведения о защищенных сообщениях](https://support.office.com/article/Learn-about-protected-messages-in-Office-365-2baf3ac7-12db-40a4-8af7-1852204b4b67) .

Подробный список различий между предыдущей версией OME и новыми возможностями OME приведен в статье [Сравнение версий OME](ome-version-comparison.md).
  
## <a name="steps-to-manually-set-up-the-new-capabilities-for-ome"></a>Действия по настройке новых возможностей для OME вручную

Новые возможности OME автоматически включены для большинства организаций Office 365. Если в вашей организации автоматически не включен OME, или если вы включили новые возможности OME, выполните следующие действия, чтобы вручную настроить новые возможности для OME.
  
### <a name="to-manually-set-up-the-new-capabilities-for-ome"></a>Настройка новых возможностей для OME вручную

1. Убедитесь, что у вас есть подправная подписка для вашей организации. Сведения о подписках в разделе "какие подписки необходимо использовать для создания нового OME Капабилитиес_км_" в статье [вопросы и ответы по шифрованИю сообщений Office 365.](ome-faq.md). Сведения о приобретении подписки на Azure Information Protection можно узнать в статье [Azure Information Protection](https://azure.microsoft.com/services/information-protection/).

2. Решите, хотите ли вы, чтобы корпорация Майкрософт управляла корневым ключом для Azure Information Protection (по умолчанию) или создавала и управляла этот ключ самостоятельно (под названием "переместить свой ключ" или "БЙОК"). Если вы хотите самостоятельно создать этот ключ и управлять им, необходимо выполнить некоторые действия перед настройкой новых возможностей для OME. Для получения дополнительных сведений ознакомьтесь с разделом [планирование и реализация ключа клиента Azure Information Protection](https://docs.microsoft.com/information-protection/plan-design/plan-implement-tenant-key). Перед настройкой OME Корпорация Майкрософт рекомендует выполнить указанные ниже действия.

3. Активируйте новые возможности для OME, активируя управление правами Azure. Инструкции см в разделе [Активация службы управления правами Azure](https://docs.microsoft.com/azure/information-protection/deploy-use/activate-service). После этого Office 365 автоматически включает новые возможности OME.

    > [!TIP]
    > Outlook в Интернете кэширует свой пользовательский интерфейс, поэтому рекомендуется подождать некоторое время, прежде чем попытаться применить новые возможности для OME к сообщениям электронной почты с помощью этого клиента. Перед обновлением ПОЛЬЗОВАТЕЛЬСКОГО интерфейса в соответствии с новой конфигурацией новые возможности OME не будут доступны. После обновления ПОЛЬЗОВАТЕЛЬСКОГО интерфейса пользователи могут защищать сообщения электронной почты с помощью новых возможностей OME.
  
4. Необязательно Настройте новые правила для почтовых ящиков или обновите существующие правила для обработки почты, определяющие, как и когда вы хотите, чтобы Office 365 зашифровать сообщения, отправленные из вашей организации.

## <a name="verify-that-the-new-capabilities-for-ome-are-configured-properly-by-using-windows-powershell"></a>Проверка правильности настройки новых возможностей OME с помощью Windows PowerShell

Выполните следующие действия, чтобы убедиться, что клиент правильно настроен для использования новых возможностей OME через Exchange Online PowerShell.
  
1. С помощью рабочей или учебной учетной записи с разрешениями глобального администратора в организации Office 365 запустите сеанс Windows PowerShell и подключитесь к Exchange Online. Инструкции можно найти [в статье подключение к Exchange Online PowerShell](https://aka.ms/exopowershell).

2. Запустите командлет Test-IRMConfiguration, используя следующий синтаксис:

     ```powershell
     Test-IRMConfiguration [-Sender <email address >]
     ```  

   Пример:

     ```powershell
     Test-IRMConfiguration -Sender securityadmin@contoso.com
     ```

    Где адрес электронной почты это адрес электронной почты пользователя в организации Office 365. При необязательном предоставлении электронного адреса отправителя заставляет систему выполнять дополнительные проверки. Результаты должны выглядеть следующим образом:

     ```text
    Results : Acquiring RMS Templates ...
                - PASS: RMS Templates acquired.  Templates available: Contoso  - Confidential View Only, Contoso  - Confidential, Do Not 
            Forward.
            Verifying encryption ...
                - PASS: Encryption verified successfully.
            Verifying decryption ...
                - PASS: Decryption verified successfully.
            Verifying IRM is enabled ...
                - PASS: IRM verified successfully.

            OVERALL RESULT: PASS
    ```

    Где *contoso* заменяется именем вашей организации Office 365.

    Имена шаблонов по умолчанию, возвращаемые в результатах, могут отличаться от имен, отображаемых в приведенных выше результатах.

    Общие сведения о шаблонах и сведения о шаблонах по умолчанию приведены в статье [Настройка шаблонов для Azure Information Protection и управление ими](https://docs.microsoft.com/information-protection/deploy-use/configure-policy-templates). Сведения о параметрах "не переСылать", "только шифровать", а также о том, как создавать дополнительные шаблоны или узнать, какие права включены в существующий шаблон, можно узнать в статье [Настройка прав использования для управления правами Azure](https://docs.microsoft.com/information-protection/deploy-use/configure-usage-rights).

3. Запустите командлет reMove – PSSession, чтобы отключиться от службы управления правами.
    
     ```powershell
     Remove-PSSession $session
     ```

## <a name="next-steps-define-new-mail-flow-rules-that-use-the-new-ome-capabilities"></a>Дальнейшие действия: определение новых правил для почтовых ящиков, использующих новые возможности OME

Это необязательный шаг для новых развертываний OME, но этот шаг необходим для существующих развертываний OME, для которых уже настроены правила обработки почтового ящика для шифрования исходящей почты. Если вы хотите использовать новые возможности OME, необходимо обновить существующие правила для обработки почты. В противном случае ваши пользователи продолжат получать зашифрованные сообщения, в которых используется предыдущий формат вложений HTML, а не новые, неудобство работы с OME.
  
Правила для почтового процесса определяют, при каких условиях сообщения электронной почты должны шифроваться, а также условия удаления этого шифрования. Когда вы настраиваете действие для правила, все сообщения, которые удовлетворяют условиям правила, шифруются при отправке.
  
Дополнительные сведения о правилах для обработки почтового процесса [можно узнать в статье Определение правил обработки почтового ящика для шифрования сообщений электронной почты в Office 365](define-mail-flow-rules-to-encrypt-email.md).
