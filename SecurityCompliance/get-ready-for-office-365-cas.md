# <a name="get-ready-for-office-365-cloud-app-security"></a>Подготовка к использованию Office 365 Cloud App Security

При подготовке к включению и внедрению Microsoft Office 365 Cloud App Security для вашей организации необходимо учесть несколько моментов. Используйте эту статью в качестве руководства по планированию Office 365 Cloud App Security.
    
## <a name="step-1-identify-and-protect-your-global-and-security-administrator-accounts"></a>Шаг 1: идентификация и защита глобальных учетных записей и учетных записей администраторов безопасности

Глобальные администраторы, администраторы безопасности и средства чтения безопасности могут получать доступ к порталу Cloud App Security (Office 365) для просмотра политик, проверки оповещений и использования отчетов. Глобальные администраторы и администраторы безопасности могут определять политики и выполнять другие действия для защиты Организации. (Дополнительные сведения см. [в разделе разрешения в центре &amp; безопасности Office 365](permissions-in-the-security-and-compliance-center.md).) ИзУчите учетные записи пользователей организации с повышенными правами в качестве меры предосторожности. 
  
 **[Защитите учетные записи глобального администратора Office 365](https://docs.microsoft.com/office365/enterprise/protect-your-global-administrator-accounts)**. 
  
## <a name="step-2-turn-on-audit-logging-for-your-organization"></a>Шаг 2: Включение ведения журнала аудита для Организации

Для правильной работы Office 365 Cloud App Security необходимо включить ведение журнала аудита. Это обычно делается администратором Exchange Online или глобальным администратором.
  
 **[Включение и отключение поиска в журнале аудита Office 365](turn-audit-log-search-on-or-off.md)**. 
  
## <a name="step-3-go-to-the-office-365-cloud-app-security-portal"></a>Шаг 3: перейдите на портал Cloud App Security (Office 365)

Вы можете перейти на портал Office 365 Cloud App Security, перейдя по [https://portal.cloudappsecurity.com](https://portal.cloudappsecurity.com) адресу и войдя в систему. 

Вы также можете получить его из центра безопасности &amp; Office 365. Вот один хороший способ выполнения этой задачи:

1. Перейдите на [https://protection.office.com](https://protection.office.com) страницу и выполните вход. (откроется центр соответствия требованиям безопасности &amp; .)
    
2. Перейдите к разделу **оповещения** \> **Управление расширенными оповещениями**.
    
3. Выберите **Перейти к office 365 Cloud App Security** , чтобы перейти на портал Office 365 Cloud App Security.<br> ![Выберите Управление расширенными оповещениями, чтобы перейти к Office 365 Cloud App Security](media/958632d4-03e3-4ade-8e22-d5509db6fca7.png)<br>При переходе на портал Cloud App Security (Office 365) первая показанная страница является страницей "политики", которая напоминает следующий рисунок:<br>![При переходе на портал Cloud App Security (Office 365) вы можете начать со страницы "политики"](media/5cb8833c-4e08-438c-bab3-91b5106f6f3f.png)<br>
  
## <a name="step-4-define-policies-and-set-up-alerts-amp-actions"></a>Шаг 4: определение политик и Настройка действий с &amp; оповещениями

Глобальные администраторы и администраторы безопасности определяют политики в Office 365 Cloud App Security. В процессе определения политик также устанавливаются оповещения и действия. Предупреждение — это уведомление на основе критериев, которое отображается в представлении или отправляется по электронной почте. 
  
Существует два типа оповещений в Office 365 Cloud App Security: оповещения обнаружения аномалий, которые определяют подозрительные действия и оповещения о действиях, которые определяются для действий, которые могут быть нетипичными для Организации. Оповещения уведомляют глобальных администраторов и администраторов безопасности о том, что в вашей среде Office 365 есть нестандартные действия в вашей организации.
  
Чтобы узнать больше, ознакомьтесь со следующими материалами:
  
- [Политики действий и оповещения в Office 365 Cloud App Security](activity-policies-and-alerts.md)
    
- [Политики обнаружения аномалий в Office 365 Cloud App Security](anomaly-detection-policies-in-ocas.md)
    
- [Проверка оповещений системы безопасности облачного приложения Office 365 и выполнение действий с ними](review-office-365-cas-alerts.md)
    

## <a name="step-5-set-up-conditional-access-app-control"></a>Шаг 5: Настройка управления приложениями условного доступа

Настройка и принудительное применение элементов управления в приложениях организации на основе определенных условий, таких как пользователи, которые могут использовать приложения и где. Определите пользовательские приложения и политики сеансов, чтобы определить, могут ли конфиденциальные документы загружаться и шифроваться, блокировать доступ к определенным приложениям, настраивать режим только для чтения для определенных приложений и ограничивать сеансы пользователей из некорпоративных сетей.

Чтобы узнать больше, ознакомьтесь со следующими материалами:

- [Защита приложений с помощью управления условным доступом к приложениям в Office 365 Cloud App Security](ocas-conditional-access-app-control.md)

- [Развертывание управления условным доступом к приложениям для приложений Office 365](ocas-deploy-conditional-access-app-control.md)

## <a name="step-6-learn-about-your-organizations-cloud-usage"></a>Шаг 6: сведения об использовании облачной организации

Как глобальный администратор, администратор безопасности или читатель безопасности вы можете узнать об использовании облачной среды в отчетах и информационной панели облачного обнаружения (также называемой обнаружением приложений для продуктивности). На этой панели мониторинга отображаются сведения о пользователях, приложениях, веб-трафике и уровнях риска.
  
![На портале Office 365 CAS выберите пункт Обнаружение \> облачной панели мониторинга](media/61269290-fd82-4d4b-8045-aea1ebc82287.png)
  
чтобы перейти на панель мониторинга обнаружения приложений для продуктивной работы, на портале Cloud app Security (Office 365) выберите пункт **обнаружение** \> **облачной панели мониторинга**.
  
![На портале Office 365 CAS нажмите кнопку Обнаружение](media/73b5299f-94b5-49dd-a00f-154d188eb2c5.png)
  
Чтобы заполнить отчеты с нужными сведениями, отправьте файлы журнала из брандмауэров и прокси-серверов вашей организации. Чтобы узнать больше, ознакомьтесь со следующими материалами:
  
- [Создание отчетов об обнаружении приложений в Office 365 Cloud App Security](create-app-discovery-reports-in-ocas.md)
    
- [Просмотр результатов обнаружения приложений в Office 365 Cloud App Security](review-app-discovery-findings-in-ocas.md)
    
## <a name="step-7-manage-apps-that-your-organization-is-using-to-access-office-365"></a>Шаг 7: Управление приложениями, которые ваша организация использует для доступа к Office 365

Как глобальный администратор или администратор безопасности, вы можете управлять приложениями, такими как настраиваемые приложения или сторонние приложения, которые сотрудники организации используют на своих устройствах с Office 365. Например, предположим, что кто-то загрузил пользовательское приложение, которое необходимо использовать с Office 365. Вы можете просмотреть, какие приложения работают с пользователями, не доверять приложениям или пометить приложения как утвержденные для целей отслеживания. [Управление приложениями OAuth с помощью Office 365 Cloud App Security](manage-app-permissions-in-ocas.md).
  
## <a name="step-8-create-a-maintenance-plan"></a>Шаг 8: создание плана обслуживания

После настройки и настройки Office 365 Cloud App Security вы захотите выполнить определенные задачи в качестве глобального администратора Office или администратора безопасности для вашей организации. Выполняя эти задачи, вы можете убедиться, что Office 365 Cloud App Security настроен правильно, ваши политики обновлены, а ваша организация понимает значение из Office 365. ## <a name="optional-step-9-use-your-siem-server-with-office-365-cloud-app-security"></a>Необязательно Шаг 9: использование сервера SIEM с Office 365 Cloud App Security

Использует сервер управления событиями и управления событиями (SIEM)? Теперь приложение Office 365 Cloud App Security можно интегрировать с сервером SIEM для обеспечения централизованного мониторинга оповещений. Интеграция со службой SIEM позволяет улучшить защиту облачных приложений, сохраняя при этом обычный рабочий процесс безопасности, автоматизируя процедуры безопасности и согласование между облачными и локальными событиями. Агент SIEM запускается на сервере, отправляет оповещения из Office 365 Cloud App Security и передает эти оповещения на сервер SIEM. Ознакомьтесь [с интеграциЕй SIEM с Office 365 Cloud App Security](integrate-your-siem-server-with-office-365-cas.md).
