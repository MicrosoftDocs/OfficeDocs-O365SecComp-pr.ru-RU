# <a name="anomaly-detection-policies-in-office-365-cloud-app-security"></a>Политики обнаружения аномалий в Office 365 Cloud App Security

Расширенное управление безопасностью для Office 365 является теперь приложения облаке Безопасность в Office 365.
  
|Оценка **\>**|Планирование **\>**|Развертывание **\>**|Использование ***|
|:-----|:-----|:-----|:-----|
|[Начать оценку](office-365-cas-overview.md) <br/> |[Начать планирование](get-ready-for-office-365-cas.md) <br/> |Вы находитесь здесь!  <br/> [Следующее действие](integrate-your-siem-server-with-office-365-cas.md) <br/> |[Начать использование](utilization-activities-for-ocas.md) <br/> |
   
Приступая к работе с [безопасности приложения Microsoft Cloud освобождение 116](https://docs.microsoft.com/cloud-app-security/release-notes), облачных приложений Office 365 безопасности включает в себя несколько предопределенных неполадок обнаружения политики («по умолчанию»), которые включают в себя пользователей и поведения analytics сущности (UEBA) и машинного обучения (ML).
  
![Для просмотра политик обнаружения неполадок нажмите элемент управления \> политик.](media/9663baa5-98bf-45e0-9458-6e572b43ec72.png)
  
Эти политики обнаружения неполадок предоставления интерпретации результатов с указанием интерпретации обнаружения, ориентация многочисленные поведения ошибок для пользователей и компьютеров и устройств, подключенных к сети. В дополнение к этому новые политики предоставляют дополнительные данные из модуля обнаружения безопасности приложения облачных помогут вам ускорить процесс расследования и содержит существующие угрозы.
  
Как [глобальный администратор или администратор безопасности](permissions-in-the-security-and-compliance-center.md)можно просмотреть и при необходимости изменить политики по умолчанию, доступные в облаке приложения Office 365 безопасности.
  
 > [!IMPORTANT]
> Существует от начального обучения период семь (7) дней, во время которых непредвиденных поведение оповещения не запускаются. Алгоритм обнаружения неполадок оптимизирован для уменьшения количества ложных положительных оповещения. 
  
## <a name="before-you-begin"></a>Перед началом работы

Убедитесь, что:
  
- Вашей организации есть [Облачных приложений Office 365 безопасности](office-365-cas-overview.md)и служба является [включенным](turn-on-office-365-cas.md).
    
- [Ведение журнала аудита](turn-audit-log-search-on-or-off.md) для вашей среде Office 365 включено. 
    
- Вы являетесь глобального администратора или администратора безопасности для Office 365.
    
## <a name="view-your-anomaly-detection-policies"></a>Просмотр политик обнаружения неполадок

1. Как глобальный администратор или администратор безопасности, перейдите к [https://security.microsoft.com](https://security.microsoft.com) и выполнить вход с помощью учетной записи рабочего или школы. 
    
2. В разделе Безопасность &amp; центре соответствия требованиям, выберите **оповещения** \> **Расширенное Управление оповещениями**.
    
3. Выберите, **перейдите к безопасности приложения Office 365 облака**.
    
    Откроется страница политики безопасности Office 365 облачных приложений.
    
4. В списке **Тип** выберите **политику обнаружения неполадок**.
    
    Вашей организации по умолчанию (или имеющиеся) отображаются политики обнаружения неполадок.
    
    ![Несколько политик обнаружения неполадок доступны в облаке приложения Office 365 безопасности по умолчанию](media/2e0ee770-787a-4d4a-bea8-389dc765d4c6.png)
  
5. Выберите политику, чтобы просмотреть или изменить его параметры.
    
6. Нажмите кнопку **Update** (Обновить), чтобы сохранить изменения. 
    
## <a name="learn-more-about-anomaly-detection-policies"></a>Узнайте больше о политиках обнаружения неполадок

Политики обнаружения неполадок автоматически включены; Тем не менее безопасности Office 365 облаке приложения имеет от начального обучения период семь дней, во время которых не все неполадок создаются обнаружения оповещения. После этого каждого сеанса сравнивается с действия, когда пользователи были активны, IP-адресов, устройствах, обнаруженных за последние месяц и оценки риска из этих действий и т. д. Обнаружение этих программ входят в состав модуля обнаружения неполадок эвристическое, профили среды и отправляет оповещения по отношению к базовой, который был получен по деятельности вашей организации. Обнаружение этих программ также использовать алгоритмы обучения компьютера предназначена для профилей пользователей и шаблоны в журнал для сокращения числа ошибочных срабатываний фильтра.
  
Ошибки обнаруженные при проверке активности пользователей. Риск оценивается, посмотрев более 30 различных риска индикаторов, сгруппированы в нескольких риска факторов, например рискованный IP-адрес, ошибки входа, действия администратора, неактивных учетных записей, расположение, невозможно поездки, устройства и агента пользователя и скорость активности.
  
На основании результатов политики, которые запускаются оповещение системы безопасности. Безопасности облаке приложения Office 365 рассматривается каждого сеанса пользователя в Office 365 и предупреждением каждый раз, когда что-то, которое будет отличаться из базового вашей организации или регулярного активности пользователя.

## <a name="triage-anomaly-detection-alerts"></a>Функция обнаружения неполадок рассмотрения оповещений
<a name="triage"> </a>

Как бывают оповещения можно быстро рассмотрения эти оповещения и определить, какие из них следует обрабатывать сначала. Наличие контекста для оповещения позволяет увидеть полную картину и определить, является ли что-то вредоносных действительно сведений о событиях. Используйте следующую процедуру для начала работы изучение оповещения:
  
1. Как глобальный администратор или администратор безопасности, перейдите к [https://security.microsoft.com](https://security.microsoft.com) и выполнить вход с помощью учетной записи рабочего или школы. 
    
2. В разделе Безопасность &amp; центре соответствия требованиям, выберите **оповещения** \> **Расширенное Управление оповещениями**.
    
3. Выберите, **перейдите к безопасности приложения Office 365 облака**.
    
4. Выберите **оповещения** для просмотра оповещений. 
    
5. Чтобы получить контекст для оповещения, выполните следующие действия:
    
1. Нажмите кнопку **Проверить** \> **Журнал активности**.
    
2. Выберите элемент, например, пользователя или IP-адрес. Откроется ящика актуальных.
    
    ![В журнале активности можно изучить IP-адрес.](media/32a727c5-e406-4fe2-9443-c1a7fb6628fc.png)
  
3. В ящике актуальных щелкните доступные команды, например значок в разделе **Показать СЛЕДУЮЩИЙ** . 
    
    ![В ящике актуальных щелкнуть значок часов для просмотра действий, выполняемых в течение 48 часов выбранного мероприятия](media/c6c96aa0-98e5-4205-8873-45f8d6fd0843.png)
  
4. Получить представление о выбранном элементе, продолжая для просмотра сведений для этого элемента.
    
Оповещение на несколько неуспешные действительно могут быть подозрительные и можно указать потенциальных перебора. Тем не менее такие уведомления также может быть неправильной настройке приложения, вызывающие оповещения, чтобы быть опасности положительное значение true. Если оповещение не удалось logins несколько дополнительных подозрительных действий, существует больше вероятность, что учетной записи раскрыты. Например, предположим, что оповещение Сбой входа несколькими следуют активности из сети IP-адрес и активности невозможно travel обоих строгое индикаторы раскрытия. Можно даже увидеть, что же пользователь выполнит активности массовой загрузки, которая, как правило, индикатор злоумышленник exfiltration данных для выполнения. Например, что можно изучить безопасности Office 365 облаке приложения для просмотра и рассмотрения оповещения и выполнение действий при необходимости.
