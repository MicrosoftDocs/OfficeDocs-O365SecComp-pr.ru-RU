# <a name="search-the-office-365-audit-log-to-troubleshoot-common-scenarios"></a>Поиск в журнале аудита Office 365 для устранения распространенных сценариев

В этой статье описывается, как использовать средства поиска в журнале аудита Office 365 для помощи в устранении распространенных сценариев поддержки. Это включает в себя с помощью журнала аудита для:

- Найти IP-адрес компьютера, используемого для доступа к атаке учетной записи
- Определение пользователей, которые Настройка пересылки электронной почты для почтового ящика
- Определение, если пользователь удаленных элементов электронной почты в почтовом ящике
- Определить, если пользователь создавать правила папки «Входящие»

## <a name="using-the-office-365-audit-log-search-tool"></a>С помощью средства поиска в журнале аудита Office 365

Каждый из сценарии устранения неполадок, описанных в этой статье основаны на использование средства поиска журнала аудита в Office 365 безопасности & центре соответствия требованиям. В этом разделе перечислены разрешения, необходимые для поиск в журнале аудита и описывается, как получить доступ к и запустите поисков в журнале аудита. Разделе каждого сценария приведены конкретные инструкции по настройке запроса поиска журнала аудита и что искать подробные сведения о в записей аудита, которые соответствуют условиям поиска.

### <a name="permissions-required-to-use-the-audit-log-search-tool"></a>Разрешения, необходимые для использования средства поиска в журнале аудита

Вы должны быть назначены роли журналы аудита только для просмотра или журналов аудита в Exchange Online поиск в журнале аудита Office 365. По умолчанию эти роли, назначенные группы ролей управления организацией и Управление соответствием требованиям на странице " **разрешения** " в центре администрирования Exchange. Для получения дополнительных сведений см [групп ролей управление Exchange Online](https://go.microsoft.com/fwlink/p/?LinkID=730688).

### <a name="running-audit-log-searches"></a>Выполнение поисков в журнале аудита

В этом разделе описываются основные сведения по созданию и использованию поисков в журнале аудита. Используйте следующие инструкции в качестве отправной точки для каждого сценария по устранению неполадок в этой статье. Более подробные пошаговые инструкции в разделе [Поиск журнала аудита в Office 365 безопасности & центре соответствия требованиям ](search-the-audit-log-in-security-and-compliance.md#step-1-run-an-audit-log-search).

1. Перейдите по ссылке [https://protection.office.com](https://protection.office.com).
  
2. Войдите в Office 365 с помощью своей рабочей или учебной учетной записи.

3. В левой области безопасности & центре соответствия требованиям, нажмите кнопку **поиска и изучения** > **поиска журнала аудита**.
    
    Отображается страница **поиска журнала аудита** . 
    
    ![Настройка критериев и нажмите кнопку Поиск, чтобы выполнить поиск](media/8639d09c-2843-44e4-8b4b-9f45974ff7f1.png)
  
4. Можно настроить следующие условия поиска. Обратите внимание, что в каждом сценарии устранения неполадок в этой статье будет рекомендуем конкретные инструкции по настройке этих полей.
    
    a. **мероприятий** - нажмите кнопку раскрывающегося списка для отображения действия, которые можно выполнить поиск. После запуска поиска отображаются только записи аудита для выбранного действия. Выбрав **Показать результаты для всех действий** будет отображаться результаты для всех действий, необходимых для других критериев поиска. Также необходимо хранить оставьте это поле пустым, в некоторых сценарии устранения неполадок.
    
    б. **Дата начала** и **Дата окончания** — выберите дату и время диапазона просмотра событий, возникших в течение этого периода. По умолчанию выбрано за последние семь дней. Дата и время представлены в формате по Гринвичу (UTC). Максимальный период, которые можно выбрать — 90 дней.

    c. **пользователей** — щелкните в этом поле и затем выберите одного или нескольких пользователей, чтобы отобразить поиска по результатам. В списке результатов отображаются записи аудита для выбранной операции, выполненные для пользователей, выбранного в этом поле. Оставьте это поле пустым для возвращения записей для всех пользователей (и учетных записей служб) в организации.
    
    г., **файла, папки или веб** - введите некоторые или все имя файла или папки для поиска действия, связанные с файлов из папки, содержащей указанного ключевого слова. Можно также указать URL-адрес файла или папки. Если вы используете: URL-адрес, должен быть выбран тип полный URL-адрес или просто введите часть URL-адреса, не добавляйте любые специальные символы или пробелы. Оставьте это поле пустым для возвращения записей для всех файлов и папок в организации. Обратите внимание на то, что это поле оставлено пустым в все сценарии устранения неполадок в этой статье.
    
5. Нажмите кнопку **поиска** , чтобы запустить поиск, используя условия поиска. 
    
    Загрузить результаты поиска, а после некоторое время они отображаются в области **результатов** на странице **поиска журнала аудита** . Каждый в следующих разделах будет предоставлять инструкции о том, что следует искать определенного сценария устранения неполадок.

## <a name="finding-the-ip-address-of-the-computer-used-to-access-a-compromised-account"></a>Поиск IP-адрес компьютера, используемого для доступа к атаке учетной записи

IP-адрес, соответствующий операции, выполненные любым пользователем включается в большинстве записей аудита. Сведения о клиента также включаются в записи аудита.

Вот как настроить запрос поиска журнала аудита для этого сценария:

**Действия** — при необходимости в случай выберите определенное действие для поиска. Для устранения неполадок атаке учетных записей рассмотрите возможность выбора действий **пользователь входил почтовый ящик** в разделе **действия почтового ящика Exchange**. Возвращает записи аудита, отражающая IP-адрес, который был использования при входе в почтовый ящик. В противном случае оставьте это поле пустым, чтобы возвращать записи аудита для всех действий. 

> [!TIP]
> Оставить это поле пустым возвращает **UserLoggedIn** действий, которая представляет собой Azure Active Directory действие, которое указывает, что кто-то вошел в учетной записи пользователя в Office 365. Используйте фильтры в окне результатов поиска для отображения записей аудита **UserLoggedIn** .

**Дата начала** и **Дата окончания** — выберите диапазон дат, который можно применять для расследования.

**Пользователи** — Если исследование атаке учетную запись, выберите атаке, учетную запись пользователя. Возвращает записей аудита для действий, выполняемых с этой учетной записи пользователя.

**Файла, папки или веб** - оставьте это поле пустым.

После выполнения поиска IP-адрес для каждого действия отображается в столбце **IP-адрес** в результатах поиска. Щелкните запись в списке результатов поиска для просмотра более подробные сведения на странице всплывающее меню.

## <a name="determining-who-set-up-email-forwarding-for-a-mailbox"></a>Определение пользователей, которые Настройка пересылки электронной почты для почтового ящика

Если переадресация почты настроена для почтового ящика, сообщений электронной почты, отправленных в почтовом ящике будут переадресовываться другого почтового ящика. Можно пересылать сообщения пользователям внутри или за пределами вашей организации. При настройке пересылки электронной почты для почтового ящика командлета базовым Exchange Online, используемый — **Set-Mailbox**.

Вот как настроить запрос поиска журнала аудита для этого сценария:

**Действия** - оставьте это поле пустым, чтобы возвращаются записи аудита для всех действий. Это необходимо возвращать аудита записей, связанных с командлетом **Set-Mailbox** .

**Дата начала** и **Дата окончания** — выберите диапазон дат, который можно применять для расследования.

**Пользователи** — Если не исследование переадресация проблемы для определенного пользователя электронной почты, оставьте это поле пустым. Эта информация поможет определять, соответствует ли переадресация почты был настроен для любого пользователя.

**Файла, папки или веб** - оставьте это поле пустым.

После запуска поиска нажмите кнопку **фильтрации результатов** на странице результатов поиска. В поле под заголовком столбца **активности** введите **Set-Mailbox** для отображения только записей аудита, связанных с командлетом **Set-Mailbox** .

![Фильтрация результаты поиска журнала аудита](media/emailforwarding1.png)

На этом этапе необходимо просмотреть сведения о каждой записи аудита, чтобы определить, если действие относится к пересылки электронной почты. Нажмите кнопку записи аудита для отображения всплывающих страница **сведений о** и нажмите кнопку **Подробнее**. Следующем снимке экрана и описания основные заданные сведения, которые указывают пересылки электронной почты на почтовом ящике.

![Подробные сведения из записи аудита](media/emailforwarding2.png)

а. в поле **ObjectId** было задано псевдоним почтового ящика, который пересылки электронной почты на отображается. Этот почтовый ящик также отображается в столбце **элемента** на странице результатов поиска.

б. в поле **Параметры** значение *ForwardingSmtpAddress* указывает, что для почтового ящика электронной почты пересылать установлена. В этом примере почта перенаправляется mike@contoso.com адрес электронной почты, который находится за пределами организации alpinehouse.onmicrosoft.com.

c. значение *True* для параметра *DeliverToMailboxAndForward* указывает, что копия сообщения доставляются sarad@alpinehouse.onmicrosoft.com *и* будет переведен на адрес электронной почты, указанный с *ForwardingSmtpAddress *параметр, который в данном примере — это mike@contoso.com. Если значение параметра *DeliverToMailboxAndForward* присвоено *значение False*, электронной почты на адрес, указанный в параметре *ForwardingSmtpAddress* пересылается только. Он не удается доставить в почтовый ящик, указанный в поле **ObjectId** .

г. в поле **идентификатор пользователя** указывается пользователь, который установленный пересылки электронной почты для почтового ящика, указанное в поле поле **ObjectId** . Этот пользователь также отображается в столбце **пользователя** на странице результатов поиска. В этом случае она, кажется, что владелец почтового ящика значение пересылки электронной почты в свой почтовый ящик.

Если вам определить, что переадресация почты не следует установленный для почтового ящика, его можно удалить, выполнив следующую команду в Exchange Online PowerShell:

```
Set-Mailbox <mailbox alias> -ForwardingSmtpAddress $null 
```

Дополнительные сведения о параметрах, связанные с электронной почты переадресации в статье [Set-Mailbox](https://docs.microsoft.com/powershell/module/exchange/mailboxes/set-mailbox) .

## <a name="determining-if-a-user-deleted-email-items"></a>Определение, если пользователь удаленных элементов электронной почты

Перед удалением журнала аудита записи о электронной почты элементы сохраняются в журнал аудита Office 365, аудита почтовых ящиков должна быть включена для каждого почтового ящика пользователя в организации. Кроме того SoftDelete и HardDelete почтовых ящиков действия должны быть включены для аудита. Сведения содержатся в разделе [Включить аудит в Office 365 почтового ящика](enable-mailbox-auditing.md). Если аудита почтовых ящиков уже включен для пользователей, выполните следующие действия для поиска журнала аудита для событий, связанных с элементами удаленных сообщений.

Вот как настроить запрос поиска журнала аудита для этого сценария:

**Действия** — в разделе **действия почтового ящика Exchange**выберите один или оба следующих действия:

- **Удаленные сообщения из папки "Удаленные"** — это действие соответствует **SoftDelete** почтовых ящиков, аудит действий. Это действие также — это пользователь окончательно удаляет элемент, выбрав его и нажав клавишу **Shift + Delete**. После элемента удаляется без возможности восстановления, пользователь может восстановить до истечения срока хранения удаленного элемента.

- **Purged сообщения из почтового ящика** — это действие соответствует **HardDelete** почтовых ящиков, аудит действий. Это — это пользователь, удаляет элемент из папки элементов для восстановления. Администраторы могут использовать средство поиска контента в Office 365 безопасности & центре соответствия требованиям для поиска и восстановления очищены элементов до истечения срока хранения удаленного элемента или больше времени, если почтовый ящик пользователя находится на удержании.

**Дата начала** и **Дата окончания** — выберите диапазон дат, который можно применять для расследования.

**Пользователи** — при выборе пользователем в этом поле, средства поиска в журнале аудита возвращает записей аудита для элементов электронной почты, которые были пользователем, указываемые удаленных (SoftDeleted или HardDeleted). В некоторых случаях пользователя, который удаляет сообщение электронной почты может оказаться владельца почтового ящика.

**Файла, папки или веб** - оставьте это поле пустым.

После запуска поиска можно отфильтровать результаты поиска для отображения записей аудита для удаленные элементы или -удаленных элементов. Нажмите кнопку записи аудита для отображения всплывающих страница **сведений о** и нажмите кнопку **Подробнее**. Дополнительные сведения об удаленных элементов, таких как строку темы и расположения элемента, когда он был удален, отображается в поле **AffectedItems** . Следующие снимки экрана Показать пример **AffectedItems** поля из удаленного элемента и -удаленных элементов.

### <a name="recovering-deleted-email-items"></a>Восстановление удаленных элементов электронной почты

Пользователей можно восстановить удаленные элементы, если не истек срок хранения удаленных элементов. В Exchange Online период хранения элементов удаления по умолчанию установлено 14 дней, но администраторы могут увеличить этот параметр, чтобы не более 30 дней. Точка пользователям статье [Восстановление удаленных элементов или электронной почты в Outlook Web App](https://support.office.com/article/Recover-deleted-items-or-email-in-Outlook-Web-App-C3D8FC15-EEEF-4F1C-81DF-E27964B7EDD4) для получения инструкций по восстановлению удаленных элементов.

Как объяснялось ранее "Администраторы" может иметь возможность восстановления-удаленные элементы в случае не истек срок хранения удаленного элемента или если почтовый ящик находится на удержание, в противном случае элементы будут храниться до истечения срока действия продолжительность удержания. При выполнении поиска контента удаленные и удаленных элементов в папке элементов для восстановления возвращаются в результатах поиска, если они соответствуют поисковый запрос. Дополнительные сведения о запуске поиск видеть [Поиска контента в Office 365](content-search.md).

> [!TIP]
> Поиск удаленных элементов электронной почты, поиска полностью или частично строку темы, которая отображается в поле **AffectedItems** в записи аудита.

## <a name="determining-if-a-user-created-an-inbox-rule"></a>Определение, если пользователь создан правила папки «Входящие»

При создании правила папки «Входящие» для их почтовых ящиков Exchange Online, соответствующей записи аудита сохраняется в журнал аудита. Дополнительные сведения о правилах папки «Входящие» можно:

- [Использование правил папки «Входящие» в Outlook в Интернете](https://support.office.com/article/use-inbox-rules-in-outlook-on-the-web-8400435c-f14e-4272-9004-1548bb1848f2)
- [Управление сообщениями электронной почты в Outlook с помощью правил](https://support.office.com/article/Manage-email-messages-by-using-rules-C24F5DEA-9465-4DF4-AD17-A50704D66C59)

Вот как настроить запрос поиска журнала аудита для этого сценария:

**Действия** — в разделе **действия почтового ящика Exchange**выберите **Create New-InboxRule, изменение аудио- и включения и отключения правила папки «Входящие»**.

**Дата начала** и **Дата окончания** — выберите диапазон дат, который можно применять для расследования.

**Пользователи** — Если не исследование отдельного пользователя, оставьте это поле пустым. Эта информация поможет идентифицировать новые правила папки "Входящие", настроить любым пользователем.

**Файла, папки или веб** - оставьте это поле пустым.

После выполнения поиска записей аудита для этого действия отображаются в результатах поиска. Нажмите кнопку записи аудита для отображения всплывающих страница **сведений о** и нажмите кнопку **Подробнее**. В поле **Параметры** отображаются сведения о параметрах правила папки «Входящие». Следующие снимок экрана и описания рассматриваются со сведениями о правила папки «Входящие».

![Записи аудита для нового правила папки «Входящие»](media/NewInboxRuleRecord.png)

а. в поле **ObjectId** отображается полное имя правила папки «Входящие». Это имя включает в себя псевдоним почтового ящика пользователя (например, SaraD) и имя правила папки «Входящие» (например, «перемещение сообщений из admin»).

б. в поле « **Параметры** » отображается условие правила папки «Входящие». В этом примере условие задается параметром *из* . Значение, определенное для параметра *From* указывает, что электронной почты, отправленные с admin@alpinehouse.onmicrosoft.com обрабатывает правила папки «Входящие». Полный список параметров, которые могут использоваться для определения условий правила папки "Входящие" в статье [New-InboxRule](https://docs.microsoft.com/powershell/module/exchange/mailboxes/new-inboxrule) .

c. параметр *MoveToFolder* указывает действие правила папки "Входящие"; в этом примере сообщений, полученных от admin@alpinehouse.onmicrosoft.com перемещаются в папку с именем *AdminSearch*. Также в статье [New-InboxRule](https://docs.microsoft.com/powershell/module/exchange/mailboxes/new-inboxrule) содержит полный список параметров, которые можно использовать для определения действие правила папки «Входящие».

d. поле **UserId** указывает пользователя, создавшего правило папки "Входящие", указанное в поле **ObjectId** . Этот пользователь также отображается в столбце **пользователя** на странице результатов поиска.
