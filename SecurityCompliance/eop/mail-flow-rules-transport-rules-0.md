The main difference is mail flow rules take action on messages while they're in transit, and not after the message is delivered to the mailbox. Mail flow rules contain a richer set of conditions, exceptions, and actions, which provides you with the flexibility to implement many types of messaging policies.</span></span>

## <a name="mail-flow-rule-components"></a><span data-ttu-id="eb85a-122">Компоненты правила обработки почтового потока</span><span class="sxs-lookup"><span data-stu-id="eb85a-122">Mail flow rule components</span></span>

- <span data-ttu-id="eb85a-p103">**Условия** указывают, с какими сообщениями должны выполняться действия. Некоторые условия проверяют поля заголовков сообщений (например, поля "Кому", "От" или "Копия").
