# <a name="assign-ediscovery-permissions-in-the-office-365-security-amp-compliance-center"></a>Назначение разрешений обнаружения электронных данных в Office 365 безопасность &amp; центре соответствия требованиям

Если необходимо предоставить пользователям использования средств обеспечения обнаружения электронных данных в Office 365 безопасность &amp; центре соответствия требованиям, необходимо назначить им соответствующие разрешения. — Это самый простой способ это сделать, чтобы добавить пользователя в группе соответствующих ролей на странице " **разрешения** " в Office 365 безопасность &amp; центре соответствия требованиям. В этом разделе описаны разрешения, необходимые для выполнения задач, связанных с обнаружения электронных данных с помощью безопасности &amp; центре соответствия требованиям. 
  
Группа основной роли, связанные с обнаружения электронных данных в безопасности &amp; центре соответствия требованиям называется **eDiscovery Manager**. Существуют две подгруппы в рамках этой группе ролей. 
  
- **Диспетчеры обнаружения электронных данных** - диспетчеру eDiscovery можно использовать средство поиска содержимого в системы &amp; центре соответствия требованиям для поиска размещения содержимого в организации и выполнять различные действия, связанные с поиском, такие как просмотр и экспорт поиска результаты. Члены можно также создание и управление вариантах eDiscovery, добавлять и удалять участников в случае, создание вариантов удержаний и выполните поиск связанных с case и вариантов доступа к данным в Office 365 расширенного обнаружения электронных данных.  Обнаружение электронных данных руководители могут получать доступ только к досье, их создание и управление ими. Они не могут получить доступ к или управление случаев, созданные в другие диспетчеры обнаружения электронных данных. 
    
- **eDiscovery Администраторы** - eDiscovery администратора, является участником группы ролей диспетчер обнаружения электронных данных и могут выполнять поиск же контента и вариантов задач по управлению, которые могут выполнять диспетчеру eDiscovery. Кроме того eDiscovery администратору выполнить следующие действия. 
    
  - Доступ к всех случаев, перечисленные на странице **вариантах eDiscovery** в безопасности &amp; центре соответствия требованиям. 

  - Доступ к данным вариантов в расширенной eDiscovery для любого случая в организации.
    
  - Управлять всеми делами eDiscovery после добавления себя в качестве их члена.
  
  В разделе [Дополнительные сведения](#more-information) по причинам, которым администраторы обнаружения электронных данных в вашей организации. 

> [!NOTE]
> Для анализа данных с помощью расширенных eDiscovery, пользователя (custodian данные) должны быть назначены лицензии Office 365 E5 пользователя. Кроме того пользователи, имеющие лицензию на Office 365 E1 или E3 может быть назначен расширенной eDiscovery отдельная лицензия. Администраторы и контролеры, назначенных для случаев и использовать расширенные обнаружения электронных данных для анализа данных, не требуется лицензия на E5.  
  
## <a name="before-you-begin"></a>Перед началом работы

- Необходимо быть участником группы ролей управления организацией (или назначить роль управления ролями) для назначения разрешений обнаружения электронных данных в системы &amp; центре соответствия требованиям.
    
- Можно использовать командлет [Add-RoleGroupMember](https://technet.microsoft.com/en-us/library/dd638207%28v=exchg.160%29.aspx) в безопасности &amp; PowerShell центр соответствия требованиям для добавления группы безопасности с включенной поддержкой почты как член подгруппы диспетчеры обнаружения электронных данных в группе обнаружения электронных данных диспетчера ролей. Тем не менее невозможно добавить группу безопасности с включенной поддержкой почты подгруппы Администраторы обнаружения электронных данных. В разделе [Дополнительные сведения](#more-information) для получения дополнительных сведений. 
    
## <a name="assign-ediscovery-permissions-in-the-security-amp-compliance-center"></a>Назначение разрешений обнаружения электронных данных в системы &amp; центре соответствия требованиям

1. Перейдите по ссылке [https://protection.office.com](https://protection.office.com).
    
2. Войдите в Office 365 с помощью своей рабочей или учебной учетной записи.
    
3. В левой области безопасности &amp; центре соответствия требованиям, выберите **разрешения**и затем установите флажок рядом с элементом **обнаружения электронных данных диспетчера**.
    
4. На странице " **Обнаружение электронных данных диспетчера** всплывающее окно" выполните одно из следующих действий на основе разрешений обнаружения электронных данных, которые необходимо назначить. 
    
  - **Чтобы сделать пользователя диспетчеру eDiscovery** Рядом с элементом **обнаружения электронных данных диспетчера**нажмите кнопку **Изменить**. В разделе **Выбранные eDiscovery руководители**, нажмите кнопку **Изменить**и нажмите кнопку ![добавить значок](media/ITPro-EAC-AddIcon.gif) **Add**. Выберите пользователя (или пользователей) необходимо добавить в качестве диспетчер обнаружения электронных данных и нажмите кнопку **Добавить**. По завершении добавления пользователей нажмите кнопку **Готово**. На странице всплывающее окно **редактирования выберите eDiscovery Manager** щелкните **Сохранить** для сохранения изменений для обнаружения электронных данных диспетчера членства. 
    
  - **Чтобы сделать пользователя администратором eDiscovery** Рядом с пунктом **eDiscovery администратора**нажмите кнопку **Изменить**. В разделе **Выбранные eDiscovery администраторов**, нажмите кнопку **Изменить**и нажмите кнопку ![добавить значок](media/ITPro-EAC-AddIcon.gif) **Add**. Выберите пользователя (или пользователей) необходимо добавить в качестве eDiscovery администратора и нажмите кнопку **Добавить**. По завершении добавления пользователей нажмите кнопку **Готово**. На странице всплывающее окно **редактирования выберите eDiscovery администратора** щелкните **Сохранить** для сохранения изменений для обнаружения электронных данных членства администратора. 
    
> [!NOTE]
> Также можно использовать командлет **Add-eDiscoveryCaseAdmin** чтобы сделать пользователя администратором eDiscovery. Тем не менее пользователя необходимо назначить роль управления случае прежде чем использовать этот командлет, чтобы сделать их обнаружения электронных данных администратором. Дополнительные сведения можно [Добавить eDiscoveryCaseAdmin](https://go.microsoft.com/fwlink/p/?LinkID=798217). 
  
На странице " **разрешения** " в параметрах безопасности &amp; центре соответствия требованиям, можно также назначить пользователям разрешения для связанных с eDiscovery, добавляя их в группы ролей администратора соответствия требованиям, управление организацией и редактор. Описание связанных с eDiscovery роли RBAC, назначенные для каждой из этих групп ролей обратитесь к разделу [роли RBAC, связанные с обнаружения электронных данных](#rbac-roles-related-to-ediscovery) .

### <a name="case-management"></a>Управление делами

Эта роль позволяет пользователям создавать, изменять, удаление и управления доступом к вариантах eDiscovery в центре соответствия требованиям & безопасности. ### <a name="compliance-search"></a>Поиск соответствия

Эта роль позволяет пользователям запустить средство поиска контента в центре соответствия требованиям для поиска почтовых ящиков и общих папок, сайты SharePoint Online, OneDrive для бизнеса сайтов Скайп для бесед, Office 365 групп и рабочих групп Microsoft Business & безопасности. Эта роль позволяет пользователю получить оценку результатов поиска и создание отчетов экспорта, однако дополнительные роли необходимы для инициирования действий поиска контента, например предварительный просмотр, экспорт и удаление результатов поиска.

Обратите внимание, что пользователи, назначенным на роль поиска соответствия, но не роль предварительной версии можно выполнить предварительный просмотр результатов поиска, в котором действие предварительного просмотра начато пользователем, которая назначена роль предварительного просмотра. Пользователь без предварительного просмотра роли можно выполнить предварительный просмотр результатов для на две недели после создания начальной preview действие.

Аналогично пользователи, назначенным на роль поиска соответствия, но не экспорта роли можно загрузить результаты поиска, в которой было запущено действие экспорта пользователем, которая назначена роль экспорта. Пользователь без роли экспорта можно загрузить результаты поиска для на две недели после создания начальной экспорта действие. После этого они не сможет загрузить результаты, пока кто-то с ролью экспорта перезапускает экспорта.

Для получения дополнительных сведений см [Поиска контента в Office 365](content-search.md).

Если содержимое на удержание, владельцам содержимого по-прежнему могут изменить или удалить исходное содержимое, но контента будут сохранены, пока не будет снято удержание или не истечет продолжительность удержания.

### <a name="review"></a>проверка;

Эта роль позволяет пользователям получить доступ к вариантов данных в Office 365 расширенного обнаружения электронных данных. Эта роль основного предназначен для предоставления пользователям доступа к расширенной обнаружения электронных данных. Пользователи, которым назначена эта роль можно просмотреть и открыть список вариантов на странице электронных данных в центре соответствия требованиям, они являются участниками & безопасности. После пользователь получает доступ к дела в центре соответствия требованиям & безопасности, необходимо щелкнуть **переключатель, позволяющий расширенной eDiscovery** для доступа и анализа данных обращения в расширенной обнаружения электронных данных. Эта роль не позволяют пользователю для предварительного просмотра результатов поиска контента, связанной с обращением или для выполнения других задач Управление делами или поиска контента.

## <a name="more-information"></a>Дополнительные сведения

- **Почему создать eDiscovery администратора?** Как уже описано, eDiscovery, администратор является членом группы обнаружения электронных данных диспетчера ролей пользователей, которые могут просматривать и получить доступ к всех случаев обнаружения в вашей организации. Эта возможность доступа к всех вариантах eDiscovery имеет две важные функции: 
    
  - Если это пользователь, который является элементом единственным вариантом eDiscovery оставляет вашей организации, не (включая членов группы ролей управления организацией или другого члена группы обнаружения электронных данных диспетчера ролей) можно получить доступ, случая обнаружения электронных данных, так как они не участником случая. В этом случае будет не смогут получить доступ к данным в случае. Однако поскольку eDiscovery администратор может получить доступ к всех случаев обнаружения электронных данных в организации, они могут просматривать регистр в безопасности &amp; центре соответствия требованиям и добавьте себя или другой диспетчер обнаружения электронных данных как член обращения.
    
  - Так как eDiscovery администратор могут просматривать и получить доступ к всех вариантах eDiscovery, они могут аудита и следить всех случаев и связанных операций поиска соответствия. Это помогает предотвратить неправильное использование операций поиска соответствия или в случае обнаружения электронных данных. И так как eDiscovery администраторы могут получить доступ к потенциально конфиденциальной информации в результатах поиска соответствия, следует ограничить число пользователей, которые являются администраторами обнаружения электронных данных.
    
    Кроме того, eDiscovery администраторов безопасности &amp; центре соответствия требованиям автоматически добавляется в качестве администраторов в расширенной обнаружения электронных данных. Это означает, что пользователь должен быть eDiscovery администратора для выполнения административных задач в расширенной обнаружения электронных данных, таких как Настройка пользователей, создание случаев и импорт данных в случае.
    
- **Можно добавить группу как член группы обнаружения электронных данных диспетчера ролей безопасности &amp; центре соответствия требованиям?** Как уже упоминалось, можно добавить группу безопасности с включенной поддержкой почты как член подгруппы диспетчеры обнаружения электронных данных в группе ролей диспетчер обнаружения электронных данных с помощью командлета **Add-RoleGroupMember** в безопасности &amp; PowerShell центр соответствия требованиям. Например можно выполнить следующую команду, чтобы добавить группу безопасности с включенной поддержкой почты в группу ролей диспетчер обнаружения электронных данных. 
    
  ```
  Add-RoleGroupMember "eDiscovery Manager" -Member <name of security group>
  ```

    Обратите внимание на то, что группа рассылки Exchange или группу Office 365 не поддерживаются. Необходимо использовать группы безопасности с включенной поддержкой почты, который можно создать в Exchange Online PowerShell с помощью ` New-DistributionGroup -Type Security ` команды. Создание группы безопасности с включенной поддержкой почты (и добавлять элементы) в центре администрирования Exchange или в центре администрирования Office 365. Обратите внимание на то, что может потребоваться до 60 минут после создания нового безопасности почты можно добавить в группу ролей диспетчеры обнаружения электронных данных. 
    
    Также как уже упоминалось, не удается сделать безопасности с включенной поддержкой почты групповой eDiscovery администратором с помощью командлета **Add-eDiscoveryCaseAdmin** в безопасности &amp; PowerShell центр соответствия требованиям. Как eDiscovery администраторов можно добавлять только отдельных пользователей. 
    
    Обратите внимание, что вы также нельзя добавлять группы безопасности с включенной поддержкой почты как член дела.
