# <a name="run-a-content-search-in-the-office-365-security-amp-compliance-center"></a>Выполните поиск содержимого безопасности Office 365 &amp; центре соответствия требованиям

Можно использовать средство поиска контента обнаружения электронных данных безопасности Office 365 &amp; центре соответствия требованиям для поиска элементов, таких как электронной почты, документов и обмена мгновенными сообщениями бесед в организации Office 365. Это средство предназначено для поиска элементов в этих служб Office 365:
  
- Почтовые ящики Exchange Online и общих папок
    
- SharePoint Online и OneDrive для бизнеса сайтов
    
- Скайп для бизнеса бесед
    
- Microsoft Teams 
    
- Группы Office 365
    
Поиск контента — это новое средство поиска обнаружения электронных данных с новые и улучшенные возможности масштабирования и производительности. Используйте поиска контента для запуска операций поиска обнаружения электронных данных очень большой. Можно выполнить поиск всех почтовых ящиков, все общие папки Exchange и все сайты SharePoint Online и OneDrive для бизнеса учетных записей в одном поиска контента. Не предусмотрено ограничений на количество расположения контента, которые можно выполнить поиск. Существует также отсутствие ограничений на количество операций поиска, которые могут работать в то же время. После того как выполнение поиска контента, число размещения содержимого и предполагаемое количество результатов поиска, отображаются в области сведений на странице " **Поиск контента** ". После выполнения поиска можно предварительного просмотра результатов, получить статистику по ключевым словам для одного или несколько поисков массового редактирования контента поиска и экспортировать результаты на локальном компьютере.

В этой статье также приводятся сведения о поиске типов конфиденциальной информации и поиск содержимого, совместно с пользователями внутри и вне организации.
    
- Для доступа к странице **поиска контента** для выполнения операций поиска и предварительный просмотр и экспорт результатов поиска, администратор, специалисты по соответствию требованиям или диспетчер обнаружения электронных данных необходимо быть участником группы ролей диспетчер обнаружения электронных данных в системы &amp; соответствия требованиям Центр. У вас нет для назначения разрешений дополнительные поиска в Exchange Online, SharePoint Online, или для OneDrive для бизнеса сайтов. Дополнительные сведения можно [назначать разрешения обнаружения электронных данных в Office 365 безопасность &amp; центре соответствия требованиям](assign-ediscovery-permissions.md).
    
- Существуют ограничения, применяемые к поиска контента для обеспечения работоспособности и качества служб, предоставляемых организациями Office 365. В большинстве случаев нельзя изменять эти ограничения, но следует иметь в виду из них, чтобы можно было эти ограничения во внимание при планировании, выполнение и устранение неполадок поиска. Дополнительные сведения можно [ограничения для поиска в Office 365 безопасность &amp; центре соответствия требованиям](limits-for-content-search.md).
    
- В разделе для времени примерный поиска на основе числа почтовых ящиков, на которых выполняется поиск в одном поиска контента. 
    
- Как было сказано ранее можно выполнить поиск содержимого для поиска содержимого в Office 365 групп и группами Майкрософт. Это означает, что можно выполнить поиск группы почтовых ящиков, общий календарь и сайтов SharePoint, связанного с группу Office 365 и группы разработчиков Microsoft. Кроме того можно выполнить поиск бесед канала в группы разработчиков Microsoft. Сведения об Office 365 групп и группами Майкрософт см.

## <a name="create-a-search"></a>Создать поиск
<a name="create"> </a>

1. Перейдите по ссылке [https://protection.office.com](https://protection.office.com).
    
2. Войдите в Office 365 с помощью учетной записи рабочего или школы.
    
3. В расположенной слева области Центра безопасности и соответствия требованиям щелкните **Поиск и исследования** \> **Поиск контента**.
    
4. Нажмите кнопку **Создать**![Значок "Добавить"](media/O365-MDM-CreatePolicy-AddIcon.gif).
    
5. На странице **Новый поиск** введите имя поиска контента. Это имя должно быть уникальным в пределах организации. 
    
6. Выбор расположения контента, которые требуется найти. Можно выполнить поиск почтовых ящиков, узлов и общих папок в одном поиска.
    
    ![Выбор расположения контента, которые требуется выполнить поиск](media/da32e3f9-6cd6-4a26-8217-e97a5a7071e4.png)
  
1. **Поиск везде** Установите этот флажок, чтобы найти все расположения контента в вашей организации. При выборе этого параметра можно выбрать для поиска всех почтовых ящиков (включая неактивные почтовые ящики и почтовые ящики для всех групп Office 365 и Microsoft рабочих групп), все SharePoint и OneDrive для бизнеса сайтов (включая сайты для всех групп Office 365 и Майкрософт для групп) и все общедоступные папки.
    
    ![Установите флажок, чтобы найти все расположения контента поиска везде](media/86f132f1-0a2a-4048-900c-9f219d909ef2.png)
  
2. **Выбор расположения Custom** Установите этот флажок, чтобы выбрать почтовые ящики и сайты, которые требуется выполнить поиск. Если выбран этот параметр, у вас есть возможность поиска все расположения контента для определенной службы (например, поиск всех почтовых ящиков Exchange), или можно выполнить поиск определенного расположения контента для службы Office 365.
    
    При добавлении расположения контента для поиска необходимо помните следующее:
    
    **Почтовые ящики**
    
  - При нажатии кнопки **Добавить**![добавить значок](media/ITPro-EAC-AddIcon.gif) для указания почтовые ящики для поиска, средство выбора почтового ящика, который отображается будет пустым. Это предусмотрено при проектировании для повышения производительности. Добавление получателей в этот список, введите имя (минимум 3 символов) в поле поиска и нажмите кнопку **Найти**![значок поиска](media/5f6f9463-50e9-460b-8738-b67e759c2efc.gif).
    
  - Неактивные почтовые ящики и группы рассылки можно добавить в список почтовые ящики для поиска. Для групп рассылки поиска в почтовых членов группы. Обратите внимание на то, что динамические группы рассылки не поддерживаются.
    
  - Чтобы получить список неактивные почтовые ящики в вашей организации, выполните команду `Get-Mailbox -InactiveMailboxOnly` в Exchange Online PowerShell. Кроме того, можно перейти к **управлению данных** \> **хранения** в безопасности &amp; центре соответствия требованиям и нажмите кнопку **Дополнительные**![панели навигации эллипсы](media/9723029d-e5cd-4740-b5b1-2806e4f28208.gif) \> **неактивные почтовые ящики**.
    
  - Также можно добавить почтовый ящик, связанный с группой Office 365 или группы разработчиков Microsoft. В этом случае выполняется поиск только почтовый ящик группы или рабочая группа; не искать почтовые ящики членов группы или рабочая группа. Чтобы выполнить поиск их, необходимо специально добавьте их в результате поиска.
    
  - Если не требуется включать все почтовые ящики для поиска, выберите **Выбор определенных почтовых ящиков для поиска**, но не добавлять почтового ящика в список.
    
    **Сайты**
    
  - Нажмите кнопку **Добавить**![добавить значок](media/ITPro-EAC-AddIcon.gif) для добавления сайтов для поиска. Введите URL-адрес для каждого сайта, который требуется найти. Средство поиска контента проверить URL-адрес и добавить его в список сайтов для поиска. 
    
  - Вы можете добавить SharePoint, связанный с группой Office 365 или группы разработчиков Microsoft. В разделе инструкции о том, как найти URL-адрес для группы или рабочая группа. 
    
  - Если не требуется включать в поиск веб-сайты, выберите **Выбор определенных сайтов для поиска**, но не добавить сайт в список.
    
    **Общедоступные папки**
    
    Для общих папок вы можете выполнить поиск всех общих папок в организации Exchange Online или не все общедоступные папки поиска.
    
7. Нажмите кнопку **Далее**.
    
8. На странице **Новый поиск** можно добавить ключевые слова и условия для создания поискового запроса. 
    
    ![Создание запроса на поиск с использованием ключевых слов и условий](media/1b7cf7b5-f1e1-471a-ad5c-48aad8435b00.png)
  
1. В поле в разделе **необходимые действия "мне нравится" следует искать?**, введите в поле запрос поиска. Можно указать ключевые слова, сообщение свойства, например, отправленных и полученных даты, или свойств документа, таких как имена файлов или Дата последнего изменения документа. Можно использовать более сложные запросы, использующие логический оператор, например **AND** **или**, **не**, **NEAR**и **ONEAR**. Можно также выполнить поиск конфиденциальной информации (например, номера социального страхования) в документы или поиск документов, которые совместно извне. Если ключевое слово поле оставлено пустым, весь контент, расположенный в указанном расположении контента будут включены в результатах поиска. 
    
2. Щелкните флажок **Показать список ключевых слов** и тип ключевого слова в каждой строке. После этого ключевых слов в каждой строке, соединенных с оператором **или** в запрос поиска, которая создается. 
    
    ![Введите ключевое слово или этап ключевое слово в строку в списке ключевых слов](media/aea1a361-639d-4a82-8c3c-48645ef3fc05.png)
  
    Причины использования список ключевых слов? Можно получить статистику, показывающие количество элементов соответствуют каждого ключевого слова. Это может помочь быстро определять, какие ключевые слова эффективны наиболее (и бы). Можно также использовать ключевая фраза (заключаются в круглые скобки) в строке. Дополнительные сведения о статистике поиска можно [Просмотр статистики ключевых слов для результатов поиска контента](view-keyword-statistics-for-content-search.md).
    
    В разделе Рекомендации по использованию список ключевых слов. 
    
3. Нажмите кнопку **Проверить запрос опечаток** для проверки запроса для неподдерживаемые символы и логические операторы, которые не могут иметь разное. Неподдерживаемые символы часто являются скрытыми и обычно приводит к ошибке поиска или возвращать непредвиденные результаты. Дополнительные сведения о неподдерживаемых символов, которые проверяются увидеть [запрос поиска контента по ошибкам](check-your-content-search-query-for-errors.md).
    
4. В **условиях**добавьте условия запроса поиска для сужения области поиска и возвращать более качественных набора результатов. Каждое условие добавляет предложение в поисковый запрос KQL, который создается и при запуске поиска. Условие логически подключенной к запроса ключевого слова (указанного в поле ключевых слов) **и** оператор. Который означает, что элементы для удовлетворения запроса ключевого слова и условия должны быть включены в результаты. Это, как помочь условий, чтобы сузить результаты.

9. Щелкните **Поиск**, чтобы сохранить параметры поиска и начать поиск. 
    
    Поиск работы. После завершения поиска в области сведений отображается следующая информация.
    
    ![Статистика поиска отображаются в области сведений выбранного поиска контента](media/2046bb5e-f4cb-4ba7-b7fc-8e5e53dae567.png)
  
1. Дата и время последнего выполнения поиска.
    
2. Число (и общий размер) элементов, которые были найдены, которые соответствуют поисковый запрос. Типы элементов примеры сообщений электронной почты, элементы календаря и документы. Если элемент содержит несколько экземпляров класса ключевое слово, которое является, поиск, он только считается один раз в общее число элементов. Например если вы ищете слова «stock» или «подсказка» и сообщения электронной почты содержит три экземпляра word «stock», он только считается один раз в поле **элементы** . 
    
3. Количество и общий размер неиндексированные элементов в расположения контента поиска. Количество неиндексированные элементов, не соответствующие условиям поиска будут включены в статистические данные поиска, отображаемые в области сведений. Если неиндексированные элемента совпадения, поиск запросов (из-за других свойств сообщения или документа соответствуют критериям поиска), не будут включены в предполагаемое количество неиндексированные элементов. Тем не менее если неиндексированные элемента исключения, условия поиска, его не включены в оценку неиндексированные элементов.
    
4. Число каждого типа расположение содержимого, который был выполнен поиск. Для почтовых ящиков Обратите внимание на то, включены ли в общее количество почтовых ящиков, которые были поиск архивных почтовых ящиков. В предыдущем примере поиска четыре почтовых ящиков пользователей и включить в архивный почтовый ящик для каждого пользователя. Поэтому восемь почтовых ящиков указанных в Статистика поиска.
    
5. Ссылки на предварительного просмотра результатов поиска результаты или выполните поиск еще раз, чтобы обновить статистику поиска.
    
    При необходимости нажмите кнопку **Обновить**![значок обновления](media/O365-MDM-Policy-RefreshIcon.gif) для обновления данных в области сведений для выбранного поиска.

## <a name="export-search-results"></a>Экспорт результатов поиска
<a name="export"> </a>

После успешного выполнения поиска можно экспортировать результаты поиска на локальном компьютере. При экспорте электронной почты результатов их загрузки на компьютер как PST-файлов. При экспорте содержимого из SharePoint и OneDrive для бизнеса сайтов, экспортируются копии собственный документов Office. Существуют также дополнительные документы и отчеты, которые включены в результаты поиска экспортированного. Дополнительные сведения можно [результатов поиска экспорта из безопасности Office 365 &amp; центре соответствия требованиям](export-search-results.md).
  
## <a name="preview-search-results"></a>Просмотр результатов поиска
<a name="preview"> </a>

После успешного завершения поиска можно выполнить предварительный просмотр результатов поиска. Существует ряд ограничений, связанных с Предварительный просмотр результатов поиска контента. Дополнительные сведения можно [ограничения для поиска в Office 365 безопасность &amp; центре соответствия требованиям](limits-for-content-search.md). Обратите внимание на то, что неиндексированные элементов не доступны для предварительного просмотра.
  
1. На странице " **Поиск контента** " выберите поиск. 
    
2. В области сведений в разделе **Результаты** щелкните **Предварительный просмотр результатов поиска**. Откроется страница **Предварительный просмотр результатов поиска**, содержащая список результатов поиска. 
    
    Можно щелкните заголовок столбца для сортировки результатов на основе темы, тип, отправителя или дата получения элемента в почтовом ящике источника.
    
3. Щелкните элемент для предварительного просмотра.

 **Группами Майкрософт и группы Office 365**
  
- Группами Майкрософт, основанные на Office 365 групп. Таким образом их поиск является очень похожи. Имейте в виду следующее при поиске содержимого в Office 365 группы и группами Майкрософт.
    
  - Чтобы выполнить поиск содержимого, расположенного в Office 365 группы и группами Майкрософт, необходимо указывать почтовых ящиков и сайта SharePoint, связанных с группы или группы.
    
  - Выполните командлет **Get-UnifiedGroup** в Exchange Online для просмотра свойств для группы разработчиков Microsoft или группу Office 365. Это удобный способ получить URL-адрес для сайта, связанный с группой или группу. Например следующая команда выводит выбранных свойств для группы Office 365 с именем старший руководство: 
    
  ```
  Get-UnifiedGroup "Senior Leadership Team" | FL DisplayName,Alias,PrimarySmtpAddress,SharePointSiteUrl
  DisplayName            : Senior Leadership Team
  Alias                  : seniorleadershipteam
  PrimarySmtpAddress     : seniorleadershipteam@contoso.onmicrosoft.com
  SharePointSiteUrl      : https://contoso.sharepoint.com/sites/seniorleadershipteam
  
  ```

    > [!NOTE]
    > Чтобы выполнить командлет **Get-UnifiedGroup** , необходимо назначить роль только для просмотра получателей в Exchange Online или входить в группу ролей, которая назначена роль получателей только для просмотра. 
  
  - При поиске почтового ящика пользователя не будет искать любой группы разработчиков Microsoft или группы Office 365, который входит пользователь. Аналогично, при поиске группы разработчиков Microsoft или группу Office 365 только группы и почтового ящика сайта группы, которые можно указать выполняется поиск; Если не были добавлены к поиску явно не поиска почтовых ящиков и OneDrive для бизнеса учетные записи членов группы.
    
  - Чтобы получить список членов группы разработчиков Microsoft или группу Office 365, можно просмотреть свойства на **Домашняя страница \> группы** страницы в центре администрирования Office 365. Кроме того в Exchange Online PowerShell можно выполните следующую команду: 
    
  ```
  Get-UnifiedGroupLinks <group or team name> -LinkType Members | FL DisplayName,PrimarySmtpAddress 
  ```

    > [!NOTE]
    > Чтобы выполнить командлет **Get-UnifiedGroupLinks** , необходимо назначить роль только для просмотра получателей в Exchange Online или входить в группу ролей, которая назначена роль получателей только для просмотра. 
  
  - Бесед, которые входят в состав групп Майкрософт канала хранятся в почтовом ящике, сопоставленные с группы разработчиков Microsoft. Аналогично файлы, совместно использующих участников группы в канале хранятся на сайте SharePoint team. Таким образом необходимо добавить сайт SharePoint и почтового ящика группы разработчиков Microsoft в качестве расположения контента для поиска бесед и файлы в канале.
    
  - 
    
    Кроме того бесед, которые входят в списке беседы в группами Майкрософт, хранятся в почтовый ящик Exchange Online пользователи, участвующие в чата. И файлы, которые пользователь использует совместно в беседах чата хранятся в OneDrive для бизнеса учетной записи пользователя, который использует файл. Таким образом необходимо добавить отдельных пользователей почтовых ящиков и OneDrive для бизнеса учетных записей как расположения контента для поиска файлов и бесед в списке беседы.
    
    > [!NOTE]
    > Пользователи, участвовать в беседах, в которых являются частью список чата в группах Microsoft должны иметь почтовый ящик Exchange Online (на основе облака) в порядке для поиска бесед чата. Это, поскольку бесед, которые входят в состав список чата, хранятся в облачные почтовые ящики участников беседы. Если участник беседы не имеет почтовый ящик Exchange Online, нельзя будет выполнять поиск бесед чата. Например в гибридном развертывании Exchange, пользователи с локального почтового ящика можно принять участие в беседах, в которых являются частью список чата в группах Майкрософт. Однако в этом случае содержимого из этих беседы не доступен для поиска так как пользователи не имеют облачные почтовые ящики. 
  
  - Каждый канал группы или группы разработчиков Microsoft содержит вики-сайт заметок и совместной работы. Вики-контент автоматически сохраняются в файл с формат MHT. Этот файл хранится в библиотеке документов данных вики-сайт группы на сайте SharePoint team. Средство поиска контента используется для поиска вики-сайт, указав сайту SharePoint как расположение содержимого для поиска. 
    
    > [!NOTE]
    > Возможность поиска вики-сайта группы разработчиков Microsoft или канала (при выполнении поиска сайта SharePoint team) была выпущена 22 июня 2017. Вики-страницы, которые были сохранены или обновлены на, дата, или в конце доступны для поиска. Вики-страницы последнего сохранения или обновлены до этой даты недоступны для поиска.

Если вы только что вставьте список ключевых слов в поле ключевых слов или нажмите клавишу **Ввод** после ввода ключевого слова, они не будут подключены оператором **или** .   - Можно также подготовить список ключевых слов или фраз ключевое слово в файле Excel или обычный текстовый файл и затем скопируйте и вставьте список в список ключевых слов. Для этого необходимо установите флажок **Отображать список ключевых слов** . Затем щелкните первую строку в списке ключевое слово и вставьте список. Каждую строку из файлов Excel, или текст будет вставлен в в отдельные строки в список ключевых слов. 
    
  - После создания запроса с помощью списка ключевых слов, рекомендуется проверить синтаксис запроса поиска (в области сведений выбранного поиска) чтобы при поиске запросов — это необходимо. В запрос поиска, который отображается в области сведений в разделе **запрос** , ключевые слова, разделенных текста **(c:s)**. Это указывает на то, что ключевые слова соединены оператор **OR** . Аналогично, если запрос поиска включает условия, ключевые слова и условия, разделенных текста **(c: c)**. Это указывает на то, что ключевые слова, подключенные к условиям оператором **и** . При создании поиска контента, с помощью графического интерфейса пользователя в системы &amp; центре соответствия требованиям нейтральный язык по умолчанию. 
    
    Как узнать, если необходимо изменить значение параметра язык для поиска контента? Если вы не уверены, размещения содержимого с символами, отличный от английского, вы ищете, но поиск не возвращает результаты, параметр языка может быть несколько.
    
    Чтобы изменить параметр языка для поиска существующего контента, выполните следующую команду в безопасности &amp; PowerShell центр соответствия требованиям:
    
  ```
  Set-ComplianceSearch <name of content search> -Language <culture code value>
  ```

    Например, чтобы изменить параметр языка для китайского языка, можно использовать `zh-CN` для значения код языка и региональных параметров. После изменения параметров языка, необходимо повторно запустить поиск. [Список возможных значений кода см.](https://go.microsoft.com/fwlink/p/?LinkID=184859) Для поиска контента мы рекомендуем использовать коды двух частей языка и региональных параметров для значения параметра язык; например `ja-JP` и не `ja`.
    

  
 **Поиск неактивные почтовые ящики**
  
Как было сказано ранее можно выполнить поиск неактивные почтовые ящики в поиска контента. Вот несколько факторов, которые следует учитывать при поиске неактивные почтовые ящики.
  
- Если поиск контента включает в себя почтового ящика пользователя и почтового ящика становится неактивной, поиск контента будет продолжать поиск неактивного почтового ящика, если повторно запустите поиска после его неактивным.
    
- В некоторых случаях пользователь может иметь активного хранилища почтовых ящиков и неактивного почтового ящика, для которого один и тот же адрес SMTP. В этом случае будет осуществляться только определенного почтового ящика, выберите в качестве расположения для поиска контента. Другими словами при добавлении почтового ящика пользователя для поиска нельзя предполагать, что будет осуществляться обоих их активные и неактивные почтовые ящики; поиск будет выполняться только почтовых ящиков, которые явно добавлять для поиска.
    
- Мы настоятельно рекомендуем избежать необходимости активного хранилища почтовых ящиков и неактивного почтового ящика с тем же адресом SMTP. Если требуется повторно использовать SMTP-адрес, который назначен в настоящее время неактивного почтового ящика, мы рекомендуем восстановление неактивного почтового ящика или восстановления содержимого неактивного почтового ящика для активного хранилища почтовых ящиков (или архива active почтовых ящиков), а затем удалите неактивного почтового ящика.  **Разное**
  
- Контент операций поиска, созданные на странице **поиска контента** в безопасности &amp; центр соответствия не отображаются на **In-Place eDiscovery &amp; хранения** страницы в центре администрирования Exchange. Причина этого заключается в системы создания архитектуры поиска контента и объекты поиска &amp; центре соответствия требованиям, полностью отличаются от функции обнаружения электронных данных на месте в Exchange Online. 
    
    По этой же причине операций поиска, созданные на странице " **Поиск контента** " не отображаются на странице **поиска по** обращению eDiscovery в системы &amp; центре соответствия требованиям. 
    
- В чем разница между перезапуска и повтор попытки поиска? При перезапуске поиска, все расположения контента, определенных в результате поиска выполняется поиск еще раз в новых preview поиска. Тем не менее если вы повторить попытку поиска, только расположения контента, которые не удалось после последнего выполнения поиска производится поиск еще раз.
