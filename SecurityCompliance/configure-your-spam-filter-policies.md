# <a name="configure-your-spam-filter-policies"></a>Настройте политики защиты от спама
  
Параметры фильтра нежелательной почты базовой включают выбор действие, которое выполняется в сообщениях, помечаются как нежелательная почта и Выбор необходимости фильтровать сообщения, написанные на определенных языках или отправленные из конкретных стран или регионов. Параметры политики фильтрации нежелательной почты являются только применяться к входящим сообщениям. Вы можете изменить политику фильтрации нежелательной почты по умолчанию для настройки настройки фильтра нежелательной почты всей компании и создания политики фильтрации нежелательной почты настраиваемых и применить их для конкретных пользователей, групп или доменов в организации. Настраиваемых политик всегда имеют приоритет над политикой по умолчанию. Можно изменить порядок выполнения настраиваемых политик с помощью изменения приоритета каждого пользовательскую политику.
  
> [!IMPORTANT]
> Для изолированного клиентов Exchange Online Protection (EOP): по умолчанию, фильтры нежелательной почты EOP отправлять сообщениях с обнаруженной нежелательной почты в папку нежелательной почты каждого получателя. Тем не менее чтобы убедиться в том, что действия **переместить сообщение в папку нежелательной почты** работает для локальных почтовых ящиков, необходимо настроить правила транспорта Exchange на локальных серверах для обнаружения нежелательной почты заголовки, добавляемые в EOP. Дополнительные сведения см [Убедитесь, что нежелательной почты перенаправляется в папку нежелательной почты каждого пользователя](ensure-that-spam-is-routed-to-each-user-s-junk-email-folder.md).

Предполагаемое время для завершения: 30 минут.
  
Вы должны быть назначены разрешения, перед выполнением этой процедуры или процедуры. Чтобы увидеть, какие нужны разрешения, видеть запись защиты от нежелательной почты в разделе [Feature Permissions in Exchange Online](http://technet.microsoft.com/library/15073ce1-0917-403b-8839-02a2ebc96e16.aspx) . 
  
Сочетания клавиш для процедур, описанных в этой статье, приведены в статье **Keyboard shortcuts in Exchange 2013**.
  
## <a name="use-the-exchange-admin-center-eac-to-configure-spam-filter-policies"></a>Настройка политик фильтрации спама с помощью Центра администрирования Exchange (EAC)

1. В Центре администрирования Exchange перейдите в раздел **Защита** \> **Фильтр спама**.
    
2. На странице **Общие** выполните одно из следующих действий. 
    
      - Дважды щелкните политику по умолчанию, чтобы изменить политику, применяемую в масштабах всей компании.
    
      - Нажмите значок ![Значок добавления](media/ITPro-EAC-AddIcon.gif) **Создать** для создания новой настраиваемой политики защиты от спама, которую можно применять к пользователям, группам и доменам вашей организации. Можно также изменять существующие настраиваемые политики, дважды щелкая их. 
    
3. Для настраиваемых политик только укажите имя для этой политики. Кроме того можно также указать более подробное описание. Невозможно переименование политики по умолчанию.<br/><br/>Примечание: При создании политики все параметры конфигурации отображаются на экране. В то же время при изменении политики должны перемещаться по несколько экранов. Параметры такие же, в любом случае, но остальной части процедуры описывается, как для доступа к этим параметрам, когда изменение политики. 
  
4. На странице **Действия для нежелательной почты и массовых рассылок** в разделе **Нежелательная почта** и **Нежелательная почта (высокая достоверность)** выберите действие, выполняемое для входящей нежелательной почты и массовых рассылок. По умолчанию выбрано действие **Переместить сообщения в папку "Нежелательная почта"**. Другие возможные значения: 
    
      - **Удалить сообщение** Удаляется все сообщение, включая все вложения. 
        
      - **Отправить сообщение на карантин** Сообщение отправляется на карантин, а не указанным получателям. При выборе этого варианта в поле ввода **Хранить нежелательные сообщения (дней)** укажите количество дней, в течение которых нежелательное сообщение будет находиться на карантине. (По истечении этого времени оно будет автоматически удалено. Значением по умолчанию является максимальное время  15 дней. Минимальное значение  1 день.)<br/><br/>Совет: Сведения о том, как администраторы могут управлять сообщений электронной почты, которые находятся в карантин в центре администрирования Exchange просмотрите [карантина](quarantine.md) и [Поиск и освобождение сообщений на карантине с правами администратора](find-and-release-quarantined-messages-as-an-administrator.md). > сведения о настройке нежелательных сообщений уведомлений для отправки пользователям, в разделе [Настройка конечных пользователей защиты от нежелательной почты уведомления в EOP](configure-end-user-spam-notifications-in-eop.md) или [конечных пользователей Настройка защиты от нежелательной почты уведомлений в Exchange Online](configure-end-user-spam-notifications-in-exchange-online.md). 
  
      - **Переместить сообщение в папку "Нежелательная почта"** Сообщение отправляется в папку "Нежелательная почта" указанных получателей. Это действие по умолчанию для обоих порогов вероятности нежелательной почты.<br/><br/>**Важно: Для клиентов Exchange Online Protection (EOP): чтобы это действие для работы с локальными почтовыми ящиками, необходимо настроить два правила транспорта Exchange на локальных серверах для обнаружения нежелательной почты заголовки, добавляемые в EOP. Дополнительные сведения см [Убедитесь, что нежелательной почты перенаправляется в папку нежелательной почты каждого пользователя](ensure-that-spam-is-routed-to-each-user-s-junk-email-folder.md).**
  
      - **Добавление X-заголовка** Отправляет сообщение указанным получателям, но добавляет текст заголовка X в заголовке сообщения для идентификации сообщение как нежелательная почта. Используя этот текст в качестве идентификатора, можно при необходимости создавать правила папки "Входящие" или использование нижестоящие устройства для работы в окне сообщения. X-заголовка текст по умолчанию — **это сообщение появляется, чтобы быть нежелательной почты**.<br/>Текст заголовка X можно настроить с помощью поля ввода **Добавьте этот текст заголовка X** . Если настроить текст заголовка X, необходимо учитывать следующие условия: 
    
      - Если указать только заголовок в формате \< *заголовок*\>, в котором нет пробелов в пределах \< *заголовок*\>, двоеточие будет добавлено к настраиваемым текстом, а затем текст по умолчанию. Например, если указать «Это — это личных custom заголовков» текст заголовка X появится в виде «это — это личных custom заголовков: это сообщение появляется, чтобы быть нежелательной почты.»       
        
      - При включении пробелы в тексте пользовательский заголовок или добавить двоеточие себя (например, «X это Мои настраиваемый заголовок» или «X-This-is-my-custom-header:»), текст заголовка X возвращается значение по умолчанию «X — это-является-нежелательной почты: это сообщение появляется, чтобы быть нежелательной почты.»
    
      - Не удается задать текст заголовка в формате \< *заголовок*  \>:\<  *значение*  \>. При этом оба значения перед и после двоеточия будет игнорироваться, а вместо этого отображается текст заголовка X по умолчанию: «X-это-является-нежелательной почты: это сообщение появляется, чтобы быть нежелательной почты.» 
      
      - Обратите внимание, сообщениях X-заголовка может по-прежнему перемещен в папку нежелательной почты почтовых ящиков из-за нежелательных конфигурация почтового ящика. Это можно изменить, отключив этот компонент с помощью Set-mailboxjunkemailconfiguration служит.
        
      - **Строка темы Prepend с текстом** Отправляет сообщение указанным получателям, но вставляет строку темы с текстом, указанным в поле ввода **префиксов в строку темы на этот текст** . Используя этот текст в качестве идентификатора, можно создать правила, чтобы фильтровать или направлять сообщения при необходимости. 
        
      - **Перенаправить сообщение на адрес электронной почты** Отправляет сообщение на адрес назначенного электронной почты вместо указанным получателям. Укажите адрес «перенаправления» в поле ввода **перенаправление на этот адрес электронной почты** .<br/><br/>Примечание: Дополнительные сведения об уровнях вероятности нежелательной почты см [уровень вероятности нежелательной почты](spam-confidence-levels.md). 
  
5. В разделе **массовыми сообщениями электронной почты**вы можете выбрать порогового значения следует рассматривать как нежелательная почта массовыми сообщениями электронной почты. Это значение превышено основано на уровне жалоб массового сообщения. Можно выбрать параметр пороговое значение от 1 до 9, где 1 соответствует большинство массовыми сообщениями электронной почты, как нежелательная почта и 9 позволяет большинство массовыми сообщениями электронной почты для доставки. Служба выполняет настроенного действие, таких как отправка сообщения в папку нежелательной почты получателя. Просмотреть [значения массового жалоб уровень](bulk-complaint-level-values.md) и [в чем разница между нежелательной почтой и массовыми сообщениями электронной почты?](what-s-the-difference-between-junk-email-and-bulk-email.md) для получения дополнительных сведений. 
    
6. На странице **Списки блокировок** можно указать элементы, например отправителей или домены, которые всегда будут помечаться как нежелательная почта. Служба применяет действие, настроенное для нежелательной почты с высоким уровнем достоверности, к письмам, соответствующим этим записям. 
    
      - Добавьте нежелательных отправителей в список блокировок отправителя. Нажмите кнопку **Добавить**![Значок добавления](media/ITPro-EAC-AddIcon.gif), а затем в диалоговом окне выбора добавьте адреса отправителей, которые вы хотите заблокировать. Несколько записей можно разделить точкой с запятой или новой строкой. Нажмите кнопку **ОК**, чтобы вернуться на страницу **Списки блокировок**. 
        
      - Добавление нежелательных доменов в список блокировки доменов. Нажмите кнопку **Добавить**![Значок добавления](media/ITPro-EAC-AddIcon.gif), а затем в диалоговом окне выбора добавьте домены, которые требуется заблокировать. Несколько записей можно разделить точкой с запятой или новой строкой. Нажмите кнопку **ОК**, чтобы вернуться на страницу **Списки блокировок**.<br/><br/>**Предупреждение: Если заблокировать доменов верхнего уровня, вероятнее всего, что электронной почты, который будет помечаются как нежелательная почта.** 
  
7. На странице **Списки разрешений** можно указать элементы, например отправителей или домены, которые всегда будут доставляться в папку "Входящие". Письма из этих записей не обрабатываются фильтром нежелательной почты. 
    
      - Добавление надежных отправителей в список разрешенных отправителей. Нажмите кнопку **Добавить**![Значок добавления](media/ITPro-EAC-AddIcon.gif), а затем в диалоговом окне выбора добавьте адреса отправителей, которые вы хотите разрешить. Несколько записей можно разделить точкой с запятой или новой строкой. Нажмите кнопку "ОК", чтобы вернуться на страницу **Списки разрешений**. 
        
      - Добавление доверенных доменов в список разрешенных доменов. Нажмите кнопку **Добавить**![Значок добавления](media/ITPro-EAC-AddIcon.gif), а затем в диалоговом окне выбора добавьте домены, которые требуется разрешить. Несколько записей можно разделить точкой с запятой или новой строкой. Нажмите кнопку "ОК", чтобы вернуться на страницу **Списки разрешений**.<br/><br/>**Предупреждение: Если разрешить доменов верхнего уровня, то есть доставку электронной почты, которые не требуется для папки "Входящие".** 
  
8. На странице **Международной нежелательной почты** можно отфильтровать сообщения электронной почты, написанные на определенных языках или отправленные из конкретных стран или регионов. Вы можете настроить до 86 различных языков и 250 разных регионов. Служба будет выполняться для настроенного действия для высокой вероятности нежелательной почты. 
    
9. Установите флажок **фильтрации сообщений электронной почты на следующих языках** , чтобы включить эту функцию. Нажмите кнопку ![добавить значок](media/ITPro-EAC-AddIcon.gif)и затем в диалоговом окне Выбор настроить параметры (поддерживается множественного выбора). Например если выбрать для фильтрации сообщений, записанные в арабский (AR) и **карантин сообщения** является настроенного действия для наиболее достоверные нежелательных сообщений, все сообщения, написанные на арабский помещаются на карантин. Нажмите кнопку **ОК** для возврата в область **Международной нежелательной почты** . 
    
10. Установите флажок **фильтрации сообщений электронной почты, отправляемых с следующие страны или регионы,** Чтобы включить эту функцию. Нажмите кнопку ![добавить значок](media/ITPro-EAC-AddIcon.gif)и затем в диалоговом окне Выбор настроить параметры (поддерживается множественного выбора). Например при выборе для фильтрации все сообщения, отправленные из Австралии (AU) и настроенного действия для сообщения высокой вероятности нежелательной почты, а затем все сообщения, отправляемые из Австралия — это **сообщение** , помещаются на карантин. Нажмите кнопку **ОК** для возврата в область **Международной нежелательной почты** .<br/><br/>По умолчанию, если международные параметры нежелательной почты не выбраны, служба применяет обычную фильтрацию сообщений на всех языках и из всех регионов. Сообщения анализируются и к ним применяются настроенные действия, если они идентифицированы как нежелательные или нежелательные высокого уровня. 
  
11. На странице " **Дополнительные параметры** " можно выбрать **на**, **выключить**или **тестирования** для каждого расширенный параметр фильтрации нежелательной почты. 
    
12. **На** Сообщения активно фильтруются в соответствии с правила, который связан с соответствующий параметр. Сообщения либо помечаются как нежелательная почта или будет иметь показателей нежелательной почты увеличено, в зависимости от того, какие параметры включения. 
    
13. **Выключить**. С сообщениями, которые соответствуют критериям фильтрации нежелательной почты, не выполняются никакие действия. Все параметры по умолчанию выключены. 
    
14. **Тест** Никакие действия не предпринимаются сообщения, которые соответствуют критериям фильтра нежелательной почты. Тем не менее можно с тегами сообщения, добавив X-заголовка перед доставкой получателю. X-заголовка позволяет узнать, какой вариант ASF выполнено. При указании **тестирования** для каких-либо дополнительных параметров можно настроить следующие параметры режима теста нужно применить соответствие при попытке параметр включен для тестирования: 
    
      - **Отсутствуют** С сообщением не будут выполняться действия тестового режима. Этот параметр выбран по умолчанию. 
        
      - **Установка по умолчанию тестирование текст заголовка X** При выборе этого параметра отправляет сообщение указанным получателям, но также добавляет специальные X-заголовка сообщения для обозначения совпадении определенных расширенной фильтрации нежелательной почты параметр. 
        
      - **Отправить сообщение по этому адресу скрытой копии** При выборе этого параметра отправляет скрытую копию сообщения на адрес электронной почты, укажите в поле ввода. <br/><br/>Дополнительные сведения о Дополнительные параметры фильтрации нежелательной почты, включая описание каждого параметра и текст заголовка X, который связан с каждой из них содержатся в разделе [параметры фильтрации расширенной нежелательной почты](advanced-spam-filtering-asf-options.md). 
  
15. Для настраиваемых политик только, нажмите кнопку **Применить, чтобы** пункт меню и затем создать правило на основе условие для указания пользователей, групп и доменов, к которому следует применить эту политику. Можно создать несколько условий, если они являются уникальными. 
    
      - Для выбора пользователей, выберите **получателя**. В следующем диалоговом окне выберите одного или нескольких отправителей из вашей компании из списка выбора пользователей и нажмите кнопку **Добавить**. Чтобы добавить отправителей, отсутствующих в списке, введите их адреса электронной почты и нажмите кнопку **Проверить имена**. ## <a name="use-remote-powershell-to-configure-spam-filter-policies"></a>Использование удаленной оболочки PowerShell для настройки политик фильтрации нежелательной почты

Также можно настроить и применение политик фильтрации нежелательной почты в PowerShell. Чтобы узнать, как использовать Windows PowerShell для подключения к Exchange Online, обратитесь к разделу [подключение к Exchange Online PowerShell](https://go.microsoft.com/fwlink/p/?linkid=396554). Чтобы узнать, как использовать Windows PowerShell для подключения к Exchange Online Protection, обратитесь к разделу [подключение к Exchange Online Protection PowerShell](https://go.microsoft.com/fwlink/p/?linkid=627290).
  
- [Get-HostedContentFilterPolicy](http://technet.microsoft.com/library/d510471a-dda5-4df7-b3f8-2ee7a1948436.aspx) Просмотр параметров фильтрации нежелательной почты. 
    
- [Set-HostedContentFilterPolicy](http://technet.microsoft.com/library/f597aa65-baa7-49d0-8832-2a300073f211.aspx) Изменение параметров фильтрации нежелательной почты. 
    
- [New-HostedContentFilterPolicy](http://technet.microsoft.com/library/4d15128d-9e16-42a1-8ac5-36f07d4bbbf0.aspx) Создание настраиваемой политики фильтрации нежелательной почты. 
    
- [Remove-HostedContentFilterPolicy](http://technet.microsoft.com/library/9fe1fe03-8f83-41e3-9bf5-084a392784d6.aspx) Удаление настраиваемой политики фильтрации нежелательной почты. 
    
Чтобы применить политику фильтрации нежелательной почты к пользователям, группам или доменам, используйте командлет [New-HostedContentFilterRule](http://technet.microsoft.com/library/2df13ba9-1eb0-4da3-bd72-a79d5fa15e26.aspx) (для создания правила фильтрации, которое можно применить к настраиваемым политикам) или командлет [Set-HostedContentFilterRule](http://technet.microsoft.com/library/ba259260-ffd3-43f3-8ef4-9d8659679d02.aspx) (для изменения существующего правила фильтрации, которое можно применить к настраиваемым политикам). Командлет [Enable-HostedContentFilterRule](http://technet.microsoft.com/library/354ece28-dcde-4b5f-88ed-475115e7ea78.aspx) или [Disable-HostedContentFilterRule](http://technet.microsoft.com/library/c1f8dafc-ef5d-47e3-b0fb-71a88e145fc5.aspx) используется для включения или отключения правила, которое применяется к политике. 
  
## <a name="how-do-you-know-this-worked"></a>Как проверить, что все получилось?

Чтобы убедиться, что нежелательная почта надлежащим образом обнаруживается и обрабатывается, можно отправить сообщение GTUBE через службу. Подобно файлу антивирусного тестирования EICAR сообщение GTUBE позволяет проверить, обнаруживает ли служба входящую нежелательную почту. Фильтр нежелательной почты должен всегда определять сообщение GTUBE как нежелательное, и выполняемые над ним действия должны соответствовать настроенным параметрам.
  
Добавьте в сообщение электронной почты следующий текст GTUBE в виде одной строки, без пробелов и разрывов строки:
  
```
XJS*C4JDBQADN1.NSBN3*2IDNEN*GTUBE-STANDARD-ANTI-UBE-TEST-EMAIL*C.34X
```
