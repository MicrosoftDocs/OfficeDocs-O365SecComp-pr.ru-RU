Тем не менее некоторые приложения могут быть риска безопасности для вашей организации, в зависимости от какая информация они имеют доступ и как они работают эту информацию. ## <a name="how-to-find-the-manage-app-permissions-page"></a>Как найти страницу "Управление" разрешений приложения
<a name="findappperms"> </a>

> [!NOTE]
> Разрешения для приложений осуществляется в портале облачных приложений Office 365 безопасности. Необходимо быть глобальным администратором или администратор безопасности для выполнения следующих задач. Дополнительные сведения см. Дополнительные [разрешения безопасности Office 365 &amp; центре соответствия требованиям](permissions-in-the-security-and-compliance-center.md). 
  
1. Последовательно выберите пункты [https://protection.office.com](https://protection.office.com) и выполнить вход с помощью учетной записи рабочего или школы для Office 365. (Вы перейдете к безопасности &amp; центре соответствия требованиям.) 
    
2. Последовательно выберите пункты **оповещения** \> **Управление расширенного оповещения**.
    
3. Щелкните (или коснитесь) **перейдите к безопасности приложения Office 365 облака**.
    
    ![В разделе Безопасность &amp; центре соответствия требованиям, выберите дополнительные оповещения для перехода к безопасности Office 365 облаке приложения](media/958632d4-03e3-4ade-8e22-d5509db6fca7.png)
  
    > [!NOTE]
    > Если безопасности Office 365 облаке приложения не включен еще, можно сделать это на этой странице. В разделе [подготовиться к работе приложения облаке Безопасность в Office 365](get-ready-for-office-365-cas.md). 
  
4. Нажмите кнопку **Проверить** \> **разрешений приложения**.
    
    ![На портале O365 сервера клиентского доступа нажмите кнопку Проверить.](media/8c7b87c9-71a6-4952-adb2-185e941ffe9a.png)

## <a name="mark-an-app-as-approved"></a>Пометка приложения как Утверждено
<a name="approveapp"> </a>

На странице " **Управление разрешениями приложения** " найдите приложением, которое необходимо утвердить и выберите значок **приложение пометить как Утверждено** . 
  
![Выберите приложение пометить как утвержденные значок](media/dd1b7690-441a-48c9-9c3a-58466513c63d.png)
  
Значок включает зеленого и приложение утверждено для пользователей Office 365.
  
> [!NOTE]
> Если приложение помечены как утвержденные, нет не влияет на конечного пользователя. Визуально пометки приложений, одобренные позволяет разделять их из приложения, которые еще не были просмотрены. 
  
## <a name="ban-an-app"></a>BПричиной уязвимости является приложения
<a name="banapp"> </a>

1. На странице " **Управление разрешениями приложения** " найдите приложением, которое требуется для запрета и выберите значок **приложения пометить как запрещенный** . 
    
    ![Выберите приложение пометить как запрещенных значок](media/b9b1c5f6-fde7-46d5-8589-1564d05702b3.png)
  
2. Выберите, следует ли пользователи знали запретил, что их приложения.
    
    (Рекомендуется) Чтобы пользователи знали, выберите **уведомлять пользователей, которым предоставлен доступ к этому приложению запрещенных**и добавление или изменение настраиваемого уведомления.
    
    Чтобы не пользователи знали, снимите флажок **уведомлять пользователей, которым предоставлен доступ к этому приложению запрещенных**.
    
    ![Шаблон почты для запрещенных приложения](media/6d132700-5f7f-472c-bfb5-a44549e69c16.jpg)
  
3. Выберите **приложение запрещать**.
    
## <a name="create-an-app-query"></a>Создание запроса на приложение
<a name="createapp"> </a>

1. В панели запроса приложения Если **Дополнительно**, щелкните (или коснитесь) его для перехода в расширенном режиме. (Если отображаются Basic используется расширенный вид; сохранить представление как есть).
    
2. Используйте список **Выбор фильтра** для выбора параметра. В следующей таблице обобщаются параметры доступных фильтров. 
    
|**Используйте этот фильтр**|**Отображаемый результат**|
|:-----|:-----|
|**Приложение** <br/> |Приложения с определенными именами  <br/> |
|**Состояния приложения** <br/> |Приложения, в зависимости от их состояния (утверждено, Banned или Undetermined)  <br/> |
|**Использование сообщества** <br/> |Приложения на основе в сообществе использовать уровни (Rare, Uncommon или общий)  <br/> |
|**Уровень разрешений** <br/> |Приложения на основе определенных уровней разрешений  <br/> |
|**Разрешения** <br/> |Приложения, которым требуются определенные разрешения  <br/> |
|**Издатель** <br/> |Приложения из определенных издателей  <br/> |
|**Пользователь** <br/> |Приложения, которые право конкретного пользователя  <br/> |
   
3. Выберите **равно** или **не равно**, а затем укажите значение для фильтра.
    
4. Чтобы добавить дополнительные фильтры, выберите "плюс")![Добавьте значок фильтра для запроса приложения](media/771b2958-67cd-4e14-9302-283ef238cae5.jpg)), а затем повторите шаги 2 и 3.
    
5. Чтобы удалить фильтр, выберите x (![Удалить значок фильтра для запроса приложения](media/5339277f-555d-4749-8dcc-d2574250556e.jpg)) рядом с именем фильтра.
    
Фильтры применяются автоматически и соответствующим образом обновляется в список.
