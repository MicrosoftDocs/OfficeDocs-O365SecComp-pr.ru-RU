## <a name="mark-an-app-as-approved"></a>Пометка приложения как Утверждено

На странице " **Управление OAuth приложений** " найдите приложением, которое необходимо утвердить и выберите значок **приложение пометить как Утверждено** . 
  
![Выберите приложение пометить как утвержденные значок](media/OCAS-MarkOAuthApproved.png)
  
Значок включает зеленого и приложение утверждено для пользователей Office 365.
  
> [!NOTE]
> Если приложение помечены как утвержденные, нет не влияет на конечного пользователя. Визуально пометки приложений, одобренные позволяет разделять их из приложения, которые еще не были просмотрены. 
  
## <a name="ban-an-app"></a>BПричиной уязвимости является приложения

1. На странице " **Управление OAuth приложений** " найдите приложением, которое требуется для запрета и выберите значок **приложения пометить как запрещенный** .<br/>![Выберите приложение пометить как запрещенных значок](media/OCAS-MarkOAuthBanned.png)
  
2. В окне сообщения уведомлений сохранить существующий текст как есть или настроить текст. Выберите, следует ли пользователи знали запретил, что их приложения. <br/>![Шаблон почты для запрещенных приложения](media/6d132700-5f7f-472c-bfb5-a44549e69c16.jpg)<br/>
  
3. Выберите **приложение запрещать**.

## <a name="report-an-oauth-app-to-microsoft"></a>Сообщить о приложении OAuth в корпорацию Майкрософт

Если вы хотите отправить приложение OAuth в корпорацию Майкрософт для анализа, можно сообщить о этого приложения.

1. На странице " **Управление OAuth приложений** " найдите приложение, которое вы хотите отправить для анализа.

2. Выберите вертикальное многоточие, а затем выберите **отчет о app...**.<br/>![Сообщить о приложении OAuth](media/OCAS-MarkOAuthReported.png)<br/>

3. В диалоговом окне **отчета данного приложения** чтобы указать вашей вопросов используйте раскрывающегося списка. По умолчанию выбран **этого приложения вредоносных** . Тем не менее можно выбрать один из доступных вариантов.<br/>![Сообщить о приложении OAuth](media/OCAS-ReportOAuthApp.png)<br/>

4. (Рекомендуется) Оставьте параметр для связи с вами выбран и подтвердить (или изменить) на указанный адрес электронной почты.

5. Нажмите кнопку **Submit** (Отправить). 
    
## <a name="create-an-app-query"></a>Создание запроса на приложение

Рекомендуется использовать в расширенном режиме, которая выглядит следующим образом: 

![Расширенное представление](media/OCAS-OAuthAppsAdvQueryView.png)

В панели запроса приложения **Дополнительно**, если вы используете базового представления. Щелкните (или коснитесь) **Дополнительно** , чтобы перейти в расширенном режиме. 

![Основные представления](media/OCAS-OAuthAppsBasicQueryView.png)
    
1. В строке запроса используйте список **Выбор фильтра** для выбора параметра. 
    - **Приложения** Приложения с определенными именами
    - **Состояния приложения** Приложения, в зависимости от их состояния (утверждено, Banned или Undetermined)
    - **Используйте сообщества** Приложения на основе в сообществе использовать уровни (Rare, Uncommon или общий)
    - **Уровень разрешений** Приложения на основе определенных уровней разрешений 
    - **Разрешения** Приложения, которым требуются определенные разрешения
    - **Publisher**  Приложения из определенных издателей
    - **Пользователь** Приложения, которые право конкретного пользователя
   
2. Выберите **равно** или **не равно**, а затем укажите значение для фильтра.
    
3. Чтобы добавить дополнительные фильтры, выберите "плюс")![Добавьте значок фильтра для запроса приложения](media/771b2958-67cd-4e14-9302-283ef238cae5.jpg)), а затем повторите шаги 2 и 3.
    
4. Чтобы удалить фильтр, выберите x (![Удалить значок фильтра для запроса приложения](media/5339277f-555d-4749-8dcc-d2574250556e.jpg)) рядом с именем фильтра.
    
Фильтры применяются автоматически и соответствующим образом обновляется в список.
