# <a name="manage-mailbox-auditing"></a>Управление аудитом почтовых ящиков

Начиная с 2019 января Корпорация Майкрософт расписывает ведение журнала аудита почтовых ящиков по умолчанию для всех организаций Майкрософт 365. Это означает, что определенные действия, выполняемые владельцами, представителями и администраторами почтовых ящиков, автоматически регистрируются в журнале, и соответствующие записи аудита почтовых ящиков будут доступны при поиске их в журнале аудита почтовых ящиков. Если аудит почтовых ящиков включен по умолчанию, необходимо вручную включить его для каждого почтового ящика пользователя в Организации.

Ниже приведены некоторые преимущества аудита почтовых ящиков по умолчанию.

- Аудит автоматически включается при создании нового почтового ящика. Вам не нужно включать его вручную для новых пользователей.

- Управлять действиями почтовых ящиков, которые подлежат аудиту, не требуется. По умолчанию для каждого типа входа в систему (администратора, делегата и владельца) подписывается предварительно определенный набор действий почтового ящика.

- Когда корпорация Майкрософт выпускает новое действие почтового ящика (особенно для защиты Организации от судебного исследования), это действие автоматически добавляется в список действий почтовых ящиков, которые подлежат аудиту по умолчанию. Это означает, что вам не нужно следить за добавлением новых действий в почтовых ящиках.

- В вашей организации используется согласованная политика аудита почтовых ящиков (так как для всех почтовых ящиков выполняется аудит одних и тех же действий).

> [!TIP]
> Важно помнить, что по умолчанию при аудите почтовых ящиков не требуется никаких действий для управления аудитом почтовых ящиков. Тем не менее, чтобы узнать больше, настроить аудит почтовых ящиков по умолчанию или полностью отключить его, эта тема может помочь.

## <a name="verify-mailbox-auditing-on-by-default-is-turned-on"></a>Убедитесь, что аудит почтовых ящиков включен по умолчанию

Чтобы убедиться, что аудит почтовых ящиков включен по умолчанию для вашей организации, выполните следующую команду в [Exchange Online PowerShell](https://docs.microsoft.com/powershell/exchange/exchange-online/connect-to-exchange-online-powershell/connect-to-exchange-online-powershell):

```
Get-OrganizationConfig | Format-List AuditDisabled
```

Значение **false** указывает, что аудит почтовых ящиков включен по умолчанию для Организации. Это значение по умолчанию переопределяет параметр аудита почтовых ящиков для определенных почтовых ящиков. Например, если для почтового ящика отключена функция аудита почтовых ящиков (свойство *auditenabled указывает* имеет **значение false** в почтовом ящике), по умолчанию для почтового ящика будут применяться все действия по умолчанию, так как аудит почтовых ящиков включен по умолчанию для предприятии.

Чтобы отключить аудит почтовых ящиков для определенных почтовых ящиков, необходимо настроить обход аудита почтовых ящиков для владельца почтового ящика и других пользователей, которым был делегирован доступ к почтовому ящику. Для получения дополнительных сведений см раздел " [обход ведения журнала аудита](#bypass-mailbox-audit-logging) почтовых ящиков" этой статьи.

> [!NOTE]
> Если для Организации включено ведение аудита почтовых ящиков по умолчанию, свойство *auditenabled указывает* для затронутых почтовых ящиков не будет изменено с " **false** " на " **true**". Другими словами, аудит почтовых ящиков по умолчанию игнорирует свойство *auditenabled указывает* в почтовых ящиках.

## <a name="logon-types-and-mailbox-actions"></a>Типы входа и действия с почтовыми ящиками

Типы входа классифицируют пользователя, который выполнил аудит действий в почтовом ящике. В следующем списке описываются типы входа в систему, которые используются в журнале аудита почтовых ящиков.

- **Owner**: владелец почтового ящика (учетная запись, связанная с почтовым ящиком).

- **Delegate**:

  - Пользователь, которому назначено разрешение SendAs, SendOnBehalf или FullAccess для другого почтового ящика.

  - Администратор, которому назначено разрешение FullAccess для почтового ящика пользователя.

- **Администратор**:

  - В почтовом ящике выполняется поиск по одному из следующих средств обнаружения электронных данных Майкрософт:

    - Поиск контента в центре соответствия требованиям.

    - Обнаружение электронных данных или Расширенное обнаружение электронных данных в центре соответствия требованиям.

    - Обнаружение электронных данных на месте в Exchange Online.

  - Доступ к почтовому ящику осуществляется с помощью [редактора MAPI Microsoft Exchange Server](https://go.microsoft.com/fwlink/p/?linkId=204086).

### <a name="verify-that-default-mailbox-actions-are-being-logged-for-each-logon-type"></a>Проверка того, что для каждого типа входа в систему регистрируются действия почтового ящика по умолчанию

При аудите почтовых ящиков по умолчанию добавляется новое свойство *дефаултаудитсет* для всех почтовых ящиков. Значение этого свойства указывает, подлежат ли аудиту действия почтового ящика по умолчанию (управляется корпорацией Майкрософт) в почтовом ящике.

Чтобы отобразить значение в почтовых ящиках пользователей или общих \<почтовых ящиках, замените MailboxIdentity\> на имя, псевдоним, адрес электронной почты или имя участника-пользователя (имя пользователя) почтового ящика и выполните следующую команду в Exchange Online PowerShell:

```
Get-Mailbox -Identity <MailboxIdentity> | Format-List DefaultAuditSet
```

Для отображения значения в почтовых ящиках группы Office \<365\> замените MailboxIdentity на имя, псевдоним или адрес электронной почты общего почтового ящика и выполните следующую команду в Exchange Online PowerShell:

```
Get-Mailbox -Identity <MailboxIdentity> -GroupMailbox | Format-List DefaultAuditSet
```

Значение `Admin, Delegate, Owner` указывает на следующее:

- Выполняется аудит действий почтового ящика по умолчанию для всех трех типов входа. Обратите внимание, что это единственное значение, которое вы увидите в почтовых ящиках Office 365.

- Администратор *не* изменил действия аудита для любого типа входа в почтовом ящике пользователя или в общем почтовом ящике. Примечание Это состояние по умолчанию после аудита почтовых ящиков по умолчанию включено в Организации.

Если администратор когда-либо изменил действия почтового ящика, которые были проверены для типа входа (с помощью параметров *аудитадмин*, *аудитделегате*или *аудитовнер* в командлете **Set-Mailbox** ), то значение свойства будет другим.

Например, значение `Owner` свойства *дефаултаудитсет* почтового ящика пользователя или общего почтового ящика указывает на следующее:

- Выполняется аудит действий почтового ящика, используемых по умолчанию для владельца почтового ящика.

- Действия, выполняемые при аудите `Delegate` для `Admin` типов и типов входа, были изменены с помощью действий по умолчанию.

Пустое значение для свойства *дефаултаудитсет* указывает на то, что в почтовом ящике пользователя или общем почтовом ящике были изменены действия почтового ящика для всех трех типов входа.

Дополнительные сведения приведены в разделе [изменение или восстановление действий почтового ящика, регистрируемого](#change-or-restore-mailbox-actions-logged-by-default) по умолчанию в этой статье

## <a name="change-or-restore-mailbox-actions-logged-by-default"></a>Изменение или восстановление действий почтового ящика, регистрируемого по умолчанию

Как описывалось ранее, одно из основных преимуществ аудита почтовых ящиков по умолчанию: нет необходимости управлять действиями почтовых ящиков, для которых выполняется аудит. Корпорация Майкрософт делает это автоматически и автоматически добавляет новые действия в почтовые ящики, которые будут подвергаться по умолчанию при их отредактировании.

Однако в Организации может потребоваться аудит различных наборов действий почтовых ящиков для почтовых ящиков пользователей и общих почтовых ящиков. Процедуры, описанные в этом разделе, показывают, как изменить действия почтового ящика, проверяемые для каждого типа входа в систему, и вернуться к действиям по умолчанию, управляемым корпорацией Майкрософт.

> [!IMPORTANT]
> Если вы используете следующие процедуры для настройки действий почтового ящика, вошедших в почтовые ящики пользователей или общие почтовые ящики, все новые действия по умолчанию, выпускаемые корпорацией Майкрософт, не будут автоматически проводить аудит этих почтовых ящиков Вам потребуется вручную добавить любые новые действия почтового ящика в свой настраиваемый список действий.

### <a name="change-the-mailbox-actions-to-audit"></a>Изменение действий в почтовом ящике для аудита

Вы можете использовать параметры *аудитадмин*, *аудитделегате*или *аудитовнер* в командлете **Set-Mailbox** , чтобы изменить действия почтового ящика, проверяемые для почтовых ящиков пользователей и общих почтовых ящиков (аудит действий для группы Office 365 невозможно Настройка почтовых ящиков.

Для указания действий почтовых ящиков можно использовать два разных метода:

- *Замена* (перезаписать) существующие действия почтового ящика, используя следующий `action1,action2,...actionN`синтаксис:.

- *Добавьте или удалите* действия почтового ящика, не затрагивая другие существующие значения, `@{Add="action1","action2",..."actionN"}` используя `@{Remove="action1","action2",..."actionN"}`следующий синтаксис: или.

В этом примере показано, как изменить действия почтового ящика администратора для почтового ящика под названием "Габриела Лауреано", перезаписав действия по умолчанию на SoftDelete и HardDelete.

```
Set-Mailbox -Identity "Gabriela Laureano" -AuditAdmin HardDelete,SoftDelete
```

В этом примере показано, как добавить действие Owner MailboxLogin в почтовый ящик laura@contoso.onmicrosoft.com.

```
Set-Mailbox -Identity laura@contoso.onmicrosoft.com -AuditOwner @{Add="MailboxLogin"}
```

В этом примере показано удаление делегированного действия MoveToDeletedItems для почтового ящика группового обсуждения.

```
Set-Mailbox -Identity "Team Discussion" -AuditDelegate @{Remove="MoveToDeletedItems"}
```

Независимо от используемого метода, Настройка действий по аудиту почтовых ящиков пользователей и общих почтовых ящиков имеет следующие результаты.

- Для учетной записи, которую вы настроили, действия, выполняемые при проверке почтовых ящиков, больше не управляются корпорацией Майкрософт.

- Настраиваемый тип входа больше не отображается в значении свойства *дефаултаудитсет* почтового ящика, как [описано ранее](#verify-that-default-mailbox-actions-are-being-logged-for-each-logon-type).

### <a name="restore-the-default-mailbox-actions"></a>Восстановление действий почтового ящика по умолчанию

Если вы настроили действия почтового ящика, которые подлежат аудиту для почтового ящика пользователя или общего почтового ящика, вы можете восстановить действия почтового ящика по умолчанию для одного или всех типов входа, используя следующий синтаксис:

```
Set-Mailbox -Identity <MailboxIdentity> -DefaultAuditSet <Admin | Delegate | Owner>
```

Можно указать несколько значений *дефаултаудитсет* , разделенных запятыми.

**Примечание**: следующие процедуры не применяются к почтовым ящикам Office 365 (они ограничены действиями по умолчанию, как описано в [этой статье](#mailbox-actions-for-office-365-group-mailboxes)).

В этом примере восстанавливаются действия по умолчанию для всех типов входа в почтовом ящике mark@contoso.onmicrosoft.com.

```
Set-Mailbox -Identity mark@contoso.onmicrosoft.com -DefaultAuditSet Admin,Delegate,Owner
```

В этом примере восстанавливаются действия по умолчанию для входного почтового ящика для типа "Администратор" в почтовом ящике chris@contoso.onmicrosoft.com, но для типов входа и владельца почтовых ящиков не сохраняются.

```
Set-Mailbox -Identity chris@contoso.onmicrosoft.com -DefaultAuditSet Admin
```

Восстановление действий по умолчанию для входного почтового ящика для типа входа имеет следующие результаты.

- Текущий список действий почтового ящика заменяется действиями почтового ящика по умолчанию для типа входа.

- Все новые действия с почтовыми ящиками, выпускаемые корпорацией Майкрософт, автоматически добавляются в список проверяемых действий для типа входа.

- Значение свойства *дефаултаудитсет* для почтового ящика обновляется, чтобы включить восстановленный тип входа.

## <a name="turn-off-mailbox-auditing-on-by-default-for-your-organization"></a>По умолчанию отключить аудит почтовых ящиков в Организации

Вы можете отключить аудит почтовых ящиков по умолчанию для всей Организации, выполнив следующую команду в Exchange Online PowerShell:

```
Set-OrganizationConfig -AuditDisabled $true
```

Отключение аудита почтовых ящиков по умолчанию имеет следующие результаты.

- Аудит почтовых ящиков отключен для вашей организации.

- С момента отключения аудита почтовых ящиков по умолчанию никакие действия не подлежат аудиту, даже если аудит включен для почтового ящика (свойство *auditenabled указывает* в почтовом ящике имеет **значение true**).

- Аудит почтовых ящиков не включен для новых почтовых ящиков, и установка свойства *auditenabled указывает* для нового или существующего почтового ящика в **значение true** будет игнорироваться.

- Все параметры связи обхода аудита почтовых ящиков (настроенные с помощью командлета **Set-MailboxAuditBypassAssociation** ) игнорируются.

- Существующие записи аудита почтовых ящиков сохраняются до тех пор, пока не истечет срок хранения журнала аудита для записи.

### <a name="turn-on-mailbox-auditing-on-by-default"></a>Включение аудита почтовых ящиков по умолчанию

Чтобы включить аудит почтовых ящиков в Организации, выполните следующую команду в Exchange Online PowerShell:

```
Set-OrganizationConfig -AuditDisabled $false
```

## <a name="bypass-mailbox-audit-logging"></a>Обход журнала аудита почтовых ящиков

В настоящее время невозможно отключить аудит почтовых ящиков для определенных почтовых ящиков, если в Организации включен аудит почтовых ящиков по умолчанию. Например, установка свойства почтового ящика *auditenabled указывает* в **значение false** игнорируется.

Тем не менее, вы по-прежнему можете использовать командлет **Set-MailboxAuditBypassAssociation** в Exchange Online ** PowerShell, чтобы запретить вход в систему всех действий почтовых ящиков с указанными пользователями, независимо от того, где происходят действия. Пример:

- Действия владельца почтового ящика, выполняемые обходными пользователями, не заносятся в журнал.

- Делегирование действий, выполняемых обходными пользователями почтовых ящиков других пользователей (включая общие почтовые ящики), в журнал не ведется.

- Действия администратора, выполняемые обходными пользователями, не заносятся в журнал.

Чтобы обойти журнал аудита почтовых ящиков для \<определенного\> пользователя, замените MailboxIdentity именем, адресом электронной почты, псевдонимом или именем участника-пользователя (username) пользователя и выполните следующую команду:

```
Set-MailboxAuditBypassAssociation -Identity <MailboxIdentity> -AuditByPassEnabled $true
```

Чтобы убедиться, что аудит обходится для указанного пользователя, выполните следующую команду:

```
Get-MailboxAuditBypassAssociation -Identity <MailboxIdentity> | Format-List AuditByPassEnabled
```

Значение **true** указывает, что ведение журнала аудита почтовых ящиков для пользователя обходится.

## <a name="more-information"></a>Дополнительные сведения

- По умолчанию записи журнала аудита почтовых ящиков сохраняются в течение 90 дней до их удаления. Вы можете изменить срок хранения записей журнала аудита с помощью параметра *аудитлогажелимит* командлета **Set-Mailbox** в Exchange Online PowerShell. Однако увеличение этого значения не позволяет искать события, возраст которых превышает 90 дней, в журнале аудита Microsoft 365.

  При увеличении срока хранения необходимо использовать командлет [Search-маилбоксаудитлог](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-audit/search-mailboxauditlog) в Exchange Online PowerShell, чтобы искать записи, возраст которых превышает 90 дней, в журнале аудита почтового ящика пользователя.

- Если вы изменили свойство *аудитлогажелимит* для почтового ящика, прежде чем аудит почтовых ящиков включен по умолчанию для Организации, срок хранения журнала аудита этого почтового ящика не изменяется. Другими словами, аудит почтовых ящиков по умолчанию не оказывает силы текущего срока хранения записей аудита почтовых ящиков.

- Чтобы изменить значение *аудитлогажелимит* для почтового ящика группы Office 365, необходимо включить `-GroupMailbox` параметр в команду **Set/Mailbox** .

- Записи журнала аудита почтовых ящиков хранятся во вложенной папке (с именем *audits*) в папке "элементы с возможностью восстановления" в почтовом ящике каждого пользователя. Следует учитывать следующие моменты, касающиеся записей аудита почтовых ящиков и папки "элементы с возможностью восстановления":

  - Количество записей аудита почтовых ящиков в соответствии с квотой хранилища папки "элементы с возможностью восстановления", которая по умолчанию 30GB (квота предупреждения составляет 20 ГБ). Квота хранилища автоматически увеличивается до 100 ГБ (с квотой предупреждений 90 ГБ), когда:

    - Удержание размещается для почтового ящика.

    - Почтовый ящик назначается политике хранения в центре соответствия требованиям.

  - Кроме того, количество записей аудита почтовых ящиков не [превышает максимально допустимое для папки "элементы с возможностью восстановления](https://docs.microsoft.com/office365/servicedescriptions/exchange-online-service-description/exchange-online-limits#mailbox-folder-limits)". В подпапке audits можно хранить не более 3 000 000 элементов (записей аудита).

    > [!NOTE]
    > Маловероятно, что аудит почтовых ящиков по умолчанию влияет на квоту хранилища или максимальное количество папок для папки "элементы с возможностью восстановления".

    - В Exchange Online PowerShell можно выполнить следующую команду для отображения размера и количества элементов в подпапке "audits" в папке "элементы с возможностью восстановления":

      ```
      Get-MailboxFolderStatistics -Identity <MailboxIdentity> -FolderScope RecoverableItems | Where-Object {$_.Name -eq 'Audits'} | Format-List FolderPath,FolderSize,ItemsInFolder
      ```

    - Вы не можете напрямую получить доступ к записи журнала аудита в папке "элементы с возможностью восстановления"; Вместо этого используйте командлет **Search — маилбоксаудитлог** или найдите в журнале аудита Microsoft 365 Поиск и просмотр записей аудита почтовых ящиков.

- Если почтовый ящик размещается на удержании или назначается политике хранения в центре соответствия требованиям, записи журнала аудита по-прежнему хранятся в течение срока, определенного свойством *аудитлогажелимит* почтового ящика (по умолчанию 90 дней). Чтобы сохранять записи журнала аудита дольше для почтовых ящиков на удержании, необходимо увеличить значение *аудитлогажелимит* в почтовом ящике.
