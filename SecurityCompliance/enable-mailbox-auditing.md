# <a name="enable-mailbox-auditing-in-office-365"></a>Включение аудита почтовых ящиков в Office 365
  
В Office 365 вы можете включить ведение журнала аудита почтовых ящиков, чтобы получать доступ к почтовым ящикам с помощью владельцев, делегатов и администраторов почтовых ящиков. По умолчанию аудит почтовых ящиков в Office 365 не включен. Это означает, что события аудита почтовых ящиков не будут отображаться в результатах поиска в журнале аудита Office 365 для действия почтовых ящиков. Но после включения ведения журнала аудита почтовых ящиков для почтового ящика пользователя можно выполнить поиск по журналу аудита для активности почтовых ящиков. Кроме того, если включено ведение журнала аудита почтовых ящиков, некоторые действия, выполняемые администраторами, представителями и владельцами, записываются по умолчанию. Чтобы выполнить вход в систему (а затем найти) дополнительные действия, обратитесь к этапу 3.

## <a name="before-you-begin"></a>Перед началом работы
  
- Чтобы включить ведение журнала аудита почтовых ящиков, необходимо использовать Exchange Online PowerShell. Вы не можете использовать центр безопасности &amp; и соответствия требованиям Office 365 или центр администрирования Exchange.
    
- Вы не можете включить ведение журнала аудита почтового ящика для почтового ящика, связанного с группой Office 365 или группой в Microsoft Teams.
    
- Администратор, которому предоставлены разрешения на полный доступ к почтовому ящику пользователя, считается делегированным пользователем.
  
## <a name="step-1-connect-to-exchange-online-powershell"></a>Шаг 1: подключение к Exchange Online PowerShell

1. На локальном компьютере откройте Windows PowerShell и запустите следующую команду:
   
    ```
    $UserCredential = Get-Credential
    ```

2. В диалоговом окне **Запрос учетных данных Windows PowerShell** введите имя пользователя и пароль учетной записи глобального администратора Office 365, а затем нажмите кнопку **ОК**.
    
3. Выполните следующую команду:
    
    ```
    $Session = New-PSSession -ConfigurationName Microsoft.Exchange -ConnectionUri https://outlook.office365.com/powershell-liveid/ -Credential $UserCredential -Authentication Basic -AllowRedirection
    ```

3. Выполните следующую команду:

    ```
    Import-PSSession $Session
    ```
   
4. Убедитесь, что подключение к организации Exchange Online установлено, получив список всех почтовых ящиков в организации с помощью следующей команды:
    
    ```
    Get-Mailbox
    ```
  
Дополнительные сведения или варианты решения проблем при подключении к организации Exchange Online см. в статье [Подключение к Exchange Online с помощью удаленной оболочки PowerShell](https://go.microsoft.com/fwlink/p/?LinkId=517283).
  
## <a name="step-2-enable-mailbox-audit-logging"></a>Этап 2. Включение журнала аудита почтовых ящиков

После подключения к организации Exchange Online используйте PowerShell, чтобы включить ведение журнала аудита почтовых ящиков для почтового ящика. Кроме того, вы можете включить аудит почтовых ящиков для всех почтовых ящиков в Организации.
  
В этом примере включается ведение журнала аудита почтовых ящиков для почтового ящика Pilar Pinilla.
  
```
Set-Mailbox -Identity "Pilar Pinilla" -AuditEnabled $true
```

В этом примере мы включили ведение журнала аудита для почтовых ящиков всех пользователей в организации.
  
```
Get-Mailbox -ResultSize Unlimited -Filter {RecipientTypeDetails -eq "UserMailbox"} | Set-Mailbox -AuditEnabled $true
```
  
## <a name="step-3-specify-owner-actions-to-audit"></a>Этап 3. Выбор действий владельца, регистрируемых в журнале аудита

Когда вы включаете аудит для почтового ящика, некоторые действия, выполняемые владельцем почтового ящика, подлежат аудиту по умолчанию. Для аудита необходимо указать другие действия владельца. В таблице в разделе [действия аудита](#mailbox-auditing-actions) поЧтовых ящиков приведен список и описание действий владельцев, которые регистрируются по умолчанию, а также другие действия, которые можно подвергать аудиту. 
  
В этом примере показано, как добавить действия владельца **MailboxLogin** и **HardDelete** в аудит почтовых ящиков для почтового ящика Pilar Pinilla. В этом примере предполагается, что аудит почтовых ящиков уже включен для этого почтового ящика. 

```
Set-Mailbox "Pilar Pinilla" -AuditOwner @{Add="MailboxLogin","HardDelete"}
```

В этом примере включается ведение журнала аудита почтовых ящиков для почтового ящика Дон зал и указывается, что только действие **MailboxLogin** , выполняемое владельцем почтового ящика, будет протоколироваться. Обратите внимание на то, что в этом примере перезаписывается действие UpdateFolderPermissions по умолчанию. 
  
```
Set-Mailbox "Don Hall" -AuditEnabled $true -AuditOwner MailboxLogin
```
   
В этом примере добавляются действия **MailboxLogin**, **HardDelete**и **SoftDelete** для всех почтовых ящиков в Организации. В этом примере предполагается, что аудит почтовых ящиков уже включен для всех почтовых ящиков. 
  
```
Get-Mailbox -ResultSize Unlimited -Filter {RecipientTypeDetails -eq "UserMailbox"} | Set-Mailbox -AuditOwner @{Add="MailboxLogin","HardDelete","SoftDelete"}
```
  
## <a name="how-do-you-know-this-worked"></a>Как убедиться, что все получилось?

Чтобы убедиться, что ведение журнала аудита для почтового ящика успешно активировано, с помощью командлета **Get-Mailbox** получите параметры аудита этого почтового ящика. 
  
В этом примере показано получение параметров аудита для Pilar Pinilla.

```
Get-Mailbox "Pilar Pinilla"| FL Audit*
```
   
В этом примере показано получение параметров аудита для почтовых ящиков всех пользователей в организации.

```
Get-Mailbox -ResultSize Unlimited -Filter {RecipientTypeDetails -eq "UserMailbox"} | FL Name,Audit*
```
   
Значение **true** для свойства **auditenabled указывает** проверяет, включено ли ведение журнала аудита почтовых ящиков.

- Считается, что администраторы получают доступ к почтовым ящикам только в следующих случаях:
    
  - Обнаружение электронных данных на месте в Exchange Online или поиск контента в Office 365 используется для поиска в почтовом ящике.
    
  - [Средство MAPI Editor для Microsoft Exchange Server](https://go.microsoft.com/fwlink/p/?linkId=204086) используется для доступа к почтовому ящику. 
    
- Включая ведение журнала аудита для почтового ящика, вы можете указать действия (например, доступ к почтовому ящику, перемещение или удаление сообщений), которые должны регистрироваться для каждого типа входа (с помощью учетной записи администратора, делегата или владельца). 
    
- Чтобы отключить ведение журнала аудита почтовых ящиков, выполните следующую команду:

  ```
  Set-Mailbox -Identity <identity of mailbox> -AuditEnabled $false
   ```

- Действия, которые подлежат аудиту для каждого типа пользователей, не отображаются при запуске командлета **Get/Mailbox** . Однако можно выполнить следующие команды, чтобы отобразить все действия аудита для определенного типа входа пользователя. 

    ```
    Get-Mailbox <identity of mailbox> | Select-Object -ExpandProperty AuditAdmin
    ```

    ```
    Get-Mailbox <identity of mailbox> | Select-Object -ExpandProperty AuditDelegate
    ```

    ```
    Get-Mailbox <identity of mailbox> | Select-Object -ExpandProperty AuditOwner
    ```

- Кроме того, можно экспортировать журнал аудита почтовых ящиков и указать записи, включаемые для одного или нескольких пользователей. Каждая запись в отчете и в журнале аудита содержит сведения о том, кто выполнил действие, а также о том, когда действие выполнено и что действие выполнено успешно. Дополнительные сведения можно найти в статье [Экспорт журналов аудита](https://go.microsoft.com/fwlink/p/?LinkID=404104)почтовых ящиков.
