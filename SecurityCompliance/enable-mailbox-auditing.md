# <a name="enable-mailbox-auditing-in-office-365"></a>Включение аудита почтовых ящиков в Office 365
  
В Office 365 можно включить ведение журнала аудита почтовых ящиков для регистрировать доступ владельцев, делегатов и администраторов. По умолчанию аудита почтовых ящиков в Office 365 не включено. Это означает, что при выполнении поиска журнал аудита Office 365 для почтового ящика действия аудита почтовых ящиков не будет отображаться в результатах. Но после включения почтового ящика ведения журнала аудита для почтового ящика пользователя можно выполнить поиск журнала аудита для почтового ящика активности. Кроме того когда аудита почтовых ящиков ведения журнала, некоторые действия, выполняемые администраторами, делегаты, и владельцев заносятся по умолчанию. Для входа (и затем найдите) Дополнительные действия, содержатся в шаге 3.

## <a name="before-you-begin"></a>Перед началом работы
  
- Вам следует использовать для включения почтовых ящиков Exchange Online PowerShell ведения журнала аудита. Нельзя использовать безопасности Office 365 &amp; центре соответствия требованиям или центра администрирования Exchange.
    
- После включения ведения журнала для почтового ящика аудита доступа для почтового ящика и некоторые admin и делегат действия записываются по умолчанию. Чтобы войти в действиях, выполненных владельца почтового ящика, необходимо определить, какие действия владельца для аудита. Чтобы просмотреть список действий, которые записываются после ведения журнала аудита почтового ящика включена и действий, доступные для каждого типа входа пользователя в разделе «Дополнительные сведения».
    
- Не удается включить ведение журнала для почтового ящика, сопоставленные с группу Office 365 или группы в группах Microsoft аудита почтовых ящиков.
    
- Администратор, которому предоставлены разрешения на полный доступ к почтовому ящику пользователя, считается делегированным пользователем.
  
## <a name="step-1-connect-to-exchange-online-powershell"></a>Шаг 1: Подключение к Exchange Online PowerShell

1. На локальном компьютере откройте Windows PowerShell и запустите следующую команду:
   
    ```
    $UserCredential = Get-Credential
    ```

2. В диалоговом окне **Запрос учетных данных Windows PowerShell** введите имя пользователя и пароль учетной записи глобального администратора Office 365, а затем нажмите кнопку **ОК**.
    
3. Выполните следующую команду:
    
    ```
    $Session = New-PSSession -ConfigurationName Microsoft.Exchange -ConnectionUri https://outlook.office365.com/powershell-liveid/ -Credential $UserCredential -Authentication Basic -AllowRedirection
    ```

3. Выполните следующую команду:

    ```
    Import-PSSession $Session
    ```
   
4. Убедитесь, что подключение к организации Exchange Online установлено, получив список всех почтовых ящиков в организации с помощью следующей команды:
    
    ```
    Get-Mailbox
    ```
  
Дополнительные сведения или варианты решения проблем при подключении к организации Exchange Online см. в статье [Подключение к Exchange Online с помощью удаленной оболочки PowerShell](https://go.microsoft.com/fwlink/p/?LinkId=517283).
  
## <a name="step-2-enable-mailbox-audit-logging"></a>Этап 2. Включение журнала аудита почтовых ящиков

После подключения к организации Exchange Online с помощью PowerShell для включения почтового ящика ведения журнала аудита для почтового ящика. Кроме того можно включить аудит почтовых ящиков для всех почтовых ящиков в организации.
  
В этом примере включается почтового ящика ведения журнала аудита для почтового ящика пользователя Марты Артемьевой.
  
```
Set-Mailbox -Identity "Pilar Pinilla" -AuditEnabled $true
```

В этом примере мы включили ведение журнала аудита для почтовых ящиков всех пользователей в организации.
  
```
Get-Mailbox -ResultSize Unlimited -Filter {RecipientTypeDetails -eq "UserMailbox"} | Set-Mailbox -AuditEnabled $true
```
  
## <a name="step-3-specify-owner-actions-to-audit"></a>Этап 3. Выбор действий владельца, регистрируемых в журнале аудита

Если включить аудит для почтового ящика, только одна действия ( **UpdateFolderPermissions** ) владельца почтового ящика записывается по умолчанию. Необходимо указать другие действия владельца для аудита. В разделе таблицы в разделе «Действия» список и описание владелец действий, которые могут быть проверены. 
  
В этом примере добавляется владелец действий **MailboxLogin** и **HardDelete** к почтовому ящику, аудит для почтового ящика пользователя Марты Артемьевой. В этом примере предполагается, что почтовый ящик уже включен аудит для этого почтового ящика. 

```
Set-Mailbox "Pilar Pinilla" -AuditOwner @{Add="MailboxLogin","HardDelete"}
```

В этом примере включается почтового ящика ведения журнала аудита для почтового ящика Дон зал и задает регистрацию только **MailboxLogin** действия владельца почтового ящика. Обратите внимание на то, что в этом примере перезаписывает UpdateFolderPermissions действия по умолчанию. 
  
```
Set-Mailbox "Don Hall" -AuditEnabled $true -AuditOwner MailboxLogin
```
   
В этом примере добавляется владелец действия **MailboxLogin**, **HardDelete**и **SoftDelete** ко всем почтовым ящикам в организации. В этом примере предполагается, что почтовый ящик уже включен аудит для всех почтовых ящиков. 
  
```
Get-Mailbox -ResultSize Unlimited -Filter {RecipientTypeDetails -eq "UserMailbox"} | Set-Mailbox -AuditOwner @{Add="MailboxLogin","HardDelete","SoftDelete"}
```
  
## <a name="how-do-you-know-this-worked"></a>Как проверить, что все получилось?

Чтобы убедиться, что ведение журнала аудита для почтового ящика успешно активировано, с помощью командлета **Get-Mailbox** получите параметры аудита этого почтового ящика. 
  
В этом примере показано получение параметров аудита для Pilar Pinilla.

```
Get-Mailbox "Pilar Pinilla"| FL Audit*
```
   
В этом примере показано получение параметров аудита для почтовых ящиков всех пользователей в организации.

```
Get-Mailbox -ResultSize Unlimited -Filter {RecipientTypeDetails -eq "UserMailbox"} | FL Name,Audit*
```
   
Значение **True** для **атрибута auditenabled** проверяет, аудита почтовых ящиков включено ведение журнала.

- Считается, что администраторы получают доступ к почтовым ящикам только в следующих случаях:
    
  - Обнаружение электронных данных на месте в Exchange Online или поиска контента в Office 365 используется для поиска в почтовом ящике.
    
  - [Средство MAPI Editor для Microsoft Exchange Server](https://go.microsoft.com/fwlink/p/?linkId=204086) используется для доступа к почтовому ящику. 
    
- Включая ведение журнала аудита для почтового ящика, вы можете указать действия (например, доступ к почтовому ящику, перемещение или удаление сообщений), которые должны регистрироваться для каждого типа входа (с помощью учетной записи администратора, делегата или владельца). 
    
- Чтобы отключить ведение журнала аудита почтовых ящиков, выполните следующую команду:

  ```
  Set-Mailbox -Identity <identity of mailbox> -AuditEnabled $false
   ```

- Действия, которые были проверены для каждого типа пользователя не отображаются при запуске командлета **Get-Mailbox** . Однако можно выполните следующие команды для отображения всех аудита действий для типа входа в систему определенного пользователя. 

    ```
    Get-Mailbox <identity of mailbox> | Select-Object -ExpandProperty AuditAdmin
    ```

    ```
    Get-Mailbox <identity of mailbox> | Select-Object -ExpandProperty AuditDelegate
    ```

    ```
    Get-Mailbox <identity of mailbox> | Select-Object -ExpandProperty AuditOwner
    ```

- Также можно экспортировать журнал аудита почтовых ящиков и указать записей для одного или нескольких пользователей. Каждый элемент в отчете и журнал аудита включает сведения о, выполнившего действия и при выполнении действия, а также будет ли операция выполнена успешно. Дополнительные сведения можно [экспортировать журналы аудита почтовых ящиков](https://go.microsoft.com/fwlink/p/?LinkID=404104).
