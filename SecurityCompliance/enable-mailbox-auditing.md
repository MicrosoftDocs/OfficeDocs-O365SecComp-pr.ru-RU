# <a name="manage-mailbox-auditing"></a>Управление аудитом почтовых ящиков
  
Начиная с 2019 января Корпорация Майкрософт расписывает ведение журнала аудита почтовых ящиков по умолчанию для всех организаций Майкрософт 365. Это означает, что определенные действия, выполняемые владельцами, представителями и администраторами почтовых ящиков, автоматически регистрируются в журнале, и соответствующие записи аудита почтовых ящиков будут доступны при поиске их в журнале аудита почтовых ящиков. Если аудит почтовых ящиков включен по умолчанию, необходимо вручную включить его для каждого почтового ящика пользователя в Организации. 

Ниже приведены некоторые преимущества аудита почтовых ящиков по умолчанию.

- При создании нового почтового ящика аудит будет включен по умолчанию. Вам не потребуется включать его вручную для новых пользователей. 

- Вы не будете управлять действиями почтовых ящиков, для которых выполняется аудит. Определенный набор действий почтовых ящиков по умолчанию подлежит аудиту для каждого типа входа. Эти [типы входа](#mailbox-actions-logged-by-default) являются владельцами, представителями и администраторами.

- По умолчанию аудит новых действий почтового ящика, выпущенных корпорацией Майкрософт, будет выполняться по умолчанию Когда корпорация Майкрософт выпускает новое действие с почтовым ящиком (в частности, которое помогает защитить организацию и помогить расследованию в судебном расследовании), оно автоматически добавляется в список действий по умолчанию, проверяемых в почтовые ящики. Это означает, что добавлять новые действия в список действий почтовых ящиков, выполняемых владельцами, представителями или администраторами, не нужно. 

- Убедитесь, что аудит выполняется для всех почтовых ящиков, чтобы политика аудита почтовых ящиков была единообразно настроена в Организации.

> [!TIP]
> Важно помнить, что при выпуске аудита почтовых ящиков по умолчанию нет необходимости выполнять какие-либо действия для управления аудитом почтовых ящиков. Тем не менее, если вы хотите узнать больше, измените поведение по умолчанию или полностью выключите его, эта статья поможет вам в этом.

## <a name="verify-mailbox-auditing-on-by-default-is-turned-on"></a>Убедитесь, что аудит почтовых ящиков включен по умолчанию

Чтобы убедиться, что аудит почтовых ящиков включен по умолчанию для вашей организации, выполните следующую команду в [Exchange Online PowerShell](https://docs.microsoft.com/powershell/exchange/exchange-online/connect-to-exchange-online-powershell/connect-to-exchange-online-powershell):

```
Get-OrganizationConfig | FL AuditDisabled
``` 

Значение **false** указывает, что аудит почтовых ящиков включен по умолчанию для вашей организации. 

Если включено ведение аудита почтовых ящиков по умолчанию (если для свойства *аудитдисаблед* задано **значение false**), организационный параметр будет переопределять параметр аудита почтовых ящиков для определенного почтового ящика. Например, если для свойства *auditenabled указывает* почтового ящика задано **значение false**, но для вашей организации включен аудит почтовых ящиков по умолчанию, для этого почтового ящика будут применяться действия по умолчанию для этого почтового ящика. Если аудит почтовых ящиков был явно отключен для определенного почтового ящика, но его все еще требуется отключить, можно настроить обход аудита почтовых ящиков для владельца почтового ящика и других пользователей, которым был делегирован доступ к почтовому ящику. Для получения дополнительных сведений обратитесь к разделу " [обход ведения журнала аудита](#bypass-mailbox-audit-logging) почтовых ящиков" этой статьи.

> [!NOTE]
> Если включено ведение аудита почтовых ящиков по умолчанию, свойство *auditenabled указывает* для почтового ящика не будет изменено на **true** , если для него в данный момент задано значение **false**. Другими словами, аудит почтовых ящиков по умолчанию игнорирует свойство *auditenabled указывает* для почтового ящика.

## <a name="mailbox-actions-logged-by-default"></a>Действия почтового ящика, регистрируемые по умолчанию

Набор действий почтового ящика записывается по умолчанию для каждого из следующих типов входа:  

   - **Owner** — владелец почтового ящика. 
   
   - **Delegate** — другой пользователь, которому назначено разрешение SendAs, SendOnBehalf или FullAccess для почтового ящика пользователя. Обратите внимание, что администратор, которому назначено разрешение FullAccess для почтового ящика пользователя, также считается делегированным пользователем.
   
    - **Административные** почтовые ящики считаются доступными только в следующих сценариях.
    
       - При поиске почтового ящика с помощью одного из следующих средств обнаружения электронных данных Майкрософт: 

         - Поиск контента в центре соответствия требованиям.
       
         -  Обнаружение электронных данных или Расширенное обнаружение электронных данных в центре соответствия требованиям.
       
         - Обнаружение электронных данных на месте в Exchange Online.
       - Если для доступа к почтовому ящику используется [Редактор MAPI Microsoft Exchange Server](https://go.microsoft.com/fwlink/p/?linkId=204086) .

### <a name="verify-that-default-mailbox-actions-are-being-logged-for-each-logon-type"></a>Проверка того, что для каждого типа входа в систему регистрируются действия почтового ящика по умолчанию

После снятия аудита почтовых ящиков по умолчанию было добавлено новое свойство почтового ящика с именем *дефаултаудитсет* . Данное свойство указывает, выполняется ли аудит действий почтовых ящиков по умолчанию (управляемых корпорацией Майкрософт) для каждого типа входа в систему для указанного почтового ящика. Для отображения значений этого свойства можно выполнить следующую команду:

```
Get-Mailbox <username> | FL DefaultAuditSet
```

Значение `Admin, Delegate, Owner` указывает на то, что аудит действий почтовых ящиков по умолчанию для всех трех типов входа в систему и администратор в организации *не* изменил действия для любого типа входа. Примечание Это состояние по умолчанию после аудита почтовых ящиков по умолчанию включено в Организации. 

Если администратор в Организации изменил действия почтового ящика, которые подлежат аудиту для типа входа (с помощью командлета **Set-Mailbox** с параметрами *аудитадмин*, *аудитделегате*или *аудитовнер* ), то значение для параметра Свойство *дефаултаудитсет* будет отличаться от значения по умолчанию. Например, значение `Owner` указывает, что подлежат аудиту только действия почтового ящика по умолчанию для владельца почтового ящика, и что `Delegate` действия `Admin` для и были изменены. Если для свойства *дефаултаудитсет* не отображаются значения (также называемые *нулевым* значением), то действия почтового ящика для всех трех типов входа были изменены.

Для получения дополнительных сведений об изменении действий почтовых ящиков, которые подлежат аудиту, ознакомьтесь с разделом [изменение или восстановление действий почтового ящика](#change-or-restore-mailbox-actions-logged-by-default) , регистрируемого по умолчанию.

### <a name="display-a-list-of-mailbox-actions-logged-by-default"></a>Отображение списка действий почтового ящика, регистрируемого по умолчанию

В Exchange Online PowerShell можно выполнить следующие команды для отображения списка действий почтовых ящиков, которые в настоящее время находятся для почтового ящика для каждого типа входа:

**Действия владельца**

```
Get-Mailbox <username> | Select-Object -ExpandProperty AuditOwner
```

**Делегирование действий**

```
Get-Mailbox <username> | Select-Object -ExpandProperty AuditDelegate
```

**Действия администратора**

```
Get-Mailbox <username> | Select-Object -ExpandProperty AuditAdmin
```

## <a name="change-or-restore-mailbox-actions-logged-by-default"></a>Изменение или восстановление действий почтового ящика, регистрируемого по умолчанию

Как было сказано выше, одним из основных преимуществ аудита почтовых ящиков по умолчанию является то, что вам не нужно управлять действиями почтовых ящиков, для которых выполняется аудит. Корпорация Майкрософт делает это автоматически и автоматически добавляет новые действия в почтовые ящики, которые будут подвергаться аудиту по умолчанию при их выпуске. Тем не менее, в организации могут приникать причины для аудита набора действий почтовых ящиков, которые отличаются от указанных по умолчанию. В этом разделе показано, как изменить действия почтового ящика, проверяемые для каждого типа входа, и вернуться к действиям по умолчанию, управляемым корпорацией Майкрософт.

> [!IMPORTANT]
> Если вы вносите изменения в действия почтового ящика для пользователя, который вошел в систему по умолчанию (как описано в следующем разделе), все новые действия с почтовыми ящиками, выпущенные корпорацией Майкрософт, не будут подвергаться аудиту для этих почтовых ящик Необходимо явным образом добавить новое действие почтового ящика в список действий, для которых выполняется аудит для типа входа.

### <a name="change-the-mailbox-actions-to-audit"></a>Изменение действий в почтовом ящике для аудита

С помощью командлета **Set – Mailbox** с параметрами *аудитадмин*, *аудитделегате*или *аудитовнер* можно изменить действия почтовых ящиков, для которых выполняется аудит, в зависимости от типа входа. Так как параметры, связанные с аудитом, являются многозначными параметрами (то есть они могут иметь более одного значения), их можно изменить двумя разными способами.

- Вы можете указать несколько действий почтового ящика, которые перезапишут существующие действия с помощью `action1,action2,...actionN`следующего синтаксиса:.

- Вы можете добавить или удалить одно или несколько действий с почтовым ящиком, не затрагивая существующие записи, используя следующий `@{Add="action1","value2"}` синтаксис `@{Remove="action1","action2"}`: или.

Независимо от того, какой метод используется для изменения действий почтовых ящиков, действия по умолчанию, которые проверяются по умолчанию (для типа изменения при входе), больше не будут управляться корпорацией Майкрософт. Кроме того, значение типа входа, которое вы изменили, не будет отображаться в [ранее описанном](#verify-that-default-mailbox-actions-are-being-logged-for-each-logon-type)параметре почтового ящика *дефаултаудитсет* .

Ниже приведено несколько примеров использования одного из этих методов для изменения действий в почтовых ящиках для каждого из типов входа в систему. 

В этом примере показано изменение действий администратора почтового ящика путем замены действий по умолчанию на SoftDelete и HardDelete. 

```
Set-Mailbox <username> -AuditAdmin HardDelete,SoftDelete
```

В этом примере добавляется действие Owner MailboxLogin. 

```
Set-Mailbox <username> -AuditOwner @{Add="MailboxLogin"}
```

В этом примере удаляется действие делегата MoveToDeletedItems.

```
Set-Mailbox <username> -AuditDelegate @{Remove="MoveToDeletedItems"}
```

#### <a name="checking-the-defaultauditset-property"></a>Проверка свойства Дефаултаудитсет

После того как вы измените действия почтового ящика по умолчанию для типа входа, свойство *дефаултаудитсет* в почтовом ящике будет автоматически Обновлено в соответствии с этим изменением. Например, при `Get-Mailbox <username> | FL DefaultAuditSet` первом добавлении или удалении действия владельца почтового ящика команда возвращает только значение `Admin, Delegate`. Это означает, что действия почтового ящика по умолчанию для владельца изменены это также означает, что все новые действия владельца почтового ящика, выпущенные корпорацией Майкрософт, не будут автоматически добавляться в этот почтовый ящик. То же самое справедливо для изменения действий почтового ящика для типа учетной записи администратора или делегата.

### <a name="restore-the-default-mailbox-actions"></a>Восстановление действий почтового ящика по умолчанию

При внесении изменений в действия почтового ящика, которые подлежат аудиту для типа входа, можно восстановить действия по умолчанию, которые проверяются `Set-Mailbox -DefaultAuditSet` при выполнении команды. После этого произойдет следующее:

- Текущий список действий почтового ящика будет заменен действиями почтового ящика по умолчанию для соответствующего типа входа.
 
- Все новые действия с почтовыми ящиками, выпускаемые корпорацией Майкрософт, автоматически добавляются в список для соответствующего типа входа.

- [Свойство почтового ящика дефаултаудитсет](#checking-the-defaultauditset-property) будет Обновлено, чтобы включить соответствующий тип входа.

Чтобы восстановить действия почтового ящика по умолчанию для всех типов входа в систему, выполните следующую команду:

```
Set-Mailbox <username> -DefaultAuditSet Admin,Delegate,Owner
```

Эту команду можно использовать для восстановления действий почтового ящика по умолчанию для любого типа входа (с использованием значений **Admin**, **Delegate**или **owner** для параметра *дефаултаудитсет* ).

## <a name="turn-off-mailbox-auditing-on-by-default-for-your-organization"></a>По умолчанию отключить аудит почтовых ящиков в Организации

Если по каким-либо причинам организация решила, что не нужно выполнять аудит действий почтового ящика, вы можете отключить аудит почтовых ящиков по умолчанию для всей Организации, выполнив следующую команду в Exchange Online PowerShell:

```
Set-OrganizationConfig -AuditDisabled $true
```

Если аудит почтовых ящиков включен по умолчанию (для свойства *аудитдисаблед* задано **значение true**) для Организации, выполняются следующие действия:

- Аудит поЧтовых ящиков отключен для вашей организации.

- Никакие действия с почтовыми ящиками не будут проверяться (начиная с момента отключения аудита для организации), даже если для свойства *auditenabled указывает* почтового ящика задано **значение true**.

- Аудит поЧтовых ящиков не будет включен для новых почтовых ящиков, и установка свойства *auditenabled указывает* для нового (или существующего) почтового ящика в **значение true** будет игнорироваться.

- Все параметры связи обхода аудита почтовых ящиков (настроенные с помощью командлета **Set-MailboxAuditBypassAssociation** ) будут игнорироваться.

- Существующие записи аудита почтовых ящиков будут храниться до тех пор, пока не истечет срок хранения журнала аудита для записи.

### <a name="turn-on-mailbox-auditing-on-by-default"></a>Включение аудита почтовых ящиков по умолчанию

Чтобы включить аудит почтовых ящиков в Организации, просто выполните следующую команду в Exchange Online PowerShell:

```
Set-OrganizationConfig -AuditDisabled $false
```

## <a name="bypass-mailbox-audit-logging"></a>Обход журнала аудита почтовых ящиков

В настоящее время невозможно отключить аудит почтовых ящиков для определенных почтовых ящиков, если в Организации включен аудит почтовых ящиков по умолчанию. Например, установка свойства почтового ящика *auditenabled указывает* в **значение false** игнорируется.  Тем не менее, вы по-прежнему можете использовать командлет **Set-MailboxAuditBypassAssociation** в Exchange Online PowerShell, чтобы предотвратить действия почтовых ящиков, выполняемые определенными пользователями, в журнал. Если вы обходите аудит почтовых ящиков, действия почтовых ящиков, выполняемые определенным пользователем, не подлежат аудиту независимо от того, в каком почтовом ящике выполняются эти действия. Если вы обходите аудит почтовых ящиков для определенного пользователя, действия владельца почтового ящика, выполняемые этим пользователем, не будут заноситься в журнал. Если ему назначены разрешения для почтового ящика другого пользователя (или общего почтового ящика), то действия делегирования, выполняемые первым пользователем, также не будут регистрироваться.

Чтобы обойти журнал аудита почтовых ящиков для определенного пользователя, выполните следующую команду:

```
Set-MailboxAuditBypassAssociation -Identity <username> -AuditByPassEnabled $true
```

Чтобы убедиться, что аудит обходится для указанного пользователя, выполните следующую команду:

```
Get-MailboxAuditBypassAssociation -Identity <username> | FL AuditByPassEnabled
```

Значение **true** указывает, что ведение журнала аудита почтовых ящиков для этого пользователя обходится.

## <a name="more-information"></a>Дополнительные сведения

- По умолчанию записи журнала аудита почтовых ящиков сохраняются в течение 90 дней, а затем удаляются. Вы можете изменить срок хранения записей журнала аудита с помощью команды **Set-Mailbox-аудитлогажелимит** в Exchange Online PowerShell. Обратите внимание на то, что увеличение срока хранения по умолчанию для записей аудита почтовых ящиков не влияет на срок хранения в 90 дней для записей журнала аудита почтовых ящиков в журнале аудита Microsoft 365. Например, если вы увеличите ограничение возраста для записей журнала аудита почтовых ящиков до 365 дней, запись аудита почтового ящика будет доступна для поиска в журнале аудита Microsoft 365 только в течение 90 дней после возникновения соответствующего события. В этом случае вам придется искать записи, возраст которых превышает 90 дней, в журнале аудита почтового ящика пользователя. Дополнительные сведения о поиске в журналах аудита почтовых ящиков приведены в статье [Search — маилбоксаудитлог](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-audit/search-mailboxauditlog).

- Если вы изменили свойство *аудитлогажелимит* для почтового ящика, прежде чем аудит почтовых ящиков включен по умолчанию для Организации, срок хранения журнала аудита для этого почтового ящика не изменится; другими словами, аудит почтовых ящиков по умолчанию не оказывает силы текущего срока хранения записей аудита почтовых ящиков.

- Записи журнала аудита поЧтовых ящиков хранятся во вложенной папке (с именем *audits*) в папке "элементы с возможностью восстановления" в почтовом ящике каждого пользователя. Следует учитывать следующие моменты, касающиеся записей аудита почтовых ящиков и папки "элементы с возможностью восстановления".
   
    - Количество записей аудита поЧтовых ящиков в соответствии с квотой хранилища папки "элементы с возможностью восстановления", которая по умолчанию 30GB (квота предупреждения составляет 20 ГБ). Обратите внимание на то, что квота хранилища для папки "элементы с возможностью восстановления" автоматически увеличивается с 100 ГБ (квота предупреждений 90 МБ), когда удержание помещается в почтовый ящик, или если почтовый ящик назначен политике хранения в центре соответствия требованиям.

    - Кроме того, записи аудита почтового ящика подсчитывается на [Максимальное число папок для папки "элементы с возможностью восстановления"](https://docs.microsoft.com/office365/servicedescriptions/exchange-online-service-description/exchange-online-limits#mailbox-folder-limits). Максимальное число элементов (которые являются записями аудита в данном случае), которые могут храниться в подпапке audits, составляет 3 000 000 элементов. 

      > [!NOTE]
      > Маловероятно, что аудит почтовых ящиков по умолчанию влияет на квоту хранилища или максимальное количество папок для папки "элементы с возможностью восстановления". 

    - В Exchange Online PowerShell можно выполнить следующую команду для отображения размера и количества элементов в подпапке "audits" в папке "элементы с возможностью восстановления": 
       ```
       Get-MailboxFolderStatistics <username> -FolderScope RecoverableItems | Where-Object {$_.Name -eq 'Audits'} | FL FolderPath,FolderSize,ItemsInFolder
       ```

     - Вы не можете напрямую получить доступ к записи журнала аудита в папке "элементы с возможностью восстановления"; Вместо этого используйте командлет **Search — маилбоксаудитлог** или найдите в журнале аудита Microsoft 365 Поиск и просмотр записей аудита почтовых ящиков.

- Если почтовый ящик размещается на удержании или назначается политике хранения в центре соответствия требованиям, записи журнала аудита по-прежнему хранятся в течение срока, определенного свойством *аудитлогажелимит* для почтового ящика (по умолчанию задано значение 90 дней). Чтобы сохранять записи журнала аудита дольше для почтовых ящиков на удержании, необходимо увеличить срок хранения, увеличив значение свойства *аудитлогажелимит* .
