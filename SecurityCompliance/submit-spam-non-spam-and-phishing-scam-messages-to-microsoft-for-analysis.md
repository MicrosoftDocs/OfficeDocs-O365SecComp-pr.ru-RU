# <a name="submit-spam-non-spam-and-phishing-scam-messages-to-microsoft-for-analysis"></a>Отправка обычных, нежелательных и фишинговых сообщений в корпорацию Майкрософт для анализа

Она может быть неудобства при пользователей в вашей организации Получение нежелательных сообщений и фишинга сообщение в папки входящих сообщений, или если они не получают сообщение надежные сообщения электронной почты, поскольку она помечена как нежелательная почта. Мы постоянно точно настроить наших фильтров нежелательной почты более. Пользователи могут помочь этот процесс, отправка ложных отрицательных и значение false, положительное нежелательных сообщений в корпорацию Майкрософт для анализа. «False» минус является нежелательное сообщение, которое должно было, но не был распознан как нежелательная почта. «Ложное срабатывание» — это сообщение надежные сообщения электронной почты, который был неправильно помечаются как нежелательная почта. 
  
> [!NOTE]
> Из-за большой поток отправки, которые мы получили мы могут не знать ответа все запросы для анализа. 
  
## <a name="submit-junk-or-phishing-messages-that-passed-through-the-spam-filters"></a>Отправка нежелательных или фишинговых сообщений, прошедших через фильтры нежелательной почты
<a name="sectionSection0"> </a>

При получении сообщения, который передается через нежелательной почты фильтр, который и следует рассматривать как нежелательные или мошенничество, вы можете отправить сообщение «ложных отрицательных» для анализа Microsoft нежелательной почты и фишинга анализа Microsoft командами, соответствующим образом. Аналитиков будет просмотрите сообщение и добавление его в фильтры всей службы, если они соответствуют условиям, классификации. 
  
Дополнительные параметры нежелательной почты, применяемые ко всей организации в разделе [Блокировать нежелательных сообщений электронной почты с фильтром нежелательной почты Office 365 для предотвращения ложных отрицательных проблемы](https://go.microsoft.com/fwlink/p/?LinkId=534225). В этой статье содержатся советы для предотвращения ложные отрицательные результаты.
  
Вы можете отправить нежелательные сообщения для анализа следующими способами:
  
- Для Outlook и Outlook на веб-пользователей используйте сообщения отчета в программе Microsoft Outlook. Сведения о том, как устанавливать и использовать это средство содержатся в разделе [Включение надстройки сообщения отчета](https://support.office.com/article/4250c4bc-6102-420b-9e0a-a95064837676). 
        
- Можно также использовать электронной почты для отправки сообщений в корпорацию Майкрософт, который следует рассматривать как мошенничество в список нежелательной почты и фишинга, как описано в следующей процедуре.
    
### <a name="use-email-to-submit-junk-spam-or-phishing-scam-messages-to-microsoft"></a>Использование электронной почты для отправки в Майкрософт нежелательных или фишинговых сообщений для анализа
<a name="Useemailtosubmitjunkspamorphishingscammessages"> </a>

Чтобы отправить в Майкрософт нежелательное или фишинговое сообщение для анализа:
  
1. Создайте пустое сообщение.
    
2. Адрес получателя для группы разработчиков Microsoft, который просматривает сообщения, как показано ниже: 
    
  - Для нежелательных сообщений: junk@office365.microsoft.com
    
  - Для сообщений, сообщение фишинга: phish@office365.microsoft.com
    
3. Скопируйте и вставьте сообщение мошенничество список нежелательной почты и фишинга в новое сообщение как вложение. 
    
    > [!NOTE]
    > Вы можете прикрепить несколько сообщений в новое сообщение. Убедитесь в том, что все сообщения должны того же типа — сообщение фишинга или нежелательных сообщений электронной почты. > Оставьте текст нового сообщения пустым. 
  
4. Щелкните **Отправить**.
    
## <a name="submit-messages-that-were-tagged-as-junk-but-should-have-been-allowed-through"></a>Отправка сообщений для анализа, которые помечены как нежелательная почта, но должны были быть пропущены
<a name="sectionSection1"> </a>

Если сообщение неправильно был распознан как нежелательная почта, можно отправить сообщение «Ложное срабатывание», чтобы группы разработчиков анализа нежелательной почты Microsoft. Аналитиков будет оценить и проанализировать сообщения. В зависимости от результаты анализа правил фильтрации содержимого нежелательной почты всей службы могут быть настроены для отправки сообщения через.
  
Администратор может просмотреть дополнительные сведения о настройке нежелательной почты, применяется ко всей организации. Узнайте, [как гарантировать, что сообщение не будет помечено как нежелательное](https://go.microsoft.com/fwlink/p/?LinkId=534224). Эта информация будет полезно при наличии управление доступом на уровне администратора, и вы хотите запретить ложные срабатывания.
  
Вы можете отправить сообщения, не являющиеся нежелательными, следующими способами.
  
- Если вы используете действия **переместить сообщение в папку нежелательной почты** , при настройке фильтров содержимого (это действие по умолчанию), пользователи могут освобождать ложных положительных сообщений в папке Outlook или OWA Нежелательная почта. 
    
  - Outlook пользователи могут освобождать ложных положительных сообщений с помощью команды меню щелкните правой кнопкой мыши **Не является нежелательным** . Тем не менее они должны отправлять сообщения в корпорацию Майкрософт по электронной почте, как показано в процедуру, описанную в этой статье. 
    
  - Пользователи Outlook Web Access можно освободить ложных положительных сообщений и отправлять их в корпорацию Майкрософт для анализа с помощью команды **Пометить как не являющееся нежелательным** . Дополнительные сведения см в [отчет о нежелательной почте и фишинге в Outlook в Интернете ](report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md).
    
- Если вы используете действие **сообщения карантин** вместо действия **переместить сообщение в папку нежелательной почты** , при настройке фильтров содержимого: 
    
  - Администраторы могут освобождать сообщения, помещенные на карантин и о них как о ложных срабатываний в центре администрирования Exchange. Для получения дополнительных сведений см. [Поиск и освобождение сообщений на карантине с правами администратора](find-and-release-quarantined-messages-as-an-administrator.md).
    
  - Пользователи могут освобождать сообщения, помещенные на карантин нежелательной почты и о них как о ложных срабатываний по каналам следующие: 
    
  - Пользовательский интерфейс центра администрирования Exchange (EAC). Дополнительные сведения см. в статье [Find and Release Quarantined Messages (End Users)](http://technet.microsoft.com/library/e439b560-827a-4807-abd3-6b861c1ff786.aspx).
    
  - Конечный пользователь нежелательной почты сообщений уведомлений (если они в случае включен администратором). 
    
- Можно также использовать электронной почты для отправки сообщений в корпорацию Майкрософт, не следует рассматривать как нежелательная почта. При этом убедитесь в том, выполните действия из следующей процедуры.
    
### <a name="use-email-to-submit-false-positive-messages"></a>Использование электронной почты для отправки сообщений с ложным положительным результатом

Выполните действия, описанные в «[Использование электронной почты для отправки нежелательной или фишинга мошенничество в корпорацию Майкрософт ](submit-spam-non-spam-and-phishing-scam-messages-to-microsoft-for-analysis.md#Useemailtosubmitjunkspamorphishingscammessages)», чтобы отправить сообщение в not_junk@office365.microsoft.com.
  
## <a name="spam-evaluation-and-rules-deployment"></a>Оценки нежелательной почты и развертывание правил
<a name="sectionSection2"> </a>

Группа анализа нежелательной почты проверяет сообщения, отправка и настраивает фильтр нежелательной почты для предотвращения будущих нежелательной почты. В результате нежелательной почты Office 365 фильтр areconstantly уточнение. Любой добавленных элементов вычисляются на уровне сети. Ложных положительных отправок рассматриваются и оценки для возможных правила Коррекция разрешить последующих сообщений через фильтр нежелательной почты. Таким образом уведомление службы ложные срабатывания, а также ложные отрицательные результаты (без фильтра нежелательной почты) заключаются в следующем для вас и все пользователи, использующие глобальной сети. Группа нежелательной почты проверяет индикаторов в пределах каждого отправленное сообщение, включая следующие:
  
- С адреса
    
- Отправляющий IP-адрес
    
- Ключевые слова
    
- Фразы
    
- Частота передачи
    
- Другие тенденции и шаблоны
    
После их просмотрите эти сведения, группа нежелательной почты могут вносить изменения в службы фильтрации слои нежелательной почты. Дополнительные сведения о группе нежелательной почты вы можете в следующем видео только на английском языке:
  
[Microsoft Exchange группой разработчиков видео](https://youtu.be/-TpX_-GMC7o?hd=1)
  
Оценка нежелательной почты — это непрерывный процесс, который применяется независимо от исходного языка или кодировки. Из-за возможной неопределенности или даже нехватки текста в теме или самом сообщении группа анализа полагается при фильтрации на другие характеристики сообщений. Это означает, что после того как группа помечает данное сообщение как нежелательное и вносит необходимые изменения в свою базу правил, сообщение будет заблокировано в будущем, пока его характеристики не будут изменены для обхода наших фильтров. Новые правила для нежелательной почты разворачиваются непрерывно. Временные рамки для правил в отдельных отправлениях варьируются в зависимости от количества и качества отправлений. Так как новые правила для нежелательной почты устанавливаются глобально для всех клиентов, не все индивидуальные отправления нежелательной почты для анализа приводят к созданию нового правила для нежелательной почты.
