# <a name="content-search-in-office-365"></a>Поиск контента в Office 365

Средство обнаружения электронных данных поиска контента в центре безопасности &amp; и соответствия требованиям Office 365 используется для поиска элементов на месте, таких как электронная почта, документы и беседы для обмена мгновенными сообщениями в организации Office 365. Используйте это средство для поиска элементов в этих службах Office 365:
  
- Почтовые ящики Exchange Online и общие папки
    
- Сайты SharePoint Online и учетные записи OneDrive для бизнеса
    
- Беседы Skype для бизнеса
    
- Microsoft Teams 
    
- Группы Office 365
    
После выполнения поиска контента в профиле поиска отображается количество расположений контента и предполагаемое количество результатов поиска. Кроме того, можно быстро просмотреть статистику, например расположение содержимого, в котором есть большинство элементов, которые отвечают поисковому запросу. После выполнения поиска можно выполнить предварительный просмотр результатов или экспортировать их на локальный компьютер.


## <a name="create-a-new-search"></a>Создание нового поиска

Чтобы получить доступ к странице " **Поиск контента** " для выполнения поиска и предварительного просмотра и экспорта результатов поиска, администратор, руководитель соответствия требованиям или диспетчер обнаружения электронных данных должны быть членом группы ролей "Диспетчер обнаружения электронных &amp; данных" в центре безопасности и соответствия требованиям. Дополнительные сведения см в разделе [Назначение разрешений обнаружения электронных данных в центре безопасности &amp; Office 365](assign-ediscovery-permissions.md).
  
1. Перейдите по ссылке [https://protection.office.com](https://protection.office.com).
    
2. Войдите в систему, используя свой адрес электронной почты и пароль Office 365. 
    
3. В центре безопасности &amp; и соответствия требованиям выберите Поиск \> **контента**для ** &amp; расследования** .
    
4. На странице " **Поиск** " щелкните стрелку рядом с ![кнопкой Добавить значок](media/8ee52980-254b-440b-99a2-18d068de62d3.gif) " **создать поиск**". 
    
    ![Раскрывающийся список "создать Поиск"](media/76b25861-55c5-4f50-9d48-9e2be2d0d078.png)
  
    На вкладке Внутренние DNS-запросы можно указать, какие DNS-серверы использовать для разрешения адресов почтовых серверов для внутренней доставки почты — DNS-серверы, настроенные на сетевом адаптере, установленном на этом сервере, или конкретные DNS-серверы. Внутренние DNS-серверы используются для разрешения IP-адресов для серверов внутри организации.
    
  - " **Интерактивный поиск** " — Этот параметр запускает мастер, который поможет вам создать поиск. Пользовательский интерфейс для выбора расположений контента и создания поискового запроса совпадает с новым параметром **поиска** . 
    
  - **Новый поиск** — этот параметр отображает обновленный пользовательский интерфейс для создания нового поиска. Этот вариант используется по умолчанию при нажатии кнопки **создать поиск**.
    
  - **Поиск по идентификатору списка** — этот параметр позволяет выполнять поиск определенных сообщений электронной почты и других элементов почтовых ящиков с помощью списка идентификаторов Exchange. Чтобы создать запрос на поиск списка ИДЕНТИФИКАТОРов (формально называется целевой поиском), необходимо предоставить файл данных с разделителями-запятыми (CSV), который определяет конкретные элементы почтового ящика для поиска. Инструкции можно найти [в разделе Подготовка CSV-файла для поиска контента списка идентификаторов в Office 365](csv-file-for-an-id-list-content-search.md).
    
    Оставшаяся часть действий, описанных в этой процедуре, будет выполняться после нового рабочего процесса поиска по умолчанию.
    
5. В раскрывающемся списке выберите пункт **Новый поиск** . 
    
6. В разделе **поисковый запрос**укажите следующие моменты.
    
    ![Указание ключевых слов, условий и расположений для поиска](media/1e6de9dd-eac9-4e2a-819d-9740cf6c9106.png)
  
- **Ключевые слова для поиска** — введите поисковый запрос в **Ключевые слова** . Можно указать ключевые слова или свойства сообщений, например даты отправки и получения, или свойства документов, например имена файлов или дату последнего изменения. Можно использовать более сложные запросы, которые используют логический оператор, например **and**, **or**, **Not**и **NEAR**. Кроме того, можно искать конфиденциальные сведения (например, номера социального страхования) в документах или искать документы, к которым предоставлен доступ извне. Если оставить поле ключевое слово пустым, в результаты поиска будут включены все содержимое, расположенное в указанных расположениях контента.
    
    Кроме того, можно щелкнуть флажок **Показать список ключевых слов** и ввести ключевое слово в каждой строке. В этом случае ключевые слова в каждой строке соединяются логическим оператором ( **к:с**), который аналогичен оператору **or** в созданном запросе поиска. 
    
    Зачем использовать список ключевых слов? Вы можете получить статистику по количеству элементов, которые совпадают с каждым ключевым словом. Это поможет быстро определить, какие ключевые слова являются самыми эффективными (и, как минимум) эффективными. Кроме того, в строке можно использовать ключевую фразу (заключенную в круглые скобки). Дополнительные сведения о статистике поиска можно узнать в статье [Просмотр статистики ключевых слов для результатов поиска контента](view-keyword-statistics-for-content-search.md).

    > [!NOTE]
    > Для сокращения проблем, вызванных списками с большими ключевыми словами, в списке ключевых слов можно ограничить до 20 строк.
    
- **Условия** — вы можете добавлять условия поиска для сужения поиска и возвращать более уточненный набор результатов. Каждое условие добавляет в запрос поиска предложение, созданное и выполняемое при запуске поиска. Условие логически подключается к запросу ключевого слова (указанному в поле ключевое слово) логическим оператором ( **к:к**), который аналогичен оператору **and** . Это означает, что элементы должны удовлетворять как запрос ключевого слова, так и одно или несколько условий, включаемых в результаты. Таким образом условия помогают сузить область результатов поиска. Список и описание условий, которые можно использовать в поисковых запросах, представлены в разделе "условия поиска" в разделе [запросы и условия поиска контента](keyword-queries-and-search-conditions.md#search-conditions).
    
- **Locations (расположения** ) — выберите расположения контента для поиска.
    
  - **Все расположения** — используйте этот параметр для поиска всех расположений контента в Организации. Это относится ко всем почтовым ящикам Exchange (включая все неактивные почтовые ящики, почтовые ящики для всех групп Office 365, почтовые ящики для всех Microsoft Teams), все беседы Skype для бизнеса, все сайты SharePoint и OneDrive для бизнеса (включая сайты). для всех групп Office 365 и Microsoft Teams) и элементов во всех общедоступных папках Exchange.
    
  - **Конкретные расположения** — используйте этот параметр для поиска определенных расположений контента. Можно выполнить поиск по всем расположениям для определенной службы Office 365 (например, поиск во всех почтовых ящиках Exchange или поиск всех сайтов SharePoint) или можно выполнить поиск в определенных расположениях в любой из отображаемых служб Office 365. 
    
    ![Пользовательский интерфейс для выбора расположений контента для поиска](media/9a09708b-f8a2-4382-8c4e-2c610ec33c72.png)
  
    Обратите внимание, что вы также можете добавить группы рассылки в список почтовых ящиков Exchange для поиска. Для групп рассылки выполняется поиск почтовых ящиков членов группы. Обратите внимание, что динамические группы рассылки не поддерживаются.
    
    **Важно!** При поиске всех расположений почтовых ящиков или только определенных почтовых ящиков данные из MyAnalytics и других приложений Office 365, сохраненных в почтовых ящиках пользователей, будут включены при экспорте результатов поиска контента. Эти данные не будут включены в оценочные результаты поиска, и они не будут доступны для предварительного просмотра. Оно будет включено только при экспорте и скачивании результатов поиска; сведения об [экспорте данных из MyAnalytics и других приложений Office 365](#exporting-data-from-myanalytics-and-other-office-365-applications) в разделе "Дополнительные сведения о поиске контента". 
    
7. После настройки поискового запроса нажмите кнопку **Сохранить &amp; выполнение**.
    
8. На странице **Сохранение поиска** введите имя поиска и необязательное описание, помогающее определить Поиск. Обратите внимание, что имя поиска должно быть уникальным в Организации. 
    
9. Нажмите кнопку **сохранить** , чтобы начать поиск. 
    
    После сохранения и запуска поиска все результаты поиска отображаются в области результатов. В зависимости от того, как настроен параметр Preview, отображаются результаты поиска или для их просмотра потребуется щелкнуть Предварительный **Просмотр результатов** . Более подробную информацию можно найти в следующем разделе. 
    
Чтобы снова получить доступ к этому содержимому или просмотреть другие операции поиска контента, перечисленные на странице **Поиск контента** , выберите Поиск и нажмите кнопку **Открыть**. 
  
Чтобы очистить результаты или создать новый поиск, нажмите ![добавить значок](media/O365-MDM-CreatePolicy-AddIcon.gif) **Новый поиск**. 

  
## <a name="preview-search-results"></a>Предварительный просмотр результатов поиска

Для предварительного просмотра результатов поиска можно настроить два параметра. После выполнения нового поиска или открытия существующего поиска щелкните * * отдельные результаты * *, чтобы просмотреть следующие параметры просмотра: 
  
![Предварительный просмотр параметров результатов поиска](media/83519477-1c85-4442-8886-481f186fd758.png)
  
1. **ПредварительНый Просмотр результатов** : этот параметр отображает результаты поиска после выполнения поиска.
    
2. **ПредварительНый Просмотр результатов вручную** — этот параметр отображает заполнители в области результатов поиска и отображает кнопку **предварительного просмотра** , которую нужно щелкнуть для отображения результатов поиска. Это параметр по умолчанию; Это помогает улучшить производительность поиска, не отображая результаты поиска автоматически при открытии существующего поиска. 
    
Количество элементов, доступных для предварительного просмотра, связано с пределами. Дополнительные сведения см. [в статье Limits for Search в центре безопасности &amp; и соответствия требованиям Office 365](limits-for-content-search.md). 
  
Список поддерживаемых типов файлов, которые можно просмотреть, приведен в разделе [Предварительный просмотр результатов поиска](#previewing-search-results) в разделе "Дополнительные сведения о поиске контента". Если для предварительной версии или загрузки копии документа тип файла не поддерживается, можно нажать кнопку **загрузить исходный файл** , чтобы скачать его на локальный компьютер. Для веб-страниц. aspx URL-адрес страницы включается, несмотря на то что у вас могут отсутствовать разрешения на доступ к странице. 
  
Кроме того, обратите внимание, что неиндексированные элементы недоступны для предварительного просмотра.
  
## <a name="view-information-and-statistics-about-a-search"></a>Просмотр сведений и статистики о поиске

После создания и запуска поиска контента можно просмотреть статистику по предполагаемым результатам поиска. Сюда входит сводка результатов поиска, Статистика запросов, например количество расположений контента с элементами, которые соответствуют поисковому запросу, и имена расположений контента с наиболее подходящими элементами. Можно отобразить статистику для одного или нескольких поисков контента. Это позволяет быстро сравнивать результаты для нескольких поисков и принимать решения относительно эффективности поисковых запросов.
  
Вы также можете скачать статистику поиска и статистику ключевых слов в CSV-файл. Это позволяет использовать функции фильтрации и сортировки в Excel для сравнения результатов и подготовки отчетов для результатов поиска.
  
Чтобы просмотреть статистику поиска:
  
1. На странице " **Поиск контента** " в центре &amp; безопасности и соответствия требованиям нажмите кнопку **Открыть** , а затем щелкните Поиск, для которого нужно просмотреть статистику. 
    
2. На странице вылет нажмите кнопку **открыть запрос**. 
    
3. В раскрывающемся списке **отдельные результаты** выберите **профиль поиска**.
    
4. В раскрывающемся списке **тип** выберите один из следующих вариантов в зависимости от статистики поиска, которую нужно просмотреть. 
    
  - **Сводка** — отображает статистику для каждого типа найденных расположений контента. В этом содержимом запрашивается количество расположений контента, соходящих из элементов, сопоставляемых с поисковым запросом, а также общее число и размер элементов результатов поиска. Это устанавливается по умолчанию.
    
  - **Запросы** — отображает статистику поискового запроса. Сюда входит тип расположения содержимого, к которому относится статистика запроса, часть поискового запроса, к которой применяется статистика (Обратите внимание на то, что **основной** указывает весь поисковый запрос), количество расположений контента, содержащих элементы, которые Сопоставьте поисковый запрос, а также общее количество и размер и количество найденных элементов (в указанном расположении контента), которые совпадают с поисковым запросом. Обратите внимание, что также отображаются статистические данные для неиндексированных элементов (также называемых частично индексированными элементами). Однако в статистику включаются только частично индексированные элементы почтовых ящиков. Частично индексированные элементы из SharePoint и OneDrive не включены в статистику.
    
  - **Основные расположения** — отображает статистику по количеству элементов, которые совпадают с поисковым запросом в каждом расположении контента, в котором выполнялся поиск. Отобразятся первые 1 000 расположений.
    
Более подробную информацию о статистике поиска можно узнать в статье [Просмотр статистики ключевых слов для результатов поиска контента](view-keyword-statistics-for-content-search.md).
  
  
## <a name="export-search-results"></a>Экспорт результатов поиска

После успешного выполнения поиска вы можете экспортировать результаты поиска на локальный компьютер. При экспорте результатов электронной почты они могут быть загружены на компьютер в виде PST-файлов или отдельных сообщений (MSG-файлы). При экспорте контента из сайтов SharePoint и OneDrive копируются копии собственных документов Office. Кроме того, в экспортированные результаты поиска включаются дополнительные документы и отчеты. Кроме того, вы можете просто экспортировать отчет по результатам поиска, а не фактические элементы.
  
Чтобы экспортировать результаты поиска, выполните указанные ниже действия.
  
1. На странице " **Поиск контента** " в центре &amp; безопасности и соответствия требованиям выберите Поиск, для которого требуется экспортировать результаты поиска. 
    
2. На странице вылет нажмите кнопку ![экспорт результатов](media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) **** поиска, а затем нажмите кнопку **экспортировать результаты**. Обратите внимание, что вы также можете экспортировать отчет о результатах поиска.
    
3. ЗаПолните разделы на странице "вылетает **Результаты экспорта** ". Обязательно используйте полосу прокрутки, чтобы просмотреть все параметры экспорта. 
    
Подробные инструкции и советы по устранению неполадок приведены в следующих статьях:

### <a name="searching-microsoft-teams-and-office-365-groups"></a>Поиск в Microsoft Teams и Office 365 Groups

Вы можете выполнить поиск в почтовом ящике, связанном с группой Office 365 или группой Майкрософт. Так как Microsoft Teams построено на основе групп Office 365, их поиск очень похож. В обоих случаях выполняется поиск только в почтовом ящике группы или группы; почтовые ящики группы или членов группы не ищутся. Чтобы выполнить поиск, необходимо добавить их в поиск.
  
При поиске контента в Microsoft Teams и группах Office 365 учитывайте следующие моменты.
  
- Для поиска контента, расположенного в группах Microsoft Teams и Office 365, необходимо указать почтовый ящик и сайт SharePoint, связанные с группой или группой.
    
- Выполните командлет **Get-UnifiedGroup** в Exchange Online, чтобы просмотреть свойства группы Майкрософт или группы Office 365. Это хороший способ получения URL-адреса для сайта, связанного с группой или группой. Например, следующая команда отображает выбранные свойства для группы Office 365 с именем "старший лидер". 
    
  ```
  Get-UnifiedGroup "Senior Leadership Team" | FL DisplayName,Alias,PrimarySmtpAddress,SharePointSiteUrl
  DisplayName            : Senior Leadership Team
  Alias                  : seniorleadershipteam
  PrimarySmtpAddress     : seniorleadershipteam@contoso.onmicrosoft.com
  SharePointSiteUrl      : https://contoso.sharepoint.com/sites/seniorleadershipteam
  
  ```

    > [!NOTE]
    > Для запуска командлета **Get-UnifiedGroup** необходимо назначить роль получателей "только просмотр" в Exchange Online или быть членом группы ролей, которой назначена роль получателей "только просмотр". 
  
- При поиске в почтовом ящике пользователя все группы Microsoft Office или Office 365, участником которых является пользователь, не будут выполнять поиск. Аналогично, при поиске в Microsoft Team или группе Office 365 Поиск выполняется только для группового почтового ящика и сайта группы, которые вы указали. почтовые ящики и учетные записи OneDrive для бизнеса участников группы не ищутся, пока вы не добавите их в поиск.
    
- Чтобы получить список членов группы Майкрософт или группы Office 365, можно просмотреть свойства на странице " **домашние \> группы** " в центре администрирования Office 365. Кроме того, можно выполнить следующую команду в Exchange Online PowerShell: 
    
  ```
  Get-UnifiedGroupLinks <group or team name> -LinkType Members | FL DisplayName,PrimarySmtpAddress 
  ```

    > [!NOTE]
    > Для запуска командлета **Get-унифиедграуплинкс** необходимо назначить роль получателей "только просмотр" в Exchange Online или быть членом группы ролей, которой назначена роль получателей "только просмотр". 
  
- Беседы, являющиеся частью канала Microsoft Teams, хранятся в почтовом ящике, связанном с командой Microsoft Teams. Аналогично, файлы, которые совместно используются участниками группы в канале, хранятся на сайте SharePoint группы. Поэтому необходимо добавить почтовый ящик группы Майкрософт и сайт SharePoint в качестве расположения контента для поиска бесед и файлов в канале.
    
- Кроме того, беседы, являющиеся частью списка чата в Microsoft Teams, хранятся в почтовом ящике Exchange Online для пользователей, участвующих в чате. И файлы, которые пользователь предоставляет в беседах беседы, хранятся в учетной записи OneDrive для бизнеса пользователя, имеющего общий доступ к файлу. Поэтому необходимо добавить индивидуальные почтовые ящики пользователей и учетные записи OneDrive для бизнеса в качестве расположений контента для поиска бесед и файлов в списке чата.
    
    > [!NOTE]
    > В гибридном развертывании Exchange пользователи с локальным почтовым ящиком могут участвовать в беседах, входящих в список чата в Microsoft Teams. В этом случае контент из этих бесед также может быть доступен для поиска, так как он сохраняется в облачной области хранилища (которая называется *облачным почтовым ящиком для локальных пользователей*) для пользователей, имеющих локальный почтовый ящик. Дополнительные сведения см. [в статье Поиск в облачных почтовых ящиках для локальных пользователей в Office 365](search-cloud-based-mailboxes-for-on-premises-users.md).
  
- Каждая команда Майкрософт или канал группы содержит вики-сайт для заметок и совместной работы. Содержимое вики-сайта автоматически сохраняется в файл в формате MHT. Этот файл хранится в библиотеке документов "вики-данные Teams" на сайте SharePoint. Вы можете использовать средство поиска контента для поиска на вики-сайте, указав сайт SharePoint группы в качестве расположения контента для поиска. 
    
    > [!NOTE]
    > Возможность поиска на вики-сайте для группы или канала Майкрософт (при поиске на сайте SharePoint группы) выпущена 22 июня 2017 г. Вики-страницы, которые были сохранены или обновлены на указанную дату или позже, будут доступны для поиска. Вики-страницы были сохранены или обновлены до того, как эта дата недоступна для поиска. 
 
- Сводные данные о собраниях и вызовах в канале Microsoft Teams также хранятся в почтовых ящиках пользователей, набираемых в собрании или вызове. Это означает, что вы можете использовать поиск контента для поиска в этих сводных записях. Сводная информация включает в себя следующее: 
  - Дата, время начала, время окончания и длительность собрания или звонка

  - Дата и время, когда каждый участник присоединился или оставил собрание или звонок.

  - Звонки, отправленные в голосовую почту

  - Пропущенные или неотвеченные звонки

  - Передачи звонков, которые представлены двумя отдельными звонками

  Обратите внимание, что для поиска собраний и записей сводки вызовов может потребоваться до 8 часов.

  В результатах поиска сводка собраний идентифицируется как **встреча** в **поле Тип**; Сводка вызовов идентифицируется как **Call**. Кроме того, беседы, которые являются частью канала Teams и сеансов 1xN, определяются как **IM** в поле **тип** .

### <a name="searching-inactive-mailboxes"></a>Поиск неактивных почтовых ящиков

Вы можете искать Неактивные почтовые ящики в поиске контента. Чтобы получить список неактивных почтовых ящиков в Организации, выполните команду `Get-Mailbox -InactiveMailboxOnly` в Exchange Online PowerShell. Кроме того, вы можете перейти **** \> к разделу **Хранение** данных в &amp; центре безопасности и выбрать пункт **Дополнительные**![линии](media/9723029d-e5cd-4740-b5b1-2806e4f28208.gif) \> навигации Неактивные почтовые **ящики**.
  
Ниже приведены некоторые моменты, которые необходимо учитывать при поиске неактивных почтовых ящиков.
  
- Если поиск контента включает почтовый ящик пользователя, а затем этот почтовый ящик затем становится неактивным, то при повторном выполнении поиска поиск контента продолжит выполнять поиск в неактивном почтовом ящике после того, как он станет неактивным.
    
- В некоторых случаях у пользователя может быть активный почтовый ящик и неактивный почтовый ящик с одним и тем же SMTP-адресом. В этом случае поиск будет выполняться только для определенного почтового ящика, выбранного в качестве расположения для поиска контента. Другими словами, если вы добавите почтовый ящик пользователя к поиску, вы не можете считать, что будет выполняться поиск как активных, так и неактивных почтовых ящиков; Поиск будет выполняться только для почтового ящика, который вы явно добавляете в поиск.
    
- Мы настоятельно рекомендуем избегать использования активного почтового ящика и неактивного почтового ящика с одним и тем же SMTP-адресом. Если необходимо повторно использовать SMTP-адрес, который в настоящее время назначен неактивному почтовому ящику, рекомендуется восстановить неактивный почтовый ящик или восстановить содержимое неактивного почтового ящика в активном почтовом ящике (или архиве активного почтового ящика), а затем удалить неактивный почтовый ящик. Для получения дополнительных сведений см один из следующих разделов:
    
  - [Восстановление неактивного почтового ящика в Office 365](recover-an-inactive-mailbox.md)
    
  - [Восстановление неактивного почтового ящика в Office 365](restore-an-inactive-mailbox.md)

### <a name="partially-indexed-items"></a>Частично индексированные элементы

- Как описывалось ранее, частично индексированные элементы в почтовых ящиках включаются в оценочные результаты поиска; элементы с частичным индексированием из SharePoint и OneDrive не включены в результаты оценки результатов поиска. 
    
- Если частичный элемент совпадает с поисковым запросом (так как другие свойства сообщений или документов соответствуют критериям поиска), он не будет включен в оценочное количество неиндексированных элементов. Если часть элемента исключена из критериев поиска, она также не будет включена в оценочное количество частично индексированных элементов. Дополнительные сведения см в разделе " [частично индексированные элементы" в разделе Поиск контента в Office 365](partially-indexed-items-in-content-search.md).
    
### <a name="exporting-data-from-myanalytics-and-other-office-365-applications"></a>Экспорт данных из MyAnalytics и других приложений Office 365

- Данные из MyAnalytics (например, сведения о том, как пользователи тратят свое время на основе данных электронной почты и календаря в своем почтовом ящике), а данные из других приложений Office 365 сохраняются в скрытом расположении (в поддереве без IPM) в облачном почтовом ящике пользователя. После выполнения поиска контента эти данные не включаются в оценочные результаты поиска, статистику запросов и недоступны для предварительного просмотра. Однако эти данные будут экспортированы при экспорте результатов поиска.
    
- Данные MyAnalytics и данные из других приложений Office 365 экспортируются в папку с именем "другие данные Office 365". Эта папка включает подпапки для каждого пользователя.
