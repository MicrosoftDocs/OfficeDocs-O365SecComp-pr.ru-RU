# <a name="content-search-in-office-365"></a>Поиск контента в Office 365

Можно использовать средство поиска контента обнаружения электронных данных безопасности Office 365 &amp; центре соответствия требованиям для поиска элементов на месте, такие как электронной почты, документов и обмена мгновенными сообщениями бесед в организации Office 365. Это средство предназначено для поиска элементов в этих служб Office 365:
  
- Почтовые ящики Exchange Online и общих папок
    
- Сайты SharePoint Online и OneDrive для бизнеса учетных записей
    
- Скайп для бизнеса бесед
    
- Microsoft Teams 
    
- Группы Office 365
    
После того как выполнение поиска контента, число размещения содержимого и предполагаемое количество результатов поиска, отображаются в профиле поиска. Можно также быстро просмотра статистики, например размещения содержимого, которые имеют большинство элементов, которые соответствуют поисковый запрос. После выполнения поиска можно предварительного просмотра результатов и экспортировать их на локальном компьютере.


## <a name="create-a-new-search"></a>Создать новый поиск

Для доступа к странице **поиска контента** для запуска операций поиска и предварительный просмотр и экспорт результатов поиска, администратор, специалисты по соответствию требованиям или диспетчер обнаружения электронных данных необходимо быть участником группы ролей диспетчер обнаружения электронных данных в системы &amp; центре соответствия требованиям. Дополнительные сведения можно [назначать разрешения обнаружения электронных данных в Office 365 безопасность &amp; центре соответствия требованиям](assign-ediscovery-permissions.md).
  
1. Последовательно выберите пункты [https://protection.office.com](https://protection.office.com).
    
2. Вход при помощи Office 365 адрес электронной почты и пароль. 
    
3. В разделе Безопасность &amp; центре соответствия требованиям, нажмите кнопку **поиска &amp; расследования** \> **поиска контента**.
    
4. На странице **поиска** щелкните стрелку рядом с элементом ![значком Добавить](media/8ee52980-254b-440b-99a2-18d068de62d3.gif) **новый поиск**. 
    
    ![Новый список раскрывающийся список поиска](media/76b25861-55c5-4f50-9d48-9e2be2d0d078.png)
  
    Можно выбрать один из следующих параметров.
    
  - **Интерактивная поиска** — этот параметр-мастер руководство по созданию поиска. Пользовательский интерфейс для выбора размещения содержимого и создайте запрос поиска, такие же, как вариант **новый поиск** . 
    
  - **Новый поиск** — этот параметр отображает обновленный пользовательский интерфейс, чтобы создать новый поиск. Это параметр по умолчанию, если нажать кнопку **Создать поиска**.
    
  - **Поиск по список Идентификаторов** — этот параметр позволяет поиска для конкретного сообщения и другие элементы почтового ящика с помощью списка идентификаторов Exchange. Для создания поискового списка ID (официально называется целевого поиска), отправка значений, разделенных запятыми (CSV) файл, определяющий элементы определенного почтового ящика для поиска. Для получения инструкций см. [Подготовка CSV-файла для списка идентификатор поиска контента в Office 365](csv-file-for-an-id-list-content-search.md).
    
    В оставшейся части действия, описанные в этой процедуре выполним по умолчанию, новый рабочий процесс поиска.
    
5. Выберите в раскрывающемся списке **новый поиск** . 
    
6. В разделе **запрос поиска**укажите следующие параметры.
    
    ![Укажите ключевые слова, условия и расположений для поиска](media/1e6de9dd-eac9-4e2a-819d-9740cf6c9106.png)
  
- **Ключевые слова для поиска** - тип запроса поиска в поле **ключевые слова** . Можно указать ключевые слова, сообщение свойства, например, отправленных и полученных даты, или свойств документа, таких как имена файлов или Дата последнего изменения документа. Можно использовать более сложные запросы, использующие логический оператор, например **и**, **или**, **не**и **NEAR**. Можно также выполнить поиск конфиденциальной информации (например, номера социального страхования) в документы или поиск документов, которые совместно извне. Если ключевое слово поле оставлено пустым, весь контент, расположенный в указанном расположении контента будут включены в результатах поиска.
    
    Кроме того щелкните флажок **Показать список ключевых слов** и тип ключевого слова в каждой строке. После этого ключевых слов в каждой строке, соединенных с логический оператор ( **c:s**), аналогичную функциональность оператор **OR** в запрос поиска, которая создается. 
    
    Причины использования список ключевых слов? Можно получить статистику, показывающие количество элементов соответствуют каждого ключевого слова. Это может помочь быстро определять, какие ключевые слова эффективны наиболее (и бы). Можно также использовать ключевая фраза (заключаются в круглые скобки) в строке. Дополнительные сведения о статистике поиска можно [Просмотр статистики ключевых слов для результатов поиска контента](view-keyword-statistics-for-content-search.md).
    
- **Условия** - можно добавить условия поиска для сужения области поиска и возвращать более качественных набора результатов. Каждое условие добавляет предложение в поисковый запрос KQL, который создается и при запуске поиска. Условие по логический оператор ( **c: c**), аналогичную функциональность **и** оператор логически подключение для запроса ключевого слова (указанного в поле ключевых слов). Который означает, что элементы для удовлетворения запроса ключевого слова и одно или несколько условий, которые будут включены в результаты. Это, как помочь условий, чтобы сузить результаты. Список и описание условий, которые можно использовать в запросе поиска обратитесь к разделу «Условия поиска» в [запросах по ключевым словам и условия поиска для поиска контента](keyword-queries-and-search-conditions.md#search-conditions).
    
- **Расположения** - выберите размещения содержимого для поиска.
    
  - **Все расположения** — используйте этот вариант, чтобы найти все расположения контента в вашей организации. Это включает в себя электронной почты в всех почтовых ящиков Exchange (включая все неактивные почтовые ящики, почтовые ящики для всех групп Office 365, почтовые ящики для всех групп Майкрософт), все Скайп для бизнеса бесед, все SharePoint и OneDrive для бизнеса сайты (включая сайты для всех групп Office 365 и Microsoft рабочих групп) и элементы в общих папок Exchange.
    
  - **Определенные расположения** — используйте этот вариант для поиска конкретных размещения содержимого. Можно выполнить поиск все расположения контента для определенной службы Office 365 (таких как поиск все почтовые ящики Exchange или поиск по всем сайтам SharePoint) или можно выполнить поиск определенного расположения в любом из служб Office 365, которые отображаются. 
    
    ![Пользовательский интерфейс для выбора размещения содержимого для поиска](media/9a09708b-f8a2-4382-8c4e-2c610ec33c72.png)
  
    Обратите внимание, что можно также добавлять группы рассылки в список почтовые ящики Exchange для поиска. Для групп рассылки поиска в почтовых членов группы. Обратите внимание на то, что динамические группы рассылки не поддерживаются.
    
    **Важные:** При выполнении поиска все расположения почтового ящика или только определенных почтовых ящиков, данные из MyAnalytics и других приложений Microsoft Office 365, сохраненным почтовые ящики пользователей включаются при экспорте результатов поиска контента. Эти данные не будут включены в оценке результатов поиска и не будут доступны для просмотра. Оно будет включен только при экспорте и загрузка результатов поиска; в разделе [Экспорт данных из MyAnalytics и других приложений Microsoft Office 365](#exporting-data-from-myanalytics-and-other-office-365-applications) в разделе «Дополнительные сведения о поиск контента». 
    
7. После настройки запрос поиска, нажмите кнопку **Сохранить &amp; запустите**.
    
8. На странице **Сохранить поиска** введите имя для поиска и необязательное описание, которое помогает определить поиска. Обратите внимание, что имя поиска должно быть уникальным в пределах организации. 
    
9. Нажмите кнопку **Сохранить** для запуска службы поиска. 
    
    После сохранения и выполнить поиск любые результаты, возвращенные операцией поиска отображаются в области результатов. В зависимости от того, как у вас есть настроен параметр предварительного просмотра результаты поиска, отображаемое или нужно щелкнуть **Предварительный просмотр результатов** для их просмотра. Для получения дополнительных сведений см. 
    
Чтобы снова откройте этот поиска контента или получить доступ к другим контента запросы поиска, перечисленные на странице " **Поиск контента** ", выберите Поиск и нажмите кнопку **Открыть**. 
  
Чтобы очистить результаты или создать новый поиск, нажмите кнопку ![значком Добавить](media/O365-MDM-CreatePolicy-AddIcon.gif) **новый поиск**. 

  
## <a name="preview-search-results"></a>Просмотр результатов поиска

Существует два параметров конфигурации для предварительного просмотра результатов поиска. Запустите новый поиск новый или откройте существующий поиск, нажмите кнопку ** отдельных результатов ** для просмотра следующие параметры предварительного просмотра: 
  
![Параметры результатов поиска в предварительной версии](media/83519477-1c85-4442-8886-481f186fd758.png)
  
1. **Предварительный просмотр результатов автоматически** — этот параметр отображает результаты поиска после можно выполнить поиск.
    
2. **Предварительный просмотр результатов вручную** — этот параметр отображает заполнители в области результатов поиска и кнопка **Предварительный просмотр результатов** , который нужно щелкнуть для отображения результатов поиска. Это значение по умолчанию; Это помогает повысить производительность поиска, автоматически с результатами поиска при открытии существующей поиска. 
    
Существуют ограничения, связанные с количество элементов, доступны для просмотра. Дополнительные сведения можно [ограничения для поиска в Office 365 безопасность &amp; центре соответствия требованиям](limits-for-content-search.md). 
  
Список поддерживаемых типов файлов, которые можно просматривать в разделе [Previewing результаты поиска](#previewing-search-results) в разделе «Дополнительные сведения о поиск контента». Если тип файла не поддерживается для предварительного просмотра или загрузить копию документа, нажмите кнопку **загрузить исходный файл** для загрузки на локальный компьютер. URL-адрес страницы для веб-страницах .aspx, включен на то, что может не иметь разрешения на доступ к странице. 
  
Также Обратите внимание на то, что неиндексированные элементов не доступны для предварительного просмотра.
  
## <a name="view-information-and-statistics-about-a-search"></a>Просмотр сведений о и статистики о поиске

После создания и выполнения поиска контента, можно просмотреть статистических данных об оценке результатов поиска. Этот компонент включает результатов поиска, статистику запросов, такие как количество размещения содержимого с элементами, которые соответствуют запросов поиска и имя расположения контента, которые имеют наиболее подходящие элементы. Вы можете отобразить статистики для поиска контента. Это позволяет быстро сравнение результатов для нескольких операций поиска и принятия решений о эффективность поисковых запросов.
  
Также можно загрузить Статистика поиска и статистику по ключевым словам в CSV-файл. Это позволяет использовать функции фильтрации и сортировки в Excel для сравнения результатов и подготовки отчетов для результатов поиска.
  
Для просмотра статистики поиска:
  
1. На странице **поиска контента** в безопасности &amp; центре соответствия требованиям, нажмите кнопку **Открыть** и нажмите кнопку поиска, для просмотра статистики для. 
    
2. Во время выполнения страницы нажмите кнопку **Открыть запрос**. 
    
3. В **отдельных результатов** раскрывающемся списке выберите **профиль поиска**.
    
4. В раскрывающемся списке **Тип** выберите один из следующих вариантов в зависимости от Статистика поиска, который вы хотите просмотреть. 
    
  - **Сводка** — отображает статистику для каждого типа контента поиск расположения. Это содержимое с количеством направлений контента, содержащие элементы, которые соответствуют условиям поиска, и общее число и размер элементов результатов поиска. Это значение по умолчанию.
    
  - **Запросы** - отображает статистику запросов поиска. Включает тип контента места, статистику запросов могут быть применены, частью статистику запросов поиска могут быть применены к (Обратите внимание на то, что **основной** указывает весь поисковый запрос), с количеством направлений контента, которые содержат элементы, Match запросов поиска и общее число и размер и элементы, которые были найдены (в выбранном расположении контента), соответствующие поисковый запрос. Обратите внимание на то, что статистика для неиндексированных элементов (также называемая частично индексированных элементов) также отображаются. Тем не менее только частично индексированных элементов из почтовых ящиков включенных в статистику. Частично индексированных элементов из SharePoint и OneDrive не включаются в статистику.
    
  - **Основные расположения** - отображает статистику о количестве элементов, которые соответствуют запросов поиска в каждой расположение содержимого, который был выполнен поиск. В начало 1000 расположения отображаются.
    
Более подробные сведения о статистике поиска содержатся в разделе [Просмотр статистики ключевых слов для результатов поиска контента](view-keyword-statistics-for-content-search.md).
  
  
## <a name="export-search-results"></a>Экспорт результатов поиска

После успешного выполнения поиска можно экспортировать результаты поиска на локальном компьютере. При экспорте результатов по электронной почте, их можно загрузить на свой компьютер как PST-файлов или отдельных сообщений (файлы MSG-файл). При экспорте содержимого из сайтов SharePoint и OneDrive экспортируются копии собственный документов Office. Существуют также дополнительные документы и отчеты, которые включены в результаты поиска экспортированного. Можно также просто экспортировать отчет о результатах поиска и не фактические элементов.
  
Экспорт результатов поиска:
  
1. На странице **поиска контента** в безопасности &amp; центре соответствия требованиям, нажмите кнопку **Открыть** и нажмите кнопку поиска, который необходимо экспортировать результаты поиска. 
    
2. Во время выполнения страницы, нажмите кнопку ![результатов поиска в экспорта значок](media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) **больше**и нажмите кнопку **экспортировать результаты**. Обратите внимание на то, что вы также можете экспортировать отчет о результатах поиска.
    
3. Заполните разделы на всплывающей странице **Экспорт результатов**. С помощью полосы прокрутки прокрутите содержимое страницы и просмотрите все параметры экспорта. 
    
Более подробные инструкции и советы по устранению неполадок см.
  
- [Экспорт результатов поиска из безопасности Office 365 &amp; центре соответствия требованиям](export-search-results.md)
    
- [Экспорт отчета о поиске контента](export-a-content-search-report.md)

### <a name="building-a-search-query"></a>Построение запросов поиска

Получить подробные сведения о создании поискового запроса, с помощью логические операторы поиска и условия поиска и поиск типов конфиденциальной информации и контент пользователям за пределами вашей организации просмотреть запросы [ключевого слова и условия поиска для поиска контента ](keyword-queries-and-search-conditions.md).

Если вы только что вставьте список ключевых слов в поле ключевых слов или нажмите клавишу **Ввод** после ввода ключевого слова, они не будут подключены оператором **или** . Ниже приведены неправильный и правильный пример добавления список ключевых слов. 
    
    **Неправильно**
    
    ![Неправильный способ форматирования списка ключевых слов (путем вставки в список в поле ключевых слов)](media/fb54e3df-232a-439a-b3d7-27a60ec76a4c.png)
  
    **Правильно**
    
    ![Правильный способ форматирования списка ключевых слов (путем выбора checkbox, а затем вставки списка)](media/5d511a7b-c1f9-499c-bffe-e075bfc9adec.png)
  
- Можно также подготовить список ключевых слов или фраз ключевое слово в файле Excel или обычный текстовый файл и затем скопируйте и вставьте список в список ключевых слов. Для этого необходимо установите флажок **Отображать список ключевых слов** . Затем щелкните первую строку в списке ключевое слово и вставьте список. Каждую строку из файлов Excel, или текст будет вставлен в в отдельные строки в список ключевых слов. 
    
- После создания запроса с помощью списка ключевых слов, рекомендуется убедитесь, что вы предназначен синтаксис запроса поиска, чтобы сделать поисковый запрос. В запрос поиска, который отображается в области сведений в разделе **запрос** , ключевые слова, разделенных текста **(c:s)**. Это указывает на то, что ключевые слова соединены логический оператор, аналогичную функциональность оператор **OR** . Аналогично, если запрос поиска включает условия, ключевые слова и условия, разделенных текста **(c: c)**. Это указывает на то, что ключевые слова, подключенные к условиям с логического оператора аналогично функциональных возможностей **и** оператор. Неподдерживаемые символы часто являются скрытыми и обычно приводит к ошибке поиска или возвращать непредвиденные результаты. Дополнительные сведения о неподдерживаемых символов, которые проверяются увидеть [запрос поиска контента по ошибкам](check-your-content-search-query-for-errors.md).
    
- Если у вас есть поискового запроса, который содержит ключевые слова для не английские символы (например, китайских знаков), нажмите кнопку **Query language страны или региона**![значок страны или региона языка запросов на поиск контента](media/8d4b60c8-e1f1-40f9-88ae-ee2a7eca0886.png) и выберите пункт значение кода страны и языка языка и региональных параметров для поиска. ### <a name="searching-microsoft-teams-and-office-365-groups"></a>Поиск групп Майкрософт и группы Office 365

Можно выполнить поиск почтовых ящиков, связанный с группой Office 365 или группы разработчиков Microsoft. Так как группами Майкрософт, основанные на Office 365 групп, поиск их очень похожи. В обоих случаях поиск только почтовый ящик группы или рабочая группа; не искать почтовые ящики членов группы или рабочая группа. Чтобы выполнить поиск их, необходимо специально добавьте их в результате поиска.
  
Имейте в виду следующее при поиске содержимого в Office 365 группы и группами Майкрософт.
  
- Чтобы выполнить поиск содержимого, расположенного в Office 365 группы и группами Майкрософт, необходимо указывать почтовых ящиков и сайта SharePoint, связанных с группы или группы.
    
- Выполните командлет **Get-UnifiedGroup** в Exchange Online для просмотра свойств для группы разработчиков Microsoft или группу Office 365. Это удобный способ получить URL-адрес для сайта, связанный с группой или группу. Например следующая команда выводит выбранных свойств для группы Office 365 с именем старший руководство: 
    
  ```
  Get-UnifiedGroup "Senior Leadership Team" | FL DisplayName,Alias,PrimarySmtpAddress,SharePointSiteUrl
  DisplayName            : Senior Leadership Team
  Alias                  : seniorleadershipteam
  PrimarySmtpAddress     : seniorleadershipteam@contoso.onmicrosoft.com
  SharePointSiteUrl      : https://contoso.sharepoint.com/sites/seniorleadershipteam
  
  ```

    > [!NOTE]
    > Чтобы выполнить командлет **Get-UnifiedGroup** , необходимо назначить роль только для просмотра получателей в Exchange Online или входить в группу ролей, которая назначена роль получателей только для просмотра. 
  
- При поиске почтового ящика пользователя не будет искать любой группы разработчиков Microsoft или группы Office 365, который входит пользователь. Аналогично, при поиске группы разработчиков Microsoft или группу Office 365 только группы и почтового ящика сайта группы, которые можно указать выполняется поиск; Если не были добавлены к поиску явно не поиска почтовых ящиков и OneDrive для бизнеса учетные записи членов группы.
    
- Чтобы получить список членов группы разработчиков Microsoft или группу Office 365, можно просмотреть свойства на **Домашняя страница \> группы** страницы в центре администрирования Office 365. Кроме того в Exchange Online PowerShell можно выполните следующую команду: 
    
  ```
  Get-UnifiedGroupLinks <group or team name> -LinkType Members | FL DisplayName,PrimarySmtpAddress 
  ```

    > [!NOTE]
    > Чтобы выполнить командлет **Get-UnifiedGroupLinks** , необходимо назначить роль только для просмотра получателей в Exchange Online или входить в группу ролей, которая назначена роль получателей только для просмотра. 
  
- Бесед, которые входят в состав групп Майкрософт канала хранятся в почтовом ящике, сопоставленные с группы разработчиков Microsoft. Аналогично файлы, совместно использующих участников группы в канале хранятся на сайте SharePoint team. Таким образом необходимо добавить сайт SharePoint и почтового ящика группы разработчиков Microsoft в качестве расположения контента для поиска бесед и файлы в канале.
    
- Кроме того бесед, которые входят в списке беседы в группами Майкрософт, хранятся в почтовый ящик Exchange Online пользователи, участвующие в чата. И файлы, которые пользователь использует совместно в беседах чата хранятся в OneDrive для бизнеса учетной записи пользователя, который использует файл. Таким образом необходимо добавить отдельных пользователей почтовых ящиков и OneDrive для бизнеса учетных записей как расположения контента для поиска файлов и бесед в списке беседы.
    
    > [!NOTE]
    > Пользователи, участвовать в беседах, в которых являются частью список чата в группах Microsoft должны иметь почтовый ящик Exchange Online (на основе облака) в порядке для поиска бесед чата. Это, поскольку бесед, которые входят в состав список чата, хранятся в облачные почтовые ящики участников беседы. Если участник беседы не имеет почтовый ящик Exchange Online, нельзя будет выполнять поиск бесед чата. Например в гибридном развертывании Exchange, пользователи с локального почтового ящика можно принять участие в беседах, в которых являются частью список чата в группах Майкрософт. Однако в этом случае содержимого из этих беседы не доступен для поиска так как пользователи не имеют облачные почтовые ящики. 
  
- Каждый канал группы или группы разработчиков Microsoft содержит вики-сайт заметок и совместной работы. Вики-контент автоматически сохраняются в файл с формат MHT. Этот файл хранится в библиотеке документов данных вики-сайт группы на сайте SharePoint team. Средство поиска контента используется для поиска вики-сайт, указав сайту SharePoint как расположение содержимого для поиска. 
    
    > [!NOTE]
    > Возможность поиска вики-сайта группы разработчиков Microsoft или канала (при выполнении поиска сайта SharePoint team) была выпущена 22 июня 2017. Вики-страницы, которые были сохранены или обновлены на, дата, или в конце доступны для поиска. Вики-страницы последнего сохранения или обновлены до этой даты недоступны для поиска. 
 
- Сводные данные для собраний и звонков в канале группами Майкрософт также хранятся в почтовые ящики пользователей, которым набираемым в собрание или звонок. Это означает, что поиск контента можно использовать для поиска этих сводки записей. Сводная информация включает в себя: 
  - Дата, время начала, время окончания и продолжительность вызова или собрания

  - Дата и время, когда каждый участник или устраненных собрания или звонок

  - Вызовы, направленные на голосовую почту

  - Пропущенные или неотвеченные звонки

  - Перевод звонков, которые представлены в виде двух отдельных вызовов

  Обратите внимание на то, что может потребоваться до 8 часов для собраний и вызова сводки записей будут доступны для поиска.

  В результатах поиска сводок собраний, определяются как **собрание** в **поле типа**; сводки звонков, определяются как **звонок**. Кроме того бесед, которые входят в состав группы канала и 1xN бесед, определяются как **обмен мгновенными Сообщениями** в поле **Тип** .
  
  ![В поле Тип определяются группами собрания, звонки и 1xN чаты](media/O365-ContentSearch-Teams-MessageKind.png)

- Свойство **типа** электронной почты или условие поиска **Вид сообщения** можно использовать для поиска специально для содержимого в группах Майкрософт. 
  - Чтобы использовать свойство **типа** как часть запрос поиска, в поле **ключевые слова** поискового запроса введите `kind:microsoftteams`.

    ![Используйте вид: microsoftteams в поле ключевые слова](media/O365-ContentSearch-Teams-Keywords.png)
  
  - Чтобы использовать условие поиска, добавить условие **Вид сообщения** и использовать значение `microsoftteams`. 

    ![Используйте условие вида сообщения с microsoftteams значение.](media/O365-ContentSearch-Teams-MessageKindCondition.png)

Обратите внимание, что условия логически подключены к запроса ключевого слова с оператором **и** . Это означает, что элемент должно соответствовать запроса ключевого слова и условия поиска, возвращаемых в результатах поиска. Для получения дополнительных сведений обратитесь к разделу «Рекомендации по использованию условия» в [запросах по ключевым словам и условия поиска для поиска контента.](keyword-queries-and-search-conditions.md#guidelines-for-using-conditions)

  
### <a name="searching-inactive-mailboxes"></a>Поиск неактивные почтовые ящики

Можно выполнить поиск неактивные почтовые ящики в поиска контента. Чтобы получить список неактивные почтовые ящики в вашей организации, выполните команду `Get-Mailbox -InactiveMailboxOnly` в Exchange Online PowerShell. Кроме того, можно перейти к **управлению данных** \> **хранения** в безопасности &amp; центре соответствия требованиям и нажмите кнопку **Дополнительные**![панели навигации эллипсы](media/9723029d-e5cd-4740-b5b1-2806e4f28208.gif) \> **неактивные почтовые ящики**.
  
Вот несколько факторов, которые следует учитывать при поиске неактивные почтовые ящики.
  
- Если поиск контента включает в себя почтового ящика пользователя и почтового ящика становится неактивной, поиск контента будет продолжать поиск неактивного почтового ящика, если повторно запустите поиска после его неактивным.
    
- В некоторых случаях пользователь может иметь активного хранилища почтовых ящиков и неактивного почтового ящика, для которого один и тот же адрес SMTP. В этом случае будет осуществляться только определенного почтового ящика, выберите в качестве расположения для поиска контента. Другими словами при добавлении почтового ящика пользователя для поиска нельзя предполагать, что будет осуществляться обоих их активные и неактивные почтовые ящики; поиск будет выполняться только почтовых ящиков, которые явно добавлять для поиска.
    
- Мы настоятельно рекомендуем избежать необходимости активного хранилища почтовых ящиков и неактивного почтового ящика с тем же адресом SMTP. Если требуется повторно использовать SMTP-адрес, который назначен в настоящее время неактивного почтового ящика, мы рекомендуем восстановление неактивного почтового ящика или восстановления содержимого неактивного почтового ящика для активного хранилища почтовых ящиков (или архива active почтовых ящиков), а затем удалите неактивного почтового ящика. Дополнительные сведения см.
    
  - [Восстановление неактивного почтового ящика в Office 365](recover-an-inactive-mailbox.md)
    
  - [Восстановление неактивного почтового ящика в Office 365](restore-an-inactive-mailbox.md)
    
  - [Удаление неактивного почтового ящика в Office 365](delete-an-inactive-mailbox.md)

### <a name="partially-indexed-items"></a>Частично индексированных элементов

- Как объяснялось ранее, частично индексированных элементов в почтовых ящиках, включенных в оценке результатов поиска; частично индексированных элементов из SharePoint и OneDrive не включаются в оценке результатов поиска. 
    
- Если частично элемента совпадений поиска запросов (из-за других свойств сообщения или документа соответствуют критериям поиска), не будут включены в предполагаемое количество неиндексированные элементов. Если частично элемент исключен с условия поиска, он также не будут включены в предполагаемое количество частично индексированных элементов. Для получения дополнительных сведений см [частично индексированных элементов в поиска контента в Office 365](partially-indexed-items-in-content-search.md).
    
### <a name="exporting-data-from-myanalytics-and-other-office-365-applications"></a>Экспорт данных из MyAnalytics и других приложений Microsoft Office 365

- Данные из MyAnalytics (например, полезные сведения о на как пользователи тратить свое время на основе данных почта и календарь в почтовом ящике) и данные из других приложений Office 365 — это сохраненный скрытых расположение (в ветви не IPM) в облачного почтового ящика пользователя. После запуска поиска контента эти данные не включена в оценке результатов поиска, статистику запросов и не доступна для предварительного просмотра. Тем не менее эти данные будут экспортироваться при экспорте результатов поиска.
    
- MyAnalytics данные и данные из других приложений Office 365 экспортируется в папку с именем «Другие данные Office 365». Эта папка содержит вложенные папки для каждого пользователя.
