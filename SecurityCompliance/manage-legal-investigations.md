Это может вызвать быстрого поиска и хранения для дальнейшего изучения особые сведения в электронной почты, документы, Ведение текстовых бесед и другие расположения контента, используемых пользователей в своей повседневной работе задачи. Эти и другие аналогичные действия можно выполнить с помощью средств делами обнаружения электронных данных в Office 365 безопасность &amp; центре соответствия требованиям.

## <a name="analyze-case-data-using-office-365-advanced-ediscovery"></a>Анализ данных делами, с помощью Office 365 расширенного обнаружения электронных данных

Office 365 расширенной eDiscovery построена на контента возможности поиска и обнаружения электронных данных, описанное в предыдущем разделе. После создания вариантом eDiscovery место расположения custodian на удержание и сбора данных, которые могут представлять реагировать на регистр, вы может провести дальнейший анализ данных с помощью анализа текста, машинного обучения и упреждающий написания кода возможности Дополнительно обнаружение электронных данных. Это может помочь организации быстрой обработки тысячи сообщений электронной почты, документы и другие виды данных найдите те элементы, которые, скорее всего, предназначенных для определенного случая. И мы единой расширенной eDiscovery и Управление делами, можно легко управлять же case в контексте безопасности &amp; центре соответствия требованиям.
  
> [!NOTE]
> Для анализа данных с помощью расширенных eDiscovery, пользователя (custodian данные) должны быть назначены лицензии Office 365 E5 пользователя. Кроме того пользователи, имеющие лицензию на Office 365 E1 или E3 может быть назначен расширенной eDiscovery отдельная лицензия. Администраторы и контролеры, назначенных для случаев и использовать расширенные обнаружения электронных данных для анализа данных, не требуется лицензия на E5.

### <a name="analyze-data"></a>Анализ данных

После загрузки данных поиска в случай в расширенной eDiscovery модуля анализа используется для запуска анализа. Первая часть процесса анализа состоит из упорядочение файлов в группы уникальных файлов, дубликаты и рядом с дубликаты (также известен как подобия документов). Затем будет организации данные еще раз в иерархически структурированном группы потоков электронной почты и темы и при необходимости set игнорировать текстовые фильтры для исключения определенных текст из анализа. Затем вы выполнения анализа и просмотреть результаты.
  
- [Дополнительные сведения о документе подобия](understand-document-similarity-in-advanced-ediscovery.md) для подготовки к анализ данных в расширенной обнаружения электронных данных 
    
- [Настройка параметров](set-analyze-options-in-advanced-ediscovery.md) для рядом с дубликаты, темы и адрес электронной почты, работа с потоками и затем запустите модуля анализа 
    
- [Настройка игнорировать текстовые фильтры](set-ignore-text-in-advanced-ediscovery.md) для исключения текст и текстовые строки из выполняется анализ; Эти фильтры также игнорирует текст при запуске анализа релевантности 
    
- [Просмотр результатов](view-analyze-results-in-advanced-ediscovery.md) анализа процесса
