## <a name="what-is-service-encryption-with-customer-key"></a><span data-ttu-id="219de-110">Что такое служба шифрования с ключом клиента?</span><span class="sxs-lookup"><span data-stu-id="219de-110">What is service encryption with Customer Key?</span></span>

<span data-ttu-id="219de-p104">Ключ клиента — функция, которая позволяет вам потребуется подготовить и обслуживать клавиш, используемые для шифрования данных rest в Office 365. Компонент использует службы шифрования, которая является шифрования, который выполняется с Office 365 Exchange и SharePoint. Службы шифрования является строгое мера противодействия, если он пытается обхода системы управления доступа к Office 365, который будет использоваться для обработки всех запросов на доступ к данным клиента. Это, так как служба шифрования означает, что администратор сервера не имеет элемента управления или даже доступа для шифрования разделов и не может отключить шифрование, в отличие от с BitLocker. The feature leverages service encryption, which is encryption that is performed by Office 365 Exchange and SharePoint. Service encryption provides advantages beyond what BitLocker can provide - namely, greater defense-in-depth against attackers. Service encryption is a strong countermeasure if an attacker attempts to bypass the Office 365 access control system that is used to process all requests for access to customer data. This is because service encryption means that a server administrator does not have control or even access to encryption keys and cannot disable encryption, unlike with BitLocker. Therefore, an attacker with administrative access to a server hosting customer data that was encrypted using service encryption will not be able to read the customer data and even if the encrypted data is copied off the server it will remain useless.</span></span>

<span data-ttu-id="219de-p111">Ключ доступности является корневой раздел, который был подготовлен при создании политики шифрования данных. Клавиша доступности хранения и защищенные в Office 365 и функционально похож на двух корневые ключи, предоставленные пользователем, для использования с помощью службы шифрования с ключом клиента. Хранение и доступ к разделу доступности, намеренно отличаются от ключи помещение ключ Azure по трем причинам: во-первых, ключ доступности обеспечивает возможность высокой доступности в случае, если службы Office 365, не удается подключиться к ключи, размещенных в Azure Помещение; Во-вторых ключ доступность содержит функцию «приостановить стекло» в случае, если оба ключа помещение ключ Azure не сохраняются; и в-третьих, представлены в глубокой разделение логических элементов управления и обеспечивает защиту от потери всех ключей из одного атаки или точке сбоя. В конечном счете общего доступа к ответственности для защиты ключей, во время использования различных средства защиты и процессы, используемые для управления ключами снижает риск, что все разделы (и, следовательно, данные) будут утеряны или удален. The availability key is stored and protected within Office 365 and is functionally similar to the two root keys that are supplied by you for use with service encryption with Customer Key. Unlike the keys that you provide and manage in Azure Key Vault, you can't directly access the availability key. Storage and control of the availability key are deliberately different from Azure Key Vault keys for three reasons: first, the availability key provides a high-availability capability in the event that Office 365 services are unable to reach keys hosted in Azure Key Vault; second, the availability key provides a "break glass" capability in the event that both Azure Key Vault keys are lost; and third, the separation of logical controls provides defense-in-depth and protects against the loss of all keys from a single attack or point of failure.
