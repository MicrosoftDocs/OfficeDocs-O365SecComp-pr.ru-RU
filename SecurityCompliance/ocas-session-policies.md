# <a name="session-policies-in-office-365-cloud-app-security"></a>Политики сеансов в Office 365 Cloud App Security

Политики сеансов безопасности облачных приложений Office 365 включают мониторинг на уровне сеанса в режиме реального времени, позволяя детально отображать приложения Office 365 и способность принимать различные действия в зависимости от политики, заданной для сеанса пользователя. Вместо того чтобы разрешать или блокировать доступ полностью, с помощью контроля сеансов можно разрешить доступ при наблюдении за сеансом и/или ограничения определенных действий сеансов с помощью возможностей обратного прокси-сервера для условного доступа к приложению.

Например, вы можете решить, что из неуправляемых устройств или для сеансов, поступающих из определенных расположений, вы хотите разрешить пользователю доступ к приложению, но также ограничить загрузку конфиденциальных файлов или требовать, чтобы определенные документы были защищены после загрузки. Политики сеансов позволяют настраивать эти элементы управления пользователя и разрешить доступ, а также позволяет:

- [Наблюдение за всеми действиями](#monitor-all-activities)

- [Блокировать все загружаемые файлы](#block-all-downloads)

- [Блокировка определенных действий](#block-specific-activities)

- [Защита файлов при загрузке](#protect-files-on-download)

## <a name="prerequisites-to-using-session-policies"></a>Необходимые условия для использования политик сеансов

- Лицензия Azure AD Premium P1

- Важные приложения Office 365 должны быть [развернуты с помощью элемента управления приложением условНого доступа](ocas-deploy-conditional-access-app-control.md)

- должна находиться  [политика условного доступа Azure AD](https://docs.microsoft.com/azure/active-directory/active-directory-conditional-access-azure-portal), которая перенаправляет пользователей на Office 365 Cloud App Security

## <a name="create-an-azure-ad-conditional-access-policy"></a>Создание политики условного доступа Azure AD

Политики условного доступа Azure Active Directory и политики сеанса безопасности Cloud App действуют в сочетании для изучения каждого сеанса пользователя и принятия решений по политике для каждого приложения. Чтобы настроить политику условного доступа в Azure AD, выполните следующие действия:

1. Настройка  [политики условного доступа Azure AD](https://docs.microsoft.com/azure/active-directory/active-directory-conditional-access-azure-portal)с назначениями для пользователя или группы пользователей и приложения, которым необходимо управлять с помощью элемента управления "условный доступ". Note: Эта политика влияет только на приложения, развернутые [с помощью элемента управления](ocas-deploy-conditional-access-app-control.md) "условный доступ".

2. направьте пользователей на Office 365 Cloud app Security, выбрав на странице **сеанса** принудительные  **ограничения на управление приложением условного доступа**.

## <a name="create-a-cloud-app-security-session-policy"></a>Создание политики сеанса Cloud App Security

Чтобы создать новую политику сеанса, выполните следующие действия:

1. На портале выберите **элемент Управление** , а затем **политики**.

2. На странице " **политики** " щелкните **создать политику** и выберите **Политика сеанса**.

3. В окне **Политика** сеанса назначьте имя политики.

4. В поле **тип** управления сеансом:
    
    - Выберите **следить, только** если вы хотите отслеживать только пользователей. При этом будет создана политика "только мониторинг" для приложений, которые вы выбрали, где будут загружаться все входы, эвристические загрузки и типы действий.
    
    - Выберите **Загрузка файла управления (с DLP)** , если вы хотите отслеживать действия пользователей. Можно выполнить дополнительные действия, такие как блокировка или защита загрузки для пользователей.
    
    - Выберите **блокировать действия** для блокирования определенных действий, которые можно выбрать с помощью фильтра **типов** действий. Будут отслеживаться все действия из выбранных приложений (и они будут задаваться в журнале активности). Выбранные действия будут заблокированы, если выбрано действие **блокировать** . Выбранные действия будут вызывать оповещения при выборе действия **теста** и включении оповещений.

5. В разделе **источник** действия в разделе **действия, отвечающие всем следующим** разделам выберите дополнительные фильтры активности, которые необходимо применить к политике. Эти фильтры могут включать следующие параметры:
    
    - **Теги устройств**: Используйте этот фильтр для определения неуправляемых устройств.
    
    - **Расположение**: Используйте этот фильтр для определения неизвестных (и, следовательно) расположений.
    
    - **IP-адрес**: Используйте этот фильтр для ФИЛЬТРАЦИИ по IP-адресам или использования ранее назначенных тегов IP-адреса.
    
    - **Тег агента пользователя**: Используйте этот фильтр, чтобы включить эвристику для определения приложений для мобильных устройств и настольных ПК. Для этого фильтра можно задать значение равно или не равно **Native Client**. Этот фильтр необходимо протестировать для мобильных и настольных приложений для каждого облачного приложения.<br>NOTE. политики сеансов не поддерживают мобильные и классические приложения. Вы также можете заблокировать или разрешить мобильные приложения и классические приложения, создав политику доступа.

6. Если вы выбрали параметр **Управление скачиванием файлов (с DLP)**, в разделе **источник** действия в разделе **файлы, удовлетворяющие всем следующим** разделам выберите дополнительные фильтры файлов, которые будут применяться к политике. Эти фильтры могут включать следующие параметры:
        
    - **Метка классификации —** используйте этот фильтр, если ваша организация использует Azure Information Protection, а ваши данные защищены с помощью меток классификации. Вы можете фильтровать файлы на основе назначенной им метки классификации. Дополнительные сведения об интеграции с Azure Information Protection можно найти в статье [Интеграция Azure Information Protection](https://docs.microsoft.com/cloud-app-security/azip-integration).
        
    - **Имя файла —** используйте этот фильтр для применения политики к определенным файлам.
        
    - **Тип файла —** используйте этот фильтр для применения политики к определенным типам файлов, например заблокировать загрузку для всех xls файлов.
    
    - В разделе **Проверка** содержимого укажите, следует ли включить модуль DLP для сканирования документов и содержимого файлов.
    
    - В разделе **действия**выберите один из следующих элементов:
        
        - **Test (наблюдение за всеми действиями)**: установите для этого действия явно разрешение скачивания в соответствии с заданными фильтрами политики.
        
        - **Block (блокировать загрузку и отслеживание всех действий)**: установите для этого действия явное блокирование загрузки в соответствии с заданными фильтрами политики. Для получения дополнительных сведений Узнайте, [как работает загрузка блока](https://docs.microsoft.com/en-us/cloud-app-security/session-policy-aad#block-download).
        
        - **Защита (применение метки классификации для скачивания и отслеживания всех действий)**: этот параметр доступен только в том случае, если в разделе **Политика сеанса**выбран **элемент управления Загрузка файла (с защитой DLP)** . Если в вашей организации используется Azure Information Protection, вы можете задать **действие** , которое будет применять метку классификации, заданную в Azure Information Protection, в файл. Для получения дополнительных сведений Узнайте, [как работает функция защиты скачивания](https://docs.microsoft.com/en-us/cloud-app-security/session-policy-aad#protect-download).

7. Вы можете **создать оповещение для каждого сопоставленного события с уровнем серьезности политики** и задать лимит оповещений. Выберите, нужно ли оповещать как сообщение электронной почты, текстовое сообщение или и то, и другое.

## <a name="monitor-all-activities"></a>Наблюдение за всеми действиями

При создании политики сеанса каждый сеанс пользователя, соответствующий политике, перенаправляется в управление сеансом, а не непосредственно в приложение. Пользователь увидит уведомление о мониторинге, чтобы сообщить им, что их сеансы отслеживаются.

Если вы не хотите уведомлять пользователя о том, что он отслеживается, вы можете отключить сообщение с уведомлением.

1. В разделе Параметры ког выберите пункт **Общие параметры**.

2. Затем в разделе **элемент управления приложением условного доступа**выберите пункт **мониторинг** пользователей и снимите флажок **уведомлять пользователей** .

Чтобы сохранить пользователя в сеансе, элемент управления приложением "условный доступ" заменяет все необходимые URL-адреса, скрипты Java и файлы cookie в сеансе приложения с помощью URL-адресов безопасности облачных приложений Office 365. Например, если приложение возвращает страницу со ссылками `myapp.com`, домены которых завершаются, элемент управления условного доступа заменяет ссылки на домены, заканчивающиеся `myapp.com.us.cas.ms`на что-то вроде. Таким образом, в Office 365 Cloud App Security отслеживается весь сеанс.

Элемент управления приложением условного доступа записывает журналы трафика каждого сеанса пользователя, который пересылается через него. Журнал трафика включает время, IP-адрес, агент пользователя, URL-адреса, которые были просмотрены, а также количество отправленных и загруженных байтов. Эти журналы анализируются и непрерывный отчет, **Управление приложением условНого доступа в Cloud App Security**добавляется в список отчетов о облачном обнаружении в панели мониторинга облачного обнаружения.

### <a name="to-export-these-logs"></a>Чтобы экспортировать эти журналы:

1. Перейдите к разделу Параметры Ког и выберите **элемент Управление приложением условНого доступа**.

2. В правой части таблицы нажмите кнопку Экспорт.<br>![Кнопка экспорта](media/image3.png)<br>

3. Выберите диапазон отчета и нажмите кнопку **Экспорт**. Этот процесс может занять некоторое время.

### <a name="to-download-the-exported-log"></a>Чтобы скачать экспортированный журнал:

1. После того как отчет будет готов, перейдите в раздел **Параметры** , а затем **экспортируйте отчеты**.

2. В таблице выберите нужный отчет из списка  **журналов трафика управления приложениями условного доступа**и нажмите кнопку Загрузить.<br>![Кнопка "Скачать"](media/image4.png)<br>

## <a name="block-all-downloads"></a>Блокировать все загружаемые файлы

Если в качестве **действия** , которое необходимо выполнить в политике сеанса безопасности Cloud App, задано значение **Block** , элемент управления "условный доступ" не позволяет пользователю загрузить файл в соответствии с фильтрами файлов политики. Событие загрузки распознается в Office 365 Cloud App Security для каждого приложения, когда пользователь начинает загрузку. Управление приложением условного доступа осуществляется в режиме реального времени, чтобы предотвратить его запуск. При получении сигнала, когда пользователь инициировал загрузку, элемент управления "условный доступ", он возвращает **** пользователю сообщение об ограничении загрузки и заменяет загруженный файл текстовым файлом. Пользовательское сообщение с текстовым файлом можно настраивать и настраивать из политики сеанса.

## <a name="block-specific-activities"></a>Блокировка определенных действий

Когда **блокирующие действия** задаются как **тип действия**, вы можете выбрать определенные действия для блокировки в определенных приложениях. Все действия из выбранных приложений будут отслеживаться и указываться в журнале активности. Выбранные действия будут заблокированы, если выбрано действие **блокировать** . Выбранные действия будут вызывать оповещения при выборе действия **теста** и включении оповещений.

**Блокировать определенные действия** и применять их к определенным группам для создания в организации полного режима, предназначенного только для чтения.

## <a name="protect-files-on-download"></a>Защита файлов при загрузке

Выберите **блокировать действия** для блокирования определенных действий, которые можно найти с помощью фильтра **типов** действий. Будут отслеживаться все действия из выбранных приложений (и они будут задаваться в журнале активности). Выбранные действия будут заблокированы, если выбрано действие **блокировать** . Выбранные действия будут вызывать оповещения при выборе действия **теста** и включении оповещений.

Если в качестве **действия** , выполняемого в политике сеанса безопасности Cloud App, задано значение **Protect** , элемент управления "условный доступ" применяет метки и последующую защиту файлов в соответствии с фильтрами файлов политики. Метки настраиваются в консоли Azure Information Protection и **защищаются** в метке, чтобы она отображалась в качестве варианта в политике Cloud App Security. После выбора метки и загрузки файла, соответствующего условиям политики безопасности Cloud App, подпись и соответствующая защита (с разрешениями) применяются при загрузке файла. Исходный файл остается без редактирования в облачном приложении, а загруженный файл теперь защищен. Пользователи, пытающиеся получить доступ к файлу, должны удовлетворять требованиям к разрешениям, определенным для применения защиты.
