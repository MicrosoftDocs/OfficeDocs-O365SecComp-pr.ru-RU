# <a name="compare-versions-of-ome"></a>Сравнение версий OME

В этой статье сравнивается устаревшее шифрование сообщений Office 365 и новые возможности OME. Новые возможности — это слияние и более поздние версии OME и управления правами на доступ к данным (IRM). Мы также рассмотрим, как эти два могут сосуществовать в организации Office 365.

## <a name="advantages-of-using-the-new-ome-capabilities-over-legacy-ome"></a>Преимущества использования новых возможностей OME по сравнению с прежними версиями OME

Новые возможности предоставляют следующие преимущества:

- Возможность использования только шифрования (что позволяет обеспечить безопасную совместную работу), не переСылать и настраиваемые ограничения.
- ОтПравители могут отправлять сообщения, зашифрованные с новыми возможностями, вручную из Outlook Desktop, Outlook для Mac и Outlook в веб-клиентах.
- Получатели Office 365 получают возможность использовать встроенные возможности в поддерживаемых клиентах Outlook. Кроме того, администраторы могут отображать Office 365 для пользователей с фирменной символикой.
- Учетные записи, не входящие в Office 365, такие как Gmail, Yahoo и учетные записи Майкрософт, являются федеративными с порталом OME, что обеспечивает более удобную работу пользователей для этих получателей. Все остальные удостоверения используют однократный код прохода для доступа к зашифрованным сообщениям.
- Администраторы могут настраивать фирменную символику и создавать различные шаблоны фирменного стиля.
- Администраторы могут отозвать электронные сообщения, зашифрованные с новыми возможностями.
- Новые возможности предоставляют подробные отчеты об использовании в центре безопасности &amp; и соответствия требованиям.

## <a name="coexistence-of-legacy-ome-and-the-new-capabilities-in-the-same-tenant"></a>Сосуществование устаревших OME и новых возможностей в одном клиенте

Вы можете использовать как устаревшие OME, так и новые возможности в одном клиенте. Как администратор, вы можете сделать это, выбрав версию OME, которую нужно использовать при создании правил для своего почтового процесса.

- Чтобы указать устаревшую версию OME, используйте действие правила обработки почтовых ящиков Exchange "применить предыдущую версию OME".
- Чтобы указать новые возможности, используйте действие правила обработки почтовых ящиков Exchange "применение шифрования сообщений Office 365 и защита прав".

Пользователи также могут отправлять сообщения, зашифрованные с новыми возможностями, вручную из Outlook Desktop, Outlook для Mac и Outlook в веб-клиентах.

## <a name="migrating-from-legacy-ome-to-the-new-capabilities"></a>Переход с устаревших OME на новые возможности

Несмотря на то, что обе версии OME могут совместно существовать, настоятельно рекомендуется изменить старые правила для обработки почты, которые используют действие правила "применить предыдущую версию OME", чтобы использовать новые возможности, указав действие правила обработки почты "применить шифрование сообщений Office 365 и защита прав ". Инструкции приведены в разделе [Определение правил для почтового процесса для шифрования сообщений электронной почты в Office 365](define-mail-flow-rules-to-encrypt-email.md).

## <a name="getting-started-with-ome"></a>Начало работы с OME

Как правило, новые возможности OME автоматически включены для вашей организации Office 365. Если вы готовы приступить к работе с новыми возможностями OME в вашей организации, ознакомьтесь со статьей [Настройка новых возможностей шифрования сообщений Office 365](set-up-new-message-encryption-capabilities.md).

Устаревшая версия OME автоматически включается для вашей организации Office 365, если вы включили Azure Information Protection. Ранее устаревшие OME работали, даже если не включена служба Azure Information Protection. Это больше не так.

Чтобы начать использовать устаревшее OME, если вы включили службу Azure Information Protection, все, что нужно сделать, — настроить правила для обработки почты, которые используют действие правила "применить предыдущую версию OME". Инструкции приведены в разделе [Определение правил для почтового процесса для шифрования сообщений электронной почты в Office 365](define-mail-flow-rules-to-encrypt-email.md).
