# <a name="apply-information-rights-management-irm-to-a-list-or-library"></a>Применение управления правами на доступ к данным (IRM) для списка или библиотеки

Управление правами на доступ к данным (IRM) можно использовать для управления и защитить файлы, которые загружаются из списков или библиотек.
  
## <a name="before-you-begin"></a>Перед началом работы

- Служба управления правами Azure (Azure RMS) из Azure защита информации и аналогичной локальной службы управления правами Active Directory (AD RMS) поддерживает управление правами на доступ для сайтов. Без установки отдельных или дополнительных не требуется.
    
- Перед установкой IRM для списка или библиотеки сначала она должна быть включена администратором сайта.
    
- Чтобы применить IRM для списка или библиотеки, необходимо иметь разрешения администратора для этого списка или библиотеки.
    
- При использовании SharePoint Online, пользователи могут возникнуть время ожидания при загрузке больших файлов, защищенных службой IRM. В этом случае применить защиту IRM с помощью приложений Office и хранить большие файлы в библиотеке SharePoint, который не используется служба IRM.
    
> [!NOTE]
> Если вы используете SharePoint Server 2013, администратор сервера необходимо установить компоненты защиты на всех интерфейсных веб-серверов для каждого типа файла, пользователи в вашей организации необходимо защитить с помощью IRM.

2. На ленте перейдите на вкладку **Библиотека** и нажмите кнопку **Параметры библиотеки**. (При работе в списке, перейдите на вкладку **список** и выберите **Параметры списка**).
    
    ![Параметры библиотеки SharePoint кнопки на ленте](media/cdf718fa-d792-40fc-8026-00c3b80b9e05.png)

Обратитесь к администратору сервера для получения можно включить IRM для вашего сайта. Ссылка управление правами на доступ не отображается для библиотеки рисунков.
    
4. На странице " **Параметры управления правами данными** " установите флажок **ограничить разрешения для документов этой библиотеки при загрузке** применять разрешения ограниченного доступа к документам, которые загружаются из этого списка или библиотеки.

## <a name="what-is-information-rights-management"></a>Что такое управление правами на доступ?
<a name="__toc256598175"> </a>

Управление правами на доступ к данным (IRM) позволяет ограничить действия, которые можно производить файлы, загруженные из списков или библиотек. IRM шифрует загруженных файлов и ограничения для группы пользователей и программ, которые разрешены для расшифровки этих файлов. IRM можно также ограничить права на доступ пользователей, которые могут читать файлы, поэтому они не может выполнять действия, например печати копий файлов или скопируйте текст из них.
  
Можно использовать IRM на списки или библиотеки для ограничения распространения конфиденциального содержимого. К примеру при создании библиотеки документов для предоставления сведений о предстоящих продуктов выбранный официальные представители маркетинга, можно использовать IRM чтобы запретить общий доступ к этой контент с другими сотрудниками компании пользователями.
  
На сайте Применение IRM для всего списка или библиотеки, а не в отдельные файлы. Это позволяет обеспечить согласованность уровень защиты для всего набора документов или файлов. IRM таким образом могут помочь вашей организации, для применения параметров корпоративных политик, управляющие использование и распространение конфиденциальных и личных сведений.
  
> [!NOTE]
> Сведения на этой странице относительно управления правами на доступ заменяет любые условия, ссылающиеся на «Управление правами на доступ» в любые условия соглашения Microsoft SharePoint Server 2013 и SharePoint Server 2016 лицензии. 
  
### <a name="how-irm-can-help-protect-content"></a>Как IRM помогает защитить содержимое
<a name="__toc256598176"> </a>

IRM помогает защитить содержимое с ограниченным доступом следующими способами:
  
- Помогает предотвратить авторизованным viewer, копирование, изменение, печать, отправка и прием факсов, или копировать и вставлять контент для несанкционированного использования
    
- Позволяет предотвратить копирование содержимого с помощью функции Print Screen в Microsoft Windows авторизованного средства просмотра
    
- Помогает предотвратить несанкционированного просмотра могли видеть содержимое при отправке по электронной почте после загрузки с сервера
    
- Ограничивает доступ к содержимому для указанного периода времени, после которого необходимо проверить их учетных данных пользователей и загрузки содержимого еще раз
    
- Позволяет задать корпоративной политики, управляющие использование и распространение содержимого в пределах организации
    
### <a name="how-irm-cannot-help-protect-content"></a>Как IRM не может помочь защиты контента
<a name="__toc256598177"> </a>

IRM не может защитить содержимое с ограниченным доступом из следующих вариантов:
  
- Стирания, кража, захвата или передачи по вредоносные программы, такие как троянских коней, нажатием клавиши средства ведения журнала и определенных типов шпионских программ
    
- Потеря или повреждение из-за действия компьютерных вирусов
    
- Ручное копирование или повторного ввода содержимого из отображения на экране
    
- Цифровые или фильма фотографии контент, который отображается на экране
    
- Копирование с помощью программ сторонних производителей снимок экрана
    
- Копирование содержимого метаданных (значения столбцов) с помощью программ сторонних производителей снимок экрана или действие копирования и вставки

## <a name="how-irm-works-for-lists-and-libraries"></a>Как IRM работает для списков и библиотек
<a name="__toc256598178"> </a>

Защиту IRM применяется к файлам на уровне списка или библиотеки. При включении IRM для библиотеки управления правами применяется ко всем файлы из этой библиотеки. Если IRM включена для списка, управление правами на применяется только к файлы, подключенные к элементам списка, не элементы существующих списков.
  
Когда пользователи загружать файлы в поддержкой IRM списка или библиотеки, файлы шифруются, чтобы их можно было просматривать только авторизованным пользователям. Каждый управляемый правами файл также содержит лицензию на выдачу, который задает ограничения на пользователей, которые Просмотр файла. Типичные ограничения включать создания файла только для чтения, для отключения копирование текста, пользователей из сохранение локальной копии, а также запретить пользователям печать в файл. Клиентские программы, которые могут читать типы файлов, поддерживаемые IRM используйте лицензии на выдачу в управляемом правами файле для принудительного применения этих ограничений. Это как управляемый правами файл сохраняет его защиту даже после его загрузки с сервера.
  
Типы ограничений, применяемых в файл при загрузке из списка или библиотеки основаны на отдельных пользователей разрешения на сайте, содержащую файл. В следующей таблице описаны как соответствующие разрешения на веб-сайтах разрешения IRM.
  
|**Разрешения**|**Разрешения IRM**|
|:-----|:-----|
|Управление разрешениями, управление веб-сайта  <br/> |**Полный доступ** (как определено клиентской программы): обычно это разрешение позволяет пользователям для чтения, изменение, копирование, сохранить и изменение разрешений из управляемого правами содержимого.  <br/> |
|Изменение элементов, управление списками, добавление и настройка страниц  <br/> |**Изменение**, **Копировать**и **сохранять**: пользователь может распечатывать файл только в том случае, если установлен флажок **Разрешить пользователям печатать документы** на странице Параметры управления правами сведениями для списка или библиотеки.  <br/> |
|Просмотр элементов  <br/> |**Чтение**: пользователь может читать документ, но нельзя скопировать или изменить его содержимое. Пользователя можно печатать только в том случае, если установлен флажок **Разрешить пользователям печатать документы** на странице Параметры управления правами сведениями для списка или библиотеки.<br/> |
|Другое  <br/> |Никакие другие разрешения непосредственно соответствуют разрешения IRM.  <br/> |
   
Если включить IRM для списка или библиотеки в SharePoint Server 2013 только можно защищать типов файлов в этом списке или библиотеке, для которого система защиты установлена на всех интерфейсных веб-серверах. Система защиты — это программа, которая управляет шифрования и расшифровки управляемых правами файлов формата файла. SharePoint включает компоненты защиты для следующих типов файлов:
  
- Microsoft Office InfoPath forms
    
- Форматы файлов 97-2003 для следующих программ Microsoft Office: Word, Excel и PowerPoint
    
- Форматы Office Open XML для следующих программ Microsoft Office: Word, Excel и PowerPoint
    
- Формат XML Paper Specification (XPS)
    
Если ваша организация планирует использовать IRM для защиты других типов файлов, помимо перечисленных выше, администратор сервера необходимо установить компоненты защиты для этих дополнительных форматов файлов.
