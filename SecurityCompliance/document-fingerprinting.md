## <a name="how-document-fingerprinting-works"></a><span data-ttu-id="2edb7-121">Принцип работы отпечатков документов</span><span class="sxs-lookup"><span data-stu-id="2edb7-121">How Document Fingerprinting works</span></span>

<span data-ttu-id="2edb7-p105">Вы понимаете, вероятно, уже документы не фактические отпечатков, что имя поможет разобраться с компонента. Аналогичным образом, что отпечатки другого пользователя иметь уникальные шаблоны документы, имеющие уникальные word шаблоны. При отправке файла защиты от потери данных определяет шаблон уникальных word в документе, создает отпечаток документа, на основе этого шаблона и использует этот отпечаток документа для обнаружения исходящих документов, содержащих по той же схеме. Поэтому Отправка формы или шаблон создает наиболее подходящий тип отпечаток документа. Кто заполнении формы пользователем использует тот же набор исходного слова и затем добавляет свои собственные слова в документ. Поскольку исходящего документа не защищены паролем и содержит весь текст из исходной формы, защиты от потери данных можно определить, соответствует ли документ отпечаток документа.</span><span class="sxs-lookup"><span data-stu-id="2edb7-p105">You've probably already guessed that documents don't have actual fingerprints, but the name helps explain the feature. In the same way that a person's fingerprints have unique patterns, documents have unique word patterns. When you upload a file, DLP identifies the unique word pattern in the document, creates a document fingerprint based on that pattern, and uses that document fingerprint to detect outbound documents containing the same pattern. That's why uploading a form or template creates the most effective type of document fingerprint. Everyone who fills out a form uses the same original set of words and then adds his or her own words to the document. As long as the outbound document isn't password protected and contains all the text from the original form, DLP can determine if the document matches the document fingerprint.</span></span>
  
<span data-ttu-id="2edb7-128">В примере ниже показано, что происходит, если вы создаете отпечаток документа на основе шаблона патента, но вы можете использовать любую форму в качестве основы для создания отпечатка документа.</span><span class="sxs-lookup"><span data-stu-id="2edb7-128">The following example shows what happens if you create a document fingerprint based on a patent template, but you can use any form as a basis for creating a document fingerprint.</span></span>

<span data-ttu-id="2edb7-p106">Патента шаблон содержит пустые поля «Патента title», «Изобретателей» и «Описание» и описания для каждого из этих полей — это шаблон word. При отправке исходного шаблона патента — это один из поддерживаемых типов файлов и в обычный текст. Этот шаблон word в отпечатку документа для небольших XML Юникод файл, содержащий хэш-функции уникальное значение, представляющее исходного текста и отпечаток преобразует защиты от потери данных сохраняется в виде классификации данных в службе каталогов Active Directory. DLP converts this word pattern into a document fingerprint, which is a small Unicode XML file containing a unique hash value representing the original text, and the fingerprint is saved as a data classification in Active Directory. The patent fingerprint then becomes a sensitive information type that you can associate with a DLP policy. After you associate the fingerprint with a DLP policy, DLP detects any outbound emails containing documents that match the patent fingerprint and deals with them according to your organization's policy.</span></span> 

<span data-ttu-id="2edb7-p107">Например можно настроить политики защиты от потери данных, которая не позволяет регулярных сотрудников может отправлять исходящие сообщения, содержащие патенты. Кроме того может потребоваться разрешить вашего юридического отдела, чтобы иметь возможность отправлять патенты других организаций, так как она имеет необходима для этого.
