# <a name="document-fingerprinting"></a>Отпечатки документов

Информационные работники в вашей организации обработки множество типов конфиденциальной информации во время типовой день. В разделе Безопасность &amp; центре соответствия требованиям, отпечатков документов позволяет облегчить вам защитить эти сведения, определяя стандартных форм, используемых в вашей организации. В этом разделе описываются принципы отпечатков документов и как создать с помощью PowerShell.
  
## <a name="basic-scenario-for-document-fingerprinting"></a>Основной сценарий использования отпечатков документов

Отпечатки документов — возможность предотвращения потери данных (DLP), которая обеспечивает преобразование стандартной формы в тип конфиденциальных данных, который можно использовать в правилах политики защиты от потери данных. К примеру можно Создание отпечатка документа на основе шаблона пустой патента и создайте политики защиты от потери данных, которая обнаруживает и блокирует все шаблоны исходящей патента конфиденциальный контент заполнено. При необходимости можно настроить [Советы по политикам](use-notifications-and-policy-tips.md) для уведомления отправителей, что они могут отправляться конфиденциальной информации, и отправитель должен убедитесь, что получателей для получения патенты. Этот процесс работает с помощью любого текстового форм, используемых в вашей организации. Дополнительные примеры форм, которые можно загрузить. 
  
- Правительственные формы
    
- Формы соответствия требованиям обязательного медицинского страхования
    
- Формы сведений о сотрудниках для отделов кадров
    
- Настраиваемые формы, созданные специально для вашей организации
    
В идеальном варианте вашей организации уже имеет установленный business методика, с помощью некоторых форм для передачи конфиденциальных данных. После отправки пустая форма преобразуется в отпечаток документа и настройка соответствующую политику защиты от потери данных обнаруживает все документы в исходящей почты, соответствующие, отпечаток пальца.
  
## <a name="how-document-fingerprinting-works"></a>Принцип работы отпечатков документов

Вы понимаете, вероятно, уже документы не фактические отпечатков, что имя поможет разобраться с компонента. Аналогичным образом, что отпечатки другого пользователя иметь уникальные шаблоны документы, имеющие уникальные word шаблоны. При отправке файла защиты от потери данных определяет шаблон уникальных word в документе, создает отпечаток документа, на основе этого шаблона и использует этот отпечаток документа для обнаружения исходящих документов, содержащих по той же схеме. Поэтому Отправка формы или шаблон создает наиболее подходящий тип отпечаток документа. Кто заполнении формы пользователем использует тот же набор исходного слова и затем добавляет свои собственные слова в документ. Поскольку исходящего документа не защищены паролем и содержит весь текст из исходной формы, защиты от потери данных можно определить, соответствует ли документ отпечаток документа.
  
В примере ниже показано, что происходит, если вы создаете отпечаток документа на основе шаблона патента, но вы можете использовать любую форму в качестве основы для создания отпечатка документа.
  
**Пример патентного документа, соответствующего отпечатку документа для шаблона патента**

![Document_Fingerprinting_diagram.PNG](media/Document_Fingerprinting_diagram.png)
  
Патента шаблон содержит пустые поля «Патента title», «Изобретателей» и «Описание» и описания для каждого из этих полей — это шаблон word. При отправке исходного шаблона патента — это один из поддерживаемых типов файлов и в обычный текст. Этот шаблон word в отпечатку документа для небольших XML Юникод файл, содержащий хэш-функции уникальное значение, представляющее исходного текста и отпечаток преобразует защиты от потери данных сохраняется в виде классификации данных в службе каталогов Active Directory. (Из соображений безопасности сам исходный документ не хранятся в службе; сохраняется только значения хэш-функции и исходный документ не может быть восстанавливается из значения хэш-функции.) Патента отпечатка становится тип конфиденциальных сведений, которые можно связать с политикой защиты от потери данных. После отпечатка связать с политикой DLP, защиты от потери данных обнаруживает исходящих сообщений, которые содержат документы, соответствующие патента отпечатка и обрабатывает их в соответствии с политикой вашей организации. 

Например можно настроить политики защиты от потери данных, которая не позволяет регулярных сотрудников может отправлять исходящие сообщения, содержащие патенты. Защиты от потери данных будет использоваться патента отпечаток для обнаружения патенты и заблокировать сообщения электронной почты. Кроме того может потребоваться разрешить вашего юридического отдела, чтобы иметь возможность отправлять патенты других организаций, так как она имеет необходима для этого. Можно разрешить определенного подразделения для отправки конфиденциальной информации путем создания исключения для тех отделов в политике защиты от потери данных, или можно разрешить их переопределил подсказку политики с бизнес-требований.
  
### <a name="supported-file-types"></a>Поддерживаемые типы файлов

Отпечатки документов поддерживает же типы файлов, поддерживаемые в правилах транспорта. Список поддерживаемых типов файлов [Поддерживаемые типы файлов для проверки содержимого правил потока обработки почты](https://docs.microsoft.com/en-us/exchange/security-and-compliance/mail-flow-rules/inspect-message-attachments#supported-file-types-for-mail-flow-rule-content-inspection)см. Один небольшое примечание о типах файлов: ни правил транспорта ни отпечатков документов не поддерживают .dotx тип файла, который может быть некоторая путаница, поскольку именно в этот файл шаблона в Word. При появлении «шаблон» в этом и другие разделы, отпечатков документов word, она ссылается на документ, определенными в качестве стандартной формы не тип файла шаблона.
  
#### <a name="limitations-of-document-fingerprinting"></a>Ограничения относительно отпечатков документов

Отпечатки документов не определяет конфиденциальные сведения в следующих случаях:
  
- Файлы защищены паролем.
    
- Файлы содержат только изображения.
    
- Документы не содержат весь текст из исходной формы, которая использовалась для создания отпечатка документа.
    
## <a name="use-powershell-to-create-a-classification-rule-package-based-on-document-fingerprinting"></a>Создание пакета правил классификации на основе отпечатков документов с помощью PowerShell

Обратите внимание на то, что в настоящее время можно создать отпечаток документа только с помощью PowerShell в системы &amp; центре соответствия требованиям. Чтобы подключиться, обратитесь к разделу [подключение к Office 365 безопасность и соответствие требованиям центр PowerShell](https://docs.microsoft.com/en-us/powershell/exchange/office-365-scc/connect-to-scc-powershell/connect-to-scc-powershell).

Защиты от потери данных с помощью пакетов правил классификации для определения конфиденциального содержимого. Создание пакета правил классификации на основании отпечаток документа, используйте командлеты **New-DlpFingerprint** и **New-DlpSensitiveInformationType** . Так как результаты **New-DlpFingerprint** не сохраняются за пределами правила классификации данных, вы всегда выполняются **New-DlpFingerprint** и **New-DlpSensitiveInformationType** или **Set-DlpSensitiveInformationType** в том же Сеанс PowerShell. В следующем примере создается новый отпечаток документа на основе файла C:\My Documents\Contoso Employee Template.docx. Отпечаток нового хранилища как переменную, поэтому их можно использовать с помощью командлета **New-DlpSensitiveInformationType** в том же сеансе PowerShell. 
  
```
$Employee_Template = Get-Content "C:\My Documents\Contoso Employee Template.docx" -Encoding byte -ReadCount 0
$Employee_Fingerprint = New-DlpFingerprint -FileData $Employee_Template -Description "Contoso Employee Template"
```

Давайте создадим новое правило классификации данных с именем "Contoso Employee Confidential", использующее отпечаток документа из файла C:\My Documents\Contoso Customer Information Form.docx.
  
```
$Employee_Template = Get-Content "C:\My Documents\Contoso Customer Information Form.docx" -Encoding byte -ReadCount 0
$Customer_Fingerprint = New-DlpFingerprint -FileData $Customer_Form -Description "Contoso Customer Information Form"
New-DlpSensitiveInformationType -Name "Contoso Customer Confidential" -Fingerprints $Customer_Fingerprint -Description "Message contains Contoso customer information." 
```

Теперь можно использовать командлет **Get-DlpSensitiveInformationType** для поиска всех пакетов правил классификации данных защиты от потери данных, а в этом примере «Contoso Customer Confidential» входит в список пакетов правил классификации данных. 
  
Наконец, добавьте «Contoso Customer Confidential» пакет правил классификации данных в политику защиты от потери данных в системы &amp; центре соответствия требованиям. В этом примере добавляется правило существующую политику защиты от потери данных с именем «ConfidentialPolicy».

```
New-DlpComplianceRule -Name "ContosoConfidentialRule" -Policy "ConfidentialPolicy" -ContentContainsSensitiveInformation @{Name="Contoso Customer Confidential"} -BlockAccess $True
```

Также можно использовать пакет правил классификации данных в правилах транспорта в Exchange Online, как показано в следующем примере. Чтобы выполнить эту команду, сначала необходимо [подключение к Exchange Online PowerShell](https://docs.microsoft.com/en-us/powershell/exchange/exchange-online/connect-to-exchange-online-powershell/connect-to-exchange-online-powershell). Также Обратите внимание на то, что требуется времени для пакета правила для синхронизации с безопасности &amp; центре соответствия требованиям в центр администрирования Exchange.
  
```
New-TransportRule -Name "Notify :External Recipient Contoso confidential" -NotifySender NotifyOnly -Mode Enforce -SentToScope NotInOrganization -MessageContainsDataClassification @{Name=" Contoso Customer Confidential"}

```

Теперь защиты от потери данных обнаруживает документы, соответствующие отпечатку документа Contoso Customer Form.docx.
