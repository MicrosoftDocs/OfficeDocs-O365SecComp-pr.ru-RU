# <a name="configure-supervision-policies-for-your-organization"></a>Настройка политик контроля для организации

Используйте политики контроля для захвата коммуникации сотрудников для изучения внутренних или внешних рецензентов.
  
> [!NOTE]
> С помощью политики контроля требуется подписка на Office 365 E5 для вашей организации. Если нет этого плана и попытайтесь контроля, можно [подписаться на пробную версию Office 365 корпоративный E5](https://go.microsoft.com/fwlink/p/?LinkID=698279). 
  
Выполните следующие действия, чтобы настроить и использовать контроля в организации Office 365. 
  
- [Настройка групп для контроля](configure-supervision-policies.md#exampledist)
    
    Прежде чем начать, с помощью контроля, определение пользователей, которые будут их communications просмотра и пользователей, которые будут выполнять эти проверки. Если необходимо начать разработку только несколько пользователей, чтобы увидеть, как работает контроля можно пропустить настройку групп в данный момент.
    
- [Сделать доступными в вашей организации контроля](configure-supervision-policies.md#SRavailable)
    
    Добавьте себя в группу ролей контролирующий проверки, можно настроить политики. Любой пользователь, имеющий этой роли, назначенной можно получить доступ к странице **контроля** в области **Управления данными** в системы &amp; центре соответствия требованиям. 
    
- [Настройка политики контроля](configure-supervision-policies.md#setupsuper)
    
    Вы создадите политики контроля безопасности &amp; центре соответствия требованиям. Эти политики определить, какие подключения могут быть проверки в вашей организации и указывает, кто должен выполнять проверки. Коммуникации включают электронной почты, а также связи платформы сторонних производителей (например, Facebook, Twitter, и т.д.)
    
- [Использование Outlook web app для проверки связи, определяемую средством политика контроля](configure-supervision-policies.md#UseOutlook)
    
    Надстройка контроля предоставляет рецензентов доступ к функции контроля вправо в Outlook web app, чтобы оценить и распределения по категориям каждого элемента. Поддержка для настольных систем версии Outlook в ближайшее время.
    
- **Запуск отчета контроля**
    
    Использование отчетов о контроля для просмотра действий Обзор на уровне политики и редактор. Для каждой политики можно также просмотреть live статистики на текущее состояние проверки активности. Дополнительные сведения см [контроля отчеты](supervision-reports.md).
    
## <a name="set-up-groups-for-supervision"></a>Настройка групп для контроля
<a name="exampledist"> </a>

 При создании политики контроля необходимо решить, кто будет иметь связь анализа и пользователей, которые будут выполнять их проверки. В политику адресов электронной почты используется для идентификации отдельных пользователей или групп пользователей. Чтобы упростить настройку, создание группы для пользователей, которые будут иметь связь, их анализа и групп для пользователей, которые будут просмотрите эти связи. Если вы используете групп, может потребоваться несколько — например, если вы хотите отслеживать взаимодействия между двух различных групп пользователей или если вы хотите указать группу, которая не будет иметь управляет. Просмотрите [Пример группы рассылки](configure-supervision-policies.md#GroupExample) для получения дополнительных сведений о том, как это работает. 
  
Для контроля за коммуникации между группы или групп в вашей организации, настроить группы рассылки в центре администрирования Exchange (последовательно выберите пункты **Получатели** \> **группы**). Дополнительные сведения о настройке группы рассылки можно [Управление группами рассылки](http://go.microsoft.com/fwlink/?LinkId=613635)
  
> [!NOTE]
> Можно также использовать динамические группы рассылки или групп безопасности для контроля при необходимости. Чтобы помочь вам определить, если они лучше подходит вашей организации должен видеть [Управление группами безопасности с включенной поддержкой почты](http://go.microsoft.com/fwlink/?LinkId=627033)и [Управление динамические группы рассылки](http://go.microsoft.com/fwlink/?LinkId=627058). 
  
### <a name="example-distribution-groups"></a>Примеры групп рассылки
<a name="GroupExample"> </a>

В этом примере включает в себя группы рассылки, которые были настроены для финансовые организации вызове финансовый International Contoso. 
  
В Contoso Financial International необходимо отслеживать выборку сообщений между брокерами в США. Тем не менее лица, ответственные за соблюдение требований в этой группе, не требуют наблюдения. Для этого примера можно создать следующие группы:
  
|**Группа рассылки**|**Адрес группы (псевдоним)**|**Описание**|
|:-----|:-----|:-----|
|Все брокеры США  <br/> |US_Brokers@Contoso.com  <br/> |Эта группа содержит адреса электронной почты всех брокеров из США, которые работают в компании Contoso.  <br/> |
|Все ответственные за соответствие требованиям в США  <br/> |US_Compliance@Contoso.com  <br/> |Эта группа включает в себя адреса электронной почты для всех США контролеры, работающие для компании Contoso. Поскольку эта группа подмножество всех брокеры США, можно использовать этот псевдоним для освобождения контролеры из политики контроля.  <br/> |
   
В разделе [Настройка политика контроля](configure-supervision-policies.md#setupsuper) описывается, как можно использовать эти группы во время настройки политики. 
  
## <a name="make-supervision-available-in-your-organization"></a>Сделать доступными в вашей организации контроля
<a name="SRavailable"> </a>

Чтобы сделать **контроля** в меню параметров в безопасности &amp; центре соответствия требованиям, необходимо быть членом роли администратора контролирующий проверки. 
  
Для этого можно либо добавить себя как член группы ролей контролирующий Просмотр, или можно создать новую группу ролей.
  
### <a name="add-members-to-the-supervisory-review-role-group"></a>Добавление членов в группу ролей контролирующий проверки

1. Вход в [https://protection.office.com](https://protection.office.com) с использованием учетных данных для учетной записи администратора в организации Office 365. 
    
2. В разделе Безопасность &amp; центре соответствия требованиям, чтобы перейти на **разрешения**.
    
3. Выберите группу ролей **Контролирующий Review** и щелкните значок Правка. 
    
4. В разделе **Участники** добавьте пользователей, которые вы хотите управлять контроля для вашей организации. 
    
### <a name="create-a-new-role-group"></a>Создание новой группы ролей

1. Вход в [https://protection.office.com](https://protection.office.com) с использованием учетных данных для учетной записи администратора в организации Office 365. 
    
2. В разделе Безопасность &amp; центре соответствия требованиям, перейдите к **разрешениями** и нажмите кнопку Добавить ( **+**).
    
3. В разделе **ролей** нажмите кнопку Добавить ( **+**) и прокрутки вниз до **Контролирующий Review администратора**. Добавьте эту роль в группу ролей.
    
4. В разделе **Участники** добавьте пользователей, которые вы хотите управлять контроля для вашей организации. 
    
Дополнительные сведения о группах ролей и разрешений можно [разрешения безопасности Office 365 &amp; центре соответствия требованиям ](permissions-in-the-security-and-compliance-center.md).
  
## <a name="set-up-a-supervision-policy"></a>Настройка политики контроля
<a name="setupsuper"> </a>

> [!IMPORTANT]
> Перед созданием политика контроля, необходимо сначала удалить любые существующие политики контролирующий проверки. 
  
1. Вход в [https://protection.office.com](https://protection.office.com) с использованием учетных данных для учетной записи администратора в организации Office 365. 
    
2. В разделе Безопасность &amp; центре соответствия требованиям, чтобы перейти на нажмите кнопку **управления данными** \> **контроля**.
    
    > [!NOTE]
    > Предыдущие версии компонента может отображаться в левой панели навигации как **контролирующий Review (закончится ближайшее время)**. Эта версия будет вскоре устаревшие и удалены. Новая версия, называется **контроля** будет выполнено его. 
  
3. Нажмите кнопку **Создать** и затем следуйте указаниям мастера для настройки политики на следующих страницах. 
    
### <a name="policy-name-and-description"></a>Название и описание политики

Введите имя и описание политики. В качестве примера мы будем имя политики брокеры Contoso "мне Нравится".
  
### <a name="choose-users-to-supervise"></a>Выберите пользователей для контроля за

- В поле **Supervise эти пользователи или группы** выберите пользователей или группы, коммуникации, необходимо для контроля за. Использовать именно в нашем примере для брокеров Contoso "мне Нравится", мы будем Выбор группы US_Brokers@Contoso.com здесь. 
    
- При выборе группы для контроля за выбор членов группы, для которых отменено контроля можно использовать поле **исключение этих пользователей** . В примере мы будем исключение группы US_Compliance@Contoso.com здесь. 
    
### <a name="choose-communications-to-review"></a>Выберите коммуникаций для просмотра
<a name="CommsToReview"> </a>

По умолчанию условие **идет в направлении** отображается и не может быть удалена. Чтобы ограничить область действия дальнейшей проверки (например, только просмотр контента, который содержит определенные слова или фразы), нажмите кнопку **Добавить условие**. При необходимости можно выбрать несколько условий.
  
Условия, выбранного будут применяться связь из источников электронной почты и сторонних производителей в вашей организации (как из Facebook или общего банка данных). Для получения дополнительных сведений об импорте communications сторонних производителей в организации Office 365 отображаются [архивации сторонних производителей в Office 365](https://technet.microsoft.com/EN-US/library/mt621583.aspx).
  
В приведенной ниже таблице описаны дополнительные о каждом из условий.
  
|**Условие**|**Как использовать это условие**|
|:-----|:-----|
|Указание  <br/> |Выберите **Входящие** для просмотра передаваемых **для** людей, выбранное для контроля за **из** людей не включенный в политике.  <br/> Выберите **исходящего соединителя** , если вы хотите просмотреть коммуникаций, которые отправляются **из** людей, выбранный для контроля за ** для ** людей, не включенный в политике.  <br/> Выберите **Внутренний** Просмотр связям **между** людей, которых был определен в политике.  <br/> |
|Сообщение содержит любое из этих слов  <br/> Сообщение не включающего ни одно из этих слов  <br/> |Чтобы применить политику, когда определенные слова или фразы включаются или исключаются в сообщение, введите каждого слова или фразы в отдельной строке. Каждой строки слова, введенные будет применяться отдельно (только один из этих строк необходимо применить для политику, применяемую к сообщению). Дополнительные сведения о вводе слов или фраз, содержится в следующем разделе [соответствующие слова и фразы по электронной почте или вложений](configure-supervision-policies.md#Matchwords).<br/> |
|Вложение содержит любое из этих слов  <br/> Вложение не включающего ни одно из этих слов  <br/> |Чтобы применить политику, когда определенные слова или фразы включаются или исключаются во вложении сообщения (например, документ Word), введите каждого слова или фразы в отдельной строке. Каждой строки слова, введенные будет применяться отдельно (только одну строку необходимо применить политики для применения к вложения). Дополнительные сведения о вводе слов или фраз, содержится в следующем разделе [соответствующие слова и фразы по электронной почте или вложений](configure-supervision-policies.md#Matchwords).<br/> |
|Вложение представляет любой из этих типов файлов  <br/> Вложение представляет ни один из этих типов файлов  <br/> |Для контроля за коммуникаций, включение и исключение определенных типов вложений, введите расширения файлов (например, .exe или PDF-файл). Если вы хотите включить или исключить несколько расширений имен файлов, введите в отдельные строки. Расширение только один вложения должно соответствовать для применения политики.  <br/> |
|Размер сообщения превышает  <br/> Размер сообщения не превышает  <br/> |Просмотр сообщений в зависимости от определенного размера, используйте эти условия, чтобы задать минимальный и максимальный размер сообщения, по истечении которого зависит от проверки. Например, если указать **размер сообщения превышает** \> **1.0 МБ**, все сообщения, которые являются 1.01 МБ и размер которых будет подчиняться проверки. Можно выбрать в байтах, килобайт, мегабайт или гигабайт для этого условия.<br/> |
   
#### <a name="matching-words-and-phrases-to-emails-or-attachments"></a>Поиск слов и фраз в сообщениях или вложениях
<a name="Matchwords"> </a>

Каждой строки слова, введенные будет применяться отдельно (только одну строку необходимо применить для условия политики для применения к электронной почте или вложения). Например используем условие, **сообщение содержит любое из этих слов**, с ключевыми словами «банка» и «изнутри торговыми» на отдельных строках. Политика будет применяться к все сообщения, которые содержит слово «банка» или фразу «изнутри торговыми». Только один из этих слов или фраз, должны выполняться для этого условия политики для применения. Можно ввести слова в сообщении или вложения должен полностью совпадать.
  
#### <a name="entering-multiple-conditions"></a>Ввод нескольких условий
<a name="Matchwords"> </a>

При вводе нескольких условий, Office 365 все условия вместе используется для определения, когда следует применить политику к элементам обмена данными. При указании нескольких условий, они все необходимые для политики применить, если не был введен исключение. Например предположим, необходимые для создания политики, который следует применить, если сообщение содержит слово «торговые», а превышает 2 МБ. Тем не менее если сообщение также содержит слова «Утверждено по финансам Contoso», политика не должна применяться. Таким образом в этом случае этих трех условий может быть следующим образом: 
  
- **Сообщение содержит любое из этих слов**, с ключевыми словами «торговые»
    
- **Размер сообщения превышает**, на значение 2 МБ
    
- **Сообщение не включающего ни одно из этих слов**, с ключевыми словами «Утверждено группой по финансам Contoso».
    
### <a name="specify-percentage-to-review"></a>Укажите процент для просмотра
<a name="CommsToReview"> </a>

Если для сокращения объема содержимого для просмотра следует укажите процент. Случайным образом мы будем выберите, объем контента всего, который соответствует условиям, выбранное. Если вы хотите проверяющих для просмотра всех элементов, введите **100%**.
  
### <a name="choose-reviewers"></a>Выберите рецензентов
<a name="CommsToReview"> </a>

Пользователи и группы, выбранного будет использовать приложение контроля в Outlook web app для проверки связи, которые возвращаются для этой политики. Можно включить адреса электронной почты для внутренних или внешних рецензентов. 
  
### <a name="review-your-settings"></a>Проверьте параметры
<a name="CommsToReview"> </a>

После выполнения всех разделах политика контроля, проверьте параметры и нажмите кнопку **Готово** , чтобы сохранить политику. Выполнение может занять несколько часов для политики для запуска перехвата сообщений. Контроля предоставляет все данные для просмотра в общую папку, рецензентов можно получить доступ в Outlook web app. 
  
## <a name="use-outlook-web-app-to-review-communications-identified-by-a-supervision-policy"></a>Использование Outlook web app для проверки связи, определяемую средством политика контроля
<a name="UseOutlook"> </a>

Проверяющие будет использовать контроля надстройки для Outlook web app для просмотра коммуникаций. Надстройка устанавливается автоматически в Outlook web app для всех рецензентов, указанному в политике. Поддержка для настольных систем версии Outlook в ближайшее время.
  
 **Проверка связи в Outlook web app**
  
1. В Outlook web app, разверните узел **контроля - \<имя политики\> ** папки. 
    
2. В ** \<имя политики\> ** вложенной папке, рецензентов будут видеть все данные, определяемую средством, политика контроля. 
    
    ![Надстройка контроля в Outlook web app отображением выбранных подпапку политики контроля](media/eef329bf-2bd0-477e-a715-76ca19b3347f.jpg)
  
3. Откройте элемент, чтобы просмотреть и нажмите кнопку надстройки **контроля** . 
    
4. Используйте надстройку для классификации элемент как **соответствует**, **Не соответствует**, **Questionable** или **разрешено**. После классификации элемент перемещается в подпапку в разделе ** \<имя политики\> ** папки.
