# <a name="monitor-devices-in-microsoft-365-security"></a>Мониторинг устройств в Центре безопасности Microsoft 365

Обеспечьте безопасность и актуальность ваших устройств, а также возможность выявления потенциальных угроз в центре безопасности Microsoft 365.

## <a name="view-device-alerts"></a>Просмотр оповещений устройства

Получение обновленных предупреждений о действиях с нарушением безопасности и других угрозах на устройствах из пакета ATP для защитника Windows (доступно с помощью лицензии "по"). Центр безопасности Microsoft 365 имеет несколько карточек, позволяющих эффективно отслеживать эти оповещения на высоком уровне, в зависимости от предпочтительного рабочего процесса.

### <a name="monitor-high-impact-alerts"></a>Отслеживание оповещений о высокой степени влияния

Каждое оповещение для защитника Windows ATP имеет соответствующую степень серьезности, среднюю, низкую или информационную, которая указывает на потенциальное влияние на сеть, если оно оставлено необслуживаемым.  

Используйте карточку серьезности оповещений для **устройств** , чтобы сосредоточиться на более серьезных оповещениях и может требовать немедленный отклик. Из этой карточки вы можете просмотреть дополнительные сведения на портале Центра безопасности защитника Windows.

![Карточка серьезности оповещений устройств](./media/security-docs/device-alerts-severity.png)

### <a name="understand-sources-of-alerts"></a>Общие сведения об источниках оповещений

"Защитник Windows" использует данные из широкого диапазона датчиков безопасности и источников аналитики для создания оповещений. Например, он может использовать сведения об обнаружении антиВирусной программы "Защитник Windows" и стороннего антивредоносного по, а также собственную логику, предоставляемую через API веб-службы.

В карточке источников **обнаружения оповещений для устройств** показано распределение оповещений по источнику. С помощью этой карточки можно отслеживать действия, связанные с определенными источниками, в частности настраиваемые источники. Кроме того, вы можете сосредоточиться на оповещениях, поступающих от датчиков, которые не настроены для автоматической блокировки вредоносных действий или компонентов.

![Карточка источников обнаружения оповещений для устройств](./media/security-docs/device-alert-detection-sources.png)

Из этой карточки вы можете просмотреть дополнительные сведения на портале Центра безопасности защитника Windows.

### <a name="understand-the-types-of-threats-that-trigger-alerts"></a>Сведения о типах угроз, инициирующих оповещения

Пакет ATP для защитника Windows сортирует каждое оповещение в категории, представляющей определенный этап в цепочке атак, или тип компонента угрозы. Например, обнаруженные действия с угрозой могут быть отнесены к категории "Латерал Replace", чтобы указать, что действие приводило к попытке получить доступ к другим устройствам в сети и, возможно, произошло после того, как злоумышленник получит первоначальный фусолд. При обнаружении компонент для работы с угрозой может быть классифицирован в широком смысле как "Вредоносная" или как можно точнее, например "", "Перенос учетных данных" или других типов вредоносных или нежелательных программ.

В карточке **категории угроз для устройств** показано распределение оповещений по этим категориям. Эти сведения можно использовать для определения действий по угрозам, например попыток кражи учетных данных, которые могут значительно повлиять на появление попыток социального проектирования, например. Кроме того, вы можете использовать эту возможность для отслеживания потенциально необратимых угроз, таких как шантажистом.

![Карточка категорий угроз для устройств](./media/security-docs/device-threat-categories.png)

### <a name="monitor-active-alerts"></a>Отслеживание активных оповещений

Карточка **состояния оповещения для устройства** указывает количество оповещений, которые не были разрешены и могут потребовать внимания. Из этой карточки вы можете просмотреть дополнительные сведения на портале Центра безопасности защитника Windows.

![Карточка состояния оповещения для устройства](./media/security-docs/device-alert-status.png)

### <a name="monitor-classification-of-resolved-alerts"></a>Отслеживание классификации разрешенных оповещений

При разрешении оповещения о продолжении защитника Windows сотрудники отдела безопасности могут указать, было ли оповещение проверено следующим образом:

* Истинное оповещение, идентифицирующее реальные действия или компоненты угроз
* Ложное оповещение, которое неправильно определило нормальные действия

На карточке **классификация оповещенИй устройств** показано, были ли разрешенные оповещения классифицированы как "истина" или "ложные". Из этой карточки вы можете просмотреть дополнительные сведения на портале Центра безопасности защитника Windows.

Примечание. в некоторых случаях сведения об классификации недоступны для определенных оповещений.

![Карточка классификации оповещений для устройств](./media/security-docs/device-alert-classification.png)

### <a name="monitor-determination-of-resolved-alerts"></a>Отслеживание определения разрешенных оповещений

В дополнение к классификации того, является ли оповещение истинным или ложным во время разрешения, персонал по обеспечению безопасности может определить тип обычной или вредоносной активности, обнаруженной при проверке оповещения.

На карточке **определения оповещения для устройства** отображается определение, предоставленное для каждого оповещения, а именно:

* **Апт** — Расширенная постоянная угроза, указывающая на то, что обнаруженные действия или компоненты угроз являются частью сложного нарушения безопасности, разработанного для получения фусолд в сети, подверженной уязвимости  
* **ВредоносНая программа** — вредоносный файл или код
* **Персонал по безопасности** — обычные действия, выполняемые персоналом по обеспечению безопасности
* **Тестирование безопасности** — действия или компоненты, предназначенные для моделирования фактических угроз и ожидаемые для запуска датчиков безопасности и создания оповещений
* **Нежелательное программное обеспечение** — приложения и другое программное обеспечение, которые не считаются вредоносными, но в противном случае нарушают политику или допустимые стандарты использования
* **** Другие — любое другое определение, которое не относится к предоставленным типам

Из этой карточки вы можете просмотреть дополнительные сведения в центре безопасности защитника Windows.

![Карточка определения оповещения для устройства](./media/security-docs/device-alert-determination.png)

### <a name="understand-which-devices-are-at-risk"></a>Сведения о том, какие устройства подвержены риску

**Защита устройства** показывает уровень риска для устройств. Уровень риска зависит от таких факторов, как тип и серьезность оповещений на устройстве.

![Карта защиты устройства](./media/security-docs/device-protection.png)

## <a name="monitor-and-report-status-of-intune-managed-devices"></a>Отслеживание и отправка отчетов о состоянии устройств, управляемых Intune

Следующие мониторинг и отчеты содержат данные от устройств, зарегистрированных в Intune. Данные из незарегистрированных устройств не входят в состав. Просматривать эти карты могут только глобальные администраторы.

Данные устройства, зарегистрированные в Intune, включают:

* Соответствие устройства требованиям
* Устройства с активным вредоносным по
* Типы вредоносных программ на устройствах
* Вредоносные программы на устройствах
* Устройства с обнаружением вредоносных программ
* Пользователи с обнаружением вредоносных программ

## <a name="monitor-and-manage-asr-rule-deployment-and-detections"></a>Мониторинг развертывания и обнаружения правил ASR и управление ими

[Правила предотвращения снижения уязвимости (ASR)](https://docs.microsoft.com/en-us/windows/security/threat-protection/windows-defender-exploit-guard/attack-surface-reduction-exploit-guard) помогают предотвратить действия и приложения, которые обычно используются вредоносной программой для проникновения компьютеров. Эти правила определяют, когда и как могут запускаться исполняемые файлы. Например, можно запретить JavaScript или VBScript запускать загруженный исполняемый файл, блокировать вызовы Win32 API из макросов Office или блокировать процессы, которые запускаются с USB-дисков.

![Карточка о снижении уязвимости](./media/security-docs/attack-surface-reduction-rules.png)

В карточке **правила уменьшения уязвимой зоны** представлен обзор развертывания правил на всех устройствах.

В верхней панели карточки показано общее количество устройств, которые находятся в следующих режимах развертывания:

* **Режим блокировки** — устройства с как минимум одним правилом, настроенными для блокирования обнаруженных действий
* **Режим аудита** — устройства без правил, не настроенные для блокирования обнаруженных действий, но у которых есть по крайней мере один набор правил для аудита обнаруженных действий  
* **Выкл** . — устройства со всеми правилами ASR отключены.

В нижней части этой карточки показаны параметры по правилам для всех устройств. На каждой полосе отображается количество устройств, которые настроены на блокировку или обнаружение, а также полностью отключено правило.

### <a name="view-asr-detections"></a>Просмотр обнаруженных АВАРИЙных ASR

Чтобы просмотреть подробные сведения об обнаружении правил ASR в сети, выберите **Просмотр обнаружений** в карточке **правила уменьшения уязвимой зоны** . Откроется вкладка **обнаружения** на странице подробный отчет.

![Вкладка "обнаружения"](./media/security-docs/detections-tab.png)

На диаграмме, расположенной в верхней части страницы, показано обнаружение с обнаружением стеков по времени, которые были заблокированы или проверены. В таблице внизу приведен список самых последних обнаружений. Используйте следующие сведения в таблице, чтобы определить природу обнаружений:

* **Обнаруженный файл** — файл, обычно сценарий или документ, содержимое которого активировало вероятные действия по атакам
* **Rule (правило** ) — имя, описывающее действия, которые разрабатываются правилом для перехвата. Сведения о существующих правилах ASR
* **Исходное приложение** — приложение, которое загрузило или выполнило содержимое, вызывающее вероятные действия по атакам. Это может быть легальное приложение, например веб-браузер, приложение Office или системное средство, например PowerShell.
* **Издатель** — поставщик, который освободил исходное приложение

### <a name="review-device-asr-rule-settings"></a>Проверка параметров правил для устройства ASR

На странице отчета " **правила снижения уязвимости** " перейдите на вкладку **Конфигурация** , чтобы просмотреть параметры правил для отдельных устройств. Выберите устройство, чтобы получить подробные сведения о том, находится ли каждое правило в режиме блокировки, режиме аудита или полном отключении.

![Вкладка "Конфигурация"](./media/security-docs/configuration-tab.png)

Microsoft Intune предоставляет функции управления для правил ASR. Если вы хотите обновить параметры, выберите **начать работу** в разделе **Настройка устройств** на вкладке, чтобы открыть компонент "Управление устройствами в Intune".

### <a name="exclude-files-from-asr-rules"></a>Исключение файлов из правил ASR

За счет исключения файлов из обнаружений можно предотвратить нежелательные ложные срабатывания и более уверенно развернуть правила снижения уязвимости в режиме блокировки.

Несмотря на то, что исключения файлов для правил сокращения направлений атак управляются в Microsoft Intune, центр безопасности Microsoft 365 предоставляет средство анализа, которое поможет вам определить файлы, активируемые для обнаружения. Кроме того, он помогает собирать имена файлов, которые можно исключить из системы.

Чтобы начать анализ обнаружений и сбор файлов для исключения, перейдите на вкладку " **Добавление исключений** " на странице отчета " **правила уменьшения уязвимой зоны** ".

![Вкладка "Добавление исключений"](./media/security-docs/add-exclusions-tab.png)

В таблице перечислены все имена файлов, обнаруженные правилами сокращения для атак на уязвимую зону. После выбора файла или нескольких файлов вы можете узнать, как добавлять эти файлы к исключениям.

* Сокращение общего числа обнаружений
* Сокращение общего числа устройств, на которые влияет обнаружение

Чтобы получить список выбранных файлов с полными путями для исключения, выберите пункт **Получение путей исключения**.

Для получения дополнительных сведений об исключении и подробных инструкциях по их добавлению прочитайте статью [Устранение неполадок с правилами снижения уязвимости](https://docs.microsoft.com/en-us/windows/security/threat-protection/windows-defender-exploit-guard/troubleshoot-asr).
