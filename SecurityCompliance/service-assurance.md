# <a name="service-assurance-in-the-office-365-security-amp-compliance-center"></a>Службы Software assurance безопасности Office 365 &amp; центре соответствия требованиям

Использование службы Software assurance безопасности Office 365 &amp; центре соответствия требованиям на доступ к документам, которые описывают различные разделы, в том числе: 
  
- По обеспечению безопасности Microsoft для пользовательских данных, которые хранятся в Office 365. 
    
- Отчеты аудита независимой сторонних производителей Office 365. 
    
- Реализация и тестирования сведения о безопасности, конфиденциальности и элементов управления соответствием требованиям Office 365 использует для защиты данных. 
    
Можно также выяснить, как Office 365 заказчики могут соответствовать требованиям стандартов, законами и правилам отраслей, таких как:
  
-  Международной организации по стандартизации (ISO) 27001 и 27018 
    
- Акт 1996 года о передаче и защите данных учреждений здравоохранения (HIPAA)
    
- Федеральное риск и программе управления авторизации (FedRAMP)
    
## <a name="who-can-access-office-365-service-assurance-and-how"></a>Доступ к контролю службы Office 365 и как?

 **New клиентов и клиентов, оценка Microsoft online services** можно получить доступ к службе Software assurance, который входит в состав планы Office 365 для предприятий E3 и E5 (пробной версии и платной подписки). Если не использовать одну из следующих планов и попытайтесь Software assurance службы, можно [регистрации на пробную версию Office 365 корпоративный E5](https://go.microsoft.com/fwlink/p/?LinkID=698279). 
  
 **Существующие клиенты Office 365 для бизнеса** можно обращаться к службе Software assurance. Если вы глобальный администратор Office 365 (иногда называемую администратор компании) для вашей организации будет уже имеется доступ к службе Software assurance, и вы можете встроенный другим пользователям. Если вы не глобального администратора Office 365 для вашей организации, и требуется доступ к службе Software assurance, обратитесь к администратору добавить вас в группу ролей пользователей Software Assurance службы. 
  
 **Члены подписки O365** имеют доступ к в разделе службы Software assurance в центре защиты Office 365 по умолчанию, если службы доступом на основе ролей Software assurance не были реализованы в вашей подписки. Службы Software assurance предоставляет отчеты и документы, которые описывают корпорации Майкрософт по обеспечению безопасности для пользовательских данных, которые хранятся в Office 365. Он также содержит независимые аудита сторонних производителей отчетов об Office 365. 
  
> [!NOTE]
> Примечание: Если вашей компании производитель реализовал Service Assurance Roles-Based доступа в подписки Office 365 (который может быть, если ваша организация предоставляется доступ к настраиваемых отчетов) и необходим доступ к службе Software assurance, но он не включен в слева область безопасности &amp; центру соответствия требованиям, попросите администратора Office 365 можно добавить в группу ролей пользователей Software Assurance службы на странице " **разрешения** ". Для получения дополнительных сведений см Onboard службы Software assurance пользователи или группы. 
  
 **Службы на основе ролей Software assurance для доступа к подготовке пользователей** При реализации службы доступом на основе ролей Software assurance вашей компании можно предоставлять службы Software assurance доступ всем сотрудникам безопасность и соответствие требованиям включая сведения о безопасности, управление рисками, соответствия требованиям и аудита рабочих групп и персонала в рамках вашей организация. Дополнительные сведения см [Onboard службы Software assurance пользователи или группы,](service-assurance.md#addother).
  
Службы Software assurance доступна с помощью безопасности &amp; центре соответствия требованиям. Вот как получить для него.
  
1. Перейдите по ссылке [https://protection.office.com](https://protection.office.com).
    
2. Войдите в Office 365 с помощью учетной записи рабочего или школы. 
    
3. В левой области выберите **службы Software assurance**. Далее, [выберите отрасли и региональных параметров](service-assurance.md#Chooseyourindustryregional) и [Onboard службы Software assurance пользователи или группы,](service-assurance.md#addother).
    
    > [!NOTE]
    > Если необходим доступ к службе Software assurance, и он не включен в левой области безопасности &amp; центру соответствия требованиям, попросите администратора Office 365 можно добавить в группу ролей пользователей Software Assurance службы на странице " **разрешения** ". 
  
## <a name="choose-your-industry-and-regional-settings"></a>Выбор отрасли и региональных параметров
<a name="Chooseyourindustryregional"> </a>

При получении доступа к службе Software assurance в первый раз, первым этапом является настройка отрасли и региональных параметров. Эти параметры можно изменить в любое время. Настройка этих параметров позволяет Software assurance службы для предоставления контента, наиболее подходящих для вашей организации. Чтобы настроить параметры отрасли и области:
  
1. После выхода Software assurance службы выберите **Параметры** и регион и отрасли параметры на странице отображается, как показано на следующем снимке экрана. 
    
    ![Показана страница параметров центр защиты.](media/101716e8-9c0a-4839-a2c0-f6aacf64eb9d.png)
  
2. На странице " **Параметры** " выберите стрелку вниз рядом с полем **региона** и проверьте соответствующую регионов для вашей организации. 
    
3. Выберите стрелку вниз рядом с пунктом **отрасли** и проверьте соответствующую отраслях для вашей организации. 
    
4. После выбора областей и отраслях, выберите команду **Сохранить**.

## <a name="onboard-other-service-assurance-users-or-groups"></a>Встроенный другие службы Software assurance пользователей или групп
<a name="addother"> </a>

Для доступа к отчетам по умолчанию, доступные для всех подписок на облаке Майкрософт, не нужно внедрять модели службы Software Assurance доступом на основе ролей, и этот шаг можно пропустить. Тем не менее если ваша организация предоставляется доступ к настраиваемых отчетов затем добавьте другие пользователи или группы Software Assurance службы ролей. Чтобы добавить другие пользователи или группы:
  
1. В разделе Безопасность &amp; центре соответствия требованиям, выберите **разрешения** в области слева, как показано на следующем снимке экрана. 
    
    ![Безопасность &amp; меню разрешения соответствия требованиям](media/8b479ad4-5c91-4e4c-a1db-372d50526db6.png)
  
2. В правой области выберите **Пользователей Software Assurance службы**, а затем выберите **Изменить группу ролей**и выберите в разделе **элементы** выберите **Изменить** для добавления членов в роль пользователя Software Assurance службы, как показано на следующем снимке экрана. 
    
    ![Обслуживания пользователей Software Assurance - Добавление](media/c8b1978e-9b6e-4200-860a-a1704f8e2355.png)
  
3. В следующем диалоговом поиск и выбор отдельных пользователей или групп, которым требуется просматривать отчеты соответствия Software assurance службы и доверие ресурсы, а затем выберите **Добавить** для каждого выбранного сделать и нажмите кнопку **X** в правом верхнем углу области при закончено. 
    
    ![Обслуживания пользователей Software Assurance: Выбор пользователя](media/33f13ef6-1e10-4d49-af06-8e1ad5ffbfba.png)
  
4. Каждый пользователь или группа, добавлены к роли пользователя службы Software Assurance теперь поиск Software assurance службы и загрузка отчеты и другие документы в системы &amp; центре соответствия требованиям.
    
Вернуться к странице " **разрешения** " в любое время, чтобы добавить несколько пользователей или удалить существующие.
