# <a name="web-traffic-logs-and-data-sources-for-office-365-cloud-app-security"></a>Журналы веб-трафика и источники данных для Office 365 Cloud App Security

|:-----|:-----|:-----|:-----|

Большое число файлов журнала трафика web и источники данных можно использовать с Office 365 облачных приложений безопасности. Тем не менее файлам журнала веб-трафика необходимо включить определенные сведения и иметь формат определенным образом, чтобы они работали с отчетами обнаружения приложения безопасности Office 365 облаке приложения и панель мониторинга обнаружения облака. Используйте эту статью как справочное руководство по веб-трафика журналов и источников данных, которые вы будете использовать с Office 365 облачных приложений безопасности.
  
> [!NOTE]
> Необходимо быть глобальным администратором, администратор безопасности или чтения безопасности доступа к безопасности &amp; центре соответствия требованиям и безопасности Office 365 облаке приложения портала. Просмотреть [разрешения безопасности Office 365 &amp; центре соответствия требованиям](permissions-in-the-security-and-compliance-center.md). 
  
## <a name="web-traffic-log-requirements"></a>Требования к журнала трафика в Интернете

Используете данных использует безопасности облаке приложения Office 365 в вашей веб-трафика журналов помогут вам понять, какие приложения людей в организации. Дополнительные сведения, которые включены в файлы журнала, лучше контролировать, которые следует иметь в действие пользователя.
  
В следующей таблице приведены требования и атрибуты, которые необходимы для вашего веб-трафика журналов для правильной работы с Office 365 облачных приложений безопасности:
  
|**Атрибуты**|**Дополнительные требования **|
|:-----|:-----|
| Дата операции  <br/>  Исходный IP-адрес  <br/>  Источник пользователя (рекомендуется)  <br/>  Конечный IP-адрес  <br/>  URL-адрес конечного (рекомендуется: URL-адреса обеспечивают выше точность для облачных приложений обнаружения чем IP-адресов)  <br/>  Общий объем данных (рекомендуется)  <br/>  Объем загрузку и передачу данных (рекомендуется: предоставляет полезные сведения о об облаке особенности использования приложения)  <br/>  Действие, выполняемое (разрешенные или заблокированные)  <br/> | Источник данных для хранения файлов журналов должно поддерживаться.  <br/>  Формат, который использовать файлы журналов должен соответствовать стандартного формата. При загрузке файла приложение проверит это.  <br/>  События в журнале должно произойти не более 90 дней назад.  <br/>  Файл журнала должны включать сведения о исходящий трафик, можно проанализировать для сети.  <br/> |
   
Если атрибуты не включаются в журналы, которые загружаются, облачных приложений Office 365 безопасности не удается показать или анализа сведений для вас. Например стандартный формат Cisco ASA брандмауэра не включает объем загруженному байт для каждой операции, имя пользователя или конечный URL-адрес (только конечного IP). Так как эти сведения не находится в файлах журналов Cisco, безопасности Office 365 облаке приложения не будет включить при анализе вашей организации сетевого трафика.
  
> [!NOTE]
> Для некоторых видов брандмауэры необходимо задать уровень сведений для веб-трафика журналов для включения обязательные атрибуты. Например брандмауэры Cisco ASA необходимо иметь уровень сведения задайте значение 6. Убедитесь в том, чтобы убедиться, что брандмауэров установлено значение для доставки на правильные сведения в журналах трафика web.

## <a name="troubleshoot-errors-when-log-files-are-uploaded"></a>Устранение ошибок при передаче файлов журнала

После отправки файлов веб-трафика журналов Проверьте журнал управления, чтобы просмотреть все ошибки. При наличии ошибок, используйте сведения в следующей таблице для устранения этих ошибок.
  
|**Ошибка**|**Описание**|**Решение**|
|:-----|:-----|:-----|
|Неподдерживаемый тип файла  <br/> |Загрузить файл не является файлом журнала. Например, для файла изображения.  <br/> |Загрузка текста, zip или файл gzip, непосредственно были экспортированы из брандмауэр или прокси-сервера.  <br/> |
|Внутренняя ошибка  <br/> |Сбой внутреннего ресурсов.  <br/> |Нажмите кнопку **Повторить** для повторного запуска задачи.  <br/> |
|Формат журнала не соответствует  <br/> |Формат журнала, загруженный не соответствует формату ожидаемый журнала для этого источника данных.  <br/> |
Убедитесь в том, что журнал не поврежден. Сравнение и соответствует формату файла журнала в формате примера, отображаются на странице загрузки. |
|Транзакции являются более чем на 90 дней  <br/> |Все операции более 90 дней и поэтому того, игнорируются.  <br/> |Экспортируйте новый журнал с последними событиями и повторно отправить его.  <br/> |
|Нет транзакций в каталоге облачных приложений  <br/> |В журнале нет транзакций для любой распознанный облачных приложений не найдены.  <br/> |Убедитесь, что журнал содержит сведения о исходящий трафик.  <br/> |
|Тип не поддерживается журнала  <br/> |При выборе **источник данных = других (не поддерживается)**, не синтаксический анализ журнала. Оно отправляется на рассмотрение в группу [Безопасности приложения Microsoft Cloud](https://aka.ms/whatiscas) технические.<br/> |Технические групп [Безопасности Microsoft Cloud приложения](https://aka.ms/whatiscas) выполняется построение выделенного средство синтаксического анализа для каждого источника данных. Самые популярные источники данных уже поддерживаются. При загрузке источника данных не поддерживается просматривается и добавлено в список возможных анализаторы нового источника данных.<br/> При добавлении нового средства синтаксического анализа для компонента уведомление включен в заметки о выпуске Microsoft Cloud приложения безопасности.  <br/> |
