# <a name="differences-between-estimated-and-actual-ediscovery-search-results-in-office-365"></a>Различия между eDiscovery фактические результаты поиска в Office 365

Этот раздел предназначен для поиска, можно запустить с помощью одного из следующих средств обнаружения электронных данных корпорации Майкрософт:  <br/>  
- Содержимое поиска безопасности Office 365 &amp; центре соответствия требованиям  <br/>  
- Обнаружение электронных данных на месте в центре администрирования Exchange (EAC)  <br/>  
- Центр обнаружения электронных данных в SharePoint Online  <br/> 
   
При выполнении поиска методом электронного обнаружения, средство, которое вы используете возвращает приблизительное число элементов (и их общий размер), которые соответствуют условиям поиска. Например, при выполнении поиска в системы &amp; центре соответствия требованиям, примерный поиска, результаты отображаются в области сведений для выбранного поиска.
  
![Рассчитанные результаты, отображенные в области сведений выбранного запроса на поиск](media/74e4ce83-40be-41a9-b60f-5ad447e79fe4.png)
  
Это же оценку общий размер и число элементов, который отображается в обнаружении электронных данных средство экспорта при экспорте результаты на локальном компьютере и в отчете Сводка экспорта, загружаемый с результатами поиска.

Тем не менее как видно в предыдущем снимок экрана Экспорт сводном отчете, размера и количества результатов фактического поиска, которые фактически загружаются отличаются от размера и количества оценке результатов поиска. 
  
![Разница между приблизительными и скачанными результатами поиска](media/84aef318-230f-430d-9d9e-02f21342d364.png)
  
Ниже перечислены некоторые причины следующие отличия:
  
- **Оценка результатов способ** - оценку результатов поиска действительно являются, оценку (не фактическая количество и) элементов, которые соответствуют требованиям запросов поиска. Чтобы компилировать оценки элементов Exchange, список сообщений идентификаторы, соответствующие критериям поиска из базы данных Exchange по запросу средство обнаружения электронных данных, которое вы используете. Но при экспорте результатов поиска, повторно выполните поиск и фактических сообщения извлекаются из базы данных Exchange. Поэтому эти различия могут привести к из-за определение предполагаемое количество элементов и фактическое число элементов. 
    
- **Изменения, которые происходят с момента, когда результаты поиска оценка и экспорт** — если экспорт результатов поиска, поиск перезапущен для сбора, последних элементов в индексе поиска, соответствующие критериям поиска. Возможно, существуют дополнительные элементы были созданы, отправки или получения, соответствующие условиям поиска в интервал между при оценке результатов поиска были собраны и когда были экспортированы результатов поиска. Можно также, что элементы, которые не были в индексе поиска результаты поиска были предполагаемое больше не существует, так как были очищены из расположение содержимого, прежде чем экспортировать результаты поиска. Чтобы указать диапазон дат для поиска обнаружения электронных данных является одним из способов устранения этой проблемы. Другим способом является помещение удержания на размещения содержимого, чтобы элементы сохраняются и не могут быть удалены. 
    
- **Неиндексированных элементов** - элементов, которые являются неиндексированные для поиска может вызвать различия между фактические результаты. Например, электронного обнаружения на месте в Exchange и центра eDiscovery в SharePoint не содержат неиндексированные элементов (, не соответствующие условиям поиска) при выполнении поиска для оценки результатов поиска. Однако вы можете добавить неиндексированные элементов при экспорте результатов поиска. При включении неиндексированные элементов при экспорте результатов поиска может быть несколько элементов, которые требуется экспортировать. Это приведет к различие между примерный и экспортированного результаты поиска. 
    
    При использовании средства поиска контента в системы &amp; центре соответствия требованиям, у вас есть параметр, чтобы включить неиндексированные элементы в оценке поиска. Количество неиндексированные элементов, возвращенных в результате поиска отображается в области сведений вместе с другими оценке результатов поиска. Все элементы неиндексированные также будет входить в общий размер оценке результатов поиска. При экспорте результатов поиска, у вас есть возможность включать или не включать неиндексированные элементов. Как настроить эти параметры может привести к различия между предполагаемое и фактических результатов, загружаются. 
    
- **Экспорт результатов поиска контента, который включает в себя все расположения контента** - в случае поиска из всех расположений контента в вашей организации, а затем только неиндексированные элементы из расположения контента, которые содержат Экспорт результатов поиска будет экспортироваться элементы, соответствующие условиям поиска. Другими словами Если результаты поиска не найдены в почтовый ящик или сайта, затем все неиндексированные элементы в этом почтового ящика или сайтов не будут экспортировать. Тем не менее неиндексированные элементов из всех расположений контента (даже те, которые не содержат элементы, соответствующие поискового запроса) будут включены в оценке результатов поиска. 
    
    Кроме того Если поиска, который выполняется импорт результаты из конкретного расположения контента, затем неиндексированные элементов (, которые не исключены с условия поиска) из всех расположений содержимого, заданного в поиске будет экспортироваться. В этом случае предполагаемое количество неиндексированные элементов и количество фактически экспортированные неиндексированные элементов должны совпадать.
    
    Причина не Экспорт неиндексированные элементов из каждого местоположения в организации является, поскольку он может повысить вероятность ошибок экспорта и увеличить время, необходимое для экспорта результатов поиска.
    
- **Форматы исходных файлов Сравнение форматов экспортируемых файлов** - Exchange для элементов, примерный размер результатов поиска рассчитать с помощью необработанные размеров сообщений Exchange. Тем не менее сообщения электронной почты требуется экспортировать в PST-файл или в виде отдельных сообщений (которые являются в формате EML файлов). Обе эти используют параметры экспорта другой файл форматирование, необработанные сообщений Exchange, которые приводит к общего экспортированного размера файла, отличного от расчетный размер файла. 
    
- **Версии документов** — документов для SharePoint, нескольких версий документа не включаются в оценке результатов поиска. Однако вы можете включить всех версий документов при экспорте результаты поиска, которые приведет к увеличению фактическое число (и общий размер), экспортированного документов. 
    
- **Дедупликация** - Exchange для элементов, дедупликация сокращает число элементов, которые требуется экспортировать. У вас есть возможность отзыва дублирующиеся результатов поиска при экспорте их. Для сообщений, Exchange это означает, что экспортируется только один экземпляр сообщения, даже если этого сообщения могут находиться в нескольких почтовых ящиков. Оценке результатов поиска включают каждый экземпляр сообщения. Если выбран параметр дедупликация при экспорте результатов поиска, фактическое число элементов, которые требуется экспортировать может быть значительно меньше, чем предполагаемое количество элементов. 
    
    Другой что следует учитывать при выборе параметра дедупликация это, что требуется экспортировать все элементы Exchange в одном PST-файл и структуру папок из исходных почтовых ящиков не будет сохранен. Только что экспортированный PST-файл содержит элементы, электронной почты. Тем не менее отчет о результатах поиска содержит запись для каждого экспортированного сообщение, указывающее исходного почтового ящика, где находится сообщение. Это помогает определить все почтовые ящики, которые содержат дублирующиеся сообщения. Если не включить дедупликация, для каждого почтового ящика, включенные в результате поиска экспортируется в отдельном PST-файл. 
    
## <a name="exporting-unindexed-items-from-the-ediscovery-center-in-sharepoint-online"></a>Экспорт неиндексированные элементов из центра обнаружения электронных данных в SharePoint Online

В центре обнаружения электронных данных в SharePoint Online, вы можете включить неиндексированные контент (из Exchange и SharePoint) при экспорте результаты поиска обнаружения электронных данных. Для этого, выбрав параметр **Включить элементы, которые шифруются или нераспознанный формат** . Неиндексированных элементов (также называемая uncrawlable в SharePoint) являются элементы в Exchange и SharePoint, для какой-либо причине не были индексироваться для поиска. Неиндексированных элементов Exchange, перечислены в отчете **Ошибок Exchange** , предоставляемую при экспорте результатов поиска. Аналогично неиндексированные элементов SharePoint, перечислены в отчет **Ошибки индекса SharePoint** . При экспорте элементов неиндексированные их загрузки на папку с именем **Uncrawlable**. Неиндексированные Exchange элементы включаются в PST-файл; слишком загрузки каждого неиндексированные документа из SharePoint. Количество неиндексированные элементов (если они существуют), перечислены в отчет ошибки каждого индекса. Количество неиндексированные элементов в отчетах должна соответствовать количество неиндексированные элементов, которые загружаются. 
  
 **Каковы по следующим причинам, если количество экспортированных неиндексированные элементов не соответствует количество элементов в отчет об ошибках индекса?** Как объяснялось ранее существует возможность элементы, удаленные из Office 365, между время выполнения оценки поиска и времени, которые были экспортированы в результатах поиска. Аналогично расхождение возникает для неиндексированных элементов. Например индекс поиска может быть масштабирование дату, когда требуется экспортировать результаты поиска. Это означает, что неиндексированные элемента, который был экспортирован с результатами поиска может не отображаются в отчете ошибки индекса элемент не был индексируется во время результаты поиска были экспортированы. В результате более неиндексированные элементов, экспорт, чем перечисленные в отчет об ошибках индекса. Аналогично неиндексированные элемента в отчет об ошибках индекса может удалены из Office 365 до обновления индекса поиска. В результате меньше неиндексированные элементов, экспорт, чем перечисленные в отчет об ошибках индекса. 
  
> [!NOTE]
> Если не выбран параметр **Включить элементы, которые шифруются или нераспознанный формат** при экспорте результатов поиска или просто загрузить отчеты, отчеты об ошибках индекса, загруженные, но у них нет все записи. Это не значит, что индексирования ошибок не. Только что означает, что неиндексированные элементов не были включены в экспорта.
