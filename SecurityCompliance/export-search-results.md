# <a name="export-content-search-results-from-the-office-365-security--compliance-center"></a>Экспорт результатов поиска контента в безопасности Office 365 и центре соответствия требованиям

После успешного запуска поиска контента, можно экспортировать результаты поиска на локальном компьютере. При экспорте электронной почты результатов их загрузки на компьютер как PST-файлов. При экспорте содержимого из SharePoint и OneDrive для бизнеса сайтов, экспортируются копии собственный документов Office. Существуют дополнительные документы и отчеты, которые включены в результаты поиска экспортированного.
  
Кроме того любые шифрования RMS сообщений, которые включаются в результаты поиска контента будут расшифрованы, при экспорте их (как отдельные сообщения). Эта возможность расшифровки включен по умолчанию для членов группы обнаружения электронных данных диспетчера ролей. Это роль управления расшифровки RMS назначается этой группе ролей. В разделе [Дополнительные сведения](#more-information) для получения дополнительных сведений о расшифровки RMS при экспорте результатов поиска. 
  
Экспорт результатов поиска контента включает в себя Подготовка результатов, а затем загрузкой на локальном компьютере.
  
## <a name="before-you-begin"></a>Перед началом работы

- Чтобы экспортировать результаты поиска, необходимо назначить роль управления экспорта безопасности Office 365 &amp; центре соответствия требованиям. Эта роль назначается группу встроенных обнаружения электронных данных диспетчера ролей. Он не будет назначен по умолчанию в группу ролей управления организацией. Дополнительные сведения можно [назначать разрешения обнаружения электронных данных в Office 365 безопасность &amp; центре соответствия требованиям](assign-ediscovery-permissions.md).
    
- Компьютер, который вы используете для экспорта результатов поиска, должен соответствовать указанным ниже требованиям к системе.
    
  - 32- или 64-разрядный выпуск Windows 7 и более поздней версии.
    
  - Microsoft .NET Framework 4.7
    
  - Поддерживаемый браузер:
    
     - Microsoft Edge
    
        ИЛИ
    
     - Microsoft Internet Explorer 10 и более поздних версий
    
    **Примечание:** Корпорация Майкрософт не производит расширения сторонних производителей или надстройки для приложения ClickOnce. Экспорт результатов поиска, используете неподдерживаемый браузер с расширениями сторонних производителей или надстройки не поддерживается. 
    
- При загрузке результатов поиска (описано в шаге 2) может увеличить скорость загрузки, настроив параметр реестра Windows на компьютере, используемом для экспорта результатов поиска. Дополнительные сведения можно [повысить скорость загрузки при экспорте результаты поиска обнаружения электронных данных в Office 365](increase-download-speeds-when-exporting-ediscovery-results.md).
    
- При экспорте результатов поиска данных временно хранится в уникальное место хранения Microsoft Azure в облаке Майкрософт перед загрузкой на локальный компьютер. Убедитесь, что ваша организация может подключаться к конечной точке в Azure, который является ** \*. blob.core.windows.net** (подстановочный знак представляет уникальный идентификатор для экспорта). Результаты поиска удален из места хранения данных Azure две недели после его создания. 
    
- Если в организации используется прокси-сервер для связи с Интернетом, необходимо определить параметры прокси-сервера на компьютере, которая используется для экспорта результатов поиска (чтобы проверка подлинности средство экспорта может быть прокси-сервера). Для этого откройте файл *machine.config* в папке, которая соответствует вашей версии Windows. 
    
  - **32-разрядная версия** - `%windir%\Microsoft.NET\Framework\[version]\Config\machine.config`
    
  - **64-разрядная версия** - `%windir%\Microsoft.NET\Framework64\[version]\Config\machine.config`
    
    Добавьте следующие строки в файл *machine.config* , где-нибудь между `<configuration>` и `</configuration>` тегов. Убедитесь, что для замены `ProxyServer` и `Port` с правильными значениями для вашей организации; например `proxy01.contoso.com:80` . 
    
    ```
    <system.net>
       <defaultProxy enabled="true" useDefaultCredentials="true">
         <proxy proxyaddress="http://ProxyServer :Port " 
                usesystemdefault="False" 
                bypassonlocal="True" 
                autoDetect="False" />
       </defaultProxy>
    </system.net>
    ```

- В разделе Описание ограничений для экспорта результатов поиска. 
    
- Максимальный размер в PST-файл, который можно экспортировать, составляет 10 ГБ. Если вы хотите изменить этот размер по умолчанию, можно изменить реестра Windows на компьютере, которая используется для экспорта результатов поиска. В разделе [Изменение размера PST-файлов при экспорте результаты поиска обнаружения электронных данных](change-the-size-of-pst-files-when-exporting-results.md).
    
## <a name="step-1-prepare-search-results-for-export"></a>Шаг 1. Подготовка результатов поиска к экспорту

Первый шаг состоит в подготовке результатов поиска для экспорта. При подготовке результаты, их можно было выгрузить в Azure хранилище в облаке Майкрософт. Обратите внимание, что содержимое из почтовых ящиков и сайтов загруженные с максимальной скоростью 2 ГБ в час.
  
1. Перейдите по ссылке [https://protection.office.com](https://protection.office.com).
    
2. Войдите в Office 365 с помощью своей рабочей или учебной учетной записи.
    
3. В расположенной слева области Центра безопасности и соответствия требованиям щелкните **Поиск и исследования** \> **Поиск контента**.
    
4. На странице " **Поиск контента** " выберите поиск. 
    
5. В области сведений в разделе **Экспорт результатов на компьютер** щелкните **Начать экспорт**.
    
    > [!NOTE]
    > Если результаты поиска сформированы более 7 дней назад, вам будет предложено обновить их. В этом случае отмените экспорт, в области сведений щелкните **Обновить результаты поиска** для выбранного поиска. После обновления результатов еще раз запустите экспорт.  
  
6. На странице " **Экспорт результатов поиска** " в разделе **Включить эти элементы из поиска**выберите один из следующих вариантов:
    
    - Экспортировать только индексированные элементы.
    
    - Экспорт индексированных и частично индексированных элементов
    
    - Экспорт только частично индексированных элементов
    
    В разделе [Дополнительные сведения](#more-information) для описания о том, как частично индексированных элементов требуется экспортировать. Дополнительные сведения о частично индексированных элементов можно [частично индексированных элементов в поиска контента](partially-indexed-items-in-content-search.md).
    
7. В разделе **Exchange экспортировать содержимое в виде**выберите один из следующих вариантов:
    
    - **Один PST-файл для каждого почтового ящика** - экспортирует один PST-файл для каждого почтового ящика пользователя, содержащий результаты поиска. В одной PST-файл включены все результаты из архивного почтового ящика пользователя. Обратите внимание, что этот параметр воспроизводит структуру папок почтового ящика из исходного почтового ящика. 
    
    - **Один PST-файл, содержащий все сообщения** - экспортируется в одном PST-файл (с именем *Exchange.pst* ), содержащий результаты поиска из всех исходных почтовых ящиков, включенных в поиск. Обратите внимание, что этот параметр воспроизводит структуру папок почтового ящика для каждого сообщения. 
    
    - **Один PST-файл, содержащий все сообщения в одной папке** - Экспорт результатов поиска одного PST-файл, где все сообщения, находятся в одной папке с верхнего уровня. Этот параметр позволяет рецензентов Просмотр объектов в том порядке, (элементы сортируются по Дата отправки) без необходимости перехода по структуре папок исходного почтового ящика для каждого элемента. 
    
    - **Отдельные сообщения** - Экспорт результатов поиска как отдельные сообщения электронной почты, используя формат MSG-файл. Если выбран этот параметр, результаты поиска по электронной почте, экспортированные в папку в файловой системе. Путь к папке для отдельных сообщений — это то же, что используется при экспорте результаты в PST-файлов. 
    
      > [!IMPORTANT]
      > Для расшифровки сообщений, зашифрованных службы управления правами, когда они в случае экспортированные, необходимо экспортировать результаты поиска электронной почты как отдельные сообщения. Зашифрованные сообщения останутся зашифрованные экспорта результатов поиска в PST-файл. 
  
8. Установите флажок **Включить дедупликация** для исключения повторяющихся сообщений. Этот параметр доступен только в том случае, если источники контента поиска включает в себя почтовые ящики Exchange или общих папок. 
    
    Если выбран этот параметр, только одна копия сообщения будет экспортироваться даже в том случае, если несколько копий одного сообщения находятся в почтовых ящиках поиска. Отчет о результатах экспорта (Results.csv) будет содержать строку для каждой копии повторяющихся сообщений, таким образом, можно определить почтовых ящиков (или общие папки), которые содержат копии повторяющихся сообщений. Для получения дополнительных сведений о дедупликация и как повторяющихся элементов определяются, см [дедупликация в результатах поиска обнаружения электронных данных](de-duplication-in-ediscovery-search-results.md).
    
9. Установите флажок **Включить версий документов SharePoint** для экспорта всех версий документов SharePoint. Этот параметр доступен только в том случае, если источники контента поиска включает в себя SharePoint или OneDrive для бизнеса сайтов. 
    
10. Установите флажок **Экспорт файлов в сжатых ZIP-папку** для экспорта результатов поиска в сжатых папок. Этот параметр доступен только в том случае, когда как отдельные сообщения при экспорте элементов Exchange и SharePoint или OneDrive документы включать в результаты поиска. Этот параметр в основном используется для обхода 260 знаков в именах файлов путь Windows при экспорте элементов. В разделе «Имена файлов экспортируемые элементы» в разделе [Дополнительные сведения](#more-information) . 
    
11. Нажмите кнопку **Начать экспорт**.
    
    Результаты поиска подготавливаются для загрузки, что означает, что вы не были отправлены расположение хранилища Azure в облаке Майкрософт. Если результаты поиска будут готовы для загрузки, ссылка **загрузки экспортировать результаты** отображается в списке **экспортировать результаты на компьютер** в области сведений. 
  
## <a name="step-2-download-the-search-results"></a>Шаг 2. Скачивание результатов поиска

Следующим шагом является можно загрузить результаты поиска из расположения хранилища Azure для локального компьютера.
  
Как объяснялось ранее можно увеличить скорость загрузки, настроив параметр реестра Windows на компьютере, используемом для экспорта результатов поиска. Дополнительные сведения можно [повысить скорость загрузки при экспорте результаты поиска обнаружения электронных данных в Office 365](increase-download-speeds-when-exporting-ediscovery-results.md).
  
1. В области сведений для поиска, для которого вы запустили операцию экспорта, в разделе **Экспорт результатов на компьютер** щелкните **Скачать экспортированные результаты**.
    
    Окно **загрузки экспортировать результаты** отображаются, содержит следующие сведения о результатах поиска, которые будут загружаться на своем компьютере. 
    
    - Количество скачиваемых элементов.
    
    - Предполагаемый общий размер скачиваемых элементов.
    
    - Сведения о том, какие элементы будут экспортированы, индексированные или неиндексированные. Неиндексированные элементы — это элементы, которые имеют распознаваемый формат, зашифрованы или не проиндексированы по другим причинам. Дополнительные сведения см. в статье [Unindexed items in Content Search](partially-indexed-items-in-content-search.md).
    
    - Сведения о том, будут ли скачаны версии документов SharePoint.
    
    - Сведения о состоянии процесса подготовки экспорта. Вы можете начать скачивать результаты поиска даже тогда, когда подготовка данных еще не завершена.
    
2. В разделе **Ключ экспорта** щелкните **Скопировать в буфер обмена**. Вы используете этот ключ для скачивания результатов поиска в действии 5.
    
    > [!NOTE]
    > Так как любой пользователь может установить и запустить средство экспорта службы обнаружения электронных данных, а затем использовать этот ключ для скачивания результатов поиска, то для защиты ключа необходимо предпринять такие же меры, как и для защиты паролей и другой информации, связанной с обеспечением безопасности.  
  
3. Нажмите кнопку **Скачать результаты**.
    
4. Если вам будет предложено установить **eDiscovery MicrosoftOffice 365 средство экспорта**, нажмите кнопку **установить**.
    
5. В **средстве экспорта службы обнаружения электронных данных** в соответствующее поле вставьте ключ экспорта, который вы скопировали в действии 2. 
    
6. Нажмите кнопку **Обзор** и укажите расположение, в которое вы хотите скачать файлы результатов поиска. 
    
    > [!NOTE]
    > Из-за высокой жесткому диску (чтение и запись) необходимо загрузить результаты поиска на локальный жесткий диск; не загружать их для сетевого диска или другое сетевое расположение. 
  
1. Нажмите кнопку **Запустить**, чтобы скачать результаты поиска на свой компьютер. 
    
    **Обнаружение электронных данных средство экспорта** сведения о состоянии процесса экспорта, включая оценку числа (и размера) оставшихся элементов веб-сайта. После завершения экспорта имели доступ к файлам в месте, где они были загружены.

 ### <a name="export-limits"></a>Ограничения для экспорта
  
- Экспорт результатов поиска из безопасности &amp; центре соответствия требованиям имеет следующие ограничения:
    
  - Не более 2 ТБ данных можно экспортировать из одного поиска контента. Если результаты поиска, размер которых превышает 2 ТБ, рассмотрите возможность использования диапазонов или других типов фильтров для сокращения общий размер результатов поиска.
    
  - Вашей организации можно экспортировать не более 2 ТБ данных в течение одного дня.
    
  - В организации может выполняться не более 10 операций экспорта одновременно.
    
  - В то же время одного пользователя можно запустить не более трех экспорта.
    
  - Экспорт отчетов поиска контента не учитывается какие-либо ограничения экспорта. 
    
- Как уже указано, результаты поиска из почтовых ящиков и веб-узлы можно было выгрузить в расположение хранилища Azure (как описано в статье [Шаг 1: Подготовка результаты для экспорта поиска](#step-1-prepare-search-results-for-export)) с максимальной скоростью 2 ГБ в час.
    
- Максимальный размер, который можно экспортировать в PST-файл по умолчанию — 10 ГБ. Это означает, размер которых превышает 10 ГБ результатов поиска из почтового ящика пользователя, результаты поиска для почтового ящика будет экспортироваться в двух (или более) отдельных PST-файлов. Кроме того Если выбран для экспорта всех результатов поиска в одном PST-файл, PST-файл будет иметь разделения дополнительных в дополнительных PST-файлы Если общий размер результатов поиска превышает 10 ГБ. Если вы хотите изменить этот размер по умолчанию, можно изменить реестра Windows на компьютере, которая используется для экспорта результатов поиска. В разделе [Изменение размера PST-файлов при экспорте результаты поиска обнаружения электронных данных](change-the-size-of-pst-files-when-exporting-results.md).
    
    Кроме того результаты поиска из определенного почтового ящика не распределить несколько PST-файлов Если содержимого из одного почтового ящика не более 10 ГБ. Если вы хотите экспортировать результаты поиска в один PST файл, который содержит все сообщения в одной папке и результаты поиска, размер которых превышает 10 ГБ, элементы по-прежнему организованы в том порядке, поэтому они будут разделения дополнительных в дополнительных PST-файлы на основании отправленной d ate.
     
 ### <a name="export-reports"></a>Экспорт отчетов
  
- При экспортировать результаты поиска, следующие отчеты будут включены в дополнение к результатов поиска.
    
  - **Экспорт Сводка** Документ Excel, который содержит сводку экспорта. Это включает данные, например число источников контента, поиска, размеры примерный и загруженные из результатов поиска и оценка и загруженного число элементов, которые были экспортированы. 
    
  - **Манифест** Файл манифеста (в формате XML), который содержит сведения о каждом элементе, включенные в результатах поиска. 
    
  - **Результаты** Документ Excel, который содержит сведения о каждого элемента, который можно загрузить в качестве результата поиска. Для электронной почты, журнал результатов содержит сведения о каждом сообщении, включая: 
    
      - Расположение сообщения в исходном почтовом ящике (включая сведения о том, в каком почтовом ящике находится сообщение, в основном или в архивном).
        
      - Дата отправки или получения сообщения.
        
      - Тема сообщения.
        
      - Отправитель и получатели сообщения.
        
      - Сообщение является ли повторяющихся сообщения Если этот параметр включен параметр дедупликация при экспорте результатов поиска. Дублирующиеся сообщения будут иметь значение в столбце **повторяющихся элементов** , определяющий сообщение как дубликат. Значение в столбце **дублирующиеся элемента** содержит идентификатор элемента сообщения, который был экспортирован. Для получения дополнительных сведений см [дедупликация в результатах поиска обнаружения электронных данных](de-duplication-in-ediscovery-search-results.md).
        
      Для документов из SharePoint и OneDrive для бизнеса сайтов журнал результатов содержит сведения о каждого документа, включая:
        
      - URL-адрес документа.
        
      - URL-адрес семейства веб-сайтов, в котором расположен документ.
        
      - Дата последнего изменения документа.
        
      - Имя документа (которое указано в столбце "Тема" журнала результатов).
    
  - **Неиндексированных элементов** Документ Excel, который содержит сведения о любой частично индексированных элементов, которые должны быть включены в результатах поиска. Если не включить частично индексированных элементов при создании отчета результаты поиска, в этом отчете будет загружен, но будет пустым. 
    
  - **Ошибки и предупреждения** Содержит ошибки и предупреждения для файлов во время экспорта. В разделе столбец сведений об ошибках для сведения, относящиеся к каждой отдельных сообщение об ошибке или предупреждение. 
    
  - **Пропущено элементов** После экспорта результатов поиска из SharePoint и OneDrive для бизнеса сайтов экспорта обычно включает пропущенные элементы отчета (SkippedItems.csv). Элементов, указанных в этом отчете обычно являются элементы, которые не будут загружать, например, папки или документа задать. Экспортирует данные в этом типов элементов — это предусмотрено при проектировании. Для других элементов, которые были пропущены поле «Тип ошибки» и «Сведения об ошибке» в отчете о пропущенных элементов показывают причине элемента была пропущена, а не был файл для загрузки с результатами поиска. 
    
  - **Журнал трассировки** Содержит подробное ведение журнала сведения о процессе экспорта и может помочь Открывание проблемы во время экспорта. 
    
    > [!NOTE]
    > Только что можно экспортировать в этих документах без необходимости экспортировать результаты фактического поиска. В разделе [Экспорт отчетов поиска контента](export-a-content-search-report.md). 
  
 ### <a name="exporting-partially-indexed-items"></a>Экспорт частично индексированных элементов
  
- При экспорте элементов почтового ящика из поиска контента, которое возвращает все элементы почтового ящика в результатах поиска (так как ключевые слова не там, где включены в запрос поиска), частично индексированных элементов не копируются PST-файл, содержащий неиндексированные элементов. Это, так как все элементы, включая любые частично индексированных элементов, автоматически включается в результаты поиска регулярного. Это означает, что частично индексированных элементов будут включены в PST-файл (или как отдельные сообщения), который содержит другие, индексированных элементов.
    
    Кроме того при экспорте индексированных и частично индексированных элементов или экспорт только индексированных элементов из поиска контента, которое возвращает все элементы, будет загружено такое же число элементов. Это происходит, несмотря на то, что результаты примерный поиска для поиска контента (отображаемых в статистики поиска по безопасности &amp; центре соответствия требованиям) по-прежнему будет включать отдельные оценки числа частично индексированных элементов. Например предположим, что оценки для поиска, включающее все элементы (без ключевых слов в запросе поиска) отображается обнаружения 1 000 элементов, а также найдены 200 частично индексированных элементов. В этом случае 1 000 элементов включают частично индексированных элементов, так как поиск возвращает все элементы. Другими словами существует 1000 общее элементы, возвращенные операцией поиска и не 1200 элементов (как можно было бы ожидать). Если Экспорт результатов поиска и выбрать для экспорта индексируются и частично индексированных элементов (или просто индексированных элементов), а затем будет загружен 1 000 элементов. Опять же это, поскольку частично индексированных элементов включены в регулярных (индексированные) результаты при использовании пустой поисковый запрос для возврата всех элементов. В этом примере же при выборе экспорт только частично индексированных элементов, затем только 200 неиндексированные элементы будет загрузить.
    
    Также Обратите внимание на то, что в предыдущем примере (при экспорте индексированных и частично индексированных элементов или экспорт только индексированных элементов), **Экспортировать сводном** отчете, включенные в результатах поиска экспортированного бы 1 000 элементов примерный элементы списка и 1 000 для загрузки элементы по же причинам, как описано выше. 
    
- Если поиска, который выполняется импорт результаты из поиска определенные расположения содержимого или все расположения контента в вашей организации, будут экспортироваться только частично элементы из расположения контента, которые содержат элементы, соответствующие условиям поиска. Другими словами Если результаты поиска не найдены в почтовый ящик или сайта, затем какие-либо частично индексированных элементов в этот почтовый ящик или сайтов не будут экспортированы. Причина этого —, что экспорт частично индексированных элементов из большое количество расположений в организации может повысить вероятность ошибок экспорта и увеличить время, необходимое для экспорта результатов поиска.
    
    Чтобы экспортировать частично индексированных элементов из всех расположений контента для поиска, Настройка поиска для возвращения всех элементов (путем удаления все ключевые слова из поискового запроса) и экспорта только частично индексированных элементов при экспорте результатов поиска.
    
    ![Используйте параметр экспорта thrid для экспорта только неиндексированных элементов](media/5d7be338-a0e5-425f-8ba5-92769c24bf75.png)
  
- При экспорте результатов поиска из SharePoint или OneDrive для бизнеса сайтов, возможность экспорта неиндексированные элементов также зависит от выбранного параметра экспорта и содержит ли сайт, который был выполнен поиск индексированный элемент, который соответствует условиям поиска. Например если поиск определенных SharePoint или OneDrive для бизнеса сайтов и не найдены результаты поиска, затем неиндексированные элементов из этих сайтов не будут экспортированы при выборе второго параметра экспорта для экспорта элементов индексированных и неиндексированные. Индексированный элемент из сайта соответствующие условиям поиска, все неиндексированные элементы из этого сайта будут экспортироваться, при экспорте индексированных и неиндексированные элементов. Ниже описываются параметры экспорта, в зависимости от того, является ли сайт содержит индексированный элемент, который соответствует условиям поиска.
    
    ![Выберите параметр экспорта на основании того, является ли сайт содержит индексированный элемент, который соответствует условиям поиска](media/94f78786-c6bb-42fb-96b3-7ea3998bcd39.png)

    
    A - только индексированный элемент, соответствующий критерию поиска требуется экспортировать. Частично индексированных элементов не требуется экспортировать.
    
    B — Если не индексированных элементов на сайте условиям поиска, не экспортировать частично индексированных элементов из того же сайта. Если индексированных элементов на сайте возвращаются в результатах поиска, экспортируются частично индексированных элементов из этого сайта. Другими словами экспортируются только частично индексированных элементов из сайтов, которые содержат элементы, соответствующие условиям поиска.
    
    C - экспортируются всех частично индексированных элементов со всех сайтов в поиск, независимо от того, является ли сайт содержит элементы, соответствующие условиям поиска.
    
    При выборе для экспорта частично индексированных элементов в отдельном файле PST-файлов, независимо от того, параметр, выберите в разделе **содержимое Экспорт Exchange как**экспорте элементов частично индексированных почтового ящика.

- Если частично индексированных элементов, возвращенных в результате поиска результаты (так как другие свойства частично индексированных элементов, отвечающих условиям поиска), а затем эти частично индексированных экспортируются с результатами поиска регулярного. Таким образом Если выбран для экспорта индексированных элементов и частично индексированных элементов (путем выбора параметра экспорта **всех элементов, включая те, которые имеют нераспознанный формат шифруются, или не были индексированы по другой причине** ), частично индексированных элементов экспорта с регулярным результаты будут отображены в отчете по Results.csv. Они не отображаются в отчете о неиндексированные items.csv.
    
 ### <a name="exporting-individual-messages-or-pst-files"></a>Экспорт отдельных сообщений или PST-файлов
  
- Если имя файла и путь сообщения превышает максимально допустимое число знаков для Windows, усекается путь к файлу. Однако исходное имя файла и путь отображается в манифест и ResultsLog.
    
- Как объяснялось ранее электронной почты поиска, который требуется экспортировать результаты в папку в файловой системе. Путь к папке для отдельных сообщений будет реплицировать путь к папке в почтовом ящике пользователя. Например, для поиска с именем «ContosoCase101» сообщений в папке "Входящие" пользователя должны быть расположены в путь к папке `~ContosoCase101\\<date of export\Exchange\user@contoso.com (Primary)\Top of Information Store\Inbox`. 
    
- При выборе Экспорт сообщений электронной почты в один PST-файл, содержащий все сообщения в одной папке папки " **Удаленные** " и папку **Папки поиска** включены в сайте верхнего уровня в папку PST-файлов. Эти папки будет пустым. 
  
 ### <a name="decrypting-rms-encrypted-messages"></a>Расшифровка сообщений, зашифрованных службы управления правами
  
- Как объяснялось ранее для расшифровки сообщений, зашифрованных службы управления правами, при экспорте их, необходимо экспортировать результаты поиска в виде отдельных сообщений. Если экспортировать результаты поиска в PST-файл, останутся зашифрованных сообщений, зашифрованных службы управления правами.
    
- Компонент расшифровки RMS в поиск контента не расшифровки сообщений, зашифрованных с помощью шифрования сообщений Office 365 (OME) при экспорте результатов поиска. Тем не менее если зашифрован с помощью OME сообщение отправлено пользователем в вашей организации, копии сообщения в папке Отправленные пользователя не шифруются и возможность просмотра после его экспорта. Тем не менее если сообщения, зашифрованные с OME получает пользователей в вашей организации, они не будут расшифрованы, после их в случае экспорта. Дополнительные сведения о OME можно [Шифрования сообщений Office 365](https://go.microsoft.com/fwlink/p/?linkid=844966).
    
- Сообщения, которые расшифрованы определяются в отчете о **ResultsLog** . В этом отчете содержится столбец с именем **Декодирования состояние**и значение **читаемости** в этой статье приводятся сообщения были расшифрованы. 
    
- На данный момент эта возможность расшифровки сюда не включены зашифрованное содержимое из SharePoint и OneDrive для бизнеса сайтов. Только сообщения электронной почты службы управления правами зашифрованных будут расшифрованы, при экспорте их.
    
- Если сообщение электронной почты службы управления правами зашифрованных вложений (например, документа или другое сообщение электронной почты), который также шифрования, будут расшифрованы, только сообщения электронной почты верхнего уровня.
    
- Не удается выполнить предварительный просмотр сообщения электронной почты, зашифрованных службы управления правами. Для просмотра зашифрованного сообщения, необходимо экспортировать его.
    
- Если вам требуется для предотвращения расшифровки сообщений, зашифрованных службы управления правами, необходимо создать пользовательскую группу ролей (, скопировав eDiscovery встроенные группы ролей диспетчера) и удалите роль управления расшифровки RMS на пользовательскую группу ролей. Затем добавьте человека, который требуется для расшифровки сообщения как член группы настраиваемых ролей.
  
 ### <a name="filenames-of-exported-items"></a>Имена файлов экспортируемые элементы
  
- Полный путь имени для сообщений электронной почты и сайта документов, экспортированные с локальным компьютером существует ограничение 260 знаков (налагаемые операционной системы). Полный путь для элементов, экспортированный включает в себя исходное расположение этого элемента и местоположение папки на локальном компьютере, где загружаются результатов поиска. Например, если указать можно загрузить результаты поиска, `C:\Users\Admin\Desktop\SearchResults` в средстве Экспорт электронных данных, затем полный путь для элемента загруженного электронной почты может быть `C:\Users\Admin\Desktop\SearchResults\ContentSearch1\03.15.2017-1242PM\Exchange\sarad@contoso.com (Primary)\Top of Information Store\Inbox\Insider trading investigation.msg`.
    
    В случае превышения 260 знаков усекаются полный путь для элемента.
    
  - Если полное имя пути больше 260 знаков, имя файла уменьшается для получения в разделе знаков; Обратите внимание на то, что усеченных имя файла (не включая расширение файла) не может быть меньше, чем из 8 знаков.
    
  - Если полное имя пути по-прежнему слишком много времени после сокращения имени файла, перемещено из текущего места в родительской папке. Если путь по-прежнему слишком много времени, то процесс повторяется: сокращение имени файла, и при необходимости еще раз перемещение в родительской папке. Этот процесс повторяется до полный путь в разделе 260 знаков.
    
  - Если имя усеченных полный путь уже существует, номер версии будет добавлен в конец имени файла; например `statusmessage(2).msg`.
    
    Для снижения этой проблемы решения о загрузке результатов поиска в папке с именем короткий путь. Например, загрузка результатов поиска в папку с именем `C:\Results` добавить меньшего числа символов имена путей экспортированного элементов, превышающее загрузкой на папку с именем `C:\Users\Admin\Desktop\Results`.
    
- При экспорте сайта документов можно также, что имя исходного файла документа будут изменены. Это происходит специально для документов, которые были удалены из SharePoint или OneDrive для бизнеса сайта, ставится на удержание. После удаления документов, который находится на сайт, который находится на удержании удаленного документа автоматически перемещаются в библиотеке удержания хранения для сайта (который создается, если узел был помещенные на удержание). При перемещении удаленного документа в библиотеку удержания хранения, случайным образом полученное и уникальный идентификатор используется в качестве исходного файла документа. Например, если имя файла для документа `FY2017Budget.xlsx` и более поздних версий удаляется и перемещено в библиотеке удержания хранения, изменяется имя файла документа, который перемещается в библиотеку удержания хранения, чтобы что-то вроде этого документа `FY2017Budget_DEAF727D-0478-4A7F-87DE-5487F033C81A2000-07-05T10-37-55.xlsx`. Если документа в библиотеке удержания хранения сопоставляет запрос поиска контента и экспортировать результаты поиска, экспортированный файл будет иметь имя измененного файла; в этом примере имя файла экспортированного документа может быть `FY2017Budget_DEAF727D-0478-4A7F-87DE-5487F033C81A2000-07-05T10-37-55.xlsx`.
    
    Кроме того при изменении документа, расположенные на сайте, который находится на удержании (и управления версиями для библиотеки документов на сайте была включена) копию файла автоматически создаваемых в библиотеке удержания хранения. В этом случае случайным образом полученное и уникальный идентификатор также добавляется к имени файла документа, который копируется в библиотеке удержания хранения.
    
    Причина, почему имена файлов документов, перемещения или копирования библиотеки удержания хранения — это во избежание конфликта имен файлов. Дополнительные сведения о размещения удержания на сайты и библиотека удержания хранения содержатся в разделе [Общие сведения о месте хранения в SharePoint Server 2016](https://support.office.com/article/5e400d68-cd51-444a-8fe6-e4df1d20aa95).
    
 ### <a name="miscellaneous"></a>Разное
  
- Все результаты поиска и экспорт отчетов включены в папку с именем поиска контента. Сообщения электронной почты, которые были экспортированы, находятся в папке с именем **Exchange**. Документы, находятся в папке с именем **SharePoint**. 
    
- Метаданные файловой системы для документов в SharePoint и OneDrive для бизнеса сайтов сохраняется при экспорте документы на локальный компьютер. Который означает, что свойства документа, например, созданные и последнего изменения дат, остаются прежними при экспорте документов.
