# <a name="office-365-security-roadmap---top-priorities-for-the-first-30-days-90-days-and-beyond"></a>Обзор безопасности Office 365 — верхней приоритеты для первого 30 дней, 90 дней и за ее пределами

В этой статье содержатся основные рекомендации из группы cybersecurity корпорации Майкрософт для реализации возможностей безопасности для защиты среды Office 365. В этой статье заимствован из сеанса Microsoft Ignite — [безопасного Office 365 как cybersecurity pro: наиболее распространенные приоритеты для первого 30 дней, 90 дней и за ее пределами](https://www.youtube.com/watch?v=luignzNyR-o). Этот сеанс был разработан и доклад Mark Simos и Kemelhar Мэтт, архитекторы Cybersecurity предприятия.

## <a name="roadmap-outcomes"></a>Схема деятельности
<a name="Roadmap"> </a>

Эти рекомендации схема распределить на три этапа в логический порядок для следующих целей.

|||
|:-----|:-----|
| |Результаты
|30 дней|Быстрое конфигурации:  <br/> • Защиту от базового администрирования  <br/> • Ведение журналов и аналитика  <br/> • Защиту от базового удостоверений  <br/> Конфигурация клиента  <br/>  Подготовка заинтересованных лиц  <br/> |
|90 дней|Дополнительные средства защиты:  <br/> • Учетные записи администраторов  <br/>  • Данных &amp; учетных записей пользователей  <br/>  Возможность просмотра потребностей соответствия требованиям, угроз и пользователей  <br/>  Адаптировать и реализации политики по умолчанию и защиты  <br/> |
|Помимо тех|Настройка и оптимизация основные политики и элементов управления  <br/> Расширение возможностей защиты в зависимостей локальной  <br/> Интеграция с процессами бизнеса и безопасности (legal, внутренних угроз, и т.д.)  <br/> |
  

   
## <a name="30-days--powerful-quick-wins"></a>30 дней, мощные быстрого wins
<a name="Thirdaydays"> </a>

Эти задачи выполняются быстро и почти не влияют на пользователей.
  
|||
|:-----|:-----|
|Area  <br/> |Задачи  <br/> |
|Управление безопасностью  <br/> |• Проверка безопасного счета и запомните текущее показателя ( [https://securescore.office.com](https://securescore.office.com)).  <br/>  • Включить ведение журнала аудита для Office 365. Просмотреть [Поиск в журнале аудита безопасности Office 365 &amp; центре соответствия требованиям](search-the-audit-log-in-security-and-compliance.md).<br/> • [Настройка клиента Office 365 для повышения уровня безопасности](tenant-wide-setup-for-increased-security.md) .  <br/>  • Регулярно проверять панелей мониторинга и отчетов в Office 365 безопасности и центре соответствия требованиям и безопасности приложения в облаке.  <br/> |
|Защита от угроз  <br/> |[Подключение Office 365 для безопасности приложения Microsoft Cloud](https://docs.microsoft.com/cloud-app-security/connect-office-365-to-microsoft-cloud-app-security) запуск мониторинга с помощью политик обнаружения угроз по умолчанию для непредвиденных поведения. Он принимает семь дней для построения базовые обнаружение неполадок.<br><br/>  Применить защиту для учетных записей администратора.  <br/> • Admin использования выделенных учетных записей для действия администратора.  <br/>  • Обеспечение многофакторная проверка подлинности (многофакторной проверкой Подлинности) для учетных записей администратора.  <br/>  • Использовать [устройства Windows 10 с повышенной защитой](https://docs.microsoft.com/windows-hardware/design/device-experiences/oem-highly-secure) для действия администратора.  <br/> |
|Управление удостоверениями и доступом  <br/> |• [Включить защиту учетных данных Azure Active Directory](https://docs.microsoft.com/azure/active-directory/active-directory-identityprotection-enable).  <br/> • Для сред федеративных удостоверений, обеспечения безопасности учетной записи (длина пароля, срок хранения, сложности, и т.д.).  <br/> |
|Защита информации  <br/> | Изучите рекомендации по защите сведения о примере. Защита информации требует координации внутри организации. Начало работы с следующие ресурсы:<br/> • [Защита office 365 информации для GDPR](http://aka.ms/o365gdpr) <br/> • [Безопасного SharePoint Online сайтами и файлами](https://docs.microsoft.com/Office365/enterprise/secure-sharepoint-online-sites-and-files) (включая общий доступ к, классификации, защиту от потери данных и защита информации Azure)  <br/> |
   
## <a name="90-days--enhanced-protections"></a>90 дней — улучшенные средства защиты
<a name="Ninetydays"> </a>

Планирование и реализация этих задач занимают немного больше времени, но это значительно повысит безопасность ваших данных. 
  
|||
|:-----|:-----|
|Area  <br/> |Task  <br/> |
|Управление безопасностью  <br/> | • Проверка безопасного показатель рекомендуемых действий для вашей среды ( [https://securescore.office.com](https://securescore.office.com)).  <br/>  • Продолжать регулярно проверять панели мониторинга и отчеты в безопасности Office 365 и центре соответствия требованиям, облачных приложений, средства безопасности и SIEM.  <br/>  • Поиск и выполнять обновление программного обеспечения.  <br/>  • Моделирования атаки поведения для направленных фишинга, распылителя пароль и пароль перебора атаки с помощью [Имитатора атаки](https://support.office.com/article/attack-simulator-office-365-da5845db-c578-4a41-b2cb-5a09689a551b) (входит в состав [Анализ угроз Office 365](office-365-ti.md)).  <br/>  • Внешнего вида для общего доступа к риск, просмотрев встроенные отчеты в облаке безопасности приложения (на вкладке проверить).  <br/>  • Проверка [Соответствия Manager](meet-data-protection-and-regulatory-reqs-using-microsoft-cloud.md) для просмотра состояния для правила, которые применяются к вашей организации (например, GDPR NIST 800 171).  <br/> |
|Защита от угроз  <br/> | Реализация расширенной защиты для учетных записей администратора:  <br/>  • Настройка [Привилегированной рабочих станций доступа](https://docs.microsoft.com/windows-server/identity/securing-privileged-access/privileged-access-workstations) (лапами) для действия администратора.  <br/>  • Настройка [инфраструктуры управления удостоверениями привилегиями Azure AD](https://docs.microsoft.com/azure/active-directory/active-directory-privileged-identity-management-configure).  <br/>  • Настройка безопасности сведения и события (SIEM) средство управления для сбора данных в журналах с Office 365, безопасность приложения в облаке и другие службы, включая службы федерации Active Directory. Журнал аудита Office 365 хранит данные только 90 дней. Эти данные в SIEM средство захвата позволяет хранить данные на длительный период.<br/> |
|Управление удостоверениями и доступом  <br/> | • Включение и примените многофакторной проверкой Подлинности для всех пользователей.  <br/>  • Реализовать набор [условного доступа и связанных с ними политик](https://docs.microsoft.com/en-us/microsoft-365/enterprise/microsoft-365-policies-configurations). |
|Защита информации  <br/> | Адаптировать и Внедрите политики защиты информации. Следующие ресурсы содержат примеры:<br/> • [Защита office 365 информации для GDPR](http://aka.ms/o365gdpr) <br/> • [Безопасного SharePoint Online сайтами и файлами](https://docs.microsoft.com/Office365/enterprise/secure-sharepoint-online-sites-and-files) <br/> <br> Используйте политики предотвращения потери данных и средства мониторинга в Office 365 для данных, хранящихся в Office 365 (а не безопасности приложения облако). <br><br>Использование безопасности облаке приложения с помощью Office 365 сложные функции (кроме защиту от потери данных) оповещения.  <br/> |
   
## <a name="beyond"></a>Помимо тех
<a name="Beyond"> </a>

Это меры безопасности, основанные на предыдущей работы. 
  
|||
|:-----|:-----|
|Area  <br/> |Task  <br/> |
|Управление безопасностью  <br/> |• Продолжить планирование следующего действия с помощью Secure показатель ( [https://securescore.office.com](https://securescore.office.com)).  <br/>  • Продолжать регулярно проверять панели мониторинга и отчеты в безопасности Office 365 и центре соответствия требованиям, облачных приложений, средства безопасности и SIEM.  <br/>  • Продолжать искать и выполнять обновление программного обеспечения.  <br/>  • Интегрировать обнаружения электронных данных в вашей юридическим и процессов ответа угроз.  <br/> |
|Защита от угроз  <br/> | • Реализация [Безопасного привилегированный доступ](https://docs.microsoft.com/windows-server/identity/securing-privileged-access/securing-privileged-access) (SPA) для компонентов удостоверение при локальном (AD, AD FS).  <br/>  • Использовать облачных приложений безопасности для наблюдения за внутренних угроз.  <br/>  • Обнаружить использование теневой SaaS ИТ с помощью безопасности приложения в облаке.  <br/> |
|Управление удостоверениями и доступом  <br/> | • Уточнение сведения по защите:  <br/>  • Azure защита информации и Office 365 защита от потери данных (DLP).  <br/>  • Политики безопасности приложения облако и оповещения.  <br/> |
|Защита информации  <br/> | • Уточнение политики и рабочих процессов.  <br/>  • Защиту использования Azure AD для идентификации внутренних угроз.  <br/> |
