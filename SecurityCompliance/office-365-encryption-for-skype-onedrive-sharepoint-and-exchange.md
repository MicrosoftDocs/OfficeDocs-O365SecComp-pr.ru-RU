# <a name="office-365-encryption-for-skype-for-business-onedrive-for-business-sharepoint-online-and-exchange-online"></a>Шифрование Office 365 для Скайп для бизнеса, OneDrive для бизнеса, SharePoint Online и Exchange Online

Office 365 — это очень безопасной среде, которая обеспечивает расширенную защиту в нескольких уровней: центра обработки данных физических, безопасности, безопасность сети, безопасность доступа, безопасность приложений и безопасность данных.

## <a name="skype-for-business"></a>Skype для бизнеса
Скайп для клиента бизнес-данных может храниться в rest в виде файлов или презентации, отправленные участниками собрания. Сервер веб-конференций выполняет шифрование данных клиента, с помощью AES с 256-разрядный ключ. Зашифрованные клиента данные хранятся в общей папке. Каждой части пользовательских данных шифруется с помощью различных случайным 256-разрядный ключ. Если часть данных клиента общих в конференции, сервер веб-конференций этот параметр клиентов конференц-связи для загрузки данных зашифрованные клиента с помощью HTTPS. Он отправляет соответствующий ключ для клиентов, чтобы данные клиента могут быть расшифрованы. Сервер веб-конференций также выполняет проверку подлинности клиентов конференц-связи перед тем как разрешить клиентам доступа к данным клиента конференции. При присоединении к веб-конференции, каждый клиент конференц-связи устанавливает диалоговое окно SIP с компонентом фокус конференц-связи, выполняется внутри интерфейсного сервера через TLS, сначала. Фокус конференц-связи передает клиенту конференции файл cookie проверки подлинности, созданные сервером веб-конференций. Клиент конференц-связи осуществляет подключение к серверу веб-конференций, презентацию файла cookie проверки подлинности на проверку подлинности на сервере.

## <a name="sharepoint-online-and-onedrive-for-business"></a>SharePoint Online и OneDrive для бизнеса
Все файлы клиентов в SharePoint Online защищены уникальный, каждого файла Нажимаемые всегда являются исключительно для одного клиента. Ключи либо создан и управляемых службой SharePoint Online либо при использовании ключа клиента создаются и управляются с клиентами. При загрузке файла шифруются с SharePoint Online внутри контекста запроса загрузки перед отправкой хранилища Azure. При загрузке файла SharePoint Online извлекает зашифрованные клиента данные из хранилища Azure на основе идентификатора уникальных документов и расшифровывает данные клиента перед отправкой для пользователя. Хранилище Azure имеет возможность расшифровать, или даже определить или понять данные клиента. Шифрование и дешифровка произойти же систем, обеспечивающие изоляция клиента, которые являются Azure Active Directory и SharePoint Online.

Несколько рабочих нагрузок в Office 365 хранения данных в SharePoint Online, включая группами Майкрософт, который хранит все файлы в SharePoint Online и OneDrive для бизнеса, который использует SharePoint Online для его хранилища. Все данные клиента, хранящимся в SharePoint Online зашифрован (с помощью одного или нескольких ключей AES 256-разрядная версия) и распределены по центра обработки данных, как показано ниже. (Каждый шаг этот процесс шифрования является FIPS 140-2 уровня 2 проверены. Для получения дополнительных сведений о соответствия требованиям FIPS 140-2 см [соответствия требованиям FIPS 140-2](https://docs.microsoft.com/previous-versions/sql/sql-server-2008-r2/bb326611(v=sql.105))).
- Каждый файл разбивается на один или несколько фрагментов, в зависимости от размера файла. Каждый блок шифруется с помощью собственный уникальный AES 256-разрядный ключ.
- При обновлении файла обновление обрабатывается так же, как: изменение разбивается на один или несколько фрагментов, а каждый блок зашифрован с помощью отдельной уникальный ключ.
- Следующие фрагменты – файлы, части файлов и обновление дельты – сохраняются как большие двоичные объекты в хранилище Azure случайным образом распределены по несколько учетных записей хранилища Azure. 
- Набор ключи шифрования для этих фрагментов данных клиента является шифрование.
   - Клавиши, которые используются для шифрования большие двоичные объекты хранятся в SharePoint Online базы данных контента.
   - База данных контента был защищен с шифрование статических и элементов управления доступа к базе данных. Шифрование выполняется с помощью [Прозрачное шифрование данных](https://docs.microsoft.com/sql/relational-databases/security/encryption/transparent-data-encryption-tde) в [Базе данных SQL Azure](https://docs.microsoft.com/azure/sql-database/sql-database-technical-overview). (База данных azure SQL — это служба общего назначения реляционной базы данных в Microsoft Azure, которая поддерживает структуры, такие как реляционные данные, JSON Пространственные и XML). Эти секреты, на уровне службы для SharePoint Online, не на уровне клиента. Эти секреты (иногда называется главного ключа) хранятся в отдельных безопасное хранилище вызывается ключом хранилища. TDE обеспечивает безопасность на уровне rest для активной базы данных и резервных копий баз данных и журналов транзакций. 
   - При клиенты предоставляет дополнительный ключ, ключа клиента хранится в хранилище ключа Azure и служба использует ключ шифрования с ключом клиента, который используется для шифрования ключом сайта, который затем используется для шифрования ключей уровня файла. По сути введенным новой иерархии ключей при клиент предоставляет ключ.
- Повторно сборки в файл карты хранится в базе данных контента, а также зашифрованных разделов, отдельно от основного ключа, необходимые для их расшифровки.
- Каждой учетной записи хранилища Azure имеет свои собственные уникальные учетные данные на тип доступа (чтение, запись, перечисление и удаление). Каждый набор учетных данных остается в secure Store ключа и регулярно обновляются. Как описано выше, существует три различных типа хранилищ, каждый из которых различные функции.
- Данные клиента сохраняется как зашифрованные большие двоичные объекты в хранилище Azure. Ключ для каждого блока данных клиента шифруются и отдельно хранятся в базе данных контента. Данные клиента самого содержит понять как расшифрован.
- База данных контента представляет собой базу данных SQL Server. Он содержит карты, необходимых для поиска и заново собрать данные клиента большие двоичные объекты, которые проводятся в хранилища Azure, а также клавиш, необходимым для шифрования эти большие двоичные объекты. Тем не менее набор разделов является зашифрован (как описано выше) и элемент хранится в отдельном хранилище ключа.
- Хранилище ключ физического разделения из хранилища базы данных контента и Azure. Он содержит учетные данные для каждого контейнера хранилища Azure и главный ключ набор зашифрованных разделов, хранящихся в базе данных контента.

Каждый из этих трех компонентов хранения – Azure BLOB-хранилища базы данных контента и хранилище ключа – физического разделения. Информация, хранящаяся в одном из компонентов не может использоваться самостоятельно. Без доступа для всех трех пользователь не сможет получить с помощью клавиш блоки, расшифровки клавиши, чтобы сделать их можно использовать, связать ключи с их соответствующие блоки, расшифровки каждого блока или восстановить документ из его составные блоки.

BitLocker сертификаты, которые защитить томами физических дисков на компьютерах в центре обработки данных, хранимых в безопасное хранилище (SharePoint Online секретный внутренний), который был защищен с помощью ключа фермы.

Ключи TDE, защиты ключей в больших двоичных объектов, хранятся в двух местах:
- Безопасный репозитория, который содержит сертификаты BitLocker и был защищен с ключом фермы; и
- В безопасное хранилище управляется базы данных SQL Azure.

Учетные данные, используемые для доступа к контейнерам хранилища Azure также хранятся в SharePoint Online секрет хранения и передать каждой ферме SharePoint Online, при необходимости. Эти учетные данные, хранилища Azure SAS подписей с отдельные учетные данные, используемые для чтения или записи данных и политики применен, чтобы они auto-истечения срока действия каждые 60 дней. Другие учетные данные используются для чтения или записи данных (но не оба) и фермах SharePoint Online не назначаются разрешения на перечисление.

> Примечание для Office 365 США правительственных клиентов, больших двоичных объектов, хранятся в хранилище государственных организаций США Azure. Кроме того персонала Office 365, в частности экранироваться ограничен доступ к SharePoint Online ключи в государственном секторе США Office 365. Azure правительства США персоналу нет доступа к хранилищу SharePoint Online ключа, который используется для шифрования данных больших двоичных объектов.

### <a name="list-items-in-sharepoint-online"></a>Элементы списка в SharePoint Online
Элементы списка, более мелких блоках клиента, данные, которые создаются точечную или, можно дополнительно динамически live в пределах сайта, такие как строк в списке, созданные пользователем, отдельных записей в SharePoint Online блог или записи в SharePoint Online вики-страниц. Элементы списка хранятся в базе данных контента (базы данных SQL Azure) и защищенные с помощью TDE.

## <a name="encryption-of-data-in-transit"></a>Шифрование транзитных данных
В OneDrive для бизнеса и SharePoint Online используются два сценария для входящих и исходящих данных в центрах данных.
- **Клиент обмена данными с сервером** - связи в OneDrive для бизнеса через Интернет с помощью подключения SSL или TLS. Все SSL подключений с помощью клавиш 2048 бит.
- **Перемещение данных между центрами обработки данных** — основная причина для перемещения данных между центрами обработки данных — это географическая репликации возможность аварийного восстановления. Например дельты хранилища больших двоичных объектов и журналы транзакций SQL Server бюро по этой канала. Хотя эти данные уже передаются с помощью частной сети, дальнейшей защищен с помощью шифрования лучшие в своем классе.


## <a name="exchange-online"></a>Exchange Online
Exchange Online использует BitLocker для всех данных почтового ящика и конфигурации BitLocker представлено в [BitLocker для шифрования](office-365-bitlocker-and-distributed-key-manager-for-encryption.md). Шифрование на уровне службы шифрует все данные почтовых ящиков на уровне почтового ящика. 

В дополнение к шифрование службы Office 365 поддерживает ключа клиента, которая построена на основе службы шифрования. Ключ клиента это управляемых Microsoft ключа для Exchange Online службы шифрования, который также является на стратегия корпорации Майкрософт. Этот метод шифрования предоставляет усиления защиты, не предоставляемые BitLocker, поскольку он обеспечивает разделение администраторов серверов и ключи шифрования, необходимые для расшифровки данных, а шифрование применяется непосредственно к данным (в контрастность с BitLocker применяется шифрование на том логический диск) остается зашифрованные данные клиента скопирован с сервера Exchange.

Область для Exchange Online службы шифрования — пользовательских данных, которые хранятся в rest в Exchange Online. (Скайп для бизнеса хранятся почти все вызванные пользователем содержимого в Exchange Online почтового ящика пользователя и поэтому наследует функции шифрования службы версией Exchange через Интернет).
