# <a name="office-365-message-encryption"></a>Шифрование сообщений Office 365

## <a name="how-office-365-message-encryption-works"></a>Как работает шифрование сообщений Office 365

Шифрование сообщений Office 365 — это сетевая служба, встроенной в Microsoft Azure Rights Management (Azure RMS) которого является частью защита информации Azure. Office 365 для администраторов можно определить правила потока обработки почты для определения условий для шифрования. Например правило может потребовать шифрования все сообщения, адресованные определенного получателя.
  
Когда кто-то отправляет сообщение электронной почты в Exchange Online, которая соответствует правило поток почты шифрования, шифруется сообщение перед отправкой. Всех конечных пользователей Office 365, использующие клиенты Outlook для чтения почты получать машинный, первого класса чтения каждый раз для почты зашифрованные и защищенные правами доступа, даже в том случае, если они не в той же организации, что отправителя. Поддерживаемые клиенты Outlook включают Outlook на рабочем столе, Outlook Mac, Outlook mobile на iOS и Android (en) и Outlook Web App.
  
Получатели зашифрованных сообщений, которые получают зашифрованные и защищенные правами почта, передаваемая учетных записей Outlook.com, Gmail и Yahoo можно легко выполнять проверку подлинности на портале OME, с помощью своей учетной записи Майкрософт или Gmail или Yahoo учетные данные.
  
Конечные пользователи, которые читать зашифрованные или защищенной почты на клиентами, отличными от Outlook также использовать портал OME просматривать зашифрованные и защищенные правами доступа сообщения, которые они получают.
  
Мы повысили ограничения размера сообщений и вложений, которые могут быть зашифрованы с помощью шифрования сообщений Office 365. Дополнительные сведения об ограничениях можно [ограничения Exchange Online.](https://technet.microsoft.com/en-us/library/exchange-online-limits.aspx)
  
## <a name="defining-rules-for-office-365-message-encryption"></a>Определение правил для шифрования сообщений Office 365

— Это один из способов включения новых возможностей для шифрования сообщений Office 365 для администраторов Exchange Online и Exchange Online Protection для определения правил потока обработки почты. Эти правила определяют, в разделе какие условия электронной почты следует зашифрованных сообщений. Если для правила шифрования действие, все сообщения, которые соответствуют условиям правила шифруются перед отдельными.
  
Правила для потока обработки почты отличаются гибкостью, позволяющей объединять условия, поэтому требованиям безопасности в одно правило. Например можно создать правила для шифрования всех сообщений, содержащих ключевые слова и предназначено для внешних получателей. Новые возможности для шифрования сообщений Office 365 также шифровать ответы получатели зашифрованных сообщений электронной почты.
  
Дополнительные сведения о создании почтового потока обработки правил, чтобы воспользоваться преимуществами новых возможностей OME можно [Определение правил для шифрования сообщений Office 365](define-mail-flow-rules-to-encrypt-email.md).
  
## <a name="sending-viewing-and-replying-to-encrypted-email-messages"></a>Отправка, просмотр и ответ на зашифрованные сообщения электронной почты

С помощью шифрования сообщений Office 365 пользователи могут отправлять зашифрованное сообщение электронной почты в Outlook и Outlook в Интернете. Кроме того "Администраторы" можно настроить правила для потока обработки почты в Office 365 для автоматического шифрования сообщений электронной почты на основе сопоставления или других условий ключевого слова.
  
Получатели зашифрованных сообщений, которые находятся в Office 365 организации смогут прочитать эти сообщения общения в любой версии Outlook, включая Outlook для ПК, Outlook для Mac, Outlook в Интернете, Outlook для операций ввода-вывода и Outlook для Android. Пользователи, которые получают зашифрованные сообщения на других клиентов электронной почты можно просмотреть сообщения на портале OME.
