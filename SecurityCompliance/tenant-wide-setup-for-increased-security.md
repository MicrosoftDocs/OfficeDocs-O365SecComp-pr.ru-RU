# <a name="configure-your-office-365-tenant-for-increased-security"></a>Настройка вашего клиента Office 365 для улучшения безопасности

В этом разделе описывается рекомендуемая конфигурация для параметры на уровне клиента, которые влияют на безопасность вашей среде Office 365. Увеличить или уменьшить безопасности может потребоваться требованиями безопасности. Используйте эти рекомендации в качестве отправной точки.
  
## <a name="check-office-365-secure-score"></a>Проверьте безопасной показатель Office 365

Office 365 безопасного показатель служит для анализа безопасности организации Office 365 на основании регулярного действий и параметров безопасности и назначает оценки. Начните с Обратите внимание на ваш текущий отчет. Настройка некоторые параметры на уровне клиента будет увеличиваться результатов. Цель — не для достижения максимальной показателя, но для поддержки возможностей для защиты среды, не влияют отрицательно сказаться на производительности для пользователей. В разделе [Общие сведения об безопасных показатель Office 365](office-365-secure-score.md).

## <a name="configure-tenant-wide-sharing-policies-in-sharepoint-admin-center"></a>Настройка политик общего доступа всей клиента в центре администрирования SharePoint

Microsoft рекомендациям по настройке сайтов групп SharePoint в повышение уровня защиты, начиная с защитой базового. Для получения дополнительных сведений см [безопасного SharePoint Online сайтами и файлами](https://docs.microsoft.com/microsoft-365-enterprise/secure-sharepoint-online-sites-and-files)
  
Сайты групп SharePoint, настроенных на уровне базовой разрешить совместное использование файлов с внешними пользователями с помощью ссылки анонимный доступ. Такой подход рекомендуется вместо Отправка файлов по электронной почте. 
  
Для достижения целей для защиты от базового, настройте политики общего доступа клиента всей соответствии с рекомендациями здесь. Совместное использование параметров для отдельных сайтов можно более строгий, чем этот клиент политики, но не менее строгий.

Рекомендуется использовать следующие параметры:<br/>  • Разрешить общий доступ к внешним пользователям прошедшим проверку подлинности и с помощью ссылки анонимный доступ (по умолчанию).  <br/>  • Ссылки анонимный доступ через столько дней. ## <a name="configure-settings-in-azure-active-directory"></a>Настройка параметров в Azure Active Directory

Убедитесь, что посетите этих двух областей в Azure Active Directory для завершения установки на уровне клиента для более безопасной среды.
  
### <a name="configure-named-locations-under-conditional-access"></a>Настроить именованные расположения (в разделе условного доступа)

Если в организации имеется офисы с безопасный доступ к сети, добавьте Azure Active Directory надежные диапазоны IP-адресов как именованный расположения. Эта функция позволяет сократить число переданные ложные срабатывания для события входа рисков. 
  
См.: [имя расположения в Azure Active Directory](https://docs.microsoft.com/azure/active-directory/active-directory-named-locations)
  
### <a name="block-apps-that-dont-support-modern-authentication"></a>Блокировать приложения, которые не поддерживают современных проверки подлинности

Многофакторная проверка подлинности требует приложения с поддержкой современных проверки подлинности. Нельзя блокировать приложения, которые не поддерживают современных проверки подлинности с помощью правила условного доступа.
  
Для безопасной среды убедитесь, что отключение проверки подлинности для приложений, которые не поддерживают современных проверки подлинности. Вы можете сделать в Azure Active Directory с элементом управления, в ближайшее время.
  
В это время используйте один из следующих методов для выполнения этой задачи для SharePoint Online и OneDrive для бизнеса:
  
- С помощью PowerShell, см [Блокировать приложения, которые используют современных проверки подлинности](https://docs.microsoft.com/intune-classic/deploy-use/block-apps-with-no-modern-authentication).
    
- Настройки в центре администрирования SharePoint «страницы доступа устройств — «Управления доступа из приложений, не использующих современных проверки подлинности». Выберите блок.
