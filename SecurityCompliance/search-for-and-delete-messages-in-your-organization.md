# <a name="search-for-and-delete-email-messages-in-your-office-365-organization---admin-help"></a>Для поиска и удаления сообщений электронной почты в организации Office 365 - Admin справки

**Эта статья предназначена для администраторов. Вы пытаетесь найти элементы в почтовом ящике пользователя, который требуется удалить? Просмотреть [найдите сообщение или элемента с помощью мгновенного поиска](https://support.office.com/article/69748862-5976-47b9-98e8-ed179f1b9e4d)**|
   
Можно использовать функцию поиска контента в Office 365 для поиска и удаления сообщения электронной почты из всех почтовых ящиков в организации. Это может помочь поиск и удаление потенциально вредоносных или высокого риска электронной почтой, такие как:
  
- Сообщения, которые содержат опасные вложения или вирусов
    
- Фишинговые сообщения
    
- Сообщения, содержащие конфиденциальные данные
    
> [!CAUTION]
> Поиск и удаление — это мощное средство, позволяющее всем пользователям, назначенная необходимые разрешения для удаления сообщений электронной почты из почтовых ящиков в организации. 
  
## <a name="before-you-begin"></a>Перед началом работы

- Создание и выполнение поиска контента, необходимо быть участником группы **eDiscovery диспетчер** ролей или назначить роль управления **Поиска соответствия** . Чтобы удалить сообщения, необходимо быть участником группы ролей **Управления организацией** или назначить роль управления **Поиска и удалить** . Сведения о добавлении пользователей в группу ролей, можно [предоставления пользователям доступа к Office 365 безопасность &amp; центре соответствия требованиям](grant-access-to-the-security-and-compliance-center.md).
    
- Используйте безопасности &amp; PowerShell центр соответствия требованиям для удаления сообщений. Инструкции о подключении содержатся в разделе [Шаг 2](#step-2-connect-to-security-amp-compliance-center-powershell) .
    
- Одновременно можно удалить не более 10 элементов на один почтовый ящик. Так как возможности для поиска и удаления сообщений, предназначенного для формирования средство аварийно ответа, это ограничение гарантирует, что Быстрое удаление сообщений из почтовых ящиков. Этот компонент не предназначен для очистки почтовые ящики пользователей. Чтобы удалить более 10 элементов, можно использовать команду **Search-Mailbox-DeleteContent** в Exchange Online PowerShell. В разделе [Поиск и удаление сообщений - Admin справки](search-for-and-delete-messagesadmin-help.md).
    
- Максимальное количество почтовых ящиков в поиск содержимого, можно удалять элементы в, выполняя поиск и удаление действие — 50 000. Если поиск контента (, создаваемого на [этапе Step 1](#step-1-create-a-content-search-to-find-the-message-to-delete)) имеет более 50 000 исходных почтовых ящиков, очистку (, создаваемого в шаге 3) завершится с ошибкой. В разделе [Дополнительные сведения](#more-information) подсказки на выполнение поиска и очистить операции для более чем 50 000 почтовых ящиков. 
    
- Процедуру, описанную в этой статье можно использовать только для удаления элементов из почтовых ящиков Exchange Online и общедоступные папки. Его нельзя использовать для удаления содержимого из SharePoint или OneDrive для бизнеса сайтов.
    
## <a name="step-1-create-a-content-search-to-find-the-message-to-delete"></a>Шаг 1. Создание поиска контента для поиска сообщения, которое необходимо удалить

Первым шагом является создание и выполнение поиска контента, чтобы найти сообщение, которое требуется удалить из почтовых ящиков в организации. Поиск можно создать с помощью безопасности &amp; центре соответствия требованиям или с помощью командлетов **New-ComplianceSearch** и **ComplianceSearch Пуск** . Сообщения, которые соответствуют запрос для поиска будут удалены, выполнив командлет **New-ComplianceSearchAction** на [шаге 3](#step-3-delete-the-message). Сведения о создании поиска контента и настройке поисковых запросов в следующих разделах: 
  
- [Поиск контента в Office 365](content-search.md)
    
- [Запросы ключевое слово для поиска контента](keyword-queries-and-search-conditions.md)
    
- [New-ComplianceSearch](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-content-search/New-ComplianceSearch)
    
- [Start-ComplianceSearch](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-content-search/Start-ComplianceSearch)
    
> [!NOTE]
> Размещения содержимого, поиск в поиск содержимого, создаваемого на этом этапе не может содержать SharePoint или OneDrive для бизнеса сайтов. Можно добавить только почтовых ящиков и общих папок поиска контента, который будет использоваться для сообщения электронной почты. Если поиск контента включает сайты, вы получите ошибку в шаге 3 при запуске командлета **New-ComplianceSearchAction** . 
  
### <a name="tips-for-finding-messages-to-remove"></a>Советы по поиску сообщения, чтобы удалить

Цель поискового запроса — ограничить результаты поиска сообщениями, которые необходимо удалить. Ниже приведено несколько советов.
  
- Если вы знаете точный текст или фразу, используемые в строке темы сообщения, используйте свойство **Subject** в поисковые запросы. 
    
- Если вы знаете точную дату (или диапазон дат) получения сообщения, укажите в поисковом запросе свойство **Received**. 
    
- Если вы знаете, кто отправил сообщение, укажите в поисковом запросе свойство **From**. 
    
- Предварительный просмотр результатов поиска, чтобы убедиться, что поиск только сообщения (или сообщения), который требуется удалить.
    
- Используйте статистику оценки поиска (отображаются в области сведений в системы поиска &amp; центре соответствия требованиям или с помощью командлета [Get-ComplianceSearch](https://go.microsoft.com/fwlink/p/?LinkId=517934) ) для получения счетчик общее число результатов. 
    
Ниже приведены два примера поиска подозрительных сообщений электронной почты.
  
- Этот запрос возвращает сообщения, полученные пользователями 13–14 апреля 2016 г., в которых есть слова "action" и "required" в строке темы.
    
    ```
    (Received:4/13/2016..4/14/2016) AND (Subject:'Action required')
    ```
   
- Этот запрос возвращает сообщения, отправленные с адреса chatsuwloginsset12345@outlook.com, которые содержат фразу "Update your account information" ("Обновите сведения об учетной записи") в строке темы.
    
    ```
    (From:chatsuwloginsset12345@outlook.com) AND (Subject:"Update your account information")
    ```

## <a name="step-3-delete-the-message"></a>Шаг 3: Удаление сообщения

После создания и уточнение поиска контента, чтобы получить сообщение, которое нужно удалить и подключенные к безопасности &amp; PowerShell центр соответствия требованиям, завершающий шаг состоит выполните командлет **New-ComplianceSearchAction** , чтобы удалить сообщение. Удаленные сообщения перемещаются папки элементов для восстановления. 
  
Приведенная в следующем примере команда удаляет результаты поиска, возвращенные в результате поиска контента с именем Remove Phishing Message. 

```
New-ComplianceSearchAction -SearchName "Remove Phishing Message" -Purge -PurgeType SoftDelete
```
  
Поиск, указанного с помощью параметра *SearchName* является поиска контента, созданной на шаге 1. 
  
Дополнительные сведения содержатся в разделе [New-ComplianceSearchAction](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-content-search/New-ComplianceSearchAction).
  

## <a name="more-information"></a>Дополнительные сведения

- **Как получить сведения о состоянии на поиск и удаление операции**

    Запустите **Get-ComplianceSearchAction** , чтобы получить состояние на операцию удаления. Обратите внимание на то, что объект, созданный при выполнении командлета **New-ComplianceSearchAction** называется использованием следующего формата: `<name of Content Search>_Purge`. 
    
- **Что происходит после удаления сообщения?**

    Сообщение, в котором будет удален с помощью `New-ComplianceSearchAction -Purge -PurgeType SoftDelete` команда перемещается в папку Удаленные элементы в папки элементов для восстановления. Не будет удален сразу же с Office 365. Пользователь может восстановить сообщений в папки "Удаленные" во время выполнения, в зависимости от срока хранения удаленного элемента, настроенных для почтового ящика. После этого срока их хранения (или если пользователь удаляет сообщение до истечения срока действия), сообщение перемещено в папку удаляет и больше не может осуществляться пользователем. Один раз в папке удаляет сообщение еще раз сохраняется во время выполнения, в зависимости от срока хранения удаленного элемента, настроенных для почтового ящика, если этот параметр включен восстановления отдельных элементов для почтового ящика. (В Office 365 функцию восстановления отдельных элементов включена по умолчанию при создании нового почтового ящика.) По истечении срока хранения удаленного элемента сообщение помечено для окончательного удаления и будут очищены из Office 365 в будущем обработки помощником для управляемых папок почтового ящика. 
    
- **Как узнать, что сообщения удаляются и перемещено в папку элементов для восстановления?**

    При выполнении поиска же контента после удаления сообщения, вы по-прежнему будут видеть такое же число результатов поиска (и предположить, что сообщение не был удален из почтовых ящиков пользователей). Тем, что поиск контента осуществляет поиск папки элементов для восстановления, которая является, где удаленное сообщение перемещается после запуска `New-ComplianceSearchAction -Purge -PurgeType SoftDelete` команды. Чтобы убедиться в том, что сообщения были перемещены в папку элементов для восстановления, можно запустить поиск обнаружения электронных данных на месте (с использованием одной исходных почтовых ящиков и условия поиска как поиск контента, созданном на шаге 1) и затем скопировать результаты поиска в почтовый ящик обнаружения. Затем можно просматривать результаты поиска в почтовом ящике обнаружения и убедитесь, что сообщение было перемещено в папку элементов для восстановления. Сведения о создании поиска обнаружения электронных данных на месте, которое использует список исходных почтовых ящиков и запросов поиска из поиска контента в разделе [Использование поиска контента в рабочий процесс обнаружения электронных данных](use-content-search-in-ediscovery.md) . 
    
- **Что делать, если необходимо удалить сообщения из более чем 50 000 почтовых ящиков?**

    Как было сказано ранее можно выполнить поиск и удалить операцию не более 50 000 почтовых ящиков. Если нужно выполнить поиск и удалить операции для более чем 50 000 почтовых ящиков, рассмотрите возможность создания временной поисковых фильтров разрешения, которые может уменьшить количество почтовых ящиков, которые будет искать не более 50 000 почтовых ящиков. Например если ваша организация содержит почтовые ящики в различных отделов, состояния или странах, можно создать разрешения фильтр для поиска почтовых ящиков на основе одного из этих свойств почтового ящика для поиска подмножество почтовых ящиков в организации. После создания фильтра поиска разрешений, нужно создать поиска (описано в шаге 1) и затем удалить сообщение (как описано в шаге 3). Затем можно изменить фильтр, поиск и удаление сообщений в другой набор почтовых ящиков. Дополнительные сведения о создании фильтров разрешения поиска содержатся в разделе [разрешения Настройка фильтрации для поиска контента](permissions-filtering-for-content-search.md).
    
- **Будет удалена неиндексированные элементов, включенных в результатах поиска?**

    Нет, `New-ComplianceSearchAction -Purge -PurgeType SoftDelete` команда не удаляет неиндексированные элементов. 
    
- **Что произойдет, если сообщения будут удалены из почтового ящика, которая размещена на удержание на месте или хранение для судебного разбирательства или назначенные политики хранения к Office 365?**

    После очистки сообщения (пользователем или по истечении срока хранения удаленного элемента), сообщение сохраняется до истечения срока действия продолжительность удержания. Если продолжительность удержания неограниченное число, элементы будут храниться, пока не будет снято удержание или изменены продолжительность удержания.
    
- **Почему рабочий процесс поиска и удалить распределяется между безопасности &amp; группы ролей в центре соответствия требованиям?**

    Как объяснялось ранее пользователь должен входить в группу eDiscovery диспетчер ролей или назначить роль управления поиска соответствия требованиям для поиска почтовых ящиков. Для удаления сообщений, пользователь должен входить в группу ролей управления организацией или назначить роль управления поиска и удалить. Это позволяет ли контролировать, кто поиска почтовых ящиков в организации и пользователей, которые могут удалять сообщения.
