# <a name="get-ready-for-the-new-microsoft-365-security-center-and-microsoft-365-compliance-center"></a>Подготовка к новому Центру безопасности Microsoft 365 и Центру соответствия требованиям Microsoft 365

**Мы рады объявить о [центре соответствия требованиям Microsoft 365](#microsoft-365-compliance-center), развертывание начиная с конца января 2019 и через 2019 марта и все новый [Центр безопасности Microsoft 365](#microsoft-365-security-center) **. В этой статье, чтобы получить общие сведения о предстоящих, [чего следует ожидать](#what-to-expect)и [необходимых лицензий и необходимые разрешения](#required-licenses-and-permissions).

## <a name="microsoft-365-security-center"></a>Центр безопасности Microsoft 365

Нового центра безопасности Microsoft 365 включает в себя панель мониторинга для управления и наблюдения за безопасности удостоверения, данные, устройств, приложений и инфраструктуры. Также необходимо доступ все новые результатов безопасного Microsoft, новых отчетов threat устройства, новых отчетов threat удостоверения и отчетов безопасности приложения в облаке. 

![Новый центр безопасности Microsoft 365](media/m365-security-center.png)

При использовании центра безопасности Microsoft 365 в первый раз, вы увидите сведения в верхней части экрана, чтобы помочь вам начать работу. Вы также увидите описывается переход легко функций безопасности вас интересует самых исследование.

После того как центр безопасности Microsoft 365 включен для вашего клиента, вы сможете получить доступ к его в [https://security.microsoft.com](https://security.microsoft.com). 

> [!NOTE]
> Необходимо присвоить действительной роли Azure Active Directory для доступа к центру безопасности Microsoft 365. Для получения дополнительных сведений обратитесь к разделу [необходимые лицензии и разрешения](#required-licenses-and-permissions) (в этой статье).

## <a name="microsoft-365-compliance-center"></a>Microsoft 365 центре соответствия требованиям

Нового центра соответствия требованиям Microsoft 365 предоставляет видимости в Microsoft соответствия диспетчер, который отражает вашей общей уровня соответствия требованиям, а также получать рекомендуемые действия по настройке параметров для удовлетворения сложных соблюдения. Вам придется легкий доступ к меток уровень конфиденциальности сообщения и хранения и политики, защита от потери данных (DLP), управления данными, обнаружения электронных данных, запросы данных тема (сигналы DSR), Управление делами и облачных безопасности приложения. Кроме того можно получить принятия решений и использовать intelligent автоматизации для уменьшения рисков соответствия требованиям и защищают цифровой офисных. 

![Microsoft 365 центре соответствия требованиям](media/m365-compliance-center.png)

При использовании в центре соответствия требованиям Microsoft 365 в первый раз, вы увидите сведения в верхней части экрана, чтобы помочь вам начать работу. Вы узнаете, как переходить легко для функции контроля соответствия требованиям вас интересует самых исследование.

После включения в центре соответствия требованиям Microsoft 365 для вашего клиента можно получить доступ к его в [https://compliance.microsoft.com](https://compliance.microsoft.com).  

> [!NOTE]
> Необходимо присвоить действительной роли Azure Active Directory для доступа к центру соответствия требованиям Microsoft 365. Для получения дополнительных сведений обратитесь к разделу [необходимые лицензии и разрешения](#required-licenses-and-permissions) (в этой статье).

## <a name="what-to-expect"></a>Чего следует ожидать

### <a name="coming-soon"></a>Ожидается в ближайшее время

Новый центр безопасности Microsoft 365 и новый центр соответствия требованиям Microsoft 365 ваши масштабирование Приступая к работе с поздней января, а также по 2019 марта. Должен иметь доступ к новый центр безопасности Microsoft 365 и центре соответствия требованиям Microsoft 365 за этот период времени.

### <a name="easy-access"></a>Простой доступ!

Улучшенная навигация, интегрированные решения и упростить работу можно будет видеть и доступ к данным наибольший интерес и воспользоваться преимуществами эффективных решений по безопасности и соответствия требованиям в Microsoft 365.

### <a name="smooth-transition"></a>Плавный переход!

Может предполагать плавный переход на новые центры. После этого изменения является полностью для развертывания, планируется удаление предыдущих версий Microsoft 365 безопасности & центре соответствия требованиям ([https://protection.microsoft.com](https://protection.microsoft.com)). Интерфейс администрирования изменится, но это не будет влиять на текущий безопасность и соответствие требованиям конфигураций.

После обновления для развертывания, если в вашей организации есть Microsoft 365 для предприятий E3 или E5, безопасность и соответствие требованиям администраторов выполнить следующие действия.

- Перейдите к [https://security.microsoft.com](https://security.microsoft.com) и [https://compliance.microsoft.com](https://compliance.microsoft.com); <br>или  
- Перейдите в центр администрирования Microsoft 365 и перейдите на новый центр безопасности Microsoft 365 и центре соответствия требованиям Microsoft 365 (ссылки находятся в разделе центры администрирования в левой панели переходов).

> [!TIP]
> Если вы используете Office 365 безопасность & центре соответствия требованиям ([https://protection.office.com](http://protection.office.com)), по-прежнему можно настроить и управление параметрами Office 365 в рамках вашей существующей безопасности Office 365 & центре соответствия требованиям. Конфигурации будет поддерживаться в существующий безопасности Office 365 & центре соответствия требованиям также как и в новый центр безопасности Microsoft 365 и центре соответствия требованиям Microsoft 365.  

## <a name="required-licenses-and-permissions"></a>Необходимые лицензии и разрешения

Чтобы получить новый центр безопасности Microsoft 365 и центре соответствия требованиям Microsoft 365, вашей организации необходимо иметь подписка на Microsoft 365 E3 или E5 или корпоративного лицензирования эквивалентный (которая состоит из Office 365 для предприятий E3 или E5, мобильности Enterprise + безопасности E3 или E5 и Windows 10 корпоративный E3/E5).

Пользователям необходимо назначить роль глобального администратора, администратора соответствия требованиям, администратор безопасности или безопасности чтения в Azure Active Directory для доступа к новый центр безопасности Microsoft 365 или центре соответствия требованиям Microsoft 365.

- Глобальные администраторы могут получить доступ к центре обеспечения безопасности и в центре соответствия требованиям

- Соответствие требованиям для администраторов в центре соответствия требованиям

- Администраторы безопасности или читатели безопасности доступа к центру безопасности

> Дополнительные роли, включая оператор безопасности и соответствия требованиям администратора данных, ожидается в ближайшее время.
