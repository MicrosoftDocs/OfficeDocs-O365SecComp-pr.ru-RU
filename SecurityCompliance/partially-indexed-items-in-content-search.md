# <a name="partially-indexed-items-in-content-search-in-office-365"></a>Частично индексированные элементы в Поиске содержимого в Office 365

Поиск контента, которую можно запускать из безопасности Office 365 &amp; центре соответствия требованиям автоматически включает в себя частично индексированных элементов в оценке результатов поиска при выполнении поиска. Частично индексированных элементов, элементы почтовых ящиков Exchange и документов на сервере SharePoint и OneDrive для бизнеса сайтов, для какой-либо причине не были полностью индексироваться для поиска. В Exchange, частично индексированный элемент обычно содержит файл — типа файлов, не могут быть индексированы —, вложенного в сообщение электронной почты. Ниже приведены некоторые другие и по следующим причинам элементов не могут быть индексированы для поиска возвращаются в виде частично индексированных элементов при выполнении поиска. 
  
- Тип файла нераспознанный или не поддерживается для индексации.
    
-  Сообщения имеют вложенный файл без допустимого обработчика, такие как файлы изображений; Это самой распространенной причиной частично индексированных электронной почты. 
    
- Тип файла поддерживает индексирование, но произошла ошибка индексирования определенного файла.
    
- В сообщение электронной почты вложено слишком много файлов.
    
- Размер файла, вложенного в сообщение электронной почты, превышает допустимый.
    
- Файл зашифрован с помощью технологий, разработанных не корпорацией Майкрософт.
    
- Файл защищен паролем.
    
> [!NOTE]
> Большинство организаций Office 365 иметь не более 1% контента и меньше, чем 12% по размеру частично индексации. Причина различие между объема и размера является то, что большие файлы выше вероятность содержащий данные, которые не могут быть индексированы полностью. 
  
Для юридических расследований вашей организации могут потребоваться для просмотра частично индексированных элементов. Также можно указать, следует ли включать частично индексированных элементов при экспорте результатов поиска на локальном компьютере или при подготовке результаты для анализа с Office 365 расширенного обнаружения электронных данных. Для получения дополнительных сведений см [Investigating частично индексированных элементов в Office 365 обнаружения электронных данных](investigating-partially-indexed-items-in-ediscovery.md).
  
## <a name="file-types-not-indexed-for-search"></a>Типы файлов, которые не поддерживают поиск

Определенные типы файлов, такие как растровое изображение или MP3-файлов, не содержат контента, могут быть индексированы. В результате поиска, индексирования серверы Exchange и SharePoint не выполняйте полнотекстового индексирования на эти типы файлов. Эти типы файлов считаются файлами неподдерживаемых типов. Существуют также типы файлов, для которых полнотекстового индексирования отключена, по умолчанию или администратором. Типы файлов не поддерживается и отключен помечаются как неиндексированные элементов в поиске содержимого. Как было сказано ранее, частично индексированных элементов может быть включен в набор результатов поиска при выполнении поиска, экспортировать результаты поиска на локальном компьютере или подготовить результатов поиска для расширенной обнаружения электронных данных. 
  
Список поддерживаемых и отключенных форматов файлов см. в следующих статьях:
  
- **Exchange** - [форматы файлов, индексированные службой поиска Exchange](https://go.microsoft.com/fwlink/p/?LinkID=386618)
    
- **Exchange** - [Get-SearchDocumentFormat](https://go.microsoft.com/fwlink/p/?LinkID=724037)
    
- **SharePoint** - [расширения имен файлов для обхода и анализируемые типы файлов в SharePoint по умолчанию](https://go.microsoft.com/fwlink/p/?LinkID=404033)
    

  
## <a name="messages-and-documents-with-partially-indexed-file-types-can-be-returned-in-search-results"></a>Сообщения и документы с частично индексируются файл типы могут возвращаться в результатах поиска

Не все сообщения электронной почты с вложением частично индексированных файлов или частично индексированных документам SharePoint автоматически возвращаются в виде частично индексированному элементу. Вот так, как другие сообщения или свойств документа, таких как свойство **Subject** в сообщениях электронной почты и **заголовок** или **автора** свойств документов, индексированные, для поиска. Например поиск по ключевому слову «финансовый» которого возвращаются элементы с частично индексированных файла вложения при появлении этого ключевого слова в теме сообщения электронной почты или имени файла или заголовок документа. Тем не менее если ключевое слово отображается только в теле файла, сообщение или документ возвращаются в виде частично индексированных элементов. 
  
Аналогично при другие сообщения или документа свойства, которые становятся доступными для поиска, соответствующие критериям поиска сообщений с помощью частично индексированных файлов вложений и документах для типа файла частично индексированных включаются в результатах поиска. Даты отправки и получения, отправителя и получателя, имя файла вложения и текст следующие сообщения свойства индексироваться для поиска в тексте сообщения. Свойства документа, индексироваться для поиска включают в себя созданного и изменения дат. Так что несмотря на то, что вложения сообщения могут быть частично индексированный элемент, сообщение будет включен в результаты поиска регулярных Если значение других свойств сообщения или документа результатов поиска.
  
Чтобы получить список свойств электронной почты и документов, которые можно выполнить поиск с помощью функции поиска в системы &amp; центре соответствия требованиям [запросов ключевых слов](keyword-queries-and-search-conditions.md)и условий поиска для поиска контента.
  

  
## <a name="partially-indexed-items-included-in-the-search-results"></a>Частично индексированных элементов, включенных в результатах поиска
<a name="unindexeditemsresults"> </a>

Ваша организация может потребоваться для идентификации и выполнить дополнительный анализ частично индексированных элементов для определения они являются, они содержат и ли они важны для конкретных расследования. Как объяснялось ранее частично индексированных элементов в расположения контента которых выполняется автоматически включены в оценке результатов поиска. У вас есть параметр для включения этих частично индексированных элементов при экспорте результатов поиска или Подготовка результатов поиска для расширенной обнаружения электронных данных. Для включения частично индексированных элементов при подготовке их расширенных обнаружения электронных данных или экспорт результатов поиска, выберите один из параметров, чтобы добавить элементы, которые имеют нераспознанный формат, шифруются или не были индексированы по другим причинам.
  
Следует помнить о частично индексированных элементов следующее:
  
- При выполнении поиска контента, общее число и размер частично индексированных элементов (возвращенных по запросу поиска) отображаются в Статистика поиска в области сведений, как с меткой «неиндексированные элементы».
    
- После экспорта результатов поиска и включить частично индексированных элементов, частично индексированных элементов Exchange экспортируются в отдельные PST-файл для каждого почтового ящика, в котором они находятся или в виде отдельных сообщений при выборе параметра, чтобы загрузить элементы Exchange, как сообщения. частично индексированных элементов SharePoint экспортируются в папку с именем **Uncrawlable**.
    
- Если поиска, который выполняется импорт результаты из поиска определенные расположения содержимого или все расположения контента в вашей организации, будут экспортироваться только неиндексированные элементы из расположения контента, которые содержат элементы, соответствующие условиям поиска. Другими словами Если результаты поиска не найдены в почтовый ящик или сайта, затем все неиндексированные элементы в этом почтового ящика или сайтов не будут экспортировать. Причина этого —, что экспорт частично индексированных элементов из большое количество расположений в организации может повысить вероятность ошибок экспорта и увеличить время, необходимое для экспорта результатов поиска.
    
    Чтобы экспортировать частично индексированных элементов из всех расположений контента для поиска, Настройка поиска для возвращения всех элементов (путем удаления все ключевые слова из поискового запроса) и затем экспортировать лишь частично индексированных элементов при экспорте результатов поиска (например, щелкнув только ** элементы, которые имеют нераспознанный формат, шифруются или не были индексированы по другим причинам** в разделе **Параметры вывода**).
    
- Если нужно включить все элементы почтового ящика в результатах поиска или поисковый запрос не указывает все ключевые слова или только указывает диапазон дат, частично индексированных элементов не может быть скопирован в PST-файл, содержащий частично индексированных элементов. Это, так как все элементы, включая любые частично индексированных элементов будет автоматически включается в результаты поиска регулярного.
    
- Частично индексированных элементов, недоступны для предварительного просмотра. Необходимо экспортировать результаты поиска, чтобы просмотреть частично индексированных элементов, возвращенных в результате поиска.

## <a name="partially-indexed-items-excluded-from-the-search-results"></a>Частично индексированных элементов, исключенных из результатов поиска

Если элемент частично индексируется, но он не соответствует условиям запроса поиска, не будут включены в качестве частично индексированных элементов в списке результатов поиска. Другими словами элемент, исключенный из результатов поиска. Например предположим, выполнение поиска и не включать все ключевые слова и свойства, так как требуется включить все содержимое. Однако включить условие диапазон дат для запроса. Если частично индексированный элемент находится за пределами этого диапазона дат, не будут включены в качестве частично индексированных элементов. Диапазон дат являются и эффективный способ исключения частично индексированных элементов из результатов поиска.
  
Аналогичным образом при выборе включения частично индексированных элементов при экспорте результаты поиска, не будут экспортированы частично индексированных элементов, которые были исключены из результатов поиска.
  
Одно исключение — это правило — при создании запроса на удержание, связанной с вариантом eDiscovery. При создании запроса на удержание, все частично индексированных элементов помещенные на удержание. Этот компонент включает частично индексированных элементов, которые не соответствуют условиям запроса поиска и частично индексированных элементов, которые могут отвечать за пределами условие диапазон дат. Дополнительные сведения о создании запроса на удержание, см шаг 4 в [вариантах eDiscovery в Office 365 безопасности & центре соответствия требованиям](ediscovery-cases.md#step-4-place-content-locations-on-hold).

## <a name="more-information-about-partially-indexed-items"></a>Дополнительные сведения о частично индексированных элементов
<a name="moreinfo"> </a>

- Как было сказано ранее так как индексированные свойства message и документов и их метаданных, поиска по ключевым словам может возвращать результаты при появлении этого ключевого слова в индексированных метаданных. Тем не менее, что же поиск ключевое слово не может возвращать тому же элементу, если ключевое слово отображается только в содержимом элемента с неподдерживаемый тип файла. В этом случае элемент возвращаются в виде частично индексированных элементов.
    
- Если частично индексированный элемент включен в результаты поиска из-за соблюдение условия запроса поиска (и не был исключен) он будет включен в качестве частично индексированный элемент в статистику примерный поиска. Кроме того он будет состав частично индексированных элементов при экспорте результатов поиска.
    
- Несмотря на то, что тип файла поддерживает индексирование и индексируются, может быть индексирования и поиска ошибок, которые будет отображено файла следует возвратить частично индексированных элементов. Например, поиск большого файла Excel возможно, частично успешно (индексируются первых 4 МБ), но не удается выполнить из-за Превышено ограничение на размер файла. В этом случае это и возможно, что возвращаются тот же файл с результатами поиска и как частично индексированных элементов.
    
- Вложенные файлы зашифрован с помощью технологий Майкрософт индексируются и может быть выполнен. Файлы, зашифрованные с помощью технологий сторонних частично индексируются.
    
- Частично индексируются сообщений электронной почты, зашифрован с помощью S/MIME. Этот компонент включает зашифрованных сообщений без вложенных файлов.
    
- Сообщения, защищенные службой управления правами на доступ к данным (IRM), индексируются и включаются в результаты поиска, если они соответствуют поисковому запросу.
