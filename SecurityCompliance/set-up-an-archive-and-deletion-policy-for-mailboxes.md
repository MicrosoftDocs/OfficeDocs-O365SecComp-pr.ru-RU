# <a name="set-up-an-archive-and-deletion-policy-for-mailboxes-in-your-office-365-organization"></a>Настроить политику архивации и удаления для почтовых ящиков в организации Office 365

 **Эта статья предназначена для администраторов. Необходимо добавить политики архивации и хранения элементов в почтовом ящике? Просмотреть [Назначение политики хранения для сообщений электронной почты](https://support.office.com/article/3e5fd2dc-633f-4a38-b313-b31b81f7cf7a) | [политики хранения и архивирования в Outlook в Интернете для бизнеса](https://support.office.com/article/465372e4-e16b-47db-bee0-aba44799085e)**
  
В Office 365 можно создать политику архивации и удаления, автоматически перемещает элементы в архивный почтовый ящик пользователя и автоматическое удаление элементов из почтового ящика. Это можно сделать, создав политику хранения, который "s назначается почтовые ящики и перемещает элементы в архивный почтовый ящик пользователя, после определенного периода времени, который также удаляет элементы из почтового ящика достижении определенного предела. Фактический правил, определяющих, какие элементы являются перемещен или удален, и в этом случае являются теги хранения называемое. Теги хранения, связаны политику хранения, назначенной в свою очередь для почтового ящика пользователя. Тега хранения применяет параметры хранения для отдельных сообщений и папок в почтовом ящике пользователя. Определяет, как долго сообщение остается в почтовом ящике и какие действия по достижении срока хранения указанного сообщения. После появления сообщения срок его хранения, либо он перемещается в архивный почтовый ящик пользователя или его удаления. 
  
Действия, описанные в этом разделе будет настроить политику архивации и хранения для вымышленной организации с именем Alpine House. Настройка эта политика включает в себя следующие задачи:
  
- Включение архивного почтового ящика для каждого пользователя в организации. Это позволяет пользователям Добавление хранилища почтовых ящиков и не требуется, чтобы политика хранения может перемещаются в архивный почтовый ящик. Он также давайте хранилища архивации сведений о пользователе путем перемещения элементов их архивный почтовый ящик. 
    
- Создание трех тегов хранения настраиваемых, сделайте следующее: 
    
  - Автоматически перемещает элементы, которые являются 3 лет архивный почтовый ящик пользователя. Перемещение элементов в архивный почтовый ящик освобождает место в основной почтовый ящик пользователя.
    
  - Автоматически удаляет элементы, которые являются 5 лет из папки "Удаленные". Это также освобождает место в основной почтовый ящик пользователя. Пользователя будут иметь возможность восстановить эти элементы, если это необходимо. В разделе нижнего колонтитула в разделе [Дополнительные сведения](set-up-an-archive-and-deletion-policy-for-mailboxes.md#moreinfo) для получения дополнительных сведений. 
    
  - Автоматически (и без возможности восстановления) удаляет элементы, которые являются 7 лет из обоих основном и архивном почтовом ящике. Из-за нормативные правила некоторые организации требуется сохранить электронной почты в течение определенного периода времени. После истечения срока действия этого периода времени, организация может потребоваться удалить эти почтовые ящики пользователей элементов без возможности восстановления. 
    
- Создание новой политики хранения и добавление новых тегов хранения настраиваемых. Кроме того будет также добавить теги хранения встроенных новой политики хранения. Этот компонент включает личных тегов, которые пользователи могут назначаться элементов в почтовом ящике. Кроме того, вы добавите тега хранения, который перемещает элементы в папке элементов для восстановления в основной почтовый ящик пользователя для папки восстанавливаемых элементов в почтовом ящике архива. Это помогает освободить место в папки восстанавливаемых элементов при их почтовых ящиков ставится на удержание.
    
Вы можете выполнить некоторые или все действия, описанные в этой статье, чтобы настроить политику архивации и удаления для почтовых ящиков в своей организации. Рекомендуется проверить этот процесс на несколько почтовых ящиков перед реализацией на всех почтовых ящиков в организации.
  
## <a name="before-you-begin"></a>Перед началом работы

- Необходимо быть глобальным администратором в организации Office 365 для выполнения действия, описанные в этом разделе. 
    
-  При создании учетной записи пользователя в Office 365 и назначить ему лицензию Exchange Online, для пользователя автоматически создается почтовый ящик. При создании почтового ящика, он автоматически назначена политика хранения по умолчанию с именем политикой управления записями сообщений по умолчанию. В этой статье будет создать новую политику хранения и назначьте его к почтовым ящикам пользователей, заменив политики управления записями сообщений по умолчанию. Почтовый ящик может иметь только одну политику хранения, заданные в любой момент времени.
    
- Для получения дополнительных сведений о тегах хранения и политики хранения в Exchange Online, просмотрите [теги хранения и политики хранения](https://go.microsoft.com/fwlink/p/?LinkId=404424).
    
## <a name="step-1-enable-archive-mailboxes-for-users"></a>Шаг 1: Включение архивные почтовые ящики пользователей

Первым этапом является возможность архивный почтовый ящик для каждого пользователя в организации. Архивный почтовый ящик пользователя должен быть включен, чтобы тега хранения с помощью действия «Переместить в архив» хранения можно переместить элемент, по истечении срока хранения. 
  
> [!NOTE]
> Можно включить любое время во время этого процесса архивные почтовые ящики, только при условии, что вы включены в определенный момент, до завершения процесса. Если этот параметр не включен архивного почтового ящика, никакие действия не предпринимаются на все элементы, которые имеют назначен политики архивирования. 
  
1. Перейдите по ссылке [https://protection.office.com](https://protection.office.com).
    
2. Войдите в Office 365 с помощью учетной записи глобального администратора.
    
    
3. В разделе Безопасность &amp; центре соответствия требованиям, чтобы перейти на **Управление данными** \> **архива**.
    
    Отображается список почтовых ящиков в организации и включена или отключена в соответствующем архивный почтовый ящик. 
    
4. Выберите все почтовые ящики, нажав кнопку на первый в списке, удерживая нажатой клавишу **Shift** и затем выбрав последним в списке. 
    
    > [!TIP]
    > На этом шаге предполагается, что включены не архивные почтовые ящики. При наличии почтовые ящики с поддержкой архив нажмите и удерживайте клавишу **Ctrl** и щелкните каждого почтового ящика, который имеет отключенный архивного почтового ящика. Или можно щелкните заголовок столбца **архивного почтового ящика** для сортировки строк на основании ли архивный почтовый ящик включено или отключено для упрощения выберите почтовых ящиков. 
  
5. В области сведений в разделе **Массового изменения**, нажмите кнопку **Включить**.
    
    Отображается предупреждение о том, что элементы, возраст которых превышает два года будут перемещены в новый архивный почтовый ящик. Это, так как политика хранения по умолчанию, который назначен новый почтовый ящик пользователя при его создании имеет тег политики по умолчанию архива, который имеет срок хранения 2 лет. Тег политики по умолчанию архивирования, которое вы создадите в шаге 2 имеет срок хранения 3 года. Это означает, что элементы, которые являются 3 года или более ранней версии перемещаются в архивный почтовый ящик.
    
6. Нажмите кнопку **Да,** чтобы закрыть предупреждающее сообщение и начать процесс Включение архивного почтового ящика для каждого выбранного почтового ящика. 
    
7. После завершения процесса нажмите кнопку **Обновить** ![обновить](media/165fb3ad-38a8-4dd9-9e76-296aefd96334.png) для обновления списка на странице " **архив** ". 
    
    Архивный почтовый ящик будет доступна для всех пользователей в организации.
    
    ![Список почтовых ящиков с поддержкой архивного почтового ящика](media/61a7cb97-1bed-4808-aa5f-b6b761cfa8de.png)
  
8. Оставьте безопасности &amp; открыть центр соответствия. Оно будет использоваться на следующем шаге.
    
## <a name="step-2-create-new-retention-tags-for-the-archive-and-deletion-policies"></a>Шаг 2: Создание новых тегов хранения для архивации и удаления политик

На этом этапе вы создадите три теги хранения настраиваемых, описанных ранее.
  
- Alpine палаты 3 года переместить в архив (политика архивирования)
    
- Alpine палаты 7 года окончательно удалить (Удаление настраиваемой политики)
    
- Alpine палаты удаленных элементов 5 лет удалить и разрешить восстановление (настраиваемые тег для папки «Удаленные»)
    
Чтобы создать новые теги хранения, используется Центр администрирования Exchange (EAC) в организации Exchange Online.
  
1. В разделе Безопасность &amp; центре соответствия требованиям, щелкните запуска приложения в верхнем левом углу и выберите плитку **администрирования** . 
    
2. В левой панели переходов центра администрирования Office 365 нажмите кнопку **центры администрирования**и щелкните **Exchange**.
    
    ![Снимке экрана показан центров центра администрирования Office 365 с помощью администратор параметр были развернуты и установлен Exchange.](media/47399df2-0bc4-42e2-b183-07750a46bc68.png)
  
3. В центре администрирования Exchange перейдите к **разделу Управление соответствием требованиям** \> **теги хранения**
    
    Отобразится список тегов хранения для вашей организации.
    
### <a name="create-a-custom-archive-default-policy-tag"></a>Создать тег политики по умолчанию архивирования
  
Во-первых вы должны создать тег политики по умолчанию архивирования (тег политики по умолчанию), который будет перемещаются в архивный почтовый ящик через 3 года. 
  
1. На странице **теги хранения** щелкните **новый тег**![новый значок](media/457cd93f-22c2-4571-9f83-1b129bcfb58e.gif)и затем выберите **применяются автоматически к всему почтовому ящику (по умолчанию)**. 
    
2. На странице **новый тег, применяются автоматически к всему почтовому ящику (по умолчанию)** заполните следующие поля: 
    
    ![Параметры, чтобы создать новый тег политики по умолчанию архива](media/41c0a43c-9c72-44e0-8947-da0831896432.png)
  
1. **Имя** Введите имя для нового тега хранения. 
    
2. **Действие хранения** Выберите **Переместить в архив** перемещаются в архивный почтовый ящик по истечении срока хранения. 
    
3. **Период хранения** Выберите **элемент по достижении следующих срок хранения (в днях)**, а затем введите длительность периода хранения. Для этого сценария элементы перемещаются в архивный почтовый ящик после 1095 дней (3 года).
    
4. **Комментарий** (Необязательно) Введите комментарий, описывает назначение тега хранения настраиваемых. 
    
3. Нажмите кнопку **Сохранить** , чтобы создать настраиваемые архив тег политики по умолчанию. 
    
    Новый тег политики по умолчанию архива отображается в списке тегов хранения.
    
### <a name="create-a-custom-deletion-default-policy-tag"></a>Создать тег политики по умолчанию настраиваемые удаления
  
Теперь вы создадите другого настраиваемого тег политики по умолчанию, но это будет удаления политики, которая используется для удаления элементов после 7 лет.
  
1. На странице **теги хранения** щелкните **новый тег**![новый значок](media/457cd93f-22c2-4571-9f83-1b129bcfb58e.gif)и затем выберите **применяются автоматически к всему почтовому ящику (по умолчанию)**. 
    
2. На странице **новый тег, применяются автоматически к всему почтовому ящику (по умолчанию)** заполните следующие поля: 
    
    ![Параметры, чтобы создать новый тег политики по умолчанию удаления](media/f1f0ff62-eec9-4824-8e7c-d93dcfb09a79.png)
  
1. **Имя** Введите имя для нового тега хранения. 
    
2. **Действие хранения** Выберите **Удалить без возможности восстановления** , чтобы удалять элементы из почтового ящика при срока их хранения. 
    
3. **Период хранения** Выберите **элемент по достижении следующих срок хранения (в днях)**, а затем введите длительность периода хранения. Для этого сценария элементы будут очищены после 2555 дней (7 лет).
    
4. **Комментарий** (Необязательно) Введите комментарий, описывает назначение тега хранения настраиваемых. 
    
3. Нажмите кнопку **Сохранить** , чтобы создать настраиваемые удаления тег политики по умолчанию. 
    
    Новый тег политики по умолчанию удаления отображается в списке тегов хранения.
    
### <a name="create-a-custom-retention-policy-tag-for-the-deleted-items-folder"></a>Создание тега политики хранения настраиваемых для папки "Удаленные"
  
Последний тега хранения, которое вы создадите является тега политики хранения настраиваемых (RPT) для папки «Удаленные». Этот тег будет удалить элементы из папки "Удаленные" после 5 лет и предоставляет периода восстановления, когда пользователи могут использовать средство восстановить удаленные элементы для восстановления элементов.
  
1. На странице **теги хранения** щелкните **новый тег** ![новый значок](media/457cd93f-22c2-4571-9f83-1b129bcfb58e.gif)и затем выберите **применяется автоматически в папке по умолчанию**. 
    
2. На странице " **новый тег применяется автоматически в папке по умолчанию** " заполните следующие поля: 
    
    ![Параметры, чтобы создать новый тег политики хранения для папки "Удаленные"](media/6f3104bd-5edb-48ac-884d-5fe13d81dd1d.png)
  
1. **Имя** Введите имя для нового тега хранения. 
    
2. **Применить этот тег в следующую папку по умолчанию** В раскрывающемся списке выберите **Deleted Items**.
    
3. **Действие хранения** Выберите **Удалить и разрешить восстановление** удалять элементы, когда истекает срок хранения, но разрешить пользователям восстановление удаленных элементов в течение периода хранения удаленных элементов (который по умолчанию установлено 14 дней). 
    
4. **Период хранения** Выберите **элемент по достижении следующих срок хранения (в днях)**, а затем введите длительность периода хранения. Для этого сценария элементы будут удалены после 1825 дней (5 лет).
    
5. **Комментарий** (Необязательно) Введите комментарий, описывает назначение тега хранения настраиваемых. 
    
3. Нажмите кнопку **Сохранить** , чтобы создать настраиваемые тег политики Хранения для папки "Удаленные". 
    
    Новые RPT отображается в списке тегов хранения.

## <a name="step-3-create-a-new-retention-policy"></a>Шаг 3: Создание новой политики хранения

После создания тегов хранения настраиваемых следующим шагом является создание новой политики хранения и добавление тегов хранения. Вы добавите три теги хранения настраиваемых, созданный на шаге 2 и встроенные теги, упомянутых в первом разделе. На шаге 4 будет назначить этой новой политики хранения к почтовым ящикам пользователей.
  
1. В центре администрирования Exchange перейдите к **разделу Управление соответствием требованиям** \> **политики хранения**.
    
2. На странице " **политик хранения** " нажмите кнопку **Создать** ![новый значок](media/457cd93f-22c2-4571-9f83-1b129bcfb58e.gif).
    
3. В поле **имя** введите имя новой политики хранения; Например, **Alpine архива дома и политики удаления**. 
    
4. В разделе **теги хранения**, нажмите кнопку **Добавить** ![новый значок](media/457cd93f-22c2-4571-9f83-1b129bcfb58e.gif).
    
    Отобразится список тегов хранения в вашей организации. Обратите внимание, что отображаются настраиваемые теги, созданные в шаге 2.
    
5. Добавьте 9 теги хранения, выделенных на следующем снимке экрана (этих тегах описаны более подробно в разделе [Дополнительные сведения](set-up-an-archive-and-deletion-policy-for-mailboxes.md#moreinfo) ). Добавление тега хранения, выберите его и нажмите кнопку **Добавить**. 
    
    ![Добавление тегов хранения в новой политики хранения](media/d8e87176-0716-4238-9e6a-7c4af35541dc.png)
  
    > [!TIP]

7. На странице " **Новая политика хранения** " нажмите кнопку **Сохранить** , чтобы создать новую политику.

## <a name="step-4-assign-the-new-retention-policy-to-user-mailboxes"></a>Шаг 4: Назначение новой политики хранения к почтовым ящикам пользователей

При создании нового почтового ящика политику хранения, с именем политикой управления записями сообщений по умолчанию назначается его по умолчанию. На этом этапе вы замените эту политику хранения (так как почтовый ящик может иметь только один назначена политика хранения) с назначением новой политики хранения, созданное в шаге 3 для почтовых ящиков пользователей в вашей организации. На этом шаге предполагается, что будет Назначение новой политики ко всем почтовым ящикам в организации.
  
1. В Центре администрирования Exchange выберите **Получатели** \> **Почтовые ящики**.
    
    Отображается список всех почтовых ящиков пользователей в вашей организации. 
    
2. Выберите все почтовые ящики, нажав кнопку на первый в списке, удерживая нажатой клавишу **Shift** и затем выбрав последним в списке. 
    
3. В области сведений в правой части центра администрирования Exchange, в группе **Массового изменения**нажмите кнопку **Дополнительные параметры**.
    
4. В разделе **Политика хранения** щелкните **Обновить**.
    
5. На странице **Массовое назначение политики хранения** в раскрывающемся списке **Выберите политику хранения** выберите политику хранения, созданной на шаге 3; Например, **Alpine архива дома и политики хранения**.
    
6. Нажмите кнопку **Сохранить** для сохранения нового назначения политики хранения. 
    
7. Чтобы убедиться, что новой политики хранения была назначена в почтовые ящики, можно сделать следующее: выберите почтовый ящик на странице "почтовые ящики" и нажмите кнопку Изменить. 
    
1. Выберите пункт почтовый ящик на странице " **почтовые ящики** " и нажмите кнопку **Изменить** ![изменить](media/d7dc7e5f-17a1-4eb9-b42d-487db59e2e21.png). 
    
2. На странице свойств почтового ящика для выбранного пользователя щелкните **функции почтового ящика**.
    
    Имя новой политики, назначенные в почтовый ящик отображается в раскрывающемся списке **Политика хранения** . 
    

  
## <a name="optional-step-5-run-the-managed-folder-assistant-to-apply-the-new-settings"></a>(Необязательно) Шаг 5: Запуск помощника для управляемых папок для применения новых параметров
<a name="step3"> </a>

После установки новой политики хранения к почтовым ящикам на шаге 4, может потребоваться до 7 дней в Exchange Online для новых параметров хранения для применения к почтовым ящикам. Это, так как этот процесс называется помощника для управляемых папок почтовых ящиков процессы один раз в неделю. А не дожидается помощника для управляемых папок для запуска, можно принудительно, чтобы это сделать, выполнив командлет **Start-ManagedFolderAssistant** в Exchange Online PowerShell. 
  
 **Что происходит при запуске помощника для управляемых папок?** Он применяет параметры в политику хранения, проверяя элементов в почтовом ящике и определения, является ли они может быть хранения. Затем Маркирует элементов, относящихся к хранения с помощью тега хранения соответствующих и затем начинает действовать указанного хранения элементов прошлые срока их хранения. 
  
Ниже перечислены действия, чтобы подключиться к Exchange Online PowerShell и запустите помощника для управляемых папок для всех почтовых ящиков в организации.
  
1. На локальном компьютере откройте Windows PowerShell и запустите следующую команду:
    
    ```
    $UserCredential = Get-Credential
    ```

    В диалоговом окне **Запрос учетных данных Windows PowerShell** введите имя пользователя и пароль для учетной записи глобального администратора Office 365 и нажмите кнопку **ОК**.
    
2. Выполните следующую команду.
    
    ```
    $Session = New-PSSession -ConfigurationName Microsoft.Exchange -ConnectionUri https://outlook.office365.com/powershell-liveid/ -Credential $UserCredential -Authentication Basic -AllowRedirection
    ```

3. Выполните следующую команду:
    
    ```
    Import-PSSession $Session
    ```

4. Убедитесь, что подключение к организации Exchange Online установлено, получив список всех почтовых ящиков в организации с помощью следующей команды:
    
    ```
    Get-Mailbox
    ```

    > [!NOTE]
    > Дополнительные сведения или варианты решения проблем при подключении к организации Exchange Online см. в статье [Подключение к Exchange Online с помощью удаленной оболочки PowerShell](https://go.microsoft.com/fwlink/p/?LinkId=517283). 
  
5. Выполните следующие две команды для запуска помощника для управляемых папок для всех почтовых ящиков пользователей в вашей организации.
    
    ```
    $Mailboxes = Get-Mailbox -ResultSize Unlimited -Filter {RecipientTypeDetails -eq "UserMailbox"}
    ```

    ```
    $Mailboxes.Identity | Start-ManagedFolderAssistant
    ```

Ну вот! Настройки политику архивации и удаления для организации Alpine House.
