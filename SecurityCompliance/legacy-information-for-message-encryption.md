# <a name="legacy-information-for-office-365-message-encryption"></a>Сведения об устаревших продуктах для службы шифрования сообщений Office 365

Новые возможности OME еще еще не перемещены организации Office 365, но вы уже развернули OME, сведения в этой статье относятся для вашей организации. Корпорация Майкрософт рекомендует, чтобы сделать план для перемещения новых возможностей OME, как только целесообразно для вашей организации. Сведения содержатся в разделе [Set up новые возможности шифрования сообщений Office 365, построенные защита информации Azure](set-up-new-message-encryption-capabilities.md). Если вы хотите узнать больше о новых возможностях сначала, ознакомьтесь со [Шифрования сообщений Office 365](ome.md). Далее в этой статье относится к OME поведение перед выпуском новых возможностей OME.
  
С помощью шифрования сообщений Office 365 организации можно отправлять и получать зашифрованные сообщения электронной почты между пользователями внутри и за пределами вашей организации. Office 365 работает шифрование сообщений с Outlook.com, Yahoo, Gmail и другие службы электронной почты. Шифрования сообщений электронной почты помогает предотвратить, решена получатели могут просматривать содержимое сообщения.
  
Вот несколько примеров:
  
- Сотрудник банка отправляет данные о банковской карте операторов для клиентов
    
- Представитель страховой компании предоставляет клиентам сведения о политике.
    
- Кредитов брокер запрашивает финансовые сведения от клиента для предоставления ссуды.
    
- Медицинская страховая компания отправляет пациентам информацию о здоровье.
    
- Адвокат передает конфиденциальные сведения клиенту или другому адвокату.
    
## <a name="how-office-365-message-encryption-works-without-the-new-capabilities"></a>Как работает шифрование сообщений Office 365 без новые возможности

Шифрование сообщений Office 365 — это сетевая служба, встроенной в Microsoft Azure Rights Management (Azure RMS). С помощью службы управления правами Azure администраторы могут определить правила для потока обработки почты для определения условий для шифрования. Например правило может потребовать шифрования все сообщения, адресованные определенного получателя.
  
Просмотрите это короткое видео, чтобы увидеть, как работает шифрование сообщений Office 365 без новых возможностей.
  
> [!VIDEO https://www.microsoft.com/videoplayer/embed/c55540e7-f7f0-42f5-b254-4b2d2fbb1d63?autoplay=false]
  
Когда кто-то отправляет сообщение электронной почты в Exchange Online, который соответствует правила шифрования, сообщение отправлено с HTML-вложение. Получатель открывает HTML-вложение и соответствует инструкции для просмотра зашифрованного сообщения на портал шифрования сообщений Office 365. Получатель сможет выбирать для просмотра сообщения с вход с учетной записью Майкрософт или рабочих или school, связанных с Office 365 или однократного секретного кода. Оба параметра гарантировать, что только получателю можно просмотреть зашифрованного сообщения. Этот процесс является очень разные для новых возможностей OME.

## <a name="defining-mail-flow-rules-for-office-365-message-encryption-that-dont-use-the-new-ome-capabilities"></a>Определение правил потока обработки почты для шифрования сообщений Office 365, не использовать новые возможности OME

Включение шифрования сообщений Office 365 без новых возможностей, администраторов Exchange Online и Exchange Online Protection определить правила потока обработки почты Exchange. Эти правила определяют, в разделе email какие условия должно быть зашифровано сообщения, а также условия для удаления шифрования сообщения. Если для правила шифрования действие, все сообщения, которые соответствуют условиям правила шифруются перед отдельными.
  
Правила для потока обработки почты отличаются гибкостью, позволяющей объединять условия, поэтому требованиям безопасности в одно правило. Например можно создать правила для шифрования всех сообщений, содержащих ключевые слова и предназначено для внешних получателей. Шифрование сообщений Office 365 также шифрует ответов от получатели зашифрованных сообщений электронной почты, и можно создать правило, которое расшифровывает эти ответы для удобства для пользователей электронной почты. Таким образом выполнить вход на портал шифрования для просмотра ответов не пользователей в вашей организации.
  
Дополнительные сведения о создании правил потока обработки почты Exchange можно [Определение правил для шифрования сообщений Office 365](define-mail-flow-rules-to-encrypt-email.md).
  
## <a name="sending-viewing-and-replying-to-encrypted-email-messages"></a>Отправка, просмотр и ответ на зашифрованные сообщения электронной почты

С помощью шифрования сообщений Office 365 сообщений электронной почты, шифруются автоматически, на основании заданных администратором правил. Сообщение электронной почты, который есть зашифрованное сообщение попадает в почтовом ящике получателя с вложенных HTML-файл.
  
Получатели следуйте инструкциям, представленным в сообщение, чтобы открыть вложение и проверку подлинности с помощью учетной записи Майкрософт или рабочих или school, связанных с Office 365. Если получатели не учетную запись, они будете перенаправлены на создание учетной записи, позволит их вход для просмотра зашифрованного сообщения Microsoft. Кроме того получатели могут выбрать для получения одноразовый проход кода для просмотра сообщения. После входа в или с использованием одноразовый проход кода, получателей можно просмотреть расшифрованного сообщения и отправки зашифрованного ответа.
  
## <a name="customize-encrypted-messages-with-office-365-message-encryption"></a>Настройка зашифрованных сообщений с помощью шифрования сообщений Office 365

Как администратор Exchange Online и Exchange Online Protection вы можете настроить зашифрованные сообщения. Например можно добавить вашей компании торговая марка Логотип, укажите сведения о и добавьте текст заявления об отказе в зашифрованных сообщений и на портале, где получатели просматривать зашифрованные сообщения. Использование командлетов Windows PowerShell, можно настроить следующие аспекты возможности просмотра для получателей зашифрованные сообщения электронной почты:
  
- Вводный текст зашифрованного сообщения электронной почты
    
- Текст заявления об отказе зашифрованного сообщения электронной почты
    
- Текст, отображающийся в портале просмотра сообщений
    
- Логотип, который будет отображаться в сообщении электронной почты и на портале просмотра
    
Кроме того, в любое время вы можете восстановить интерфейс по умолчанию.

## <a name="frequently-asked-questions-about-legacy-ome"></a>Часто задаваемые вопросы о прежних версий OME
<a name="LegacyServiceInfo"> </a>

Есть вопросы о шифровании сообщений Office 365? Ниже приведены некоторые вопросы. Если вы не можете найти, что вам нужно, проверьте на форумах сообщества Office 365 в [сообщества Office 365](http://community.office365.com/en-us/forums/default.aspx).
  
 **В. в моей пользователи отправлять зашифрованные сообщения электронной почты получателям за пределами организации. Есть ли что-либо, внешним получателям необходимо сделать, чтобы читать и отвечать на сообщения электронной почты, зашифрованные с помощью шифрования сообщений Office 365?**
  
Получатели за пределами вашей организации могут просматривать полученные зашифрованные сообщения Office 365 двумя способами:
  
- С вход с учетной записью Майкрософт или учетной записи, рабочий или школы, связанных с Office 365.
    
- С помощью одноразовый передайте кода.
    
 **В. Хранятся ли зашифрованные сообщения Office 365 в облаке или на серверах Майкрософт?**
  
Нет, зашифрованные сообщения будут сохранены на системы электронной почты получателя, а при открытии сообщения получателем временно учтена для просмотра на серверах Office 365. Сообщения сохраняются не существует.
  
 **В. Могу ли я настроить зашифрованные сообщения, применив собственную фирменную символику?**
  
Да. С помощью командлетов Windows PowerShell вы можете настроить текст, который по умолчанию будет отображаться в верхней части зашифрованных сообщений, текст заявления об отказе и логотип, который вы хотите использовать в электронных сообщениях и на портале шифрования. Подробнее см. в статье [Add branding to encrypted messages](add-your-organization-brand-to-encrypted-messages.md).
  
 **В. Требуется ли для использования службы, чтобы у каждого пользователя в моей организации была лицензия?**
  
Лицензия требуется для каждого пользователя в организации, который отправляет зашифрованное сообщение электронной почты.
  
 **В. Требуется ли подписка для внешних получателей?**
  
Нет, чтобы читать или отвечать на зашифрованные сообщения, внешним получателям не требуется подписка. 
  
 **Вопрос. насколько является шифрования сообщений Office 365 различных из службы управления правами (RMS)?**
  
Службы управления правами предоставляет возможности сведения о защите прав для внутренней организации электронной почты с указанием встроенных шаблонов, таких как: действие не пересылать и Company Confidential. Office 365 шифрования сообщений поддерживает шифрования сообщений электронной почты для сообщений, отправленных в внешних получателей, а также внутренних получателей.
  
 **Шифрование сообщений Office 365 отличается от S/MIME такое как вопрос.**
  
S/MIME — это, по сути, технология шифрования на стороне клиента, которая требует сложного управления сертификатами и инфраструктуры публикации. Шифрование сообщений Office 365 использует правила транспорта и не зависит от публикации сертификатов.
  
 **В. Можно ли читать зашифрованные сообщения на мобильных устройствах?**
  
Да, можно просмотреть сообщения на Android и операций ввода-вывода с загрузки средства просмотра OME приложений с [Google воспроизвести хранения](http://go.microsoft.com/fwlink/?LinkID=525995&amp;clcid=0x409) и [хранения Apple приложения](http://go.microsoft.com/fwlink/?LinkID=525996&amp;clcid=0x409). Откройте HTML-вложение в приложении OME средства просмотра и затем следуйте инструкциям, чтобы открыть зашифрованного сообщения. Для других мобильных устройств можно открыть HTML-вложение, до тех пор, пока почтового клиента поддерживающем Form Post.
  
 **В. Шифруются ли ответы и пересылаемые сообщения?**
  
Да. Все ответы в рамках беседы продолжают шифроваться.
  
 **В. Предоставляется ли локализация в шифровании сообщений Office 365?**
  
Входящие сообщения и HTML-контент локализуются на основе параметров электронной почты отправителя. Портал просмотра локализуется на основе параметров браузера получателя. При этом фактический текст зашифрованного сообщения (контент) не локализуется.
  
 **В. Какой метод шифрования используется службой шифрования сообщений Office 365?**
  
В службе шифрования сообщений Office 365 в качестве инфраструктуры шифрования используются службы управления правами (RMS). Выбор используемого метода шифрования зависит от того, где получены ключи RMS для шифрования и расшифровки сообщений.
  
- При использовании службы управления правами Microsoft Azure для получение ключей используется криптографический режим 2. Криптографический режим 2 является реализацией шифрования AD RMS обновленные и расширенный. Поддерживает RSA 2048 для подписи и шифрования и поддерживает SHA-256 для подписи.
    
- Если для получения ключей используются службы Active Directory (AD) RMS, применяется криптографический режим 1 или 2. Использование этого метода зависит от локального развертывания AD RMS. Криптографический режим 1 является исходной системой шифрования AD RMS. Этот режим поддерживает алгоритм подписи и шифрования RSA 1024, а также алгоритм подписи SHA-1. Этот режим и далее поддерживается всеми текущими версиями служб RMS.
    
Для получения дополнительных сведений см. [Шифрования режимы AD RMS](http://go.microsoft.com/fwlink/p/?LinkId=398616).
  
 **В. Почему в некоторых зашифрованных сообщениях указан адрес отправителя Office365@messaging.microsoft.com?**
  
При отправке зашифрованного ответа с портала шифрования или с помощью средства просмотра OME адресом отправителя указывается Office365@messaging.microsoft.com, так как зашифрованное сообщение отправляется через конечную точку Майкрософт. Благодаря этому зашифрованное сообщение не помечается как спам. Кроме того, отображаемое имя сообщения и адрес внутри портала шифрования остаются без изменений. Кроме того, эта пометка применяется только к сообщениям, отправляемым с портала, а не с помощью другого почтового клиента.
  
 **Вопрос. я являетесь подписчиком Exchange Hosted Encryption (EHE). Где можно узнать больше о обновления до шифрования сообщений Office 365?**
  
Все клиенты EHE, обновлены до шифрования сообщений Office 365. Для получения дополнительных сведений посетите [Центр обновления Exchange размещенных в узлах шифрования](http://go.microsoft.com/fwlink/p/?LinkID=511077).
  
 **Вопрос. мне нужно открыть любой URL-адреса, IP-адрес решает или порты в брандмауэре Моя организация для поддержки шифрования сообщений Office 365?**
  
Да. Необходимо добавить URL-адреса для Exchange Online в список разрешений своей организации, чтобы включить проверку подлинности сообщений, зашифрованных с помощью службы шифрования сообщений Office 365. Список URL-адресов Exchange Online см. в разделе [Office 365 URLs and IP Address Ranges](https://support.office.com/article/f57e35b7-0a45-42f0-855e-11aa5e7f13fd.aspx).
  
 **В. Скольким получателям можно отправить зашифрованное сообщение Office 365?**
  
Максимальное число получателей для зашифрованного сообщения основано на количество символов в поле **для** сообщения. При объединении (после раскрытие списков рассылки), адреса получателей в **поле** не должно превышать 11,980 символов. Так как адреса электронной почты может быть разной длина в символах, не стандартных максимальное число получателей для одного зашифрованного сообщения. 
  
 **В. Можно ли отозвать сообщение, отправленное определенному получателю?**
  
Нет. Сообщение для конкретного пользователя невозможно отменить после его отправки.
  
 **В. Можно просмотреть отчет о зашифрованных сообщениях, которые были получены и прочитаны?**
  
Не отчет, отображающий Если просмотра зашифрованного сообщения, но недоступны, что можно использовать для определения количества сообщений, которые соответствуют определенное правило транспорта, например отчетов Office 365.
